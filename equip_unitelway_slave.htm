Schneider Unitelway slave

This driver supports serial communication with Schneider Electric PLCs.

In all exchanges, PcVue is the slave and the field device is the master.

Unitelway is a token passing network. A token is passed to each network device in turn. Only when a device has the token it is allowed to make requests on the network. The token is controlled by one of the devices on the network, normally a PLC. The device controlling the token is known as the master, all other devices, including PcVue, are slaves.

Network properties

This topic only includes a reference of objects' properties.

See the topic How to configure networks devices and frames for more information about how to configure communication objects.

General properties
Name The name of the network
Description The description of the network (optional)
First virtual equipment address The slave address that PcVue is to be. Range 1 to 98.
Number of virtual equipments

The number of contiguous addresses that PcVue will occupy on the network. Range 1 to 7.

When using the Unitelway slave driver, PcVue is a slave device and only has the opportunity to make requests on the network when it has the token. This can be a problem it means it only has access to the network for short periods of time restricting the number frames it can read per second. To mitigate this problem, the driver can configured to respond to more than one slave address. For example if you configure the equipment address to be 10, and the number of virtual equipments to be 7, it will be able to respond to addresses 10, 11, 12 13, 14, 15, 16 and 17.
Serial port The COM port associated to the network
Activate at start-up If enabled, the network will start upon Equipment communication start-up

Device properties

General properties
Name The name of the device
Description The description of the device (optional)
Equipment type Select the type of equipment that best suits the field device you want communicate with.
The list of frame types and address ranges depend on the equipment type.
Network number The network number that the equipment is connected to. Range 0 to 127. A value of 0 means that the network number is ignored.
Station number The number of the station on the network. Range 0 to 63.
Gate number The gate number for the equipment. Set to 0 for access to the master PLC system objects. Set to 5 for access to slave PLC system objects.
Module number The module number of the equipment. This number is only defined if the gate number is 5, otherwise it is left set to the default of 0.
Used to define the geographical position of the master coupler of the network.
Module number = ( Rack Number * 16 ) + slot No. in the rack.
Or
255 if the coupler is integrated in the CPU (SCI connector) on TSX 47/67/87/107.
Channel number Master coupler channel number of the network. This number is only defined if the gate number is 5, otherwise its value must be 0.
Used to define the link address of the destination equipment on the network as follows:
If the bus master is a TSX37/57 PLC, then:
Channel number = Address of the destination equipment network.
Or
If the bus master is a TSX/PMX (TSX 47/67/87/107), then:
Channel number = 100 + address of the destination equipment on the network.

The Free frame can be used to send a Uni-te command (STOP, RUN, OPEN_DOWNLOAD, CLOSE_DOWNLOAD, OPEN_UPLOAD, CLOSE_UPLOAD, WRITE_DOWNLOAD etc.) to a PLC.

To use it, map a register variable to the first index in the frame. To send a command you set the variable value to the command number. For example, to send a RUN command you set the value of the variable to 9223 (0x2407 hexadecimal).

More information on Uni-te commands can be found in the documentation supplied by Schneider Electric.
