How to export a configuration subset

The Export feature of the Application Architect makes it possible to export a subset of the configuration elements you have modeled. It facilitates the reuse of application functions in other projects, reinventing therefore, the way projects are configured. This feature also enables the creation of large projects out of smaller ones, thus improving the production and generation of projects.

The configuration subset you export can contains a combination of various elements such as parameters, templates, or instances from one or more libraries.

  1. From the PcVue menu, go to Configure.Application Architect and select the tab containing the elements to export. You can export instances, templates or parameters.

  2. Right-click the elements you want to export then click Add to exported items.

  3. Select the Export tab. All added elements are listed. You can also check, uncheck and remove the elements from the list using buttons in the toolbar. The list is saved in the CurrentExportList.xml file in the TP directory of the project and is loaded when opening the Application Architect. Show picture

    If your engineering workflow includes re-iterating exports for different subsets of configuration, you have what it takes to manage several custom export lists:

    • You can save the export list using the Save button. This will create a file in a folder and under a name of your choice that can later be reused.

    • You can also reload a previously saved list by clicking the Load button. This will replace the current list of elements chosen for export.

  4. Select the file path and enter the file name in which your configuration will be exported. An empty path exports the file in the TP folder.

  5. From the Dependencies box, choose whether to export all related dependencies or none. Dependencies are elements that are needed to create elements you have selected for export such as:

    • A parameter used in an expression defined in the property of an element you want to export.

    • A parameter and/ or a template included in another template you want to export.

    • A parameter and/ or a template in an instance you want to export.

    The elements you have selected and their children elements are exported in all cases.

    • Select None if the elements you want to export are only dependent on elements that are already in the configuration where you will import. For example, if you export instances with the goal of importing them in a project where the necessary templates and parameters are already defined, you will eventually select None.

    • Select All dependencies if you want to export the definition of all the elements necessary to re-compose the elements you have selected for export. For example, if you export a template, the definition of any parameter or included template referenced in the exported template will only be exported if you select All dependencies.

  6. Click Export to export the list of elements you have selected. The exported file can then be used as input to the Import feature in another PcVue project.

It is important to save your environment regularly by clicking File in upper-left menu then Save.
