Graphic Objects in General

 See also Example Properties Methods Events Applies to 

Graphic is a generic term referring to the HMI drawing elements Poly, Shape, Text, Symbol and Group plus the Bitmap and Metafile objects. The Graphic object type as such does not exist, whilst the Graphics Collection contains Object objects.

The properties that define the appearance of all objects that belong to the Graphics Collection conform to certain rules. All properties that are related to colors follow the standard color property naming scheme.

Pattern style rules

A pattern style is defined using the following conventions:

  • A background color: the color used to fill the object.

  • A pattern style: the pattern style used to fill the object (Solid, 45° Diagonal Hatching, 135° Diagonal hatching, Diagonal Cross Hatching, Horizontal Hatching, Vertical Hatching, Cross Hatching, Transparent).

  • A hatching color: the color used to draw hatching.

Line style rules

A line style is defined using the following conventions:

  • A foreground color: the color used to draw the lines of the object.

  • A line style: the line style used to draw the lines of the object (Solid, Dashed, Dotted, Dash Dot, Dash Dot Dot, Invisible).

  • A line thickness: the line thickness used to draw the lines of the object.

Appearance rules

Appearance is defined using the following conventions:

  • An appearance: the appearance used to draw the object (Regular, Shadow, Button, colored Button, Relief, Inverse Relief).

  • A bottom-right and a top-left color: the colors used to draw the bottom-right and the top-left edges of the object when its appearance is not set to Regular.

  • An appearance thickness: the thickness of the edges of the object when its appearance is not set to Regular.

Text color rules

Text colors are defined using following conventions:

  • A foreground color: the color used to draw the text contained in the text object.

  • A background color: the color used to fill the area defined by the including rectangle of the text object.

Standard color property naming

Each color used in an object can be fixed, indexed, transparent or blinking. Thus multiple properties are needed to define a single color. The names of these properties are constructed in a uniform manner using different but consistent suffixes. These suffixes are:

  • Primary: Suffixed property returns or sets the OLE_COLORcode used for primary color (information returned or set is valid only if Primary Transparent suffixed property is set to False).

  • Alternate: Suffixed property returns or sets the OLE_COLORcode used for alternate color (information returned or set is valid only if Alternate Transparent suffixed property is set to False).

  • Primary Index: Suffixed property returns or sets the index of the indexed color cell used as primary color. Indexed color cells refer to the ones that can be found in the advanced part of the palette. (Primary Index suffixed property can vary from 1 to 32 if the chosen color cell is indexed, if the chosen color cell is fixed, Primary Index suffixed property is set to 0.)

  • Alternate Index: Suffixed property returns or sets the index of the indexed color cell used as alternate color. Indexed color cells refer to the ones that can be found in the advanced part of the palette (Alternate Index suffixed property can vary from 1 to 32 if the chosen color cell is indexed, if the chosen color cell is fixed, Alternate Index suffixed property is set to 0).

  • Primary Transparent: Suffixed property returns or sets the transparent property that indicates that the primary color is transparent (True) or not (False).

  • Alternate Transparent: Suffixed property returns or sets the transparent property that indicates that the alternate color is transparent (True) or not (False).

  • Blink: Suffixed property returns or sets the index of the blinking frequency used for this color. (The information is valid only if Primarysuffixed property differs from Alternatesuffixed property.)

Events are fired only if the EnableEvents property of the underlying object has been set to True. This happens automatically once the first event script has been associated with the object.
