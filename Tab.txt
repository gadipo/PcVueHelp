Configuring the tab control
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring the Tab Control
 
The Tab Control is enabled by ticking the Enable
 Tab Control box in the Tab Control tab of the Window Properties
 dialog. Show picture
 

 

 
 
 
 
Configuring the size and position of the tab control
 
The size and location of the tab control are determined by the following properties.

 
 
X Position and Y Position - The location of the top left hand corner of the tab, in pixels, relative to the top left hand corner of the container mimic. 

 
Height- The height of the tab control, including the tab itself, in pixels.

 
Width - The width of the tab control in pixels.

 
 
Since the tab control does not have to cover all of the container mimic, you can place drawing elements (including controls) in any unused part of the container mimic. Show picture
 

 
 
 
Automatically adjusting the size of the tab control according to the size of the container mimic

 
Using the Anchor Right and/or Anchor Bottom tick boxes, you can anchor the right and/or bottom of the tab control to the right and/or bottom borders of the container mimic. If the mimic is resized, the size of the tab control automatically follows. The Margin Right and Margin Bottom properties can be used to configure a margin between the tab control and the mimic. These properties are only enabled when the corresponding Anchor property is ticked.

 
When you tick Anchor Right the Width property is disabled. When you tick Anchor Bottom the Height property is disabled.

 
Configuring the icon size and text font
 
The Icon width and Icon height properties set the size of the icon (if any) displayed in the page's tab. The application scales the icon to suit the configured size. Icons are most commonly 32 x 32 pixels. The Font checkbox enables or disables the selection of a custom font. The Font button is used to display the Font dialog to let you select a custom font.
 When the custom font feature is not enabled, the system font is used.

Overview of the mimic tab control
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Overview of the Mimic Tab Control
 
The Mimics Tab control is configured from the Tab Control 
 tab of the Window Configuration dialog. It enables you to define multiple 
 pages in a mimic, which, at run time, are selected by tabs in a similar 
 way to some configuration dialogs. Show picture
 

 

 
 
 
 
Each page is developed as a separate mimic before you configure the tab control. For example, 
 if you had a mimic with two pages, it would require three mimics in total, 
 one for the container and the other two for the pages. A 
 mimic in a tab behaves as an included mimic but instead of being included in a 
 mimic, it is included in the tab control.
 
When opening a mimic that is using the tab control, PcVue automatically opens the included mimic that was selected as the startup 
 page. Each time a new page is selected by clicking on the corresponding 
 tab, the current included mimic is closed and the other included mimic is opened.
 
Look and feel
 
The area of the container mimic that is occupied by the 
 tab control (i.e. containing the included mimics) is configured by the settings 
 you enter in the Tab Control tab. You can choose to fill the whole of 
 the container mimic or just part of it, select a custom font for the tab captions, etc.
 
Security
 
When a page is selected and the associated mimic is opened, PcVue executes the same security checks as it does when opening 
 a mimic by any other means and checks that the mimic rights level matches 
 that of the current user. If the user does not have the required levels, 
 the included mimic is not opened and the page is shown empty.
 
Using page mimics with a branch
 
You can use the included mimics with a branch in the same as for
 any other mimic. For example, you could configure the tab control to open 
 the same mimic on each page but with a different branch.

Selecting the mimics for tabs
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Selecting the Mimics for Tabs
 
You select the mimics to be shown in tabs via the lower part of the Tab Control tab in the Window Properties dialog as follows. Show picture 
 
You can configure as many tabs as can be displayed 
 across the top of the container mimic.
 
You can left-click on a property to open an edit control for modifying its value.
 

 

 
 
 
 
 
Page - The page number 
 of the included mimic. This setting is allocated each time 
 a new page is added and is used to configure the included mimic that is first 
 displayed when the container mimic is opened.
 
Mimic - The name 
 of the mimic. The mimic name may be typed in directly or selected from 
 the mimic selector dialog. Substitution strings such as #I, #U and 
 #Mn can be used. They are processed in the same way as it would 
 in a Link Open animation.
 
Mimic branch - The 
 variable branch to be used with any animations in the mimic. The branch 
 name may be typed in directly or selected from the Branch Selector dialog. 
 Substitution strings such as #B and #WB can be used. They are processed in the same way as it would in a Link-Open animation.
 
Page caption - The 
 caption that will appear in the tab corresponding to the mimic page. The 
 size of the tab automatically adjusts to accommodate the length of the 
 string. Substitution strings such as #B, #W and #WN can be used. Substitution strings starting with #W are interpreted as information 
 about the mimic in the tab, not the container mimic.
 
Icon - Optional. An icon to be displayed on the tab. The icon file must be in the standard (.ICO) Windows icon format. Only icons that have been saved in the bitmap folder (BMP) of PcVue's libraries can be used. The icon is scaled to fit Icon width and Icon height properties.
 
 
For more information, on substitution strings, see the 
 topic Using Substitution Strings or Variable Attributes in an Animation.
 
If any of the #M (#M1 to #M20) substitution strings 
 are used for the mimic name, the corresponding branch and label are used 
 for the Mimic 
 branch and Page 
 caption respectively. See the topic The 
 Links tab for more information.
 
How to add a new page to the 
 list of included mimics
 
To add a new page to the end of the page list, left-click 
 New 
 page to select the line and enter the properties.
 
To add a new page in the middle of the page list right 
 click the line immediately below the position for the new page and select New from 
 the context menu. 
 Show picture
 

 

 
 
 
 

 
The context menu also contains commands to copy, paste, 
 cut and delete lines.
 
 
Selecting the included mimic that is first displayed when 
 the container mimic is opened
 
The Startup 
 page drop-down list box selects the first page mimic that is displayed 
 each time the container mimic is opened.
 
Configuring the fitting style for an included mimic
 
These properties specify how the page mimics appear within 
 the tab control.
 
 
Position - The Position 
 X and Position 
 Y properties of each included mimic are automatically adjusted so that 
 the top left hand corner of the included mimic is located at the top left 
 hand corner of the tab control mimic area and the mimic does not overlap 
 the page tab itself. The Position 
 property should be ticked in most, if not all, applications.
 
Size - The included mimic's 
 Width and Height are automatically adjusted to fit the tab control mimic 
 area.
 
Control - The included 
 mimic's style is modified to disable display of the border and title bar 
 so only the content of the mimic is displayed.
 
 

 

 &#160;PcVue dynamically modifies some properties of 
 the included mimics to accommodate the fitting styles. If an included mimic is 
 saved, the changes will be saved and the original settings will be lost.
