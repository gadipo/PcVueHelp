3D Camera Model Settings

You can modify the settings of a camera model in these ways:

  • Directly through the current settings of the window camera.
  • Using the model's Properties dialog.

Updating

You can quickly update the settings of a camera model as follows:

  1. In Design mode, orient the scene in the HMI so that the required camera is visible and then select it. Show picture

    Here the upper camera is selected.

  2. In the 3D Draw toolbar, select the command Set Camera Target. Show picture

The command Set Camera Target is only available when a single camera model is selected.

When you use the command, the selected camera model memorizes the current settings of the window camera. When you switch to it, it will provide that point of view.

The Properties dialog

The Properties dialog of a camera model includes two tabs.

  • The 3D Base tab is the same that you would have in any 3D model. See the book on The 3D Properties Dialog for more information.
  • The 3D Camera tab contains properties specific to behavior of a camera. Show picture

To define the camera model with a command behavior allowing direct command of the window camera.

When a camera model is visible and set for selection, a left mouse click on the camera model will set the window camera with the settings of that camera model.**

* The controls are disabled if the parameter is locked (see Locking a Parameter below).

** This only works when the window is in Run mode.

Except in the First Person view mode, when you modify one parameter the HMI always re-calculates the other parameters automatically. The re-calculation is necessary so that coherent parameter values are always displayed. For example, if you modify the Distance parameter, the HMI must re-calculate the Eye or Target parameters.

Locking a parameter

When an Anchor is ticked, it indicates that the associated parameter (Eye, Target or Distance) is locked. There is always one (and only one) parameter locked at a time.

When a parameter is locked, its values cannot be modified by the HMI when it re-calculates the parameters (following the modification of one parameter by the user).

For example, if the Target parameter is locked and you modify the Distance, the HMI re-calculates the Eye parameter.

Differences in behaviors within views

Certain parameters behave differently, depending on the camera view mode that is selected: Eye, Target, Distance and Angle. The following tables describe these parameters' behaviors for each camera mode (view mode).

Top View and Bottom View modes

Show table

Parameter

Description

Eye (X, Y & Z)

No changeable; calculated by the HMI according to the Target and Distance parameter values and the view mode.

Target (X, Y & Z)

To change the X and Z values of the camera focus point in the 3D world.
The Y value is not changeable and is always set to zero.*

Distance

To change the distance between the camera eye and the camera target.

Angle (Azimuth & Pitch)

Not changeable. The azimuth and pitch are determined by the HMI** according to the view mode:

    • Top View: Azimuth and pitch are 0° and 90° respectively.

    • Bottom View: Azimuth and pitch are 0° and -90° respectively.

* The origin (zero) of the world axis that is parallel to the aiming axis of the camera.

** The HMI determines the azimuth and pitch values so as to direct the aiming axis of the camera (model's Z axis) toward, and perpendicular to, the view mode projection plane.
