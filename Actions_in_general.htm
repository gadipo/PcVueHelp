Understanding actions

Whilst the types of action available to the SnapVue client are very different in functionality, the same basic procedure is used for configuration in PcVue and they share some common behavior. For how to add an action to a zones see the topic How to add and configure a zone.

Available action types

The types of action available is the following, they offer a variety of possibility for the user to visualize information and send commands:

  • Send command - Allow the user to send a command or a set-point for a given variable. The value sent can be pre-configured or entered by the user.
  • Variable details - Display detailed information to the user for a variable, including timestamp, status and extended attributes.
  • List of variables - Display a list of variables and their values to the user.
  • Trend - Display a trend style view of up to 16 variables to the user.
  • List of alarms - Display a list of eligible alarms to the user.
  • List of logged events - Display a list of events from a selected Log List to the user.
  • Symbol - Display a graphic symbol selected at design time to the user.
  • Mimic - Display a mimic selected at design time to the user.
  • Execute program - Allow the user to execute a SCADA Basic script on the server.
  • Open Url - Allow the user to browse to a specific Url in an embedded web browser.
  • Open chat - Allow the user to open an existing chat, or create a new one.

Whilst they provide similar functionality, the trend view, log view and alarm list view are not the same as the equivalent graphic controls in PcVue desktop client or WebVue.

Mimic and symbol rendering in mobile apps is similar to that of WebVue except for the following restrictions.

  • Mimics are displayed full size and cannot be scaled to fit the device screen. However, you can scroll a mimic using touch and drag.
  • The Link-open animations are not executed, other than to open a single pop-up mimic.
  • The Link-close animations are not available to the user.

For detailed information on the available mimic functionality, see the book Configuring a project for WebVue.

Important note about the variables filter

All action types other than List of logged events and Open Url, inherit the branch from the zone to which it is attached (if any). The branch is known as the Context and allows filtering / selecting the variables available to the action. Show picture



Some actions also have an optional local branch. Any local branch is appended to the context branch unless it is preceded with the @ character which makes it absolute. For example:

  • If the context is BMS.B01 and the local branch is Floor01, the branch filter is BMS.B01.Floor1.
  • If the context is BMS.B01 and the local branch is @BMS.B02, the branch filter is BMS.B02.

Some actions, namely Send command, Variable details and List of variables, also inherit any Sql expression from the zone. If a local Sql expression is configured it is ANDed to any branch filter.

The context branch is shown in the action but any inherited SQL expression is not.

Action visibility

The actions visible to the user of the SnapVue mobile app primarily depend on what has been configured in the PcVue server for the zone in which the user is detected. However, the following conditions must also be met.

  • Browsing level - The user, having been detected as being in a zone, must have the Browsing Level rights in their User Profile corresponding to that of the action.
  • Fully defined parameters - The parameters of the action must be fully defined.
    • Open Url - The Url must be filled in.
    • Variable details - A single variable is found.
    • Trend and List of variables - At least one variable is found.
    • Symbol - The Symbol name must be filled in.
    • Mimic - The Mimic name must be filled in.
    • Execute Program - The Program name must be filled in.
    • List of logged events - The Log list name must be filled in.
    • List of alarms - Always displayed, if no alarm matches the filter, the list of alarm is empty.
    • Send command - A single variable is found.

In general, if no matching variable is found in the zone in which the action is defined, it will not be visible to a user located at that zone.

The Tiles tab

The tiles tab configures the visual representation of each action on the SnapVue home view. Show picture

  • Miscellaneous
    • Browsing level - The browsing level, corresponding to that of a user's profile, that the user must have if the tile is to be displayed.
  • Visual representation
    • Priority - The priority determines the position of the tile on the page. Tiles with a higher priority are positioned to the top and left of those with a lower priority. Range 0 to 100.
    • Title - Specifies where to display the title on the tile (default On the left). Select Do not display if you do not want to display the title.
    • Size - Area occupied by the tile. In the following screen-shot all tiles are 2 x 2 which is the default. Show picture

    • Type
      • Image - The tile displays an image selected from a library. Images are automatically re-sized to fit the space available in the tile.
      • Symbol - The tile displays a symbol selected from a library. Symbols are automatically re-sized to fit the space available in the tile. Note that any control zone in the symbol is disabled. Tapping the symbol displays the tile's action.
      • Variable and variable value - The tile displays the value of the selected variable. It is also possible to select images as for the Image on register animation. If such a configuration is performed, the image is displayed on the side of the tile, along with the variable value.
      • Image on bit - The tile displays one of three selected images depending on the status and value of a selected bit variable (0, 1 or NS).
      • Image on register - The tile displays one of up to eight selected images depending on the value of a selected register variable. Additional images can be selected for display when the value is outside of the range or when it is not set (NS).
    • Text color - The color of any displayed text. The default is black.
    • Background color - The background color, either solid or gradient. The default is solid white.

Colors are selected from a palette or entered as a hexadecimal code, for example #FF000000
