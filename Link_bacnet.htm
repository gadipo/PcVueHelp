Linking a Variable to a BACnet Object, Property or Event/Alarm

There are two methods by which devices are added to PcVue's BACnet network configuration.

  • If you are connected to the BACnet network, you can scan the network and select the devices from those that are discovered. These are known as Classic devices.
  • You can load one or more EDE files containing the BACnet configuration. Devices can be selected from the contents of the EDE files. These are known as EDE devices.

The configuration required to link a variable to a classic device or an EDE device is different.

You can also link a variable to a BACnet object property using the BACnet Mapping dialog. See the topic on BACnet Mapping in the book on Data Acquisition for more information.

How to link a variable to a BACnet object from a classic device

  1. Display the variable's Source tab.
    1. In the Application Explorer expand the variables tree and select the variable.
    2. Select the Properties task. The Variable Properties dialog opens.
    3. Select the Source tab.
  2. Select the BACnet network, Device, Object-Type and Object Identifier (instance).
    1. Using the Source list box select a BACnet network. A BACnet network is referenced using the alias that was entered when it was configured in PcVue. Once you have selected a network as the source the BACnet tab appears in the dialog.
    2. Using the Device list box select the BACnet device. The Type field will indicate if it is a Classic or EDE device. The rest of this explanation assumes it is a Classic device. Show picture
    3. Using the Object type and Object identifier drop down list boxes select the object type and instance.
  3. Select the Variable type.
    1. To link the variable to the object's real time value this property can be left at the default setting of Object.
    2. To link the variable to a specific property use the drop down list box to select it. The properties that are available will depend on the variable type (bit, register or text) and the Object type previously selected.
    3. To link the variable to an Alarm/Event use the drop down list box to select it. For binary objects there will only be one choice Alarm/Event. For analog objects there are two choices Alarm/Event - High and Alarm/Event - Low.
  4. Select any advanced options.
    1. Select any advanced properties using the Advanced tab. See the topic Advanced properties specific to BACnet variables for details.
  5. Select OK to confirm the configuration and close the dialog.

How to link a variable to a BACnet object from an EDE device

  1. Display the variable's Source tab.
    1. In the Application Explorer expand the variables tree and select the variable.
    2. Select the Properties task. The Variable Properties dialog opens.
    3. Select the Source tab.
  2. Select the BACnet network and Device.
    1. Using the Source list box select a BACnet network. A BACnet network is referenced using the alias that was entered when it was configured in PcVue. Once you have selected a network as the source the BACnet tab appears in the dialog.
    2. Using the Device list box select the BACnet device. The Type field will indicate if it is a Classic or EDE device and the EDE file field will show the name of the file from which the device configuration was imported. The rest of this explanation assumes it is an EDE device.
  3. Select the Keyname.
    1. Using the Keyname list box select the appropriate keyname. The corresponding object type and instance are displayed in the rear-only field below it. See the note below for more information on keynames. Show picture
  4. Select the Variable type. This is the same as for classic devices.
  5. Select any other advanced options in the Advanced tab. See the topic Advanced properties specific to BACnet variables for details.
  6. Select OK to confirm the configuration and close the dialog.

About the Keyname property

The Keyname is a unique identifier for a BACnet object instance. It is only found in EDE files - it is not a property that can be retrieved from the BACnet network. PcVue uses the combination of the EDE file name and the Keyname to identify the BACnet object instance.

About the Alarm/Event variable type

If you select the Variable type as either Alarm/Event, Alarm/Event-High or Alarm/Event-Low, then the variable is configured to reflect the status of the BACnet Object's Alarm/Event Notification. At this stage, PcVue does not check that the configuration of the Alarm/Event notification of the BACnet object in the device is correct. To assist the developer to check the configuration, an Alarm Helper dialog can be displayed. The Alarm Helper dialog does not modify the configuration, but it does display the status (valid, warning or not checked). For further information, see the BACnet book in the Data Acquisition help.
