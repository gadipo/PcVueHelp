File Example 5 - Use of the ChildOf property

Example of XML file generation using the Application Architect using the File and File Item configuration elements.

This topic describes 2 usages of the ChildOf property to generate XML files where the XML hierarchy of elements does not match the topology of instances:

  • The first file shows how to generate a flattened XML file,
  • The second file shows how to group child elements together.

Desired output - File 1: The flattened list of buildings and floors

<?xml version="1.0"?>
 
<RootXmlFlat>
 
<FlatXml_ListOfBuildingsAndFloors>Building01</FlatXml_ListOfBuildingsAndFloors>
 
<FlatXml_ListOfBuildingsAndFloors>Building01.Floor01</FlatXml_ListOfBuildingsAndFloors>
 
<FlatXml_ListOfBuildingsAndFloors>Building01.Floor02</FlatXml_ListOfBuildingsAndFloors>
 
<FlatXml_ListOfBuildingsAndFloors>Building01.Floor03</FlatXml_ListOfBuildingsAndFloors>
 
<FlatXml_ListOfBuildingsAndFloors>Building02</FlatXml_ListOfBuildingsAndFloors>
 
<FlatXml_ListOfBuildingsAndFloors>Building02.Floor01</FlatXml_ListOfBuildingsAndFloors>
 
<FlatXml_ListOfBuildingsAndFloors>Building02.Floor02</FlatXml_ListOfBuildingsAndFloors>
 
</RootXmlFlat>

Desired output- File 2 - With 2 separate collections - Buildings and Floors

<?xml version="1.0"?>
 
<RootXmlGrouped>
 
<Buildings Count="2">
 
<FlatXml_CollectionsOfBuildingsAndFloors>Building01</FlatXml_CollectionsOfBuildingsAndFloors>
 
<FlatXml_CollectionsOfBuildingsAndFloors>Building02</FlatXml_CollectionsOfBuildingsAndFloors>
 
</Buildings>
 
<Floors Count="5">
 
<FlatXml_CollectionsOfBuildingsAndFloors>Building01.Floor01</FlatXml_CollectionsOfBuildingsAndFloors>
 
<FlatXml_CollectionsOfBuildingsAndFloors>Building01.Floor02</FlatXml_CollectionsOfBuildingsAndFloors>
 
<FlatXml_CollectionsOfBuildingsAndFloors>Building01.Floor03</FlatXml_CollectionsOfBuildingsAndFloors>
 
<FlatXml_CollectionsOfBuildingsAndFloors>Building02.Floor01</FlatXml_CollectionsOfBuildingsAndFloors>
 
<FlatXml_CollectionsOfBuildingsAndFloors>Building02.Floor02</FlatXml_CollectionsOfBuildingsAndFloors>
 
</Floors>
 
</RootXmlFlat>

Topology

Project template

The Project template includes 2 files:

  • For file 1: FlatXml_ListOfBuildingsAndFloors
  • For file 2: FlatXml_CollectionsOfBuildingsAndFloors. 2 File items are defined, so that 2 Xml elements are generated in the output file. One is for the XML element corresponding to the collection of Buildings, the other for the XML element corresponding to the collection of Floors. You can notice the CountAttribute file item to generate the XML attribute for the element count in the respective collection.

File element on Project template for file 1 - FlatXml_ListOfBuildingsAndFloors:

Note the Tags to include and Root element, which will be referenced in File items.

File element on Project template for file 2 - FlatXml_CollectionsOfBuildingsAndFloors:

Note the Tags to include and Root element, which will be referenced in File items.

File items for the Buildings and Floors collections:

These 2 file items are direct children of the File itself. The topology hierarchy and the desired XML file hierarchy match, therefore the property ChildOf can be left empty.

Building and Floor templates

Both are designed in the same way, they include 2 file items:

  • For file 1: FlatXml_ListOfBuildingsAndFloors
  • For file 2: FlatXml_CollectionsOfBuildingsAndFloors.

File element on the Building and Floor templates for file 1: FlatXml_ListOfBuildingsAndFloors

Note the Tags to include and Root element. We reference the desired root XML element name in the ChildOf property to flatten the XML file.

File element on the Building template for file 2: FlatXml_CollectionsOfBuildingsAndFloors

Note the Tags to include and Root element. We reference the desired Buildings XML element name (root of the Buildings collection) in the ChildOf property to group buildings together in the XML file.

File element on the Floor template for file 2: FlatXml_CollectionsOfBuildingsAndFloors

Note the Tags to include and Root element. We reference the desired Floors XML element name (root of the Floors collection) in the ChildOf property to group floors together in the XML file.

The same logic and design principles apply if you want to model and generate a file of type Predefined format for a tree-view.

The only difference is that, where you reference the XML element name in the ChildOf property for an XML file, you will reference the name of a File element or the name of a File Item element in the case of the Predefined format.
