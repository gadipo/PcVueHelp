How to configure the BIM control

How to insert a BIM control in a mimic

  1. Open the mimic in which the BIM control is to be inserted and select Design mode.
  2. Select Insert.BIM control from the menu.
  3. A BIM control is inserted, and its properties dialog opens automatically. Show picture
  4. Click OK to close the dialog and create a BIM control with default properties.

You can specify more properties to control the graphical element visibility. From the Aspect tab, behind the down arrow at the bottom of the Properties dialog box:

  • Specify zoom visibility bounds so that the graphic control is only visible for predefined zoom levels.

  • Specify layers so that the graphic control is only visible if one of the selected layers is displayed.

These are even more useful when combined with user permissions and decluttering templates.

Once you have created a BIM control, and closed its properties dialog, it can be moved, sized and copied the same as any other drawing element. The BIM control configuration dialog contains two further tabs General and Advanced.

The General tab

  • .Ifc model - The name of the .Ifc file that the BIM control displays when the mimic is opened. It is recommended that BIM files are located in the project's BIM folder, although the file browser displayed by clicking the down arrow button adjacent to the .Ifc file field allows selection of any location accessible to the current user.
    The .Ifc file property can be left empty in which case the file must be specified using SCADA Basic.

  • Configuration file - The name of the .dat file in which PcVue will save the model-specific configuration (including animation, cut planes and cameras). It is recommended that files are located in the project's BIM folder, although the file browser displayed by clicking the down arrow button adjacent to the Configuration file field allows selection of any location accessible to the current user.
    The Configuration file property can initially be left empty, and specified when you add and save the first configuration elements.

  • Edit - Opens the BIM configuration editor and loads the contents of the .Ifc file to allow the configuration of animations, plane and camera presets, for the BIM control. If the name of a configuration file has already been entered, any configuration it contains is loaded.

  • Save - Save the current configuration in the specified configuration file.

  • Load - Load the configuration from the specified configuration file.

The Advanced tab

  • File settings

    • Enable geometry cache file - Enable the use of the XBIM cache file when loading Ifc files. If not enabled, no cache of the mesh will be generated and if a cache file already exists, it is not loaded. Enabling caching boosts rendering performances at run-time.

  • View

    • Simplified rendering - Simplify rendering of the 3D meshes of the model in order to increase performances.

    • Display FPS - Display, in the top right corner of the BIM control, the number of frames per second that are being rendered at run-time.

  • Camera

    • Enable pan - Enable pan at run-time.

    • Enable zoom - Enable zoom at run-time. Sensitivity defines how the model responds to zoom requests.

    • Enable rotation - Enable rotation at run-time. Sensitivity defines how the model responds to pan requests.

  • Ifc properties window

    • Enable - Enable display of the Ifc properties window. The default keyboard shortcut Tab can be changed using the Accelerator property.

    • Accelerator - The keyboard shortcut for the user to open the Ifc properties window at run-time. Click into the field and then press the keystroke to be used (or the Del key to remove the configured accelerator).

Loading a model for the first time typically takes longer because, if enabled, this is when the cache is generated.
While the cache is created and managed automatically, it is only valid for the computer where it was generated. When deploying a project with a mimic that contains a BIM control in a multi-station system, you may notice the cache creation delay upon mimic first opening on each station.

The Ifc properties window is not modal. You can keep it open, it is refreshed upon object selection change.

The Traces tab

The Traces tab is used to enable trace messages from the BIM control. Useful when diagnosing errors or if you want to track the BIM control use at run-time. Trace messages appear in the Trace window and are logged in the trace file. The following messages can be generated.

  • System errors - If enabled, a trace is logged when an unexpected error such as a processing or rendering issue occurs.

  • Verbose - All of the above plus additional information.

  • Activate on mimic open - Activate the selected traces on mimic open. If enabled, the selection of traces is persisted and applied each time the mimic is open.

By default, only system errors are logged.
