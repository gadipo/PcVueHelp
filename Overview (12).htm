Overview of OLE

Using OLE (object linking and embedding) you can include information created in other applications, such as a spreadsheet or word processor, in the HMI’s windows. The included information is known as an Object. Show picture

The main difference between linking and embedding is where the data is stored.

  • Embedded objects become part of the window's source file.
  • Linked information, remains stored in the original source file. The window contains only the location path of the file whilst still displaying a graphic representation of the linked data.
