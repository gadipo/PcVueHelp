Using BACnet Event Log Objects

In a BACnet network, a device is able to record state changes of an object, locally within the physical BACnet device. The mechanism is often referred to as Event Logging.

Event Logging is managed by the Event Log object within the BACnet physical device. The object's properties define how logging is controlled.

  • Event Logging at the device level. What, and how, properties are recorded.
  • The conditions that lead to informing other devices that logs are available for retrieval.

There is no generally accepted way to identify which state change is handled by one or the other Event Log, and the event log (as per the standard), once retrieved, does not include enough information to match a certain PcVue log list.

As a consequence, at the time of writing, PcVue is able to retrieve Event Logs (state changes and timestamps), but does not push them to its own event logging subsystem for archiving.

As a consequence, BACnet Event Log can only be retrieved by using the Helpers for testing.
