Exchanging Images

Simple changes in process status can be indicated by change of color. For example, a pump may be displayed in green when it is running, red when it is stopped and blue when faulty. However, in some cases, changing color is not adequate. Consider the display of a float switch. The requirements are that it should appear horizontal when its corresponding bit in the variables tree is true, and at an angle of 45° when the bit is false. This can easily be achieved by using two images exchanging them depending on the state of the bit. In practice, exchanging images can create many visual effects.

The image animations all require an image as the support drawing element.

You can also use the Visibility animations with images.

Anchor point

When exchanging images the anchor point is the top left hand corner. The anchor point of the support image is not used.

Using images of different sizes

By default, images displayed using the image animations are displayed in their original size. Each of the image animations has a property called Stretch images. Selecting this will cause all images to be automatically re-sized to match that of the support image.

Specifying an image for invalid Status

On all the image animations, you have the option to select an image that will be displayed when the variable used in the animation becomes invalid. A variable can become invalid for a variety of reasons, the most common one being the loss of communication to external equipment such as a PLC or RTU.

If you do not use this option, when a variable becomes invalid the animation will display the image representing the last valid value of the variable. The user will then be unaware that they may be viewing out of date information.

It is highly recommended that you make use of this option.
