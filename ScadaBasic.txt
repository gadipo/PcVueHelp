Configuring a function key
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring a Function Key
 
 
Creating a function key and configuring the general properties.
Select the Function Keys folder in the Application Explorer configuration tree and, from the task list, select Add function key. A dialog to configure the new function key will open. Show picture

Enter a description of the function key. The description is optional and is only displayed in the Application Explorer.
Select or enter the name of a 
 bit to enable or disable the function key. The bit must be set to 1 
 for enable. The bit is optional.
Tick the Enable 
 box. It must be ticked for the function key to be enabled.
 
Select the Trigger tab and configure the trigger.
 Select the key using the drop down list box.
Select any modifiers (Shift and / or Control) to be used in conjunction with the key.
Select the key event on which the action is to be triggered (Up or Down).
 
Select the Action tab and Select the action type from either Standard Action, Function to Execute or Send Animation.
 
If triggering a Standard action then:
 Select the action using the drop down list box.
 
If triggering a Execute a function then:
Enter the name of the SCADA Basic program that 
 is to run. The ellipsis button adjacent to the Program name field displays a list of available programs. The program must exist or you will not be able to create the function key.
Enter a variable branch for use with the program. The branch is optional.
Enter the name of the function, within the program, to run. 
 If you do not specify a function, the Main function will be executed.
Enter a comma'separated list of arguments for the function. The 
 arguments can be retrieved in the function using the GETARG instruction. The list of arguments is optional.
 
If triggering a Send animation then:
Enter the name of the mimic that contains the animation. The ellipsis button adjacent to the Mimic field displays a list of available mimics. The mimic must exist or you will not be able to create the function key.
If the mimic requires a variable branch, then select or enter it.
Enter the name of the animation. The animation name can be found using the Graphic Explorer when the mimic is open.
 
Click OK 
 to confirm the configuration and close the dialog.
 
 

 
&#160;PcVue does not warn you about, or prevent you from, configuring more than one action on the same key / modifier / event combination. If you do that, all configured actions are triggered when the key is used.

Function keys overview
 

 
 
 
 
 
 
 
 
 
 
 
Function Keys Overview
 
The action of many of the keyboard keys can be configured by the developer, including function keys. This is particularly useful if, at run time, the host PC has to operate without a mouse or other pointing device. Keyboard shortcuts can be configured using either the Application Explorer or the SCADA Basic instruction KEY. This book explains how to configure shortcuts using the Application Explorer. There are over 30 keys that can be configured and most can be given more than one action by using them in conjunction with the Shift and / or Ctrl keys. In addition an action may be triggered when a key is pressed down and / or when it is released. 
 
A key action can be one of three types.
 
 
Standard action. An internal, predefined action such as to display the on-line Help or show the Configuration Menu. Several of the keys are pre-configured with standard actions when a project is first created.
 
Execute a function. A function from a SCADA Basic program may be executed. The possibilities in this case are endless, but a typical use would be to open a window.
 
Send an animation. Reproduce the action of an operator clicking on a Send type animation in a window.

Changes in function key management from version 12 onwards
 
Prior to version 12 the default configuration of the function keys with pre-configured standard actions was created automatically each time PcVue started. Any changes made as part of the project configuration were overwritten. This meant that if you wanted to delete or change the function of any of these keys permanently, it had to be done using a SCADA Basic program arranged to run each time the software was started.
 
From version 12 onwards, the default configuration of the function keys with pre-configured standard actions is created once only when the project is first created. Therefore, any changes made subsequently, as part of the project configuration, are permanent.
 

 
If you start a project created with a version prior to 12 the behavior is the same as if you were creating a new project. That is the default configuration of the function keys with pre-configured standard actions is created once only. A SCADA Basic program to delete or change the function of these keys is no longer necessary.

Using the SCADA Basic instruction EXPORT
 

 
 
 
 
 
 
 
 
 
 
 
Using the SCADA Basic Instruction EXPORT
 
If you want to automatically generate an export (without intervention of a user) then you must use a SCADA Basic script incorporating the EXPORT instruction. The EXPORT instruction is straightforward to use and has three modes.
 
 
GENERATE - Generates an Export using the default settings.
 
GENERATE_DATES - Generates an Export using specific start and end times.
 
GENERATE_PERIOD - Generates an Export using a reference time and a specific period.
 
 
There are unlimited possibilities in the way SCADA Basic can be used to generate an export but the most common are as follows. 
 
 
To generate an export on the change of a variable use the Event mechanism to run the SCADA Basic script when the expected transition occurs on a variable or alarm.
 
To generate an export according to a configurable schedule use the Scheduler mechanism to run the SCADA Basic script (typically a recurring event).
 
To generate an export using a mimic button use the Run-Program animation to run the SCADA Basic script.
 
 
For more information on SCADA Basic, see the Programming Languages help book.

The following script was written to allow the properties for the Export to be passed as arguments.
