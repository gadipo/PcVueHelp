Using the context menu
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Using the Alarm Viewer Context Menu
 
The Alarm Viewer context menu is available if the property Use context menu has been enabled in the Alarm Viewer's configuration. It is opened by right clicking anywhere in the alarm line display area.
The available tools depend on the configuration of the alarm viewer (Menu tab), and if one or more alarms are selected. Commands available in the context menu correspond to those available in the toolbar, and have the same behavior.
 
The following screen shot shows all available tools. Show picture
 

 
 
 
The tools are divided in 3 sections:
 
 
Filtering - The 1st section comprises buttons to change the alarm display filter. Any change in the filter applies to the alarm viewer as a whole.
 
User actions - The 2nd section comprises button to control the alarm status: Acknowledge, mask... Unless otherwise stated, the action applies to all selected alarms.
 
Commands - The 3rd section comprises general command buttons: To change the mode or print alarms.
 
 
Interaction with other Alarm Viewer actions
 
There are two other forms of actions from the Alarm Viewer triggered by clicking with the mouse. How these behave will depend on how Mouse inputs have been configured in the HMI General Options. 
 
 
Associated actions - If associated actions are configured for a right mouse click, and the context menu is enabled, on a right mouse click they appear in the context menu above the other tools.
 
Context mimic - If the context mimic action is configured for a right mouse click, and the context menu is enabled, on a right mouse click the name of the context mimic appears in the context menu above the other tools.
 
 

 
The context mimic and the context menu are two entirely separate features other than the interaction mentioned above.

Overview of the Alarm Viewer at run-time
 

 
 
 
 
 
 
 
 
 
 
 
Overview of the Alarm Viewer at Run-time
 
The Alarm Viewer displays, in a scrollable list, the 
 current (real-time) status of all alarms that pass its filter criteria. 
 The filter can operate on the Domain and Nature, alarm level and alarm 
 status. The filter is initially configured during development and may 
 also be changed at run-time should the configuration allow it.
 
Alarms are special bit variables that have the alarm 
 property enabled. An alarm may be generated from either the 1 or 0 state 
 of the bit and is given a level of 0 to 29. An optional masking bit or 
 alarm may also be selected. A bit with alarm attributes behaves differently 
 to a standard bit in that, instead of having three states (true, false 
 and invalid), it has five.
 
 
Off.
 
On and not acknowledged.
 
On and acknowledged.
 
Off and not acknowledged.
 
Unavailable (a general 
 term for when an alarm is not operating normally for a variety of reasons).
 
 
The colors in which alarms appear are defined by the 
 Alarm on Text tab of the Preferences.Colors dialog box. In normal mode, 
 only the foreground (font) color is used, the background color being 
 taken from the Alarm Viewer itself. In advanced mode, both the foreground 
 and background colors are used.
 
Selecting Alarms
 
Individual alarms are selected by clicking on the relevant line 
 in the display list. A selected alarm is indicated 
 by displaying it in reverse video (unless the alarm display is in advanced 
 color mode - see below). Selected alarms can be acknowledged, printed, 
 masked or un-masked using the tools on the toolbar or the context menu, and copied to the Windows clipboard using the Ctrl+C keyboard shortcut.
 

 
If you enable the Context property, it will not be possible 
 to select and hence acknowledge, mask or un-mask alarms directly from 
 the alarm display. Instead, clicking on an alarm will open the Alarm Viewer 
 context window.

If you want to change the column width or modify the toolbar,

 directly from the Alarm Viewer.
 
 
Changing the way that the list is sorted
 
Using the properties Enable sorting and Sort modification, you can temporarily sort the contents of the Alarm Viewer using the contents of any column into either ascending or descending order.
 
 
A column containing numeric data only will cause a numeric sort. For example, using ascending sorting, 2 would be before 3.
 
A column containing alphanumeric data will cause an alphabetic sort. For example, using ascending sorting, 20RedCars would be before 3GreenCars. The first character 2, comes before 3.
 
A column containing time and/or date information will cause a time/date sort.
 
 
You can use the Timestamp column as a sort key. The Display new alarms at the top of the list option is ignored if the alarms are sorted according to the timestamp. If multiple lines have the same value in the sorted column (for example when sorting on an extended attribute), the timestamp column is used as a secondary sort criterion.

TOPIC: AlarmViewer_sorting.htm
========================================
Sorting the contents of an Alarm Viewer by column
 

 
 
 
 
 
 
 
 
 
 
 
Sorting the Contents of an Alarm Viewer by Column
 
When in List mode, the Alarm Viewer contents can be sorted according to the contents of any column. By default, it is sorted according to the date and time with the most recent alarms at the bottom of the list. If configured to permit it, the sort direction and/or the column on which the sort is made can be changed at run-time by clicking on the column header.
 
The sorting is intelligent and depends on the contents of the sorted column.
 
 
A column containing numeric data only such as the Priority will cause a numeric sort. For example, using ascending sorting, priority 0 would be above priority 3.
 
A column containing alphanumeric data such as the Description will cause a character sort. For example, using ascending sorting, "BLACK CAR", would be above "GREEN CAR". The first character "B", comes before "G" in the ASCII character set. But "3 BLACKCAR", would be below "1 GREEN CAR". The first character "3", comes after "1" in the ASCII character set.
 
A column containing timestamp (time and/or date) information will cause a time/date sort. For example, using an ascending sort 10:00:13 is above 10:00:20.
 
 

 
By ascending, we mean values get greater from the top to the bottom of the list.

Using the Alarm Viewer in List Mode
 
List mode is indicated by the following icon displayed as the mode tool. 
 
When the Alarm Viewer is opened a list of eligible alarms statuses are displayed. The order in which they appear, and how new eligible alarm transitions are displayed, depends on the Alarm Viewer's configuration (execution tab).
 
What do we mean by eligible alarms?
 
An eligible alarm status or alarm transition is one that passes the filter criteria configured for the Alarm Viewer. See the topic Using the Alarm Viewer filter.
 
How the Execution tab properties affect the operation of the viewer
 
 
Display new alarms at the top of the list - When the Alarm Viewer is opened eligible alarms are sorted in chronological order with the newest at the top. New alarms (that is alarms that do not already appear in the list) are added to the top of the list. The way in which new eligible alarm transitions, for alarms that already appear on the list, are treated depends on the property Move alarms to top when status changes.
If Move alarms to the top when status changes is not set - The line displaying the alarm is updated with the new status and timestamp but remains where it is in the list. The alarms will therefore no longer appear in chronological order. Show picture

If Move alarms to the top when status changes is set - The line displaying the alarm is updated with the new status and timestamp and moved to the top of the list. The alarm statuses therefore remain in chronological order. Show picture

 
Display new alarms at the bottom of the list - When the Alarm Viewer is opened eligible alarms are sorted in chronological order with the newest at the bottom. New alarms (that is alarms that do not already appear in the list) are added to the bottom of the list. The way in which new eligible alarm transitions, for alarms that already appear on the list, are treated depends on the property Move alarms to bottom when status changes.
If Move alarms to the bottom when status changes is not set - The line displaying the alarm is updated with the new status and timestamp but remains where it is in the list. The alarms will therefore no longer appear in chronological order. Show picture

If Move alarms to the bottom when status changes is set - The line displaying the alarm is updated with the new status and timestamp and moved to the bottom of the list. The alarm statuses therefore remain in chronological order. Show picture

 to changes in value of its source or User actions. The unavailable state 
 may be caused by a number of conditions and an alarm will remain in the

In the Alarm Viewer, you can select the unavailable alarms to be displayed

The following conditions can cause an alarm to be unavailable.
 
 
 
 

 
 
Condition/Event (#E)
 
 
 
Cause
 
 
 

 
 
Alarm operator masked
 
 
 
The User has masked the alarm using the Alarm Viewer 
 or it has been masked using the SCADA Basic instruction ALARM.
 
 
 

 
 
Alarm program masked
 
 
 
The alarm has been masked by the SCADA Basic instruction 
 VARIABLE.
 
 
 

 
 
Alarm variable masked
 
 
 
The mask variable, as selected in the alarm variable's 
 configuration, is true.
 
 
 

 
 
Alarm inhibited
 
 
 
The Inhibited property has been selected in the Advanced 
 tab of the variable's configuration or it has been inhibited by script.
 
 
 

 
 
Alarm not accessible
 
 
 
The source of the alarm variable is unavailable. For example, 
 communication to a PLC has failed.
 
 
 

 
 
Alarm expression masked
 
 
 
The expression, as selected in the Advanced tab of the 
 alarm variable's configuration, is true.
 
 
 
 

 
If you select the option to display all alarms that are

Using Variables as alarm counters
 

 
 
 
 
 
 
 
 
 
 
 
Using Variables as Alarm Counters
 
Certain variable names are recognized as having special 
 significance. These variables are automatically updated to provide information 
 about the system operation that would otherwise be unavailable to the 
 User. Included in this category are a range of register variables that may be used to count and display the number of alarms by state, level, 
 Domain and Nature.
 
These variables are not created automatically, they must 
 be created as part of the project configuration by the Developer if they 
 are to be used. The variables and the information that they contain are 
 as follows.
 
Variable names of SYSTEM.<TYPE>.<PRIORITY> 
 format
 
The value of TYPE determines which alarm state is counted.
 
 
 
 

 
 
Type
 
 
 
Meaning
 
 
 

 
 
0 or ACKOFF
 
 
 
Alarm off.
 
 
 

 
 
1 or NOACKON
 
 
 
Alarm on and not acknowledged.
 
 
 

 
 
2 or NOACKOFF
 
 
 
Alarm off and not acknowledged.
 
 
 

 
 
3 or ACKON
 
 
 
Alarm on and acknowledged.
 
 
 

 
 
4 or NS
 
 
 
Alarm invalid (not Set).
 
 
 

 
 
5 or USERMASK
 
 
 
Alarms masked by the operator.
 
 
 
 
The value of PRIORITY determines which alarm level is 
 counted. The range is 0 to 29. Alternatively, ALL may be used to count 
 alarms of all priorities.
 
Variable names of SYSTEM.<TYPE>.<PRIORITY>.<DOM_NAT> 
 format
 
System alarm counters by type, priority, Domain and Nature.
 
DOM_NAT is a string that determines which Domain and 
 Nature is to be taken into consideration. The character used to separate the Domain and 
 Nature is normally an underscore but may be changed using the Project.General 
 Operation box. The wild character * may be used to signify all Domains 
 or all Natures.
 

 
Do not create both versions of an alarm counter register, 
 for example SYSTEM.3.8 and SYSTEM.ACKON.8.
 
 
Examples
 
SYSTEM.3.8 or SYSTEM.ACKON.8
 
A register variable that counts all alarms of level 8 
 that are On and Not Acknowledged.
 
SYSTEM.1.ALL.ZONE1_ELEC or SYSTEM.NOACKON.ALL.ZONE1_ELEC

Using the Alarm Viewer with alarms in maintenance mode
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Using the Alarm Viewer with Alarms in Maintenance 
 Mode
 
Maintenance Mode is used to modify the behavior of an 
 alarm so that equipment maintenance can be taken into account. A typical 
 scenario where maintenance mode might be used follows.
 
 
An alarm occurs due 
 to equipment requiring maintenance.
 
The User acknowledges 
 the alarm and calls the maintenance department.
 
The User puts the 
 alarm in maintenance mode. This action can be logged showing the condition 
 has been recognized by the User. 
 
The maintenance department 
 carries out the required action and resets the cause of the alarm so that 
 the alarm is no longer present (Alarm - off).
 
The User removes 
 the alarm from maintenance mode.
 
 
When an alarm has been set in Maintenance Mode its behavior 
 is modified as shown in the following state diagram. Show picture
 

 

 
 

 
 
The new alarm behavior may be summarized as follows.
 
 
The alarm now has 
 only two possible states, Alarm on - Ack or Alarm off.
 
The alarm is only 
 visible in an Alarm Viewer with the Maintenance option selected.
 
Changes of state 
 of the alarm are no longer logged or printed.
 
 
How to set Maintenance Mode on an alarm
 
An alarm may be set to maintenance mode using an Alarm 
 Viewer by first selecting the alarm and then clicking the Start Maintenance 
 button on the tool bar. An alarm may also be set to maintenance 
 mode using a SCADA BASIC program. An alarm may only be set to maintenance 
 mode when it is in the Alarm On - Ack state.
 
How to clear Maintenance Mode on an alarm
 
An alarm may be cleared from maintenance mode using an 
 Alarm Viewer by first selecting the alarm and then clicking the Stop 
 Maintenance button on the tool bar. An alarm may also be 
 cleared from maintenance mode using a SCADA BASIC program. An alarm may 
 only be cleared from maintenance mode when it is in the Alarm Off state.
 
How to display a list of Alarms in maintenance mode
 
You can display a list of alarms in maintenance mode 
 in an Alarm Viewer with the maintenance property selected. This is selected 
 either at design time by selecting the Maintenance property in the Filter 
 tab, or at run time using the Maintenance button on the toolbar or Maintenance 
 property in the Alarm Filer dialog.
 
Alarms that have maintenance mode selected 
 can be in one of two states, Alarm on - Ack or Alarm off.
 
Logging maintenance mode
 
Changes to alarms that are in maintenance mode are no longer logged. However, the action of selecting or clearing maintenance mode is logged and treated if a log list is configured to log such user actions in the same way as acknowledging or masking an alarm.
