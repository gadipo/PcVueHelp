How to add a data export to a template

The data export element exports PcVue historical data. This topic explains how to add a data export to a template using the Application Architect.

Refer to the sub-book Data analysis for more information.

Adding a data export

  1. In the right pane of the Application Architect, select the Templates tab.
  2. Using the templates configuration tree select the template to which the data export will be added.
  3. Using the secondary navigation tree select the Configuration elements folder or the element to which the data export is to be added.

  4. Select Add a data export. A new data export with a default name is created. Show picture

    You can edit the name of the created configuration element by directly double clicking it in the Properties pane.

  5. Configure the data export's properties using the properties grid. Right-click the grid and select Advanced to display the advanced properties.

    The Naming Rule property is useful to ensure the uniqueness of the names of the generated elements, by default, the name includes branch segments from all levels of the instantiation tree. The available property values are:

    • Full branch - Generates the full name including all branch segments. This is the default.

    • Short branch - Generates the name using the primary instance. That is the branch segment of the instance containing the element that is generated.

    • Leaf only - Generates the name using only the name of the element as defined in the template.

    Alternatively, you can enter an expression in the naming rule that is used to generate the name.

    If you use a naming rule other than Full Branch, then you must ensure the uniqueness of the generated name.

  6. After you are done, click File menu item then Save.

You can add a log page, a trend page or a file element to a data export.

Adding a log page or a trend page to a data export

  • A log page generates data from a log list.

  • A trend page generates data from one or more trended variables.

  1. In the right pane of the Application Architect, select the Templates tab.
  2. Using the templates configuration tree select the template to which the page will be added.
  3. Using the secondary navigation tree select the Configuration elements folder or the element to which the page is to be added.

  4. Select Add a log page or Add a trend page. A new log or trend page with a default name is created. Show picture

    You can edit the name of the created configuration element by directly double clicking it in the Properties pane.

  5. Configure the log or trend page properties using the properties grid. Right-click the grid and select Advanced to display the advanced properties.

    The Naming Rule property is useful to ensure the uniqueness of the names of the generated elements, by default, the name includes branch segments from all levels of the instantiation tree. The available property values are:

    • Full branch - Generates the full name including all branch segments. This is the default.

    • Short branch - Generates the name using the primary instance. That is the branch segment of the instance containing the element that is generated.

    • Leaf only - Generates the name using only the name of the element as defined in the template.

    Alternatively, you can enter an expression in the naming rule that is used to generate the name.

    If you use a naming rule other than Full Branch, then you must ensure the uniqueness of the generated name.

  6. After you are done, click File menu item then Save.
