Overview of Recording in Proprietary Archive Units

Native data recording records in, and display from, PcVue own built-in data storage system.

Where is the data recorded?

The data is recorded in one or more archive units that are created as part of configuration. There are a number of different types of archive units. The choice of which type of archive unit is used mainly depends on how the recorded data is to be retrieved. See the topic About the different types of archive unit for more information.

What types of data can I record?

Type

Data recorded

Retrieval of recorded data

Trend data

The values of register and bit variables.

Through the HMI using a historic Trend Viewer.
Programmatically using SCADA Basic.
3rd party products, word processor, spreadsheet, database etc. - depending on the chosen recording format.

Log data

The values of bit and alarm variables plus the following operator actions:

Send command, send program, alarm acknowledge, logon and logoff.

Through the HMI using a Log Viewer.
Programmatically using SCADA Basic.
3rd party products, word processor, spreadsheet, database etc. - depending on the chosen recording format.

Report data

The values of register, bit, alarm and text variables.

Through the HMI using mimics configured as reports.
Printed to a graphic printer.

VCR data

The values of register, bit, alarm and text variables.

Through the HMI using a specially configured PcVue station. The data is replayed in pseudo real time controlled using a VCR style (Video Cassette Recorder) dialog.
