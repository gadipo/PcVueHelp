Filtering Events Using the Event Filter Tab

The Event Filter tab allows filtering of events that are recorded according to the event type. Ticking an option means that the event is allowed to pass, un-ticking an option means that the event is blocked. Show picture

The following event types are supported.

  • Alarm events
    • Alarm off. The change of an alarm to the off state.
    • Alarm off, Not Ack. The change of an alarm to the off, not acknowledged state.
    • Alarm on, Ack. The change of an alarm to the on, acknowledged state.
    • Alarm on, Not Ack. The change of an alarm to the on, not acknowledged state.
    • Unavailable alarm events
      • Expression masked
      • Inhibited
      • Not accessible
      • Program masked
      • Variable masked
      • User masked
  • Bit changes
    • Change to 0. The change of a bit from any state to 0.
    • Change to 1. The change of a bit from any state to 1.
    • Unavailable. The change from any state to unavailable (invalid).
  • User actions
    • Acknowledge. Acknowledging an alarm.
    • Command. The forcing of a bit, register or text variable from a control zone animation.
    • Execute program. Running a SCADA Basic program from a control zone animation.
    • Logon/logoff. Logging on and off and failed attempts at logging on.
    • Alarm mask. Masking an alarm

Alarm events may also be filtered by a minimum and maximum alarm level.
