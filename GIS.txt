Map Position animation
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Map Positioning Animation
 
The Map Positioning animation is used to create one or more control zones from where the User can interact with a Map Control. 
 
Specifically, you can pan the map centering it on a particular map marker or latitude and longitude, and/or zoom the map to a particular zoom level. The properties that control the map position can either be fixed, or be the name of a variable when the value of the variable is used. Show picture

 
The animation can be applied to any drawing elements, symbol or image.
 
How to create a Map Positioning animation
 
 
Open the mimic containing the support drawing element and select Design mode.
 
Select the drawing element and choose the Map Positioning animation from either the right-click pop-up menu or the Animation toolbar. The Map Positioning animation is found under the Positioning animations.
 
Configure the properties as required.
 
Click OK to confirm the animation.
 
 
Position Animation properties
 
 
Map control ID - The identity of the Map Control to be affected. A combo-box displays a list of the map controls in the selected mimic. You can either choose one of these or manually enter the identity. By default, if there is only one Map Control in a mimic, its ID will be MapControl1. If the name has been changed from the default, or there is more than one Map Control in a mimic you can find the name using the Graphic Explorer.
 
Comment - An optional comment about the animation. If the text is preceded by the @ character it will appear in a tooltip at run-time.
 
Position - The new position about which the map will be centered.
Position mode - Selection of the position mode, either by Latitude & Longitude, Zoom, or (map) Marker.
Layer - The layer in which the marker resides. A combo-box displays the available layers of the selected Map Control. From the Map Control's selected map markers file. You can either select the name of one of the available layers, enter the layer name manually, or choose the name of a text variable containing the layer name. This property is only available if the Position mode is set to marker.
Marker - The name of the marker about which to position the map. A combo-box displays the available markers of the selected layer. From the Map Control's selected map markers file. You can either select the name of one of the available markers, enter the marker name manually, or choose the name of a text variable containing the marker name. This property is only available if the Position mode is set to marker.
Latitude and Longitude - The new co-ordinates about which to position the map. You can enter the values manually or choose the names of register variables containing the co-ordinates. These properties are only available if the Position mode is set to Latitude & Longitude.
Zoom level - The new zoom level for the map.
 
Mimic - The mimic in which the Map Control resides.
Mimic and Branch - The name of the mimic and its branch in the case that the Map Control is in a different mimic to the map position animation. Leave empty for the current mimic.
Region - In the case of a multi-region application, the region in which the mimic containing the Map Control is opened selected from the following.
Command region - Region for the mimic in which the animation is located. This is the default.
Current region -The pre-selected region set by either the Region animation or a script.
Region 1, Region 2 ... - One of the available regions.
 
Accelerator and Disable mouse click - Common to all control zone animations, the optional keyboard shortcut for the animation. If a shortcut is chosen, Disable mouse click can optionally be selected to disable access to the animation from the mouse.
 
Interlock condition - Common to all control zone animations, an optional interlock bit to enable or disable the animation.
 
 
Error messages
 
Error and/or warning messages and/or icons are displayed if the animation is not configured correctly and/or cannot be executed properly.
 
 
On Initialization - If the referenced mimic is open and the Map Control does not exist. No checks are made if the mimic is not open.
 
At run-time
The mimic, as identified by the Mimic, Branch and Region properties, is not open.
The Map Control does not exist in the referenced mimic.
The layer or marker selected, or provided by variable value, does not exist.
The latitude and/or longitude entered values, or provided by variable value, are outside of the range of the displayed map or are outside of the latitude and/or longitude value limits.

Binding marker properties to variables
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Binding Marker Properties to Variables
 
Many of the properties configured in the Map Markers Editor can be bound to variables. The property then takes its value from the variable. You can bind to a variable's real-time value or one of the extended attributes. When a property can be bound to a variable, the Variable Mode button appears adjacent to the property.
 
How to bind a property to a variable
 
 
Click the Variable Mode button adjacent to the property. The Variable Selector button appears.
 
Enter the name of the variable. You can either type it in directly or use the Variable Selector to select it. Show picture If a Branch has been configured the variable name is relative to that branch unless it is preceded by the direct reference character (@).

 
 
What variables and variable properties can I use?
 
You can bind a property to either a variable's real-time value or one of its extended attributes. To bind to an extended attribute the variable name is appended with the attribute reference. For example, UK.INFORMATION.TATT3#, references the third text attribute of the variable UK.INFORMATION.
 
You must respect the property type. That is, a numerical property (for example Latitude) must be bound to a register variable, and a text property (for example Mimic) to a text variable or a text attribute. The one exception to this is the color swab, which can be bound to a register variable or text variable. If you use a text variable, you can use any standard HTML color name. The standard HTML color names are widely available on the Internet.
 

 
To minimize the system resources required to display a map, markers that have been visible, but are no longer, are temporarily unbound from any variables. You can prevent this by selecting the property Keep invisible markers in memory in the Map Control's configuration dialog.

Using the map cache creator
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Using the Map Cache Creator
 
The Map Cache Creator is used to download maps from the Internet and store them locally for use by the Map Control. The advantage of locally cached maps is that the target PC does not need a connection to the Internet to display the maps. The disadvantage is that a complete set of tiles for a map of the entire world requires considerable storage space. However, the Map Cache Creator allows you to select the zoom levels and range of latitude and longitude that the cached map is to cover thereby limiting the number of map tiles and storage space that is required.
 
The Map Cache Creator takes the form of an executable file (MapCacheCreator.exe) located in PcVue's BIN folder.
 

 
The ability to incorporate map data from third party map suppliers into the PcVue software is provided as a convenience. 
 
Content, as well as terms and conditions applying to map data are not within the control and responsibility of PcVue supplier. 
 
Any expressed or implied warranties, including, but not limited to, the implied warranties of merchantability and fitness for a particular purpose are disclaimed.

 
 
Map data is subject to copyright notice and license agreement. By downloading map data, you are agreeing to be bound to the selected map data suppliers' Terms and conditions.

 
The terms and conditions of map data use are available from the applicable map supplier's web site.
 
 
How to create a local map using the Map Cache Creator
 
 
Start the Map Cache Creator by running the executable file. Show picture

 
Select the Tiles Provider using the drop down list box.
 
Select the Cache folder path. The Map Control defaults to PcVue's GIS folder when selecting a local map. It is good practice to create a sub-folder with a name relevant to the specific map cache to be created. For example, Google_UK_1-10. The sub-folder names appear in the Local Map field when configuring the Map Control.
 
Select the map view and zoom levels to be cached.
Zoom (zoom control in the lower right corner) and pan (using click and drag) the map, so that it displays approximately the area that you want the map cache to support.
Select the precise area by right clicking and dragging in the map view. The resulting shaded area is that which will be cached.
Select the Zoom Levels that are to be supported by clicking and dragging the double slider control.
Check that your selection has resulted in a realistic Estimated Database Size. If not, repeat steps a, b & c until it does. Show picture

 
Click Start to start the download of the tiles. The Map Cache Creator will start fetching the tiles for each of the zoom levels. During the process, the number of tiles required, and the estimated time to load them, is displayed. The download can be stopped anytime by clicking the Cancel button. The map cache will still be created but only with those tiles downloaded at the time the Cancel button was clicked.
 
 

 
It is essential to be selective with the zoom levels and map area to minimize the size of the cache. As an extreme example, a Google Map with zoom levels 1 to 20 and covering the entire world would require 32000 terra bytes.
 
 

 
The estimated database size is based on an average tile size for a typical road/street navigation map and is given for information only. The actual cache size may be significantly different, lower if the geographical data are scarce (ocean or desert), or higher for a urban area with a great deal of details.

Map Markers Editor
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Map Markers Editor
 
The purpose of the Map Markers Editor is to create a map marker file. You can create any number of map marker files but only one file can be used per Map Control instance. Map marker files are created in a globally recognized standard known as Global Positioning eXchange, or GPX. 
 
The Map Markers Editor can be opened using either of the following methods.
 
 
From PcVue's Main Menu using the command Display.Map Markers Editor.
 
From the Map Configuration dialog of the Map Control dialog.
 
 
Map Markers Editor main features Show picture

 
 
Map pane. The actual map view can be zoomed and dragged as required. You can change the area occupied by the map pane by clicking and dragging the vertical line that separates it from the properties pane. Shown on the screen shot by the dashed vertical red line.
 
Markers toolbar.
 
Currently selected marker. The selected markers are indicated with a translucent green highlight. The marker selected in the screen shot is a polygon.
 
Properties pane. At the top of the properties pane is the Map Provider selector. 
 
Layers toolbar. Using this toolbar, you can add a new layer, a new service layer, and/or remove selected objects.
 
Map objects (layers and markers) configuration tree. You can change the area occupied by the map objects tree by clicking and dragging the horizontal line that separates it from the marker properties. Shown on the screen shot by the dashed horizontal red line.
 
Properties for the selected map marker.
 
Save and Open command buttons.
 
Map zoom control.
 
 
The map
 
 The selected map can be panned, using click and drag, and zoomed, using the map zoom control, in a similar manner to the Map Control. Note that the map is only used as an aid to position the markers and has no link to the actual map that to be used at run time. The selected map is not saved in the map markers file and therefore the default map is always displayed when the Map Markers Editor is opened.
 
The map layers
 
Map layers are a way of grouping together the map markers. The map layers themselves do not appear on map but when they are configured, they are allocated to a specific zoom range so that, at run-time, the map markers they contain appear or disappear according to the map zoom level.
 
The map markers
 
The map markers that you configure appear at run time in any Map Control in which a map markers file has been selected. Each map marker is attached to a map layer that controls its visibility. The following Map marker types are supported.
 
 
Shape - A solid shape selected from the following forms - circle, square, rounded square, triangle, inverted triangle and rhombus. 
 
Text - A bilingual text string. 
 
Image - An image from a library.
 
Symbol - A symbol from a library.
 
Track - A multi-segmented line. Similar to the polyline drawing element.
 
Polygon - A polygon. Similar to the polygon drawing element
 
 
Many of the map marker's properties can be bound to the value of a variable. 
 
All map markers types, except symbol, have a property that allows you to select a mimic that can be directly opened by clicking on the marker in run mode (like a Link-Open animation). Symbols do not have this property as the symbol could include a Link-open animation itself.
 
Supplementary information, known as User Data, can be associated with a marker. The information is displayed in a pop-up window when the marker is right clicked at run-time. User data can include fixed text, variable's real-time values, and variable's extended attributes.
 

 
It is recommended to not exceed a maximum of 500 markers visible at the same time. Displaying more markers on a map can affect responsiveness of the user interface.
 
 
The command buttons
 
When you open the Map Markers Editor from the menu, you must use the command buttons to open or save a specific map markers file.

Map Overview
 

 
The ability to incorporate map data from third party map suppliers into PcVue is provided as a convenience. 
 
Content, as well as terms and conditions applying to map data are not within the control and responsibility of PcVue supplier. 
 
Any expressed or implied warranties, including, but not limited to, the implied warranties of merchantability and fitness for a particular purpose are disclaimed.

 
 
Map data is subject to copyright notice and license agreement. By downloading map data you are agreeing to be bound to the selected map data suppliers' Terms and conditions.

 
The terms and conditions of map data use are available from the applicable map supplier's web site.
 
 
Map Control displays interactive geographic maps, with an overlay of animated symbols representing real world objects, in mimics. This type of system is often referred to as a Geographical Information System (or GIS). Show picture

 
The geographic maps are provided by third party services such as OpenStreetMap and are generated using a set of graphic images known as tiles. There are normally several sets of tiles, each covering a range of zoom levels. The higher the zoom level the more detailed are the tiles and the more tiles you need for a particular view. The tiles can either be downloaded on demand from the Internet, or they can be loaded from local storage.
 
The map is overlaid with configurable symbols known as markers. Markers are attached to layers and each layer is assigned to a range of zoom levels. The layers, and hence the markers, are hidden and revealed according to the map zoom level.
 

 
The Map Control layers and zoom are unrelated to a mimic's layers and zoom.
 
 
The Main Map Components
 
The Map facility has three main components.
 
 
Map Control - The special animation that enables maps can be embedded in any mimic. Maps are rendered using the World Geodetic System (WGS 84) standard. Using the Map Control the developer can configure the following features.
The space that the map occupies within the mimic.
The provider of the map tiles and if they are to be downloaded from the Internet or come from a local cache.
The initial position and zoom level.
The bounds between which the map panned and zoomed.
The name of the file that provides the map markers overlay.
In addition, the Map Control supports a programmatic interface permitting manipulation by both SCADA Basic and VBA.
 
Map Markers Editor - A tool to configure the map marker overlays. Each overlay contains one or more layers to which markers are attached. The visibility of each layer (and hence the map markers) is controlled by selecting a zoom minimum and maximum. The map marker overlays are saved in a modified version of a format known as GPX. 
Map markers - A map marker is a symbolic representation of a real world object.
Each map marker is attached to a layer in a map markers overlay.
Appearance can be selected from a shape, text, track (polyline), polygon, or a symbol.
Several properties of a marker can be bound to the value of a variable.
Can be configured so that, at run time, a left click opens a mimic. 
Can be associated with the value of one or more other variables that are then displayed in a pop-up at run time (right click).
 
Map positioning animation - A position type animation that allows the user to position and zoom the map using a pre-configured (by the developer) set of co-ordinates.
 
 
In addition, to these three main components, a standalone tool is provided to download map tiles from the Internet and save them in a map cache for use offline (with no Internet access).
 
Map Storage
 
By default, local maps, map marker files and the map cache are saved in the GIS folder located in PcVue's root. Map files can be very large (many terabytes) and by locating them here, the same maps can be accessed by several projects minimizing storage requirements.

Using the Map Control at run-time
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Using the Map Control at Run-time
 
When a mimic containing a Map Control is opened at run-time, the map appears centered at the configured location and zoomed at the configured zoom level. The center of the map is indicated with a small red cross. Show Picture

 
The Map Control supports the following run-time actions.
 
 
For the map itself
Click and drag - Move the map by changing the map center within the panning bounds.
Mouse wheel - Zoom the map within the configured zoom bounds.
Zoom control (lower right corner) - Click the + and - tools, or enter a number in the text box, to zoom the map within the configured zoom bounds.
Keyboard shortcuts
Ctrl and + - Zoom in.
Ctrl and - - Zoom out.
Ctrl and 0 - Zoom out to the minimum allowable zoom level.
 
For a map marker
Mouse over - Displays the marker's tooltip (if configured).
Click - Opens the mimic (window) associated with the marker (if configured).
Right click - Opens the User Data pop-up associated with the marker (if configured).
 
 
Why are some of the map tiles not displayed?
 
If the Map Control was configured so that it allows zooming and/or panning for which map tiles do not exist, or the connection to the map tiles location has been lost, it is possible that part or all of the map will not be displayed. Instead, the unavailable map tiles are replaced by blanks.

Track and polygon markers
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Track and Polygon Markers
 
This topic explains the properties and behavior of the track and polygon markers. The track and polygon are very similar except that the track is an open shape (like the polyline drawing element) and the polygon is a closed shape. See the topic Managing Map Markers in General for general information about map marker properties and behavior.
 

 
There is a significant difference in the way that a track or polygon is rendered compared to the other markers. When a map is zoomed, any track or polygon zooms with it. All other markers stay the same size as defined when they were configured.
 
 
How to create a track or polygon
 
The process to create a track or polygon marker is similar to that of the other shapes except you left click several times, each click creating a node. You complete the marker by a single right click. Show picture

 
How to move a node
 
 
Using the Marker toolbox, select Select mode.
 
Click a single node. The selected node is highlighted. Show picture

 
Drag the node to its new location.
 
 
How to delete or add a node
 
 
Using the Marker toolbox, select Select mode.
 
Right click a single node. A context menu is displayed offering the choice of adding or deleting a node.
 
Choose Add or Delete as appropriate.
 
 
Changing the stroke (line) style using the StrokeDashArray property

The Map Objects Tree
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Map Objects Tree
 
The Map Objects Tree supports the following features and actions to manage the layers and markers. Show picture

 
 
Map Items tab - Tree display of map layers and map markers. The map markers sorted according to which layer they are attached.
The branch that represents each layer, displays the layer name, the number of markers it contains in round brackets, and the zoom range in square brackets.
For example,: Countries (5) [1;20].
Clicking the Triangle icon expands or contracts the branch displaying or hiding the markers.
Clicking the Eye icon toggles the visibility of the layer's markers both in the Map Markers Editor and at run-time in any associated map.
Clicking a layer name selects it and displays the layer properties.
Clicking a marker name highlights the marker on the map and displays the marker's properties. All other objects are de-selected.
Double clicking a marker name centers the map on the marker, highlights the marker on the map and displays the markers properties.
Double clicking a layer name, computes the bounding box of all markers in the layer and locates the map on it. If necessary, it also adjusts the zoom level so the layer is visible.
Shift & click allows the selection of multiple markers. Ctrl & click adds a marker to the selection. Alt & click removes a maker from the selection.
 
Markers tab - List of all map markers irrespective of which layer they are attached to. The markers behave in the same manner as described above.
 
 

 
The default layer, displayed at the top of the Map Items Tree, is mandatory.

Configuring user data
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring User Data
 
User Data is supplementary information that can be associated with a marker. The information is displayed in a pop-up window when the marker is right clicked at run-time. User data can include fixed text, variable's real-time values, and variable's extended attributes. Show picture

 
How to configure user data
 
 
In the Map Markers Editor, select the marker to which the User Data is to be added.
 
Click the Ellipsis button adjacent to the User Data field. The User Data Management dialog opens. Show picture

 
Select the + tool to add a new item.
Enter the Label. This appears in the left hand column of the run-time window.
Enter the name of a variable or variable's extended attribute in the Variable field, or enter a fixed string in Value. See the topic Binding marker properties to variables for information about how to reference a variable's extended attributes.
Repeat a and b until the user data is complete.
 
Click OK to confirm the configuration and close the dialog.
 
 

 
The Name field must be unique although it does not appear in the User Data window at run-time. It is only used when managing user data programmatically.
 
If a label is not defined, the user data will not appear in the pop-up window.
 
If the Variable and Value are both configured, when the marker is displayed in run mode, the variable is forced to the value even if the variable is not configured with the Command property set.
 
 
About the Label field
 
The Label field is a multi-lingual string. Clicking the Ellipsis button opens the Configuring Multilingual Strings dialog where you can enter strings for the various languages. The Label field supports the standard substitution strings. For example:
 
 
#V - Substitutes the variable name
 
#T - Substitutes the variable's Description property.
