Population Filters Overview

A Population is a filter that can be applied to the variables managed by a station, in order to control the information a user can access during operations. Population filters are combined with user rights to determine the set of accessible information. Such type of filter can be changed on the fly during operations and has been designed to accommodate needs such as:

  • Be able to hide some information during emergency or maintenance situations, in order to minimize the cognitive load for users.
  • The other way round, be able to show additional information or allow particular control when the situation requires it.
  • Be able to modify the subset of information accessible to a user depending on its role and the day. For example, the case of a security gate station, with most alarms hidden during the day, and displayed at night or on week-ends when the other teams are off.

The following features of PcVue support populations filters:

  • Alarm viewer - If alarms are excluded by the population filter, then these alarms are not displayed in alarm viewers.
  • Log viewer - If variables or alarms are excluded by the population filter, then corresponding logged events are not displayed in log viewers.
  • Log printers - If variables or alarms are excluded by the population filter, then corresponding logged events are not printed.
  • Archiving - It is possible to filter data archiving for logged events according to a population filter.
  • User commands - Users cannot send commands for variables excluded by the population filter.
  • Setting values in SCADA Basic - It is not possible to set values for variables excluded by the population filter.
  • User actions on alarms in SCADA Basic - Users cannot send commands on alarms excluded by the population filter.

A population can be created from the Population node in the Application Explorer or by using the SCADA Basic instruction POPULATION. The SCADA Basic instruction, STATION_FILTER is used to apply a population so it can be dynamically applied depending on a wide range of criteria determined by the application designer.

The maximum number of populations that can be created is 512.

How a population is defined

A population is defined by including, and/or excluding, variables using one or more of the following variable properties.

  • Type -, Bit, Alarm, Register or Text.
  • Source - Internal, Equipment, DDE, OPC, IEC 60870-5, DNP3, IEC 61850, LonWorks or SNMP.
  • Binary attribute
  • Domain (text attribute #1)
  • Nature (text attribute #2)
  • Text attributes #3 to #16
  • Full name
  • Branch (part of the name)

For example a Population could include all register variables with an equipment source with the Domain Tank1 or Tank2, not having the Nature Electrical.

Population filtering based on the source is not supported to filter in log viewers (and log printing) when data are archived using a database archive unit (HDS).

A note about the Sql terminology used in Population

 PcVue supports the following Sql operators when defining a population.

  • IN - Multiple OR condition.
  • NOT - Negate a condition.
  • EXISTS - Condition is met.
  • AND - Two or more conditions.

NOT and EXISTS are used together as an exclusion.

In version 12 the NOT EXISTS operator replaces the OUT operator used in previous versions. The behavior is unchanged and applications that use population that were developed with previous versions are unaffected.
