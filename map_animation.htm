Map Positioning Animation

The Map Positioning animation is used to create one or more control zones from where the User can interact with a Map Control.

Specifically, you can pan the map centering it on a particular map marker or latitude and longitude, and/or zoom the map to a particular zoom level. The properties that control the map position can either be fixed, or be the name of a variable when the value of the variable is used. Show picture

The animation can be applied to any drawing elements, symbol or image.

How to create a Map Positioning animation

  1. Open the mimic containing the support drawing element and select Design mode.
  2. Select the drawing element and choose the Map Positioning animation from either the right-click pop-up menu or the Animation toolbar. The Map Positioning animation is found under the Positioning animations.
  3. Configure the properties as required.
  4. Click OK to confirm the animation.

Position Animation properties

  • Map control ID - The identity of the Map Control to be affected. A combo-box displays a list of the map controls in the selected mimic. You can either choose one of these or manually enter the identity. By default, if there is only one Map Control in a mimic, its ID will be MapControl1. If the name has been changed from the default, or there is more than one Map Control in a mimic you can find the name using the Graphic Explorer.
  • Comment - An optional comment about the animation. If the text is preceded by the @ character it will appear in a tooltip at run-time.
  • Position - The new position about which the map will be centered.
    • Position mode - Selection of the position mode, either by Latitude & Longitude, Zoom, or (map) Marker.
    • Layer - The layer in which the marker resides. A combo-box displays the available layers of the selected Map Control. From the Map Control's selected map markers file. You can either select the name of one of the available layers, enter the layer name manually, or choose the name of a text variable containing the layer name. This property is only available if the Position mode is set to marker.
    • Marker - The name of the marker about which to position the map. A combo-box displays the available markers of the selected layer. From the Map Control's selected map markers file. You can either select the name of one of the available markers, enter the marker name manually, or choose the name of a text variable containing the marker name. This property is only available if the Position mode is set to marker.
    • Latitude and Longitude - The new co-ordinates about which to position the map. You can enter the values manually or choose the names of register variables containing the co-ordinates. These properties are only available if the Position mode is set to Latitude & Longitude.
    • Zoom level - The new zoom level for the map.
  • Mimic - The mimic in which the Map Control resides.
    • Mimic and Branch - The name of the mimic and its branch in the case that the Map Control is in a different mimic to the map position animation. Leave empty for the current mimic.
    • Region - In the case of a multi-region application, the region in which the mimic containing the Map Control is opened selected from the following.
      • Command region - Region for the mimic in which the animation is located. This is the default.
      • Current region -The pre-selected region set by either the Region animation or a script.
      • Region 1, Region 2 ... - One of the available regions.
  • Accelerator and Disable mouse click - Common to all control zone animations, the optional keyboard shortcut for the animation. If a shortcut is chosen, Disable mouse click can optionally be selected to disable access to the animation from the mouse.
  • Interlock condition - Common to all control zone animations, an optional interlock bit to enable or disable the animation.

Error messages

Error and/or warning messages and/or icons are displayed if the animation is not configured correctly and/or cannot be executed properly.

  • On Initialization - If the referenced mimic is open and the Map Control does not exist. No checks are made if the mimic is not open.
  • At run-time
    • The mimic, as identified by the Mimic, Branch and Region properties, is not open.
    • The Map Control does not exist in the referenced mimic.
    • The layer or marker selected, or provided by variable value, does not exist.
    • The latitude and/or longitude entered values, or provided by variable value, are outside of the range of the displayed map or are outside of the latitude and/or longitude value limits.
