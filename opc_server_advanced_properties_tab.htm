The OPC Server Advanced Tab

The Advanced tab in the OPC Server properties box is for tuning the OPC Server connection.

With the majority of OPC servers, the default settings provide the best connection. Before changing any of the properties, check that the option is supported by the server.

The properties prefixed with * are only available when using OPC-DA Classic.

The properties prefixed with ** are only available when using OPC-XML.

  • *Server type
    • *Automatic detection - The driver will attempt to detect if the OPC server is 1.0a or 2.0 compliant and make the connection accordingly. This is the default setting.
    • *OPC 1.0A compliant server - Force the driver to use OPC 1.0A for connection to the server.
    • *OPC 2.0 compliant server - Force the driver to use OPC 2.0 for connection to the server.
  • *Browse server address space
    • *Item properties browsing
      • *No browsing - No browsing of item properties.
      • *Browse standard properties - The driver attempts to get the status of read/write access and the data type of an OPC itemID by using the IOPCItemMgt::ValidateItems method.
      • *Browse additional properties - The driver attempts to get the read/write access, the data type, and the description of an OPC itemID by using the IOPCItemProperties interface or IOPCBrowse interface.
    • *Default filter criteria. A string used by the client when browsing the address space in the server with no filter. Information on which of the options is required will normally be found in the server documentation.
      • *Null filter – A filter string of "" is used. This is the default setting.
      • *'*' filter – A filter string of "*" is used.
  • Connection/Reconnection
    • Check server status period – The period at which PcVue checks the status of the OPC server. The status is reported in system variables in the database. See the topic OPC Status Variables in the Database book for further information. Range 500 to 60 000 ms. The default is 10 000 ms.
    • Try to reconnect if connection fails – If ticked, PcVue will periodically try to re-establish a failed connection with the OPC server.
      • Reconnection period – The period at which PcVue will attempt to re-establish a failed connection. Range 1 000 to 60 000 ms. The default is 60 000 ms.
      • *Use the advanced reconnection strategy - Used to clean up resources upon reconnection to a server. This is not compliant with the OPC foundation standard, which is why it is an option. The OPC foundation standard is that the OPC server should correctly manage disconnection but this does not always happen.
    • *Waiting period before group creation - The wait period after establishing a connection and before trying to create groups in the OPC server. The default is 0 ms.
    • Frozen context detection - The status of each OPC server is monitored by a program that periodically checks the OPC server's response. If the server does not respond, it is said to have a frozen context.
      • Level 1 time limit - If the OPC server does not respond to requests for this period or greater a warning message is displayed in the Event Viewer and the OPC system variable <SYSTEM.OPC.ServerAlias>.ERROR is set to 10.
      • Level 2 time limit - If the OPC server does not respond to requests for this period or greater a warning message is displayed in the Event Viewer and the OPC system variable <SYSTEM.OPC.ServerAlias>.ERROR is set to 11.
  • Interoperability issues
    • **Browsing
      • **Use null string instead of empty string - Activate this property if the OPC server only accepts a null string where the OPC standard specifies the use of an empty string in browsing requests.
  • Data interpretation - Configure how PcVue interprets value with uncertain or bad quality. See note below.
    • Use new values even when the quality is bad - Use new values from the OPC server even if the quality flag is set to bad.
    • Quality - Configure how PcVue interprets the uncertain quality flag from the OPC server.
      • As good quality - Uncertain quality is treated as good quality.
      • As bad quality - Uncertain quality is treated as bad quality.
  • **Http Basic Authentication - Tick to activate the use of Http Basic Authentication.
    • **Username - The user name to send to the OPC server. Max length is 255 characters. The Username cannot include the double-quote character (").
    • **Password - The password to send to the OPC server. Max length is 255 characters. The Password cannot include the double-quote character (").
  • Servers - A drop down list box used in multi-station applications to start the OPC server only on a specific station or list of stations. For standalone applications, the list box will be empty.

About OPC quality

An OPC server provides a minimum of three properties for each of its variables, Value, Quality and Timestamp. The Quality provides information about the status of the value and is a 16-bit word. The higher 8-bits are vendor specific and the lower 8-bits follow a standard format, QQSSSSLL, where QQ provides the most basic quality information and SSSS is the sub-status.

  • 00SSSSLL - The value is bad for the reason indicated by the sub-status.
  • 01SSSSLL - The value is uncertain for reasons indicated by the sub-status.
  • 10SSSSLL - Not used.
  • 11SSSSLL - The value is good.
