Displaying a register as a bargraph
 

 
 
 
 
 
 
 
 
 
 
 
Displaying a Register as a Bargraph
 
Using a bargraph, the value of a register is represented 
 by filling one of the solid drawing elements. You can apply a bargraph 
 to a rectangle, rounded rectangle, ellipse and polygon. The fill is calculated 
 using the value of the variable with respect to its range and the height 
 (or width) of the drawing element. In the case of a polygon, the fill does 
 not take into account the area occupied.
 
Configuring the Style
 
The color of the bar and the background are specified 
 by the configuration of the animation. However, the style of the bargraph 
 is inherited from the support drawing element. For example, the bargraph 
 can appear in relief, or as a colored button.
 
Configuring the display bounds
 
The minimum and maximum bounds are the values representing 
 0% and 100% fill of the bargraph. By default, the bounds are set to Auto. 
 This means that the HMI will attempt to get the bounds for the variable 
 from the OPC server. If the range of the variable in the OPC server changes 
 all animations linked to it using the Auto bound will automatically follow 
 without the need to change the animation.
 
If the server does not support variable bounds or you 
 want to use a different value there are two other options accessed by 
 clicking the buttons adjacent to the bound fields.
 
 
 
 

 
 

 
 
 
 
 
Clicking this button allows you to enter a fixed value 
 for the minimum and maximum bounds.
 
 
 

 
 

 
 
 
 
 
Clicking this button allows the bounds to be defined by 
 the value of another register in the server. The Variable Browser is displayed 
 to allow selection of the variable name.
 
 
 
 
Configuring the fill direction
 
The animation properties allow you to enter the range 
 of the variable for the fill, as a minimum and maximum, and the fill direction. 
 You can fill from the bottom, top, left or right.
 

 
 
 
Changing the Bargraph color
 
You can change the color of the bargraph bar at runtime 
 using a threshold system controlled by up to 4 bits. To display the threshold 
 properties you click the down arrow button at the bottom of the Bargraph 
 tab. The priority of each bit in selecting the color depends on the type 
 of threshold system that is selected. Some combinations of bit have no 
 meaning and should be avoided. For example, if you select the hihi/high/low/lolo 
 system it is meaningless to have the bits representing high and low to 
 be both true at the same time.

Color properties tabs
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Color Properties Tabs
 
Each color change animation is represented by a tab in 
 the Properties dialog of the drawing to which it is applied. Although the 
 appearance of the tab will vary depending on the type of color animation 
 chosen, all of them contain common icons representing the pattern and 
 border properties. Show picture
 

 

 
 
 
 
Using the color palette to select the colors
 
The dynamic colors used in an animation are selected 
 using the color palette in the same way as when you color a static drawing 
 element. You may use single, blinking and transparent colors.
 
Specifying a color for invalid status
 
On all the color animations you have the option to select 
 a special color that will be displayed when any of the variables used 
 becomes invalid. A variable may become invalid for a variety of reasons, 
 the most common one being the loss of communication to external equipment 
 such as a PLC or RTU. 
 
It is recommended that you make use of this option. If 
 you do not, when a variable becomes invalid the animation will display 
 the color representing the last valid value of the variable. As a consequence, 
 the user may be unaware that she/he may be viewing out of date information.
 
How many colors can I select?
 
The number of colors that you can select depends on the 
 animation chosen. You do not have to use all of the colors. For example, 
 the animation Color - Bit Group allows you to select up to 16 colors 
 using 4 bits, but you could use it to select just 4 colors using 2 bits. 
 However, whichever animation you choose, you must select at least 2 of 
 the available colors as the transition to or from an unselected state 
 is not recognized.

Double bit animation
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Double Bit Animation
 
The Double Bit animation allows the user to change the 
 state of 2 bits at the same time. This is useful when a device requires 
 separate signals to start and stop it. The command is always confirmed 
 with a dialog. Show picture
 

 

 
 
 
 
Operation
 
When the user clicks the control zone at runtime a dialog appears containing three command buttons. The first button displays 
 the label for the bit 1, the second the label for bit 2. Show 
 picture
 

 

 
 
 
 
The control operates in two ways depending on whether 
 the Pulse option has been selected.
 
 
Without the Pulse 
 option. Clicking on the first button will set the first bit to 1 and the 
 second bit to 0. Clicking on the second button will set the second bit 
 to 1 and the first bit to 0.
 
With the Pulse option. 
 Clicking on the first button will set the first bit to 1 for the defined 
 period after which it will return to 0. The second bit is not affected. 
 Clicking on the second button will set the second bit to 1 for the defined 
 period after which it will return to 0. The first bit is not affected.
 
 
The Cancel button closes the dialog without any action.

Exchanging images
 

 
 
 
 
 
 
 
 
 
 
 
Exchanging Images
 
Simple changes in process status can be indicated by 
 change of color. For example, a pump may be displayed in green when it 
 is running, red when it is stopped and blue when faulty. However, in some cases, changing color is not adequate. Consider the display of a float switch. 
 The requirements are that it should appear horizontal when its corresponding 
 bit in the variables tree is true, and at an angle of 45� 
 when the bit is false. This can easily be achieved by using two images exchanging them depending 
 on the state of the bit. In practice, exchanging images can create 
 many visual effects.
 
The image animations all require an image as the support drawing element.
 
You can also use the Visibility animations with images.
 
Anchor point
 
When exchanging images the anchor point is the top left hand corner. The anchor point of the support image is not used.
 
Using images of different sizes
 
By default, images displayed using the image animations are displayed in their original size. Each of the image animations has a property called Stretch images. Selecting this will cause all images to be automatically re-sized to match that of the support image.
 
Specifying an image for invalid Status
 
On all the image animations, you have the option to select 
 an image that will be displayed when the variable used in the animation becomes invalid. A variable can become invalid for a variety of reasons, 
 the most common one being the loss of communication to external equipment 
 such as a PLC or RTU.
 
If you do not use this option, when a variable becomes 
 invalid the animation will display the image representing the last valid 
 value of the variable. The user will then be unaware that they may be 
 viewing out of date information.
 

 
It is highly recommended that you make use of this option.

Legend animation
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Legend Animation
 
The Legend animation creates a numeric scale that may 
 be used in conjunction with Bargraph or Position animation to provide 
 the user with an indication of the value. The Legend animation uses a 
 rectangle as the support drawing element. The rectangle determines the 
 size of the legend, the color of the background and the color and width 
 of the lines. Show picture
 

 

 
 
 
 
The advantage of using the Legend over a fixed scale 
 drawn manually is that the Legend can be configured so that the maximum 
 and minimum value for the scale is obtained automatically from a variable's 
 range. If the range of the variable changes the Legend follows automatically.
 
Configuring the Scale Appearance
 
Show picture
 
 

 

 
 
 
 
The number of primary and secondary divisions that appear 
 on the scale may be selected using one of three methods.
 
 
Auto � The HMI decides 
 on the number of divisions using a built-in algorithm.
 
Units � The number of 
 primary divisions is calculated using the range of the scale divided by 
 the number entered in the primary field. For example, using a minimum 
 of 0, a maximum of 100 and a primary unit of 25 there would be 4 primary 
 divisions. The number of secondary divisions per primary division is calculated 
 using the number entered in the primary field divided by the number in 
 the secondary field. For example, using a primary unit of 25 and a secondary 
 unit of 5 there will be 5 secondary divisions.
 
Count � The number entered in the primary field is 
 the number of primary divisions. The number entered in the secondary field 
 is the number of secondary divisions per primary division.
 
 
The direction of the scale is selected using the Direction 
 command buttons. If one of the horizontal options is selected the text 
 is rotated so that it reads bottom to top.
 
The format of the scale number is selected using the 
 standard output formatting characters.
 
Selecting the Scale Range
 
By default, the maximum and minimum values that define 
 the scale range are set to Auto. This means that the HMI will attempt 
 to get them from the register definition in the server.
 
If the server does not support the maximum and minimum 
 properties, or you want to use a different value, there are two other options 
 accessed by clicking the buttons adjacent to the maximum and minimum fields.
 
 
 
 

 
 

 
 
 
 
 
Clicking this button allows you to enter a fixed value 
 for the minimum and maximum bounds. If you select a fixed value then it 
 is not necessary to select a register name.
 
 
 

 
 

 
 
 
 
 
Clicking this button allows the minimum and/or maximum 
 to be defined by the real-time value of other registers in the server. 
 The Variable Browser is displayed to allow selection of the variable name.

Displaying Texts and Labels
 
The Text Animation
 
The Text animation displays the contents of a text 
 variable. The most common use of this is to display the time, date and 
 user name. Other uses include operator prompts, warning messages and status

When using the Text animation it is normal to select

 be displayed on a number of lines, automatically broken at the most convenient

The Label Animation
 
The Label animation displays text associated with a Bit 
 variable. You can display:
 
 
The variable�s Associated 
 Label.
 
A variable�s item 
 Description.

Single axis positioning animation
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Single Axis Positioning
 
Single axis positioning is used to move a drawing according 
 to the value of a register. Single axis positioning may be applied to 
 all drawing elements. Show picture
 

 

 
 
 
 
The drawing movement is along an (imaginary) straight 
 line between two points on the screen. The co-ordinates for the start 
 of the line are supplied by the position of the support drawing element. 
 The co-ordinates for the end of the line are the position of the support 
 drawing element plus the values of the X and Y axis movement properties.
 
The X and Y axis movement properties may be positive 
 or negative. Positive X axis values cause movement from the left to the 
 right of the screen. Positive Y axis values cause movement from the top 
 to the bottom of the screen.
 
The start and end point values can be any value within 
 the range of the selected register. The start value can be less than or 
 greater than the end value.
 

 
It is possible to create diagonal movement by entering values in both the X axis moving and Y axis moving properties.

TOPIC: Anim_Position_Path.htm
========================================
Path positioning animation
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Path Animation
 
The path animation allows you to move a drawing element 
 along a path defined by a polyline, polygon, B�zier curve or B�zier shape. The path 
 animation may be applied to any drawing element or Symbol. Show picture
 

 

 
 
 
 

 
The animation is applied to the drawing that is to move, 
 not to the drawing element providing the path.
 
 

 
The animated drawing and the drawing providing the path 
 must both belong to the same depth from a grouped point of view. That 
 is, if the drawings are grouped, they must both be in the same group.
 
 
How the animated drawing moves
 
The range of register values over which the drawing moves 
 is divided equally by the number of segments in the drawing providing 
 the support path. For example, if the range is 0 to 100 and there are 
 5 segments in the path drawing, the animated drawing will move along the 
 first segment from 0 to 19, the second segment from 20 to 39 etc. The 
 length of the segments is not taken into account and so if the segments 
 are of different lengths, the drawing will appear to move faster and slower 
 as it moves along the different segments.
 
At runtime, when the register is at the start point value, 
 the anchor point of the animated drawing element is located at the start 
 position (X and Y in the drawing Properties dialog) of the path drawing. 
 The location of the animated drawing element in design mode has no relevance.
 
If the Curved property is enabled the animated drawing 
 will rotate according to the angle of the segment it is following.
 

 
 
 
Drawing Restrictions
 
The animated drawing and the drawing providing the path 
 must both belong to the same depth from a group perspective.

 takes place about the drawing�s anchor point.

With a value of 100, the drawing will rotate clockwise 90�.

Maximum angle = -90�

Running a SCADA BASIC program
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Running a SCADA BASIC Program
 
Programs can do many things from providing a simple increment 
 to a variable to changing the properties and hence behavior 
 of some of the macro animations. With careful design and by using techniques 
 such as passing arguments and a variable branch, generic programs may 
 be written which may be re-used many times. The Run Program animation 
 allows a SCADA BASIC program or function to be run from a control zone.
 
In the Run Program tab, you enter the name of the program 
 to be run and an optional function name. If you do not specify a function 
 name, the control zone will always try to start the MAIN function. Show picture
 

 

 
 
 

 
A program must be pre-loaded before it may be executed 
 from a control zone.
 
 
 

 
Functions are not re-entrant, that is they can only be 
 run once at any one time. In order to avoid problems with this function's execution time 
 should be kept short and not blocking. Avoid using instructions that cause delay in 
 execution (for example DELAY).
 
 
Using a variable branch
 
If you enter a variable branch in the Branch field of 
 the Send Program tab, any variables names in the program not 
 preceded by the @ character will be prefixed with the branch. This is 
 especially useful when developing generic programs.
 
For example, if in the program you have a variable named 
 MAX, and you run the program with the branch INLET1.FLOW the program will 
 actually refer to the variable INLET1.FLOW.MAX.
 
Passing Arguments
 
A list of up to 10 arguments, separated by commas, may 
 be entered in the Arguments field of the Run Program tab. At run-time, 
 the arguments may be retrieved from within the program using the GETARG 
 instruction. The maximum number of characters that may be entered in the 
 Arguments field is 256.

Send Bit animation
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Send Bit Animation
 
The Send Bit animation allows the user to force the state 
 of a single bit. Show picture
 

 

 
 
 
 
The behavior of the control zone is determined by the 
 following properties.
 
 
Reversed - The control 
 zone sends the bit to the opposite of its current state. That is, if the 
 bit was 0 it becomes 1 or if it was 1 it becomes 0.
 
To 1 - The control 
 zone always forces the bit to 1. If it is already 1 then there is no change.
 
To 0 - The control 
 zone always forces the bit to 0. If it is already 0 then there is no change.
 
Variable - At run 
 time when the user clicks the control zone the state of the bit to be 
 forced is copied from the state of another bit.
 
Direct - The control 
 zone sends the bit without confirmation.
 
Label - See the explanation 
 below.
 
Handled � The control 
 zone behaves as a push-to-run button. At runtime, the bit is sent to the 
 requested state while the user clicks and holds down the mouse button. 
 When the button is released, the bit is sent to the opposite state.
 
Pulse - Sends the 
 bit to the required state (0, 1 or reverse) for a period defined in seconds. 
 After the period has elapsed the bit is returned to the original state.
 
Beep - Sound the 
 PC's speaker when the command is made.
 
 

 
The use of the Reversed behavior is strongly discouraged as it may have an undefined behavior depending on:
 
 

 
The data acquisition driver: For example in cases when there are 2 separate variables, one for monitoring (read) and another one for controlling (write).
 
 

 
The nature of the monitored value: Current process state or value of the previous command, in most cases, the exact behavior depends on how commands are managed at the automation controller level (PLC).
 
 

 
The state of the monitored value: Valid or invalid. No suitable behavior can be defined if the monitored value is invalid or goes invalid while processing the handled command.
 
 
 
In particular, using the Reversed behavior combined with the Handled or Pulse sending modes is not permitted.
 
The use of the Handled sending mode is also strongly discouraged, even solely, as it relies on the ability to reverse the command and as such may have the same undefined behavior.
 
In addition, it can show potential unreliable execution timing depending on the quality of the communication links.
 
 
Confirming a Command With a Label
 
If you choose the Label send mode, at runtime when the 
 user clicks the control zone a dialog will appear in which the command may 
 be confirmed or cancelled. The box displays the bit item description plus 
 the appropriate string as entered in the Labels section of the Send Bit 
 tab. In this way, the user is provided with a native language prompt for 
 the action. Show picture
 

 

 
 
 
 
By default, the label selection is set to Auto. This means 
 that the HMI will attempt to get them from the variable definition in 
 the server.
 
If you want to 
 use something different there are two other options accessed by clicking 
 the buttons adjacent to the bound fields.
 
 
 
 

 
 

 
 
 
 
 
Clicking this button allows you to enter your own labels 
 for the To 0 and To 1 conditions.
 
 
 

 
 

 
 
 
 
 
Clicking this button allows the labels to be defined by 
 the real-time value of a text variable in the server. The variable browser 
 is displayed to allow selection of the variable name.
