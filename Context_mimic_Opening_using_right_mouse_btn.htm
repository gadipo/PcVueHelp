Opening a Context Mimic Using a Mouse Button

A context mimic may be opened by clicking with a mouse button on certain animation types. The mimic that is opened depends on the type of variable in the animation and is defined using the Context Windows dialog displayed from the menu Preferences.Context mimics. The names of up to six mimics may be entered, one for each of the variable types (bit, alarm, register and text) and for the Alarm Viewer and Log Viewer. Show picture

In general, the ability to open a context mimic is only available from animations using a single variable, plus the Alarm Viewer and Log Viewer. The following animations are supported:

When a mimic is opened from a Link Open animation, the user is aware of its presence because the support drawing element is highlighted when the cursor is positioned over it. Animations that support the opening of context mimics do not have this feature and so some other mechanism must be used to indicate to the user that a mimic may be opened from them.

The ability to open a Context mimic from an animation may be enabled and disabled using the option No context mimic in the variable definition.

Which mouse button opens context mimics is selected by the HMI option Mouse inputs accessible from the Configuration menu. The default is the right button. See HMI General Options topic for more information.
