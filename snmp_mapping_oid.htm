Mapping a Variable to SNMP Using the Mapping Dialog

Using the following procedure, you can generate a variable and map it to an OID, Trap or PING (unmanaged device) in one operation.

  1. Open the Application Explorer and expand the configuration tree to select either an SNMP network or device. Using the task bar, open the Mapping dialog.
  2. Expand the Device Tree until the OID, Trap name, or PING is visible in the top right pane. Show picture
  3. In the top right pane, right click on the row containing the OID name, and select Create Variable. The Create Variable dialog opens. Show picture
  4. Select the Polling Group using the drop down list box. This property is not required for a Trap.
  5. The variable name is generated automatically.
  6. Click OK to close the dialog and confirm the mapping.

The variable type that is generated will depend on the data type. See the topic SNMP variable data types.

How the variable name is generated if Use full hierarchical name as variable name is selected

  • For scalar variables - The variable name = Network Alias + Device Alias + Symbolic OID
  • For tabular variables - The variable name = Network Alias + Device Alias + Numerical OID

Because of the structure used by OID names it is possible that the calculated variable name will exceed either the number of characters (255) or branches (12) allowed by PcVue naming convention.

  • If the number of branches is exceeded PcVue automatically merges the rightmost branches by replacing the separators (full stop) with underscores.

  • In the number of characters is exceeded you must select Customize the variable name and manually edit the variable name.

How the OID property in the variable configuration is generated

  • If the property Use numerical OID in the SNMP Settings dialog is set, then the OID will be in the numeric format. Show picture
  • If the property Use numerical OID in the SNMP Settings dialog is not set, then the OID will be in the symbolic format. Show picture

In either case:

  • For a scalar variable, the OID is appended with a 0 (instance zero).
  • For a tabular variable, the OID is appended with the instance number.

Other variable properties that are generated

  • The Min and Max of a register variable will be taken from the OID supplementary information if available.
  • The Command property will be set if the OID is read/write.
  • The Maximum size (maximum number of characters) of a text variable will be taken from the OID supplementary information if available.

How does the polling option Trap only and ad-hoc synchronization affect the polling

In addition to any Polling Groups you have configured the Polling Group list box also contains an entry called Trap only and ad-hoc synchronization. If you choose this then the variable will only be updated when PcVue receives a trap from the device, or on ad-hoc synchronization (project start-up or communication recovery).

Limitations of mapping if the device is offline

If the device is offline, you cannot map tabular variables using the mapping dialog. This is because the MIB file does not contain information about the available instances.
