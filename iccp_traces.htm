ICCP Log Messages and Traces

There are two types of traces that will appear in the Event Viewer of PcVue, and in the Log Files: Log Messages and Traces.

Log Messages

This type of message is used for events that occur when the project starts (ICCP Control Center object creation and variable creation/modification) and with different connection events (connect, disconnect, etc.). This type of message can be configured in the ICCP.ini file with the field LogMessageLevel. There is a hierarchy of Log Message Levels for the different types of ICCP objects (Global/Family, Network, Local Control Center and Remote Control Center). Each one has its own LogMessageLevel, and if Default is selected, the specific object will use the LogMessageLevel of its parent object. During runtime it is possible to modify the LogMessageLevel of each object with the SCADA Basic verbs. However, the changes done at runtime will only persist while the project is running, they will not be saved to the ICCP.ini file.

It has five different level of details that the user can select for their purposes. A higher level includes the messages of all lower levels

  1. LogMessageLevel = 0: Default – This level is the default option for all ICCP objects (except for global). Using this level will apply the Log Message Level of the parent object.
  2. LogMessageLevel = 1: Minimum – This level will show the most basic messages that include detected errors in the inputs of the ICCP.ini file and in the variable creation. It will also display a basic message for connections/disconnections of remote control centers.
  3. LogMessageLevel = 2: Light details – This level includes more details of the control center creation, variable creation and connections/disconnections of a remote control center.
  4. LogMessageLevel = 3: Full details – This level has the most level of detail of the different events while still maintaining a tracing structure.
  5. LogMessageLevel = 4: Stack – This level includes messages directly from the underlying ICCP driver kernel. This implies that there will be less order in the messages.

It is important to note that Log Messages are not designed to display events that occur cyclically, like variable updates.

Traces

This type of messages are used for events that occur cyclically and must only be activated temporarily for diagnostic purpose. To enable these traces, the user must create a SCADA Basic program inside the PcVue project that enables the traces. The code required is the following:

To enable:

TRACE(1, 11, "03");

To disable:

TRACE(0, 11, "03");
  • The first argument can be 1 (to enable) or 0 (tor disable).
  • The second argument is the dll number that must be 11 if the ICCP dll is the first SV Manager Add-on currently loaded.
  • The third argument is an hexadecimal number that represents the bit mask that are selected for the trace to be enabled or disabled.

SCADA Basic example: To activate/deactivate Traces for bits 0, 1, 2, 3 and 4.

Sub ActivateTraces()
 
    TRACE(1,11,"1");
 
    TRACE(1,11,"2");
 
    TRACE(1,11,"4");
 
    TRACE(1,11,"8");
 
    TRACE(1,11,"10");
 
End Sub
 
 
 
Sub DeactivateTraces()
 
    TRACE(0,11,"1");
 
    TRACE(0,11,"2");
 
    TRACE(0,11,"4");
 
    TRACE(0,11,"8");
 
    TRACE(0,11,"10");
 
End Sub

The meaning of the bits are summarized in the following table. Information regarding variable updates are obtained with bits 1-4.

Bit Description
0 Errors
1 Basic information - Local changes: Local server data updates (to remote) and remote client device command updates (update to remote device)
2 Basic information - Remote changes: Remote server data updates (from remote) or local client device updates (update from remote device)
3 Detailed information for Bit1
4 Detailed information for Bit2
5 General information
6 Stack messages
7 Other
