Pima

This driver supports communication over Tcp/Ip with PIMA IPGuard Receiver software series.

Supported functions

  • Receiving unsolicited messages

    • Report

    • Idle message

    • Zone status synchronization result

    • Raw message result

  • Sending messages

    • Request information

    • Zone status for synchronization

  • Sending commands

    • Raw message with free text

  • Sending acknowledgment when a Report, Request message result and Raw message result is received

Frame addressing

Depending on the equipment type you have selected when configuring the device, you have access to the following type of frames.

The Starting address and Quantity properties define the memory address range for each frame, how it is read from and written to, and how it is interpreted in PcVue.

Offset: CID Event Code (see Annexes)

0 - Restore/Arming

1 - Alarm/Disarming

Request/Response:

Unsolicited Reports reception from Report Channel

Parameter read Command via Control Channel

Data synchronization is necessary when no data is received via Report Channel for a period of time.

For frame data synchronization, Parameter Read Command (prmrd) is used. It reads Zones Status parameter (2149) and set corresponding Events in Reports frame.

This frame is cyclic.

Mapping:

Address 0: Idle state

0 – Report Channel in the synchronized state

1 – Report Channel in the idle state

Unsolicited data reception from Report Channel

This is a service frame for Report Channel. If no data is received via Report Channel between two readings of this frame, all Reports frames become invalid and must be synchronized.
