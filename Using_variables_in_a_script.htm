Using variables in a VBA script

Before you can use the properties, methods and events of a variable configured in PcVue in a script, the variable must first be subscribed.

Variables are subscribed in the following ways:

  • Automatically when used in an animation of any open mimic
  • Automatically whenever a variables value is referenced in a VBA program with the [] syntax
  • Manually by using the Add method of the Variables collection

Subscribing a variable

Any variables used in any animation in an opened mimic are automatically made available to VBA. It is unnecessary to programmatically subscribe to the variable as described below. However, when the last mimic using a particular variable in an animation is closed, that variable is no longer available to VBA unless it is (also) programmatically subscribed.

To subscribe a variable by code, you use the Variables.Add method.

For example:

Variables.Add "Branch1.B1", fvVariableTypeBit
Variables.Add "CommentDisplay%", fvVariableTypeText

The optional second parameter describes the variable type. For more information, see the Variables Collection topic.

If the variable is already subscribed by an animation in the HMI, it may still be subscribed explicitly through VBA. When the mimic containing the animation is closed, the variable will then remain subscribed until the project is closed or it is unsubscribed in VBA.

You must use the full variable name when you subscribe a variable.

Some variables may need to be subscribed 'permanently' while the project is running. For example, variables for which a script must be executed upon value change, independently from open mimics. You can subscribe to these variables in the call-back script of the event StartupComplete, and unsubscribe in the call-back script of the event BeforeQuit.

Unsubscribing a variable

To unsubscribe a variable you use the Variables.Remove method.

For example:

Variables.Remove "Branch1.B1"

If you subscribe a variable more than once in VBA, you have to unsubscribe it that many times before the variable can become unsubscribed. Even then, it remains subscribed in the HMI so long as any mimic that uses it in animations remains open.

Each variable that you subscribe uses some slight system resources. It is recommended to unsubscribe a variable as soon as it is no longer needed.

Shortcuts for accessing variables in VBA

There is a variety of syntax shortcuts to address a Variable and the Variables collection.

The Item method

The Item method is the default for the Variables collection, and the Value property is the default for the Variable object. The default method and propety of an object can always be omitted. So:

Variables("CommentDisplay") = "Hello World"

is the equivalent of:

Variables.Item("CommentDisplay").Value = "Hello World"

The [...] syntax

The [...] syntax provides an additional shortcut to directly access a variable's value.

[Main.Branch1.B1]

is the equivalent of

Variables("Main.Branch1.B1")

It is recommended that you use this shortcut to make your code more readable.

When you use the [...] syntax, the variable is automatically subscribed and unsubscribed - there is no requirement to do it programmatically earlier in your code.

Example:

Private Sub Shape1_Click()
If [Main.Branch1.B1] = True Then
  [Main.Branch1.B1] = False
Else
  [Main.Branch1.B1] = True
End If
End Sub

Running VBA code when a variable's value changes

You can execute code when a variable's value changes, using the Dim WithEvents statement and the ValueChange event.

This is best illustrated with a simple example:

'------ This code must be placed in General Declarations
Dim WithEvents objVar As Variable

'------ Execute to set up events
Private Sub fvProject_StartupComplete()
'------ Subscribe to variables
Variables.Add "Branch1.B1", fvVariableTypeBit
' ------ Assigns an object reference to the variable and enable the events
Set objVar = [Branch1.B1]
objVar.EnableEvents = True
End Sub

'------ Execute when value of Main:Branch1.B1 changes
Private Sub objVar_ValueChange()
MsgBox objVar.Name & " = " & objVar.Value
End Sub

You must declare the Variable object using the WithEvents keyword and enable events for the object by setting the EnableEvents property to True.

If the code is contained within a mimic, any references to objects are temporary. When the mimic is closed, or its mode changed (Run/Design), the reference is deleted and the event no longer fired. For the event to be fired again, the object must be reassigned to the variable and the events enabled again.
