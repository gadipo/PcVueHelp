SNMP dynamic table management extension

 PcVue provides a way to access and retrieve information that are managed in tables by certain devices, in most cases to expose a dynamic list of alarms. This works by enabling the dynamic table management SNMP extension. A dynamic table is an OID exposing a tabular value, a table having a varying number or elements (rows). The dynamic table management extension works along the SNMP Manager of PcVue.

Activating dynamic table management

To activate the dynamic table management extension, rename the file svmgrSnmpExtensions_UsrMgr.dat located in the BIN directory of PcVue into UsrMgr.dat.

If the file svmgrSnmpExtensions_UsrMgr.dat is missing, you can simply create a text file named UsrMgr.dat with the following content:

[USRMGR\svmgrSnmpExtensions]
 
DLL=svmgrSnmpExtensions.dll

Go to the C directory of your PcVue project and create a file named svmgrSnmpExtensions.ini. You are going to use this file to configure the dynamic table management extension.

Configuring the svmgrSnmpExtensions.ini file

If an optional entry is omitted from the file, then its default value is used.
If the file or a mandatory entry is missing, then the task is disabled.

The following is the file format of the svmgrSnmpExtensions.ini file. Configure this file to define the dynamic table structure and the columns that you would like to access in your project from the device. If a dynamic table is for alarms, columns usually contain the alarm description, severity etc.

[TaskParameters]
 
AttributeNumber = 16
 
AssociationName = MyServerAssoc
 
AttributeSeparator = |
 
 
 
 
 
[VariablesPool]
 
NamePrefix = POOL.ALARM
 
Number = 20
 
 
 
 
 
[ColumnsTemplate1]
 
Name = AlarmColTemplate1
 
DeviceName = Attribute3
 
Column0 = NotUsed
 
Column1 = Timestamp 
 
Column2 = Attribute4
 
Column3 = Attribute5
 
 

The parameters in this file are:

  • AttributeNumber: Specifies the extended attribute used for configuring variables for the dynamic SNMP pieces of information. Format: Number from 3 to 16. If omitted, default is 16.

  • AssociationName: Specifies the association name in a redundant architecture. The task is only active on the active server. Generally, it is the same association as the one that manages the connection to SNMP devices and produces the SNMP I/O variables. Format: String. If omitted, default is an empty string.

  • AttributeSeparator: Specifies the separator used in the extended attribute when defining the table subscription variable. Format: String. If omitted, default to '|'.

  • [VariablesPool]: Specifies the pool of variables that will be used to pass an element of the dynamic table received.

    • NamePrefix: Specifies the prefix of variable names. Format: String. If omitted, default is empty.

    • Number: Specifies the number of variables configured in the application for the pool (with the given prefix). Format: Number. If entry omitted, default is 0. This number must be greater than the maximum number of dynamic elements you expect to receive at any given time across all dynamic table OIDs of all SNMP Agent devices configured in your application.

  • [ColumnsTemplateX]: Specifies the columns of interest for a given table. The numbers X must be contiguous if you want to specify multiple column templates. Each template is composed of:

    • Name: Specifies the name of the template that will be used at the table subscription variable. Format: String. If omitted, the variable template name is ColumnsTemplateX where X is the template instance number.

    • DeviceName: Specifies in which variable extended attribute the device name shall be stored.

    • ColumnX: Specifies the mapping of the column X of the table. The template must define contiguous column numbers starting at 0. Format: String. The possible column values are:

      • NotUsed: The column will not be passed to the variable in the variable pool.

      • Timestamp: The column value will be interpreted as the timestamp of the variable in the variable pool.

      • Attribute3 to Attribute16: The column value will be written on the specified extended attribute of the variable in the variable pool.

        The Column0 represent the instance of the row within the table received.

Configuring the variables of the variable pool

Once the Add-on is configured, you need to create and configure the variables associated to the variable pool. As an example, the logic is that if you have defined the variable pool as having the prefix POOL.ALARM and the number of variables in the pool to 20, you need to configure 20 internal variables named POOL.ALARM1, POOL.ALARM2..., POOL.ALARM20, typically of type bit or alarm.

Customize the variable properties as appropriate to fit your requirements: Domain, nature, alarm, alarm level...

Configuring the variables for the table OIDs

For each dynamic table OID you need to use, an internal variable must be configured and its extended attribute, the one specified in the task parameters, filled with the necessary information for the dynamic table management add-on to process it. This variable is used to retrieve the table elements and dynamically set properties of the variables in the variable pool.

  1. Configure the SNMP driver, at least a network and a device.

  2. Create a new internal variable.

  3. Select the Extended attributes tab and tick Enable extended attributes.

  4. Click Apply to activate the extended attributes editing.

  5. In the extended attribute specified in the [TaskParameters] section of the svmgrSnmpExtensions.ini file, enter the following information:

    • SNMPTABLE: Keyword to define that this variable is linked to a table handled with the SNMP manager extension.

    • Network: Name of the SNMP network.

    • Device: Name of the SNMP device.

    • PollingGroup: Name of the polling group to use to read the table cyclically.

    • TableOid: OID of the table you would like to read elements from.

    • ColumnsTemplate: Name of the template to apply to the specified table to map columns. It must correspond to one of the templates defined in the .ini file.

    For example, if you have specified the separator '|' in the .ini file, an example of value for the extended attribute is:

    SNMPTABLE|Network01|Device01|PollingGroup01|.1.3.6.1.2.1.1.9|AlarmColTemplate1
    		 
     

  6. Click OK to close the dialog box.

  7. Repeat steps 2 to 6 for each table OID you need to process.

To get information regarding the svmgrSnmpExtensions task while running the application, check the Event viewer for lines starting with [SnmpExtensions].
