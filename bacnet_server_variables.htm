Configuration of the Variables

To define a variable so that it is exposed as a BACnet object, you enter its BACnet definition in the variable's text extended attribute as defined in the BACnetServerConfig.xml.
It takes the form of a string with the following syntax.

Analog objects - BACnet:<ObjectType>:<InstanceNumber>::::::<COVIncrement>

Binary objects - BACnet:<ObjectType>:<InstanceNumber>:::::<ActiveText>:<InactiveText>

Where:

  • BACnet - The keyword that indicates it is a variable exposed as a BACnet object,
  • <ObjectType> - Either AI, AO or AV for analog objects and BI, BO or BV for binary objects,
  • <InstanceNumber> - The object instance number,
  • <COVIncrement> - The minimum change in Present value required to trigger a COV notification for the subscribed COV clients,
  • <ActiveText> - A character string describing the Present value active state,
  • <InactiveText> - A character string describing the Present value inactive state.

Any variable defined as a BACnet object is only exposed after the project has been stopped and re-started. Any errors resulting from invalid object creation (incorrect syntax in the text attribute for example) will be detected and traced at that time.

The following shorten syntax can also be used:

  • For analog objects - BACnet:<ObjectType>:<InstanceNumber>
  • For binary objects - BACnet:<ObjectType>:<InstanceNumber>

How a variable's real time value is exposed as the BACnet Present Value property

Change in value of a variable results in a transfer to the Present Value property for the following object types:

  • Analog Input
  • Binary Input

Change in value of a variable also results in a transfer to the Present Value property for the following commandable object types but without any Priority Value, thus the Priority Array is left unchanged:

  • Analog Output

  • Analog Value

  • Binary Output

  • Binary Value

The Priority Array property of these objects is entirely initialized with NULL values, and in doing so, the Relinquish Default Property value is given to the Present Value Property on-start up.

How a variable's Unit property is exposed as the BACnet object Units property

The Unit property of a register variable is a free text representing the unit associated with the value of the variable. For example:

  • The string can represent the symbol associated with the unit as defined in the SI or Imperial Units System such as "oC" for Celsius degrees or "oF" for Fahrenheit degrees,

  • The string can represent a commonly used abbreviation of a unit such as "mV" for millivolts or "rpm" for revolutions per minute,

  • The string can represent a complete word describing the unit (such as "meter" for American English or "metre" for British English).

The correspondence between strings used in the Unit property of exposed register variables and BACnet Engineering Unit numeric Ids has to be pre-defined and stored in the file BACnetServerUnits.xml.
A sample file is provided and more unit correspondences can be added using the following format:

<units tag="correspondence string">BACnetEngineeringUnits numeric Id</units>

For example:

<units tag="square feet">1</units>

The string comparison is case sensitive. By default, or for an invalid string, the value 95 corresponding to NoUnits is exposed as the Unit property. The same applies if the file cannot be found.

Other variable properties exposed as BACnet properties

  • Name - Directly exposed as the Name property,
  • Description - Directly exposed as the Description property,
  • Minimum value - Directly exposed as the Min Pres Value property. This property is present for the Analog Input, Analog Output and Analog Value objects,
  • Maximum value - Directly exposed as the Max Pres Value property. Those properties are present for the Analog Input, Analog Output and Analog Value objects.

An auto-completion mechanism is available to help you define the BACnet Server mapping via an Extended text attribute

It is enabled via a property in the configuration file BACnetServerConfig.xml.

After declaration of a BACnet object and restarting the project, the activated Extended String attribute offer auto-completion for the BACnet server syntax.

For instance, an Analog Input object may have the following associated Extended String attribute.

BACnet:AI:1::::::COV_I

Auto-completion suggests which parameters are available for a specific object depending on its type and by concatenating the declaration string with a string containing the mnemonics of the parameters.

The entire auto-completion strings are as follows.

For Analog Input, Analog Output and Analog Value objects:

::::::COV_I

For Binary Input, Binary Output and Binary Value objects:

:::::ACT_T:INA_T
