Maintaining PcVue 

Maintenance is the name given to the process of uninstalling, modifying, or repairing the software programs.

  • Remove - Uninstall the software.
  • Modify - Modify the list of installed features, add or remove features.
  • Repair - Repair any installed component.

Important information to know before maintaining PcVue or the Web Deployment Tools

  • Removing PcVue or the Web Deployment Tools does not remove the prerequisites. See below Maintenance of the prerequisite installation for more information.
  • Maintaining PcVue does not affect any projects that have been configured. However, as a precaution, you should always make a backup copy of any project and library you have configured, and want to keep, before using maintenance.
  • Our policy of continuous improvement means that the shared libraries supplied with PcVue are regularly updated. There may be minor changes between different updates of the same version of PcVue. The uninstallation process will remove the shared libraries and any library file or folder created by the installation, even if it has changed. If your project uses the shared libraries it is essential that you make a backup copy before the uninstall process even if you are reinstalling another copy of the same version of PcVue.

Maintenance of PcVue installation

Maintenance can be started in one of two ways.

  • Using the original installation media.
  • Using the Modify or Repair modes available via the Apps & features settings or the Add/Remove program tool in the Windows Control Panel.

After a Welcome dialog, the Program Maintenance dialog is displayed. Show picture

  1. Select either Modify, Repair or Remove as appropriate.
  2. Follow the sequence of dialogs to carry out the selected process.
    • Modify guides you through adding or removing optional features. Modify is not available for the Web Deployment Tools as there are no optional features.
    • Repair Checks the installed features and re-installs properly any that could have been damaged.
    • Remove Displays a confirmation dialog then starts the process of de-installation.

Maintenance of the prerequisite installation

Prerequisites are installed as a bundle containing one or more components. The bundle, and the components it includes, can be seen in the Apps & features settings (or Programs and Features in the Windows Control Panel).

The prerequisites bundle for PcVue is called PcVue Prerequisite, and the prerequisites bundle for the Web Deployment Tools is called Web Deployment Tools Prerequisite.

Maintenance of any of the prerequisites is started by selecting it in the Programs and Features list and then clicking Uninstall, Repair or Re-install as appropriate.

Uninstalling prerequisites

Uninstalling the prerequisites is only necessary if you wish to fully uninstall software components that were installed as part of installation. Uninstalling any of the prerequisites, whilst keeping the product installed, will prevent it from running properly.

Uninstall the prerequisites bundle does not uninstall the individual components. However, you can completely uninstall the prerequisites by uninstalling the bundle as well as the individual components. Uninstalling an individual component may fail if it is used by other software, or the other software may fail to operate.

Repairing prerequisites

You can repair a bundle or an individual component. If you repair a bundle, any missing individual components will be installed and any already installed individual components will be repaired.

Re-installing prerequisites

Any missing component can be installed individually. However, if you are missing a component, clicking the Re-install button of the corresponding bundle will re-install any missing component that are part of the bundle.
