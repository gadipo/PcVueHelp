Visibility animations

These animations provide a direct method of controlling the visibility of any graphical element using one or more variables.

They can be used on a drawing element, drawing group, symbol, image or graphic control.

Using the visibility animations you can choose to make the graphical element visible (show) or invisible (hide) or select blinking between visible and invisible at one of the pre-set blink rates.

A graphical element's visibility can also be read and written from VBA using the RunTimeVisibility property.

At the time of writing, visibility animations are not available for the following controls: Map control, Web browser control and Messaging controls.

Visibility on variable value animations

These animations provide a direct method of controlling the visibility of any graphical element using the value of one or more variables.

The table below details the behavior of each animation:

Animation

Behavior

Bit

Selection of the visibility according to the state of a bit variable - on, off or invalid. Show picture

Bit group

Selection of the visibility according to the binary combination of up to 4 bit variables, plus a further visibility state if any of the bits is invalid. Show picture

Register bit

 

Selection of the visibility according to a bit mask on a register. For example, if you use the 3rd bit, when the binary representation of the register is xxxxxxxxxxxx1xx the color selected for 1 will be displayed. When the binary representation of the register is xxxxxxxxxxxx0xx the color selected for 0 will be displayed (where x is either 1 or 0). Show picture

Alarm

Selection of the visibility according to the value of an alarm variable (Off, On - Nack, On - Ack, Off - Nack, Inhibited). Show picture

Register value

Selection of up to eight states according to value of a single register plus a further two according to the value being invalid or outside of the specified range. See the page Understanding the Thresholds Used in Color Register Value for an explanation of threshold operation. Show picture
Text value

Selection of the visibilty according to the equality of a text variable to 32 entered strings. The animation is case sensitive or not depending on the value of the following Mimic Options: Enable case sensitivity for animations that compare text strings case sensitive. For example, if set for being case insensitive, HELLO, Hello, and hELLO would be treated as the same text value. A further three states can be selected according to the text variable being empty (null), not equal to any of the entered strings, or invalid. Show picture

Visibility on variable presence animation

This animation provides a direct method of controlling the visibility of any graphical element using the existence (or absence) of a named variable in the project configuration and is similar in operation to the animations Visibility on variable value. Show picture


The animation Visibility on variable presence makes it straightforward to design highly generic graphic elements.

If you use the Application Architect to model slightly different equipment pieces using inheritance, a motor with or without a thermal alarm for example, the animation allows you to design a generic symbol or a generic popup mimic in which any graphical element associated to the thermal alarm will be automatically hidden if the thermal alarm variable does not exist for some motors (i.e. for some variable branches, the execution context of a generic symbol or popup).

Changing the blink periods

You can configure the Blink Period options from the menu using the Configure.Preferences.Blinking command, as described in the topic Selecting a Blinking color.
