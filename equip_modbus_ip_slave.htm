Modbus Ip slave

This driver supports communication over Ip with Modbus master field devices.

In all exchanges, PcVue is the slave and the field device is the master. The driver only supports the RTU transmission format.

If you need the field device to be slave and PcVue to be master, you should use the Modbus Ip driver instead.

You can use the Modbus master and Modbus slave drivers simultaneously.

Equipment type Select the type of equipment that best suits the field device you want communicate with.
The list of frame types and address ranges depend on the equipment type.
  With the Modbus Ip slave driver, the property Equipment type allows you to select between Modbus and Jbus addressing, decimal and hexadecimal encoding of memory addresses.
Modbus frame addresses start at 1, Jbus frame addresses start at 0.
Initial memory state

Defines how the slave memory areas are to be initialized at Equipment communication start-up:

  • Uninitialized - The memory will not be initialized, and the error code 0x040 is returned until values are available.

  • Initialized to 0 - The memory is zeroed.

  • Saved - The values in the memory zone are saved when Equipment communication is stopped and restored again on restart. The files in which the values are saved are found in the folder PER\CWIPXBUS\ of the project. The file name is constructed from the device name and memory area.

The length of Wordbit frames must be a multiple of 16, error 0040 is returned otherwise.

If a writable frame is configured with no polling period, all writes from a master are notified immediately with no delay refreshing the variables. This is the recommended setting unless the master is writing large amounts of data and swamping the slave.
