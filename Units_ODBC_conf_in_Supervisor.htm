Configuring an ODBC Database in PcVue 

Before an ODBC database can be used by PcVue, it must first be added to the Databases folder of the Archives Configuration Explorer. If the database is to be used just for recipes then no further configuration is required. If the database is to be used to record trend, log or report data then an ODBC archive unit must also be created.

How to add an ODBC database to the Databases folder of the Archives Configuration Explorer

  1. Open Application Explorer, expand the configuration tree and select the Databases folder.
  2. From the task list, select Add an ODBC database. The ODBC database dialog is displayed.
  3. Enter a name by which the database will be known in PcVue in the Name text box. You can enter an optional description for the database in the Description text box.
  4. Enter the name of the database, as configured in Windows' ODBC Setup dialog, in the ODBC name text box. Show picture

  5. Configure any ODBC database options (see list below).
  6. Click OK to confirm the configuration and close the dialog.

For information on how to monitor the connection to the underlying ODBC data source, see the topic ODBC status variables.

Options when configuring an ODBC database

In most applications, the following properties will not need to be changed and may be left at their default value.

In the General tab:

  • Database type - The structure and number of the tables with the database was changed between version 6 and 7 of PcVue to optimize efficiency. Select Old only if the project is to be compatible with version 6, otherwise leave at the default setting of Optimized.
  • Read only - The database is read only (no data can be written by PcVue).
  • Manual commit - Enables a manual commit of the database at the specified period.

In the Advanced tab:

  • Login name - The name of the user for connection to the ODBC source (if it was configured with one).
  • Password - The password for the connection to the ODBC source (if it was configured with one).
  • Connection time-out - The time in seconds allowed for a response from the ODBC source when an attempt at connection is made. Default 15 seconds
  • Query time-out - The time in seconds allowed for a response from the ODBC source when a query is made. Default 15 second.
  • Delimiter for identifiers - Some RDBMS require the use of a particular text delimiter in queries. Select single quote if you are using a MySQL DB.

The value of the properties Connection time-out and Query time-out are passed to the underlying ODBCdriver configured in the Windows ODBC Data Source. They may be ignored by some ODBCdrivers.

ODBC table names

 PcVueuses the following default table names to record data. If required, the table names can changed using the Tables tab.

  • Version 6.0
    • HIS - Historic (trend), Log and Report data.
    • RECH - Recipe configuration information
    • RECV - Recipe variable information.
  • Version 7.0 onwards
    • TRD - Data generated by the historic trend mechanism and played back through the Trend Viewer.
    • LOG - Data generated by the log lists and played back by the Log Viewer.
    • REP - Data generated by the report mechanism.
    • RECH - Recipe configuration information.
    • RECV - Recipe variable information.
