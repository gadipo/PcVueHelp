Other runtime behaviors

Datatype conversion

If the variable type is different to that of the BACnet object, datatype conversion takes place automatically.

  • Bit variables exposed as Analog objects - The value is converted from the Boolean representation to the numeric representation (0 or 1) for Analog Input, Analog Output and Analog Value objects.
  • Register variables exposed as Binary objects - The value is converted to true if the value is different from zero, and false if its value is equal to zero for Binary Input, Binary Output and Binary Value objects.
  • Text variables exposed as Analog or Binary objects - The value is converted from its string representation into a real value for Analog Input, Analog Output and Analog Value objects. The value is converted from its string representation into 0 or 1 for binary values for Binary Input and Binary Output objects.

Commandable objects

Variables can be exposed as commandable objects. Multiple attempts to write to the Present Value property, at the same time, may result in conflicts. Consequently, the standard BACnet arbitration system is used, based on command prioritization. The priority associated with a command is ranged from 1 to 16, with 1 being the highest.

The Priority Array property contains the 16 values associated with the prioritized commands and is entirely initialized with null values at startup. Any non-null value is considered a command. The highest priority command in the array is the active command and the value contained in the Present Value property matches this active command.

If no command is active, and therefore the Priority Array is completely filled with null values, the Relinquish Default property is used for the Present Value.

The supported commandable objects are Analog Output, Binary Output, Analog Value and Binary Value.

The property Priority Array of commandable objects is not persisted when shutting down PcVue and is initialized with null values at start-up. It is the responsibility of BACnet clients to ensure setting the priority properly when sending a command.

Clock management

Local date and time are provided by the Local Time and Local Date properties of the Device object. The time and date originate from the system running the Add-on and automatically adjust for system and time zone changes.

Current Daylight Saving Status and UTC Offset are also provided within the Device object.
