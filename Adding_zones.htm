How to Add a Zone to a Template

This topic explains how to add a zone to a template using the Application Architect.

How to add a zone to a template

  1. Navigate to a template's Configuration elements folder.
    1. In the right pane of the Application Architect, select the Templates tab.
    2. Using the templates configuration tree select the template to which the zone will be added.
    3. Using the secondary navigation tree select the Configuration elements folder.
  2. Add and configure the zone.
    1. Select Add a geographical data > Zone. A new zone with a default name is created.
    2. Edit the name to one appropriate to your application. You can only edit the name directly by clicking on it in the configuration tree.
    3. Configure the zone's properties. Show picture

After a zone is configured, further geographical elements can be added to it using either the tool bar or the right click context menu.

  • Add a zone - To add a child zone to the selected zone, follow the above method from step 2 onwards.
  • Add an action.
  • Associate a geo-tag.

How to associate a geo-tag with a zone

  1. Navigate to the zone with which the geo-tag will be associated.
    1. Right click and, from the context menu select Associate a geo-tag. A new geo-tag association is created with a default name.
    2. Edit the name to one appropriate to your application. You can only edit the name directly by clicking on it in the configuration tree.
    3. Configure the geo-tag to associate using the Tag name property. You can either select an existing geo-tag using the drop down list box or use Input, Expression or Parameter. Show picture

How to add an action to a zone

  1. Navigate to the zone to which the action will be added.
    1. Right click and, from the context menu select Add an action. A sub-menu with the available action types is displayed. Show picture
    2. Select the action type. A new action is created with a default name.
    3. Edit the name to one appropriate to your application. You can only edit the name directly by clicking on it in the configuration tree.
    4. Configure the action properties. The properties depend on the action type.
