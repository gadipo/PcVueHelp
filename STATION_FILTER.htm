STATION_FILTER

See also Example

Manage the application of population filters to functions and data.

WebVue support - Yes.

Population filtering based on the Source is not supported to filter in log viewers (and log printing) when data are archived using a database archive unit (HDS).

Mode

Mnemonic

Syntax
1 CLEAR 1
2 APPLY 2, 3
3 DUMP 1

All Syntaxes

Argument

Meaning

Distribution

A numeric code, represented as a bitfield, that defines which functions of PcVue is to be affected:

  • Bit 1 - Alarm Viewers and audible alarm.
  • Bit 2 - Log Viewers.
  • Bit 3 - Recording log data on the printer.
  • Bit 4 - Recording log data on the archive units.
  • Bit 5 - Sending values by the operator.
  • Bit 7 - Sending values from the program language.
  • Bit 8 - Acknowledging alarms using the Run macro animation.
  • Bit 9 - Acknowledging alarms using the program language.

Type INTEGER.

Combinations of functions are enabled by setting the corresponding bits in the Distribution field. For example, Distribution = 3 (binary 00000011) activates Bit 1 (Alarm viewer display) and Bit 2 (Log viewer display).
List

A comma-separated list of population names, for example ZONE1,ZONE2,COMMON. Type STR.

The list is limited to a maximum of 64 names and a total of 1,024 characters.

Syntax 1

IntVal = STATION_FILTER(Mode, Distribution);

Return type: INTEGER.

Execution

Mode

Mnemonic

Action

1

CLEAR

All population filters that have been previously applied for the function corresponding to Distribution are cleared.

3

DUMP

Display the configuration of Distribution in the results area of the SCADA BASIC Program Management window (you can display this window using F9).

 

 

Return: 1 if successful, else 0.

Syntax 2

IntVal = STATION_FILTER(Mode, Distribution, List);

Return type: INTEGER.

Execution

Mode

Mnemonic

Action

2

APPLY

The list of population filters defined in List is applied to the function defined by Distribution. An OR is applied, any item matching one or the other population filter will be accessible to the user.

Return: 1 if successful, else 0.

Syntax 3

IntVal = STATION_FILTER(Mode, Handle);

Return type: INTEGER.

Argument

Meaning

Handle

The location of a memory buffer containing a function code and a list of population filters. Type LONG.

Memory buffer example:

Format: FILTER,0,Distribution,List

Execution

Mode

Mnemonic

Action

2

APPLY

The list of population filters defined in Handle is applied to the function defined in Handle. An OR is applied, any item matching one or the other population filter will be accessible to the user.

Return: 1 if successful, else 0.
