The Timetable Animation

A Timetable is used in conjunction with the Scheduler to produce time-based events. Each Timetable is based on a standard week with a number of exception periods. For more information, on Timetables see the topic on the Scheduler in the configuration book. Timetables may be viewed, modified and saved at run-time using the Send Timetable animation. Show picture

The Timetable that may be changed is selected from the Label drop down list box.

Selecting the runtime behavior

The access that the user is given to a timetable is defined by the behavior properties.

  • Add/Modify exceptions - If ticked, the user may change the time periods of existing exception periods and add new exception periods.
  • Modify standard week - If ticked, the user may change the time periods of the standard week.
  • Enable date change - If ticked, the date of existing exception periods may be changed.
  • Enable save - If ticked, any changes made may be saved on disk.
