Overview of the Log Viewer at Run-time

A Log Viewer displays data that has been recorded through a particular Log List. The Log List is attached to the Log Viewer when the Log Viewer is configured. The types of data that can be displayed are as follows :

  • Alarm changes of state.
    • On and not acknowledged.
    • On and acknowledged.
    • Off and not acknowledged.
    • Off.
    • Unavailable.
  • Bit changes of state.
    • Change to zero.
    • Change to one.
    • Change to invalid.
  • Operator actions.
    • Commands.
    • Logon and Logoff.
    • Alarm acknowledgement.
    • Alarm masking.
    • Program start.

By using different Log Lists, Log Viewers can be configured to create historic alarms lists, process event logs and operator action logs. The Domain and Nature options may be used to configure different Log Viewers for different process areas.

When a Log Viewer is opened, log data is extracted from the archive files through the Log List and saved temporarily in the display buffer. Data is extracted for the current time backwards until the display buffer is full or there is no more data. Data is sorted for display according to the sort column that was chosen for sorting, by default the date and time.

Selecting Logs

Individual logs are selected by clicking on the line in the display list. A selected log is indicated by displaying it in reverse video. Selected logs can be printed using the Print tool on the toolbar or context menu, and copied to the Windows clipboard using the Ctrl+C keyboard shortcut.
