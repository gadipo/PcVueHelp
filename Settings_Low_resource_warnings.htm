Low Resource Warnings

Show picture

  • Monitor GDI objects usage - Monitor the number of GDI (Graphic Device Interface) objects used by the HMI. The warning threshold can be customized, even if it is recommended to keep the default value of 90%. The result is placed in the system variable SYSTEM.LOCALHOST.RESOURCE.GDI.
  • Monitor User objects usage - Monitor the number of User objects used by the HMI. The warning threshold can be customized, even if it is recommended to keep the default value of 90%. The result is placed in the system variable SYSTEM.LOCALHOST.RESOURCE.USER.
  • Monitor global system memory load - Monitor the global system memory load. The result is placed in the system variable SYSTEM.LOCALHOST.RESOURCE.RAM.
  • Monitor opened mimics count - Monitor the number of mimics currently opened in the HMI. The result is placed in the system variable - SYSTEM.LOCALHOST.MIMIC.OPENCOUNT

The operating system settings affecting the maximum number of GDI and User objects a process can allocate can be set using the SV Core Management Console. See the Deployment book for more information.
