Animation Properties in WebVue

This topic outlines the extent to which the WebVue client supports the display of animations as configured in the PcVue desktop application.

For details of how to configure animations in PcVue, please refer to the Animation book in the main Help.

The following animations are not supported or not relevant in WebVue contexts:

  • Color - Legend animation - Not supported
  • Send - Recipe animation - Not supported
  • Send - Region animation - Not applicable
  • Send - Languages animation - Not applicable, the Display language is taken from the web browser language, and the Project language from the Web back end and user profile configuration
  • Send - Timetable animation - Not applicable - Use the WebScheduler instead
  • Run - Macro animation - Not supported
  • Security device - PTZ - Not supported
  • Web browser - Commands - Not supported
  • Security - Password animation - Not applicable - Users can change their password from the WebVue login page

In addition, the following animations are partially supported or have a specific behavior in WebVue contexts.

Multiline comment displayed as tooltip at runtime is supported in WebVue.

Send text

Animation WebVue support
Send text with default Value The setting Show default value on Mouseover is not supported. This option allows the user to see the configured default value before sending it. Since it is not available, the user cannot see which value the variable will have after triggering the animation.

Send register

Animation WebVue support
Send register with default Value The setting Show default value on Mouseover is not supported. This option allows the user to see the configured default value before sending it. Since it is not available, the user cannot see which value the variable will have after triggering the animation.
Send register with Step mode In the PcVue desktop application, the field shows the current register value and on MouseOver the next step (after triggering the animation) will be displayed. As MouseOver is not supported in WebVue, it will directly show the value that would be sent to the variable if the animation were triggered.
