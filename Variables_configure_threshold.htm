Configuring a Threshold

A threshold is used to set a bit variable to 1 when a register rises above, or falls below, a specified value. The bit that is set would most commonly be used as an alarm, but there are other uses such simple on-off control.

The threshold system is configured individually for each variable from one of the following combinations.

ppphigh / pphigh / hihi / high

4 x above

ppHigh / hihi / high / low

3 x above and 1 x below

hihi / high / low / lolo

2 x above and 2 x below

high / low / lolo / pplow

1 x above and 3 x below

low / lolo / pplow / ppplow

4 x below

The exact behavior of thresholds is affected by the property Trigger on strict going beyond threshold found in the Variables Tree Settings dialog.

How to create a threshold

  1. Open the Application Explorer and use the configuration tree to select the Variables Tree and then the register variable to which the threshold is to be added.
  2. Select the Add a threshold tool from the Associated Behaviors toolbar. The Thresholds Configuration dialog opens.
  3. Select the Thresholds type using the drop down list box of the same name.
  4. Enable the required thresholds by selecting the corresponding tick boxes. The name of the associated bit variable is automatically generated. You can accept the default name, type in an alternative name, or select an existing bit variable using the Variable Select dialog, which can be opened using the Ellipsis button adjacent to the field. Unless there is good reason not to do so, it is recommended that you accept the default name.
  5. Enter a value for the threshold (in engineering units). Show picture
  6. Optionally, enter a value for the hysteresis (in engineering units).
    The value of the hysteresis acts as a deadband in absolute value and is only applied when the register variable falls below the threshold value for a high value threshold (resp. rise above for a low value threshold). For a high value threshold, the trigger bit variable will be reset if the new register value is less than threshold minus hysteresis (resp for a low value threshold, if the new register value is more than threshold plus hysteresis).
  7. Click OK to confirm the configuration and close the dialog.

Examples

With a high value threshold configuration, using the threshold value 100 and the hysteresis value of 5:

  • If the register value changes from 98 to 102, the register is above the threshold value, the threshold bit is set to 1.

  • If the register value then changes from 102 to 110, the register is still above the threshold value, the threshold bit remains set.

  • If the register value then changes from 110 to 98, the register is below the threshold value but still in the hysteresis range, the threshold bit remains set.

  • If the register value then changes from 98 to 92, the register is below the threshold value and no longer in the hysteresis range, the threshold bit is reset to 0.

Respectively with a low value threshold configuration using the threshold value 10 and the hysteresis value of 5:

  • If the register value changes from 15 to 8, the register is below the threshold value, the threshold bit is set to 1.

  • If the register value then changes from 8 to 5, the register is still below the threshold value, the threshold bit remains set.

  • If the register value then changes from 5 to 12, the register is above the threshold value but still in the hysteresis range, the threshold bit remains set.

  • If the register value then changes from 12 to 17, the register is above the threshold value and no longer in the hysteresis range, the threshold bit is reset to 0.
