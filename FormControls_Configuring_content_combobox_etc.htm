Configuring the Content of a Check-box list, Option-button list, Combo-box or List-box

The contents of a Check-box list, Option-button list, Combo-box or List-box are defined in a similar manner using a table in the control's Content tab. Show picture

Each row in the table represents an item in the form control's list. The table has the following columns.

  • Sel. - The initial state of the item. See the explanation below.
  • Index - A sequential numeric index for the item that can be linked to a variable. It is also available to a SCADA Basic program or variable. See the topic Running a SCADA Basic Program from a Form Control. The value for each item is automatically generated during the configuration and cannot be manually changed.
  • Text - The text representing the item, as displayed at run-time. If PcVue's bilingual option is selected, you can enter a text string for each of the two languages.
  • User Data - Text that is available to a SCADA Basic program or a variable. The User Data field is NOT bilingual. See the topic Linking Variables to a Form Control.

To add an item to the table, click anywhere in the row marked New Item.

Editing an item in the table

To edit an item in the table, right click on it and select one of the following commands from the pop-up menu.

Command Meaning
New Create a new item in the selected position by moving the rest of the entries downward.
Cut Cut the selected item to the Windows Clipboard.
Copy Copy the selected item to the Windows Clipboard.
Paste Paste the current content of the Windows Clipboard to replace the selected item (disabled if the Clipboard is empty).
Delete Delete the selected item.
Move up Exchange the selected item with the one before it in the list.
Move down Exchange the selected item with the one after it in the list.

Using the column Sel column

The Sel. (Selected) column defines if an item is already selected (or not) when the mimic containing the form control is opened. The behavior of this column is slightly different for each of the controls. The status of each item in the Sel. column is represented by a tick box (tick = selected, un-ticked = not selected), except for the option-button list where it is represented by an option button.

  • For a check-box list - Each item can be selected, unselected or undetermined. Undetermined is represented by a ticked but dimmed tick box. The default is unselected.
  • For an option-button list - Zero or one item only can be selected. If zero items are selected then at run-time the first item on the list is selected. The default is that zero items are selected.
  • For a combo-box - Zero or one item only can be selected. If zero items are selected then at run-time the first item on the list is selected. The default is that zero items are selected.
  • For a list-box - Zero or one item only can be selected. If zero items are selected then at run-time the first item on the list is selected. The default is that zero items are selected.

Saving the Form Control's contents

The Form Control's content is saved and the configuration dialog is closed by clicking the OK button. By default, the content is saved with the Form Control's configuration in the mimic. To save (or load) the contents to (or from) a file see the topic Configuring a Form Control's contents from a file.
