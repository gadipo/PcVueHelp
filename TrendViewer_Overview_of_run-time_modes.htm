Overview of the Trend Viewer's Run-time Modes

At run-time, the Trend Viewer operates in one of three modes.

Real-time. The traces are plotted on the chart in real-time as the value of the variables they represent change. A Trend Viewer can be configured so that it starts up in real-time or it may be selected using the Real-time tool at run-time.

Pause. A sub-mode of real-time. The traces are plotted on the chart from real-time data that has been temporarily saved in the Trend Viewer display buffer. The Trend Viewer can be switched from real-time to pause mode by using the Pause tool, or by using the scroll bar.

Historic. The traces are plotted on the chart from data recorded in the archive files. A Trend Viewer may be configured so that it starts up in historic mode or it can be selected using the Historic tool at run-time.

Using the Trend Viewer in Real-time Mode

In real-time mode, new values for each trace are plotted starting on the right hand side of the chart. Older data scrolls across the chart eventually disappearing off the left hand side.

The frequency at which the chart is refreshed depends on the Refresh period property in the Display tab.

A Trend Viewer records the real-time values in its display buffer giving the User a limited ability to scroll back in time. The buffer continues to record data all the time the trend is displayed discarding the oldest values as it becomes full. As the buffer fills, the size of the slider on the scroll bar changes to reflect the amount of data stored as a ratio of the chart period. For example, if the buffer contains 10 minutes of data and the chart is 2 minutes, the slider will be approximately one-fifth the length of the scroll bar.

Using the Trend Viewer in Pause Mode

The Trend Viewer switches to pause mode (from real-time mode) whenever you click and drag the slider button on the scroll bar, or select the Pause tool.

In pause mode the chart area no longer updates in real time, instead it displays the temporary historic data recorded in the display buffer. However, the buffer of the Trend Viewer is still updated in real-time with the oldest values being discarded. The effect of this is that the data being displayed on the chart is gradually moving through the buffer. This can be seen by the fact that the slider button on the scroll bar gradually moves towards the left. Hence, if you leave the Trend Viewer in pause mode for long enough, the data for the time period displayed will be discarded. When in pause mode you can scroll backwards and forwards through the data using the scroll bar.

Using the Trend Viewer in Historic Mode

In historic mode, the chart area displays historic data extracted from the archive files. When you open a historic Trend Display at run-time, it will initially display data from the current time backwards for the period defined by its configuration.

What is the meaning of the icon in the middle of the chart?

If an icon depicting a cross and magnifying glass appears in the Trend Viewer whilst in historical mode it means that either there is no data for that period or one or more of the variables have not been configured for trend recording.
