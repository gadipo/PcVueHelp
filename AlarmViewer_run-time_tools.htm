Configuring the Alarm Viewer's Run-time Tools

The run-time tools, to which the User has access, are configured using the Execution tab. Show picture



  • Printing - If selected, the User can print selected alarm information, on the specified printer, directly from the display. The print format can be customized using the Format property and substitution strings.
  • Acknowledgement - If selected the User can acknowledge alarms.
  • Mask/unmask - If selected the User can mask and unmask alarms.
  • Filtering - If selected the User can change the alarm filter.
  • Multi-selection - If selected the User can select more than one alarm at a time in the alarm list.
  • Size of Columns - If selected, the User can change the width of columns at run-time by clicking and dragging with the mouse on the header.
  • Toolbar Position - If selected the User can change the toolbar position at run-time.
  • Toolbar Modification - If selected, the User can change the toolbar configuration at run-time.
  • Context - If selected, when the User clicks on an alarm line at run time the context window for Alarm Viewers is opened.
  • Enable sorting - Enables sorting of the alarm list at run-time using the selected column and direction. To select a column for sorting the Sort modification property must also be selected. Enable Sorting is not available in On-line mode.
  • Sort modification - Allows the User to change the column used to sort the alarms at run-time. Enable sorting must be selected before Sort modification can be selected.

If you want to change the column width or modify the toolbar, but do not want the User to do so, enable the properties and make the changes at run-time and then disable the properties again. The changes to the display will be saved.

If you want to change the sort column, but do not want the User to do so, enable Sort modification and then disable it after the sort column has been changed.

If you enable the Context property it will not be possible to select individual alarms and hence acknowledge, mask or un-mask them directly from the Alarm Viewer.

Changing the way that the list is sorted

Using the properties Enable sorting and Sort modification, you can temporarily sort the contents of the Alarm Viewer using the contents of any column into either ascending or descending order.

  • A column containing numeric data only will cause a numeric sort. For example, using ascending sorting, 2 would be before 3.
  • A column containing alphanumeric data will cause an alphabetic sort. For example, using ascending sorting, 20RedCars would be before 3GreenCars. The first character 2, comes before 3.
  • A column containing time and/or date information will cause a time/date sort.

You can use the Timestamp column as a sort key. The Display new alarms at the top of the list option is ignored if the alarms are sorted according to the timestamp. If multiple lines have the same value in the sorted column (for example when sorting on an extended attribute), the timestamp column is used as a secondary sort criterion.
