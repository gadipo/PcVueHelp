Creating and Configuring a Map Control

How to add a Map Control to a mimic

  1. Open the mimic in which the Map will be inserted and select Edit mode.
  2. Select Insert.Map from the menu.
  3. A Map will be created and its properties dialog automatically opened. Show picture

  4. Click OK to close the dialog and create a Map with default settings.

Once you have created a Map, and closed its properties dialog, it can be moved, sized and copied the same as any other drawing element. By default, a map is created with the zoom level set to two which requires an area of approximately 1025 by 1025 pixels to display a complete world map from longitude -180 to + 180 and latitude -85 to +85.

The Map configuration dialog contains three further tabs to configure the map, and enter the run-time user constraints.

The Map Configuration tab Show picture

  • Map provider
    • Local map - Select a locally stored map file. The map can be selected from a local map file previously created using PcVue's Map Cache Creator tool (MapCacheCreator.exe). By default, maps are saved in the GIS folder in PcVue's root. The drop down list box provides a list of what is available. The Ellipsis button allows the selection of an alternative folder.
    • Online map - Select a map from one of the on-line map providers. The providers supported by the Map Control, and the map variations that they provide, are listed in the drop-down list box. An Internet connection is required when first displaying the maps.
      • Cache map files - If selected, map tiles that are viewed at run-time are saved locally so that they can viewed later without an Internet connection.
  • Map markers source - The name of the file that contains the map markers. By default, the map markers files are saved in the GIS folder in PcVue's root. The Ellipsis button allows the selection of an alternative folder.
    • Edit - Open the Map markers Edit dialog. See the topic Map Markers Editor.

Map cache files, either as a result of downloading tiles at run-time or using the Map Cache Creation tool, can be very large. The higher the zoom level you allow to be used and the greater the panning limits, the larger the number of tiles that are required. If you allow the use of the maximum zoom and no panning limits the map tiles may require terabytes of storage space.

You cannot use a local map cache if the mimic containing the Map control is to be displayed in WebVue. Only a selection of online providers are supported.

Configuring the map run mode properties

The run mode properties of the map are configured using the Execution tab.Show picture

  • Opening properties - The map properties when the mimic is opened in run mode.
    • Map center - The latitude and longitude of the center point of the map.
    • Zoom level - The zoom level of the map.
    • Persist properties - Automatically save any changes to the Map center and Zoom level properties made interactively in run mode. The next time the containing mimic is opened the map will use these properties.
  • Map constraints within which a map can be used at run-time. The map constraints can also be set interactively using the Visual editor dialog (see below).
    • Allow panning - Allow the map to be panned in run mode.
    • Panning area bounds - The limits between which the map can be dragged and moved by the user in run mode, configured by entering the latitude and longitude of the top left and bottom right corners.
    • Allow zooming - Allow the map zoom level to be changed in run mode.
    • Zoom limits - The limits between which a user can zoom the map in run mode.

It is essential to enter carefully considered map constraints as the Map Control only loads those tiles required to render the current view.

  • If using an online map this minimizes the total number of map tiles that must be downloaded.

  • If using a local map, it is likely that you will only have map tiles for a limited area and zoom levels. If you allow the map to be moved or zoomed outside of what is available, blank map tiles will be displayed.

Using the map Visual Editor dialog

The Visual Editor dialog allows the developer to configure the map constraints interactively. Show picture

  • The center initial position can be changing by left clicking and dragging the map.
  • The panning bounds can be changed by right clicking and dragging the mouse.
  • The zoom level can be changed using the mouse wheel, the zoom control located at the bottom right corner of the map, or the Zoom property in the top panel.

When the constraints have been configured, the dialog is closed using the OK button and the properties are automatically copied to the Execution tab of the Map Configuration dialog.

Advanced properties

The Map Control advanced properties are found in the Advanced tab. Show picture

  • Native markers - Select the actions that the user is allowed to perform when clicking on the native map markers at run-time. Native markers are those generated by the Map Control itself - not a symbol or bitmap from a library.

    • Allow mimic open - Allow markers to directly open a mimic (if configured to do so).
    • Allow popup open - Allow markers to directly open a popup mimic (if configured to do so).
    • Highlight markers that can be clicked - Display control zone highlight when the pointer is positioned over a marker that can be clicked.

  • Keep invisible markers in memory - By default, markers that are not visible due to the maps current zoom level and pan position, are deleted from memory. Map markers can consume considerable resources and so this minimizes the requirement. However, it does mean that the markers have to be loaded again when they become visible which takes longer than if they remained in memory.

  • User input – Customize the behavior for certain user actions at runtime
    • Zoom mode – Define how zoom is handled. Options are to zoom about the cursor which at the same time is used as the new center for the map (Cursor and center), about the cursor position (Cursor) or about the center of the map (Center).
