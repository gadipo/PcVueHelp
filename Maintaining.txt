Maintaining Columns of a Table or View
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Maintaining Columns of a Table or View
 
You can display the properties of a column of a table 
 or a view, or delete it. Show picture
 

 

 
 
 
 
Displaying a column�s properties
 
 
In tree structure, 
 expand the sub-tree of a table or view and then select the name of a column.
 
Select the Properties 
 tab to display its properties in the right-hand pane.
 
To sort the rows 
 by the values of a column, click on its header. Show picture

 
Click again to reverse 
 the sequence. Show picture

 
 
To adjust the width of a column:
 
 
Drag a margin of 
 its header to the left or right.
 
 
The changes you make to the sequence of rows or the 
 widths of columns make no difference to the database contents. They are 
 lost when you display anything else in the Database Manager or close it.
 
How to delete a column
 
You can delete a column from the database as follows:
 
 
In tree structure, 
 click on the column's name then select the Action menu (or right-click 
 on a column's name to open its context menu).
 
Select Delete to 
 open a confirmation box, then select Yes to remove the column from the 
 table or from the view of the database.
 
In the main menu 
 or context menu, select Refresh to re-display the database's structure.
 
 

 
You have to have deletion rights to delete a column.
 
Take care not to delete a column of a table or view that 
 is required by other applications.

Maintaining database tables
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Maintaining Database Tables
 
Displaying a table�s properties
 
This is the default function when you select a table 
 in the tree structure of the Database Manager: Show picture
 

 

 
 
 
 
 
Click on the '+' 
 icon for a database to expand the list of tables. Select a table's name.
 
Its properties are 
 displayed in the right-hand pane.
 
 
Displaying a table�s columns
 
You can expand a table node in the tree structure to 
 list a table�s columns: Show 
 picture
 

 

 
 
 
 
 
Click on the + icon 
 by a table's name to list its columns.
 
Select a column's 
 name to display its properties in the right-hand pane.
 
 
For information on how to display a column's properties 
 or delete it, see the topic Maintaining 
 Columns.
 
Displaying or updating the data contents
 
You can display the contents of a table, or only a portion 
 of the data, in a grid: Show picture 
 

 

 
 
 
 
 
Select a table. In 
 the right-hand pane, select the View Data tab.
 
Select either All 
 Rows or Top N Rows, where you 
 set N to the number of rows to 
 display.
 
If you select Top 
 N Rows, the Ordered By box is 
 displayed. In it, select a column name to sequence the rows by its values. 
 
 
You can use the AZ/ZA 
 button to toggle between alphabetical/ascending and reverse/descending 
 sequences.
 
Click on the Execute 
 button below the table to display the data contents.
 
Use the scroll bars 
 to move the data grid relative to the pane.
 
 
You can also directly modify the table's contents by 
 editing the data in the grid in the right-hand pane of the View Data tab:
 
 
Click on a particular 
 cell to select and change its entire contents.
 
Click between characters 
 to select an insertion point, or drag the cursor to select characters 
 to replace, then type into the cell. 
 
Press the Enter key 
 to enter a changed value and to select the next cell downwards in the 
 same column.
 
Select the Update 
 button to apply the changes to the database.
 
 
Until you select Update, you can press Ctrl+Z to undo 
 the most recent change in the current cell only. 
 

 
Exercise extreme caution when making changes, since this 
 function only checks what you enter in terms of the values permitted in 
 each cell. No other constraints or dependencies are applied.
 
 
Deleting a table
 
From its context menu, you can physically delete a table 
 from the database:
 
 
In the Database Manager, 
 right-click on a table's name to open its context menu.
 
Select Delete to 
 display a confirmation box, then select Yes to delete the table.
 
In the database's 
 context menu, select Refresh to re-display its structure.
 
 

 
You have to have deletion rights to delete a table.
 
This function does not require exclusive access to database, 
 so other applications can be connected to the database when it is run. 
 However, you must take care not to delete a table that is required by other 
 applications.

About Maintaining a Database's Contents
 

 
 
 
 
 
 
 
 
 
 
 
 
 
About Maintaining a Database's Contents
 
In the Database Manager, you can view, change and delete 
 the tables, views and columns of a database. Show picture
 

 

 
 
 
 
The action of permitted viewing of a database is harmless, 
 but direct editing and deletion of database components and contents are 
 extremely risky. Such maintenance should only be done with advanced knowledge 
 and due precautions.
 

 
You have to have the rights to perform the actions. 
 
You should first make a back-up copy of the database, 
 or part of a database, that is to be altered. 
 
You must comply with other applications' requirements 
 upon the databases. Specific warnings are included in the topics where 
 such restrictions may apply. 
 
 
Bear in mind that an otherwise 
 valid change in one part of a database may cause inconsistency with the 
 contents of other parts of that database.

Maintaining PcVue&#160;
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Maintaining PcVue&#160;
 
Maintenance is the name given to the process of uninstalling, modifying, or repairing the software programs.
 
 
Remove - Uninstall the software.
 
Modify - Modify the list of installed features, add or remove features.
 
Repair - Repair any installed component.
 
 
Important information to know before maintaining PcVue or the Web Deployment Tools
 
 
Removing PcVue or the Web Deployment Tools does not remove the prerequisites. See below Maintenance of the prerequisite installation for more information.
 
Maintaining PcVue does not affect any projects that have been configured. However, as a precaution, you should always make a backup copy of any project and library you have configured, and want to keep, before using maintenance.
 
Our policy of continuous improvement means that the shared libraries supplied with PcVue are regularly updated. There may be minor changes between different updates of the same version of PcVue. The uninstallation process will remove the shared libraries and any library file or folder created by the installation, even if it has changed. If your project uses the shared libraries it is essential that you make a backup copy before the uninstall process even if you are reinstalling another copy of the same version of PcVue.
 
 
Maintenance of PcVue installation
 
Maintenance can be started in one of two ways.
 
 
Using the original installation media.
 
Using the Modify or Repair modes available via the Apps & features settings or the Add/Remove program tool in the Windows Control Panel.

 
 
After a Welcome dialog, the Program Maintenance dialog is displayed. Show picture
 

 
 
 
 
Select either Modify, Repair or Remove as appropriate.
 
Follow the sequence of dialogs to carry out the selected process.
Modify guides you through adding or removing optional features. Modify is not available for the Web Deployment Tools as there are no optional features.
Repair Checks the installed features and re-installs properly any that could have been damaged.
Remove Displays a confirmation dialog then starts the process of de-installation.
 
 
Maintenance of the prerequisite installation
 
Prerequisites are installed as a bundle containing one or more components. The bundle, and the components it includes, can be seen in the Apps & features settings (or Programs and Features in the Windows Control Panel).
 
The prerequisites bundle for PcVue is called PcVue Prerequisite, and the prerequisites bundle for the Web Deployment Tools is called Web Deployment Tools Prerequisite.

 
Maintenance of any of the prerequisites is started by selecting it in the Programs and Features list and then clicking Uninstall, Repair or Re-install as appropriate.
 
Uninstalling prerequisites
 
Uninstalling the prerequisites is only necessary if you wish to fully uninstall software components that were installed as part of installation. Uninstalling any of the prerequisites, whilst keeping the product installed, will prevent it from running properly. 
 
Uninstall the prerequisites bundle does not uninstall the individual components. However, you can completely uninstall the prerequisites by uninstalling the bundle as well as the individual components. Uninstalling an individual component may fail if it is used by other software, or the other software may fail to operate.
 
Repairing prerequisites
 
You can repair a bundle or an individual component. If you repair a bundle, any missing individual components will be installed and any already installed individual components will be repaired.
 
Re-installing prerequisites
 
Any missing component can be installed individually. However, if you are missing a component, clicking the Re-install button of the corresponding bundle will re-install any missing component that are part of the bundle.
