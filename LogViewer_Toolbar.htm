Using the Log Viewer's Toolbar

The toolbar allows the user to select the type of data displayed, to scroll the display to a specific historic time and to print the contents.

Filtering - Display alarm transitions to Alarm on not acknowledged.

Filtering - Display alarm transitions to Alarm on acknowledged.

Filtering - Display alarm transitions to Alarm off not acknowledged.

Filtering - Display alarm transitions to Alarm off acknowedged.

Filtering - Display alarm transitions to unavailable.

Filtering - Display alarms that are on.

Filtering - Display alarms that are off.

Filtering - Display bit transitions to 1.

Filtering - Display bit transitions to 0.

Filtering - Display bit transitions to invalid.

Filtering - Display program sent user actions.

Filtering - Display variable commands user actions.

Filtering - Display alarm acknowledgment user actions.

Filtering - Display the start and end of a user session (Logging on and off).
Filtering - Display alarm mask user actions.

Command - Time filter: enter a historic time to which the display is to be scrolled.

Command - Print the selected logs.

Filtering - Display alarms that are user masked.

Filtering - Display alarms that are masked by a program.

Filtering - Display alarms that are masked by dependency on another variable.

Filtering - Display alarms that are inhibited.

Filtering - Display alarms that are not accessible.

Filtering - Display alarms that are masked by an expression.

Command - Turns any blinking color used in the display of alarms to the non-blinking primary color.
Command - Enable auto-refresh.

The modes ENABLE and DISABLE of the SCADA Basic instruction VARIABLE are the only way to change the inhibition status of an alarm. As for any other type of variable, such changes done by script will be logged if the alarm is archived and the log list is configured appropriately.

Changing the Inhibited property of a variable at configuration time is not archived as part of a log list.
