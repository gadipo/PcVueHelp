Access Property 
 
 
Access Property
&#160;&#160;&#160;&#160; &#160;&#160;&#160;&#160; &#160;&#160;&#160;&#160; 

Example 

Applies to
Description
Returns the Access property that enables or disables a mode of access
 to the extended properties. Read-only.
Syntax
object.
 AccessAsfvPropertyAccess&#160;

Part
Description

fvPropertyAccess
Values: &#160;
fvPropAccessNone= 0
fvPropAccessRead= 1
fvPropAccessWrite= 2
Notes 
In the HMI, the extended
 properties are called associated attributes. There can be up to 16 properties/attributes
 for a variable. The first two are called Domain and Nature, though they
 do not differ in function from the 14 'extended' ones.
For guidance on how to enable the use of extended attributes/properties in the HMI, refer to this topic in the main Help: Specialized Topics.21CFR Part 11.Providing an Audit trail.Using Extended Attributes to Add More Information to a Logged Action.
The values of fvPropertyAccess indicate that the extended property either cannot be accessed ('0') or can ('1' or '2'). In the latter cases, its behavior may allow for reading ('1'), writing ('2') or either/both (logical '1' OR '2') including the option of being both readable and writable.
The behavior of each associated attribute in the HMI is configured as Static (read-only), Dynamic (read and write) or Distributed (read and write, for use in multi-station architectures).

TOPIC: hmobjExPropertiesX1.htm
========================================
ExProperties Iteration Example 
 
 
ExProperties Iteration Example
'-------- Retrieve and iterate an ExProperties collection --------
Private Sub ExPropertiesEx1_Click()
Dim ExProps As VarExProperties
Dim ExProp As VarExProperty

' Retrieve the Extended Properties collection of the variable Bit1

Set ExProps = TheseVariables("Bit1").ExProperties

' Check that the collection is not empty
If ExProps.Count > 0 Then
 ' Iterate the collection
For I = 1 To ExProps.Count
 ' Retrieve the Extended Property according to the rank 
 ' Note that ExProps(I) is equivalent to ExpProps.Item(I)
 Set ExProp = ExProps(I)
 ' Check that the Extended Property can be read
 If (ExProp.Access And fvPropAccessRead) = fvPropAccessRead Then
 ' Display the Name and Value
 MsgBox ExProp.Name & " = " & ExProp.Value
 End If
Next
End If
End Sub

TOPIC: hmobjExPropertiesX2.htm
========================================
ExProperties Example to Write an Extended Property 
 
 
ExProperties Example to Write an Extended Property
'-------- Retrieve and write a particular Extended Property --------
Private Sub ExPropertiesEx2_Click()
Dim ExProps As VarExProperties
Dim ExProp As VarExProperty

' Retrieve the Extended Properties collection of the variable "Bit1"
Set ExProps = TheseVariables("Bit1").ExProperties

' Check that the collection is not empty
If ExProps.Count > 0 Then
' Retrieve the Extended Property "TATT4"
 Set ExProp = ExProps("TATT4")
 If (ExProp.Access And fvPropAccessWrite) = fvPropAccessWrite Then
 ' Write the Extended Property value
 ExProp.Value = "The name is TATT4"
 End If
End If

End Sub

TOPIC: hmobjExPropertiesX3.htm
========================================
ExProperties Example Display Access Properties 
 
 
ExProperties Example Display Access Properties
'-------- Display the Access property of an Extended Property --------
'by using fvPropAccessNone, fvPropAccessRead and fvPropAccessWrite
'as binary filters to check the value of the Access property
Private Sub Text1_Click()

Dim va As VarExProperty
Set va = Variables("Bit1").ExProperties("TATT3")

If va.Access = fvPropAccessNone Then
 'value 0
 MsgBox "No Access"
End If

'THIS NEXT TEST IS INCORRECT BECAUSE IT ONLY WORKS FOR READ-ONLY ATTRIBUTE
If va.Access = fvPropAccessRead Then
 'value 1
 MsgBox "Access Read-only"
End If

'THESE TESTS ARE CORRECT BECAUSE THEY ALSO CHECK THE ACCESS PROPERTY
If va.Access And fvPropAccessRead Then
 MsgBox "Access Read"
End If

If va.Access And fvPropAccessWrite Then
 'value 2
 MsgBox "Access Write"
End If

If va.Access = (fvPropAccessRead Or fvPropAccessWrite) Then
 'value 3 (i.e. 1+2)
 MsgBox "Access Read - Write"
End If

End Sub

TOPIC: hmobjGraphicObjectsInGeneralC.htm
========================================
Graphic Objects In General 
 
 
 
Graphic Objects in General
 
&#160;

See also&#160;

Example&#160;

Properties&#160;

Methods&#160;

Events&#160;

Applies to&#160;
 
Graphic is a generic term referring to the HMI drawing elements Poly,
 Shape, Text,
 Symbol and Group plus the Bitmap and Metafile objects. The Graphic object type as such does not exist, whilst the Graphics
 Collection contains Object objects.
 
The properties that define the appearance of all objects that belong
 to the Graphics Collection conform
 to certain rules. All properties that are related to colors follow the
 standard color property naming scheme.
 
Pattern style rules
 
A pattern style is defined using the following conventions:
 
 

 
A background color: the color used to fill the
 object.
 
 

 
A pattern style: the pattern style used to fill the
 object (Solid, 45� Diagonal
 Hatching, 135� Diagonal hatching,
 Diagonal Cross Hatching, Horizontal Hatching, Vertical
 Hatching, Cross Hatching,
 Transparent).
 
 

 
A hatching color: the color used to draw hatching.
 
 
 
Line style rules
 
A line style is defined using the following conventions:
 
 

 
A foreground color: the color used to draw the
 lines of the object.
 
 

 
A line style: the line style used to draw the lines
 of the object (Solid, Dashed,
 Dotted, Dash
 Dot, Dash Dot Dot, Invisible).
 
 

 
A line thickness: the line thickness used to draw
 the lines of the object.
 
 
 
Appearance rules
 
Appearance is defined using the following conventions:
 
 

 
An appearance: the appearance used to draw the object
 (Regular, Shadow,
 Button, colored
 Button, Relief, Inverse
 Relief).
 
 

 
A bottom-right and a top-left color: the colors
 used to draw the bottom-right and the top-left edges of the object when
 its appearance is not set to Regular.
 
 

 
An appearance thickness: the thickness of the edges
 of the object when its appearance is not set to Regular.
 
 
 
Text color rules
 
Text colors are defined using following conventions:
 
 

 
A foreground color: the color used to draw the
 text contained in the text object.
 
 

 
A background color: the color used to fill the
 area defined by the including rectangle of the text object.
 
 
 
Standard color property naming
 
Each color used in an object can be fixed, indexed, transparent or
 blinking. Thus multiple properties are needed to define a single color.
 The names of these properties are constructed in a uniform manner using
 different but consistent suffixes. These suffixes are:
 
 

 
Primary:
 Suffixed property returns or sets the OLE_COLORcode used for primary color (information returned or set is valid only
 if Primary Transparent suffixed
 property is set to False).
 
 

 
Alternate:
 Suffixed property returns or sets the OLE_COLORcode used for alternate color (information returned or set is valid only
 if Alternate Transparent suffixed
 property is set to False).
 
 

 
Primary Index:
 Suffixed property returns or sets the index of the indexed color cell
 used as primary color. Indexed color cells refer to the ones that can
 be found in the advanced part of the palette. (Primary
 Index suffixed property can vary from 1 to 32 if the chosen color
 cell is indexed, if the chosen color cell is fixed, Primary
 Index suffixed property is set to 0.)
 
 

 
Alternate Index:
 Suffixed property returns or sets the index of the indexed color cell
 used as alternate color. Indexed color cells refer to the ones that
 can be found in the advanced part of the palette (Alternate
 Index suffixed property can vary from 1 to 32 if the chosen color
 cell is indexed, if the chosen color cell is fixed, Alternate
 Index suffixed property is set to 0).
 
 

 
Primary Transparent:
 Suffixed property returns or sets the transparent property that indicates
 that the primary color is transparent (True) or not (False).
 
 

 
Alternate Transparent:
 Suffixed property returns or sets the transparent property that indicates
 that the alternate color is transparent (True) or not (False).
 
 

 
Blink: Suffixed
 property returns or sets the index of the blinking frequency used for
 this color. (The information is valid only if Primarysuffixed property differs from Alternatesuffixed property.)
 
 
 

 
Events are fired
 only if the EnableEvents property
 of the underlying object has been set to True. This happens automatically
 once the first event script has been associated with the object.

MouseDown Event 
 
 
 

 MouseDown Event
 
&#160;

Applies to&#160;
 
Occurs when the user presses a mouse button over the 
 object.
 
Syntax
 
object.MouseDown(button As Integer, shift As Integer, _
 
x As 
 Variant, y As 
 Variant)
 
 

 
 
Part
 
 
 
 
Description
 
 
 
 

 
 
object
 
 
 
 
Receiver object.
 
 
 

 
 
button
 
 
 
 
An Integer that identifies the button that 
 was pressed to cause the event. The button 
 argument is a bit field with bits corresponding to the left button (bit 
 0), right button (bit 1), and middle button (bit 2). These bits correspond 
 to the values 1, 2, and 4, respectively. Only one of the bits is set, 
 indicating the button that caused the event.
 
 
 

 
 
shift
 
 
 
 
An Integer that corresponds to the state of 
 the SHIFT, CTRL, and ALT keys when the button specified in the button 
 argument is pressed. A bit is set if the key is down. The shift 
 argument is a bit field with the least-significant bits corresponding 
 to the SHIFT key (bit 0), the CTRL key (bit 1), and the ALT key (bit 2 
 ). These bits correspond to the values 1, 2, and 4, respectively. The 
 shift argument indicates the 
 state of these keys. Some, all, or none of the bits can be set, indicating 
 that some, all, or none of the keys are pressed. For example, if both 
 CTRL and ALT were pressed, the value of shift 
 would be 6.
 
 
 

 
 
x
 
 
 
 
A number that specifies the current location 
 of the mouse pointer. The x value 
 is always expressed in terms of the co-ordinate system used by its container 
 independently of the current zoom factor applied to this one.
 
 
 

 
 
y
 
 
 
 
A number that specifies the current location 
 of the mouse pointer. The y value 
 is always expressed in terms of the co-ordinate system used by its container 
 independently of the current zoom factor applied to this one.
 
 
 
 
The MouseDown event is common 
 to all the Graphic object types.
 
Events are fired only if the EnableEvents 
 property of the underlying object has been set to True. 
 This happens automatically once the first event script has been associated 
 with the object.

 is 0 then retrieve the horizontal and vertical 
' high limit positions 
 of the scroll bar controls

 bottom right corner of the Window content)

&#160;&#160;&#160;&#160;VScrollPos

TOPIC: hmobjWindowScrollE.htm
========================================
WindowScroll Event 
 
 
WindowScroll Event &#160;
&#160;

Example&#160;

Applies to&#160;
Syntax
object. 
 WindowScroll(ByVal ScrollDirection As fvScrollDirection, ByVal ScrollRequest As 
 fvScrollRequest, 
ByVal PreviousPosition As Long, ByVal NewPosition As Long, ByVal Window As Integer)

Part
Description

ScrollDirection
Indicates for which scroll bar control the event has been fired:
fvScrollHorizontal: Horizontal scroll 
 bar control
fvScrollVertical: Vertical scroll bar 
 control.

ScrollRequest
Indicates the scroll action the User has requested:
fvScrollLine: Line scrolling. 
fvScrollPage: Page scrolling.
fvScrollWheel: Mouse wheel scrolling.
fvScrollTrackStart: The User selected 
 the scroll box, ready to move it. In that case the PreviousPosition 
 and NewPosition values are both 
 equal to the current scroll bar position.
fvScrollTrack: The User is moving the 
 scroll box.
fvScrollTrackStop: The User released 
 the scroll box. In that case the PreviousPosition and NewPosition values 
 are both equal to the current scroll bar position. 

PreviousPosition
Provides the previous scroll bar position, before the User scroll request 
 has been executed. It is set with the scroll bar current position when 
 the ScrollRequest parameter equals 
 fvScrollTrackStart or fvScrollTrackStop.

NewPosition
Provides the new (current) scroll bar position, after the User scroll 
 request has been executed.

Window
Provides the index of the window (in the Mimic Windows Collection) for 
 which the event has been fired.
Each time a User scrolls the contents of a Window, the WindowScroll 
 event is fired in both VB/VBA. That event is associated with the Mimic object to which the Window 
 object belongs.
Note The WindowScroll 
 event is not fired when you use the Zoom tools. (Even though the contents 
 may be scrolled, there is no scroll request.) Use the WindowZoom 
 event to manage the scroll bar controls while zooming.

WindowZoom Event 
 
 
WindowZoom Event &#160;
&#160;

Applies to&#160;
Syntax
object.WindowZoom(ByVal ZoomRequest As fvZoomRequest, ByVal ZoomRatio As 
 Long, ByVal Window As Integer)

Part
Description

ZoomRequest
Indicates the action the User has requested as follows:
fvZoomIn: Zoom in the Window content 
 (for Zoom In and Magnifying Glass tools).
fvZoomOut: Zoom out the Window content.
fvZoomOff: Set back the zoom ratio 
 to 100% (for Zoom In and Magnifying Glass tools).

ZoomRatio
Provides the new (current) zoom ratio stated as a percentage, after 
 the zoom request has been executed. 

Window
Provides the window index (the index in the mimic's Windows Collection)

 fired in VBA. That event is associated with the Mimic 
 object to which the Window object 
 belongs.
Zooming is subject to these Window properties: EnableZoom, MinimumZoom and MaximumZoom.
A zoom ratio of 100% does not imply that the User's request was Zoom Off.
In the HMI the command Magnifying Glass is similar in effect to Zoom In, and Cancel Magnifying Glass is similar to Zoom Off, including firing the WindowZoom event.
