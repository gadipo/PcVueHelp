The Register Send Animation

Register values may be changed by the user from a control zone created with the Register Send animation. Register values may also be changed using a Position control. Position controls do not behave in the same way as a normal control zone and are explained in a separate topic.

The animation allows a user to enter a precise value for a register from the keyboard, from a displayed keypad or using positive and negative steps. Show picture

Selecting the entry method

The Register Send animation provides several options that effect the way in which the user enters the value at run time.

  • Keyboard - The new value is entered by typing it directly into a field in a dialog.
  • Step - Similar to keyboard with the addition of plus and minus command buttons with which the user may increase and decrease the value by a defined increment. The dialog will be automatically closed after each increment, unless the Keep box property is selected.
  • Keypad - Similar to keyboard except that a representation of a numeric keypad is displayed as part of the dialog, which is used in conjunction with the mouse to enter a value without using the physical keyboard.
  • Default - The user is prompted to confirm a (configurable) default value for the register.

Selecting the control range

The input range that is accepted by the animation is defined by the Minimum Value and Maximum Value properties. By default, these properties are set to Auto in which case they will come from the register's properties. If the register's maximum or minimum values are changed, all animations linked to it will automatically take account of that without the need to edit the animations.

If you want to use a different value, there are two other options. These are accessed by clicking the buttons adjacent to the fields.
