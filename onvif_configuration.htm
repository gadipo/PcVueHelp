How to configure the ONVIF control

How to insert an ONVIF control in a mimic

  1. Open the mimic in which the ONVIF control will be inserted and select Design mode.
  2. Select Insert.ONVIF control from the menu.
  3. An ONVIF control is inserted and its properties dialog opened. Show picture
  4. Click OK to close the dialog and create an ONVIF control with default properties.

You can specify more properties to control the graphical element visibility. From the Aspect tab, behind the down arrow at the bottom of the Properties dialog box:

  • Specify zoom visibility bounds so that the graphic control is only visible for predefined zoom levels.

  • Specify layers so that the graphic control is only visible if one of the selected layers is displayed.

These are even more useful when combined with user permissions and decluttering templates.

Once you have inserted the control, and closed its properties dialog, it can be moved, sized and copied the same as any other drawing element. The ONVIF control configuration dialog contains six further tabs, Camera, Advanced, PTZ control, Snapshot, Execution and Traces.

The Camera tab

  • Device - Selection of the Security device Network and Camera alias. These properties can be left empty and then specified at run time using SCADA Basic.

  • Connect on mimic open - Select to connect to the device when the mimic is opened in run-time mode, otherwise connection must be made using SCADA Basic.

The Advanced tab

  • Network caching - The length of time, in milliseconds, that the video stream is buffered to ensure continued playback in the event of a streaming lag. The larger the buffer the less likely that there will be an interruption to the video stream. However, the displayed video is delayed by the size of the buffer. For example, a buffer of 10,000 ms will delay the displayed video by 10 seconds. The default is 1000.

  • Hardware decoding - Enable hardware decoding using the computer's GPU to process video data. The results are dependent on the video codec, graphic card model and operating system. In most cases it frees the main processor (CPU) of some of the most computationally heavy tasks involved digital video processing. The default is enabled.

The PTZ control tab

  • PTZ control - The pan, tilt and zoom speed as a percentage between 1 and 100. The default is 100.

The Snapshot tab

The Snapshot tab contains properties that define how snapshots are taken. The image format is always .jpg.

  • Folder - The folder in which a snapshot will be saved. The default is the project's TP folder.

  • Prefix - The prefix given to the image. The default is 'Snapshot' followed by a number starting a 1.

  • Accelerator - A keyboard shortcut for the user to take a snapshot at run-time. Click into the field and then press the keystroke to be used (or the Del key to remove a configured accelerator).

The Execution tab

The Execution tab contains properties that define what is available to the user in run-time mode. See the topic Using the ONVIF control at run-time.

  • Enable context menu - If enabled, a context menu is available on right-click to the user in run-time mode.

    • Selection of the actions available in the context menu at run-time.

The Traces tab

The Trace tab is used to enable trace messages from the ONVIF control. Useful when diagnosing errors or if you want to track the activity run-time. Trace messages appear in the Trace window and are logged in the trace file. The following messages can be generated.

  • System errors - If enabled, a trace is logged when an unexpected error such as a stream processing failure occurs.

  • Verbose - All activity associated with the ONVIF control.

  • Activate on mimic open - Activate the selected traces on mimic open.

By default, only system errors are logged.
