Installation and Configuration

For PcVue to work with ICCP features, the application designer must:

  • Have a license with ICCP Server option enabled. The driver can run in demo mode for one hour. However, any networking architectures do not work in demo mode.
  • Reference the svmgrICCP.dll file in the usrmgr.dat file that is in the Bin folder.
  • Have a configuration file, ICCP.ini, which must be located in the C folder of the project. It is used to configure the ICCP LCCs in PcVue.

The usrmgr.dat file should have the following content:

[USRMGR\svmgrICCP]
 
DLL=svmgrICCP.dll

ICCP.ini file

Some fields of the ICCP.ini file are mandatory. If they are missing or invalid, the driver fails to start. Other fields are optional and if they are missing or invalid, the task uses default values. The Event Viewer displays warning messages in the event of a missing or invalid field.

To comment a line in the ICCP.ini file, you can enter a single quotation mark (’) at the beginning of the line.

The following paragraphs describe the ICCP.ini file in more details.

Name fields are used as unique identifiers for the configured items. Their value must be of type String with a max length of 20 characters.
Allowed characters are: [a-z], [A-Z], [0-9] and underscore.

Application References sections

The sections [ARn] in the ICCP.ini file define the addressing information necessary to define LCCs or RCCs. Depending on the use of the AR, the fields are associated to either local or remote addresses.

There are different functions to the application references depending on the Local Control Center type (Calling Client/Server, Called Client/Server, etc). Also, most of the fields in the ARs are optional, which can have different meanings:

  • If it is a Called type, omitting a specific field implies that upon remote connection, the local control center does not require an exact match of that field to accept or deny the connection.
  • If it is a Calling type, omitting a specific field implies that when the local control center sends the connection request to the remote, it does not include that field.

The exact requirements of each connection to a remote is handled between the two parties.

The TSelector field is particularly important. To avoid connection issues, it is recommended to use different TSelectors for different LCCs.

To differentiate application references, the user must create different blocks in the ICCP.ini file in a section called [ARX] where X is an increment starting from 1.

The Network is mainly useful to separate different PcVue stations and behaviors. To differentiate networks, the user must create different sections in the ICCP.ini file, each named [NetworkX] where X starts at 1.

LCCs sections

The sections [NetworkX/LocalControlCenterY] in the ICCP.ini file define the information of a local control center to be created. Multiple local control center can be created.

If multiple LCCs are configured, it is important to pay close attention to:

  • The AR, in particular the IP Address and the Port that must be different for the different configured LCCs when not using Full 7 Layer Stack addressing.
  • All the AR fields when using Full 7 Layer Stack addressing. All the LCCs can share the same port in this case.

The TSelector field in the ARs is especially important to differentiate the different LCCs that are being created.

All LCCs must be configured linked to a Network. To differentiate LCCs, the user must configure a different section for each LCC (in the ICCP.ini file) called [NetworkX/LocalControlCenterY] where X and Y increase starting from 1.

Security

Defines the information that is required to setup security mechanisms. The Local Control Center and the RCC sections have a field called AuthMechanism where you can choose which authentication mechanism is used.

If the you choose password based security or certificate based security, then a security section can be added for each Local Control Center or RCC.

Both Local Control Center and RCC security sections are the same and require the exact fields. However, the fields are different depending on the AuthMechanism.

All Security sections must be created linked to either an LCC or an RCC. To differentiate Security sections, you must create different blocks in the ICCP.ini file.

The security section has a structure like the following:

For security applied to the Local Control Center:

  • [NetworkX/LocalControlCenterY/Security]

For security applied to the RCC:

  • [NetworkX/LocalControlCenterY/RemoteControlCenterZ/Security]

where X, Y and Z increase starting from 1.

The security section is loaded at startup with the parameters of the ICCP.ini at that moment. However, two ICCP Driver requests are available to dynamically reload the security configuration. Refer to the topic ICCP System Variables and commands for more information.

Password based security

This section of inputs defines the information required for a Security section to be created corresponding to a specific LCC or RCC.

If Password based security is selected, then any Certificate based security fields are ignored.

Field Data type Value Description
WeakPassword

String

<Password>

Default to empty string

Optional

It is case sensitive.

Determines the password for ACSE authentication.

For example: The ICCP.ini file should contain:

[Network1/LocalControlCenter1/Security]
 
 
 
WeakPassword    = Secpw123!
 
 
 
[Network1/LocalControlCenter1/RemoteControlCenter1/Security]
 
 
 
WeakPassword = ClientSecpw123!

Certificate based security

This section of inputs defines the information required for a Security section to be created corresponding to a specific LCC or RCC.

If Certificate based security is selected, then any Password based security fields are ignored.

Field Data type Value Description
PKI_Directory

String

<FolderPath>

Default to the built-in PKI folder

Optional

It is case sensitive.

Indicates the general path used by the fields in this section that reference files. If the field is not defined, then any file that is used in the next fields uses the PcVue built-in PKI structure. The information for each file can change and is described individually.

If a path is defined, then any file that is used in the next fields uses this path as a base.

For example: For a file called Cert.pem:

  • PKI_Directory = C:\PKI\

  • FileField = Cert.pem

The driver searches for the combination of the path and the file name. C:\PKI\Cert.pem
CA_VerifyDepth

Integer

0..31

Default to 0

Optional

The Certificate Authority Chaining Verification Depth.

CAs can refer to other CAs for authority.

For example: Company X can sign Company Y, who can sign a certificate. If you have Company X’s CA certificate and have a depth of 1, you can accept Company Y. With a depth of 0 you cannot.
CA_CertificateFile

String

 <FilePath>

Mandatory

It is case sensitive.

The path to the file containing the Certificate Authority Certificate. If the PKI_Directory field is the built-in PKI structure, then the file should be in the “PKI\Trusted\Certs” directory.

Only the filename should be written in the field.
CA_CRL

String

 <FilePath>

Mandatory

It is case sensitive.

The path to the file containing the Certificate Authority Certificate Revocation List. If the PKI_Directory field is the built-in PKI structure, then the file should be in the PKI\Trusted\CRL directory.

Only the filename should be written in the field.
MMS_CommonName

String

<Name>

Mandatory

It is case sensitive.

The common name to expect in incoming MMS certificates.

Empty string disables MMS related parameters.
MMS_PrivateKeyFile

String

<FilePath>

Mandatory

It is case sensitive.

The path to the file containing the private key for signing ACSE Authentication. If the PKI_Directory field is the built-in PKI structure, then the file should be in the PKI\Own\Private directory.

Only the filename should be written in the field.
MMS_PrivateKeyPassPhrase

String

<PassPhrase>

Default to empty string

Optional

It is case sensitive.

The PassPhrase for decrypting the private key.
MMS_PublicCertificateFile

String

<FilePath>

Mandatory

It is case sensitive.

The path to the file containing the public certificate. If the PKI_Directory field is the built-in PKI structure, then the file should be in the PKI\Own\Certs directory.

Only the filename should be written in the field.
MMS_DisablesSecurity

Boolean

True/False

Default to False

Allows you to disable certificate security on the MMS layer.
If set to True, security related to MMS layer is not used.
TLS_DisableSecurity

Boolean

True/False

Default to False

Allows you to disable certificate security on the TLS layer.

If set to True, security related to TLS layer is not used.
TLS_CommonName

String

<Name>

Default to empty string

Optional

It is case sensitive.

The common name to expect in incoming TLS certificates.

Empty string disables TLS related parameters.
TLS_RSA_PrivateKeyFile

String

 <FilePath>

Mandatory

It is case sensitive.

The path to the file containing the private key for RSA TLS ciphers. If the PKI_Directory field is the built-in PKI structure, then the file should be in the PKI\Own\Private directory.

Only the filename should be written in the field.
TLS_RSA_PrivateKeyPassPhrase

String

<PassPhrase>

Default to empty string

Optional

It is case sensitive.

The PassPhrase for decrypting the private key for RSA TLS ciphers.
TLS_RSA_PublicCertificateFile

String

 <FilePath>

Mandatory

It is case sensitive.

The path to the file containing the public certificate for key for RSA TLS ciphers. If the PKI_Directory field is the built-in PKI structure, then the file should be in the PKI\Own\Certs directory.

Only the filename should be written in the field.

Example Security section the ICCP.ini file

[Network1/LocalControlCenter1/Security]
 
 
 
    PKI_Directory         = C:\PKI\
 
    CA_VerifyDepth        = 1
 
    CA_CertificateFile    = sample_ca_rsa_public_certificate.pem
 
    CA_CRL                = sample_ca_certificate_revocation_list.pem
 
 
 
    MMS_CommonName                 = MMS
 
    MMS_PrivateKeyFile             = sample_mms_rsa_private_key.pem
 
    MMS_PrivateKeyPassPhrase     = pwmms1234
 
    MMS_PublicCertificateFile     = sample_mms_rsa_public_cert.pem
 
 
 
    TLS_CommonName                     = TLS
 
    TLS_RSA_PrivateKeyFile             = sample_tls_rsa_private_key.pem
 
    TLS_RSA_PrivateKeyPassPhrase     = pwtls1234
 
    TLS_RSA_PublicCertificateFile     = sample_tls_rsa_public_cert.pem
 
 
 
[Network1/LocalControlCenter1/RemoteControlCenter1/Security]
 
 
 
    PKI_Directory         = C:\PKI\
 
    CA_VerifyDepth        = 1
 
    CA_CertificateFile    = client_sample_ca_rsa_public_certificate.pem
 
    CA_CRL                 = client_sample_ca_certificate_revocation_list.pem
 
 
 
    MMS_CommonName                 = MMS
 
    MMS_PrivateKeyFile             = client_sample_mms_rsa_private_key.pem
 
    MMS_PrivateKeyPassPhrase     = Clientpwmms1234
 
    MMS_PublicCertificateFile     = client_sample_mms_rsa_public_cert.pem
 
 
 
    TLS_CommonName                     = TLS
 
    TLS_RSA_PrivateKeyFile             = client_sample_tls_rsa_private_key.pem
 
    TLS_RSA_PrivateKeyPassPhrase     = Clientpwtls1234
 
    TLS_RSA_PublicCertificateFile     = client_sample_tls_rsa_public_cert.pem
