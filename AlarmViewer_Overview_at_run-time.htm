Overview of the Alarm Viewer at Run-time

The Alarm Viewer displays, in a scrollable list, the current (real-time) status of all alarms that pass its filter criteria. The filter can operate on the Domain and Nature, alarm level and alarm status. The filter is initially configured during development and may also be changed at run-time should the configuration allow it.

Alarms are special bit variables that have the alarm property enabled. An alarm may be generated from either the 1 or 0 state of the bit and is given a level of 0 to 29. An optional masking bit or alarm may also be selected. A bit with alarm attributes behaves differently to a standard bit in that, instead of having three states (true, false and invalid), it has five.

  • Off.
  • On and not acknowledged.
  • On and acknowledged.
  • Off and not acknowledged.
  • Unavailable (a general term for when an alarm is not operating normally for a variety of reasons).

The colors in which alarms appear are defined by the Alarm on Text tab of the Preferences.Colors dialog box. In normal mode, only the foreground (font) color is used, the background color being taken from the Alarm Viewer itself. In advanced mode, both the foreground and background colors are used.

Selecting Alarms

Individual alarms are selected by clicking on the relevant line in the display list. A selected alarm is indicated by displaying it in reverse video (unless the alarm display is in advanced color mode - see below). Selected alarms can be acknowledged, printed, masked or un-masked using the tools on the toolbar or the context menu, and copied to the Windows clipboard using the Ctrl+C keyboard shortcut.

If you enable the Context property, it will not be possible to select and hence acknowledge, mask or un-mask alarms directly from the alarm display. Instead, clicking on an alarm will open the Alarm Viewer context window.
