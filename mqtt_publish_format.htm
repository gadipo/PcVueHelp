How to customize the MQTT publish format

The MQTT Add-on publishes variables by sending payloads similar to those specified in Sparkplug. It also offers the possibility to customize the payload format, in particular by adding metadata such as the description, domain, nature, alarm level...

Raw publish payload

When using the keyword 'Raw' as the PayloadType value, the driver publishes unformatted messages. The message consists solely of the raw variable value, encoded as ASCII, on the MQTT topic:

  • Bit values are sent as 0 or 1.
  • Register values are sent as numeric values, e.g., 123.5.
  • Text values are sent as plain strings, e.g., Hello world.

Generic publish payload

When using the keyword 'Generic' as the PayloadType value, payloads sent for published variables have the following default fixed format.

Register variables:

{
 
    "metrics":[
 
    {
 
    "name":"MQTT.PUB_TEMP",
 
    "value":37,
 
    "timestamp":"2008-04-22T06:00:11Z",
 
    "quality":0
 
    }
 
  ]
 
}

Text variables:

{
 
"metrics":[
 
    {
 
    "name":"MQTT.PUB_TXT",
 
    "value":"mydata",
 
    "timestamp":"2019-10-29T16:14:40.284Z",
 
    "quality":0
 
    }
 
  ]
 
}

Bit variables:

{
 
"metrics":[
 
    {
 
    "name":"MQTT.PUB_BIT",
 
    "value":true,
 
    "timestamp":"2019-10-29T16:14:21.024Z",
 
    "quality":0
 
    }
 
 ]
 
}

Where:

  • name: Name of the published variable,
  • value: Current value of the variable at the time of sending the Publish message,
  • timestamp: Timestamp of the variable at the time of sending the Publish message. The format follows the ISO 8601 standard (UTC time).
  • quality: Quality of the variable at the time of sending the Publish message. See below for the list of possible values

For all register variables:

  • If the Format property is specified in the variable configuration, the MQTT applies it. For example, if the format property is "#.##", the MQTT Add-on will publish a value with 2 decimal places.

  • If the Format property is not specified in the variable configuration, the MQTT Add-on applies the number of decimal places as specified by the DefaultDecimalPlaces parameter in the file svmgrMQTT.ini.

The same applies to the following metadata: minimumValue, maximumValue, controlMinimum and controlMaximum.

This default format can be customized by defining the PublishFormat parameter. See Customizing Publish payloads with additional metadata for more information.

Predefined custom publish payload

One or more custom formats for publish payloads can be predefined in a file named PubTemplates.txt, located in the project's C directory.
Each custom format is given a name, that you use as the value of the PayloadType parameter.

Example of PubTemplates.txt file:

[Format0]
 
Name = TestFormat
 
BeginFormat
 
{
 
    "name":"&varname",
 
    "metrics": [
 
        {
 
            "value":"&value"    
 
            "timestamp":"&tsType",
 
            "quality": &quality
 
        }
 
    ]
 
}
 
EndFormat
 
 
 
[Format1]
 
Name = OtherFormatForTest
 
BeginFormat
 
Myformat1: &varname - &value - &timestamp - &quality
 
EndFormat
 
 
 
[Format2]
 
Name = Full
 
BeginFormat
 
&varname - &value - &timestamp - &quality -
 
 &tsType - &unit - &description - &source - &alarmLevel -
 
 &domain - &nature - &textAttr03 - &textAttr04 - &textAttr05 - &textAttr06 - &textAttr07 - &textAttr08 - &textAttr09 -
 
 &textAttr10 - &textAttr11 - &textAttr12 - &textAttr13 - &textAttr14 - &textAttr15 - &textAttr16 -
 
 &binAttr - &textSize
 
EndFormat

Customizing Publish payloads with additional metadata

When using the Generic payload type, It is possible to add extra information in the Publish payloads by using the PublishFormat parameter. In this case, a metadata section will be added to the payload with the specified additional information.

The syntax of the PublishFormat parameter is: keyname1#keyname2#keyname3

As any optional parameter, the PublishFormat can be specified:

  • Directly in the MQTT mapping extended attribute,
  • In a different extended attribute,
  • In the svmgrMQTT.ini file.

For example, if you want to publish a register variable named Mqtt.Pub_Temp with the following additional information:

  • Unit,
  • Description,
  • Source,
  • Minimum value,
  • Maximum value.

The PublishFormat syntax will be: unit#description#source#minimumValue#maximumValue

And a typical published payload will be:

{
 
    "metrics": [
 
    {
 
    "name":"MQTT.PUB_TEMP",
 
    "value":29,
 
    "timestamp":"2020-01-13T08:00:00.123Z",
 
    "quality":0
 
    "metaData": {
 
    "unit":"Celsius degrees",
 
    "description":"Value sent from ",
 
    "source":1,
 
    "minimumValue":0,
 
    "maximumValue":45
 
    }
 
   }
 
  ]
 
}

This table gives the list of metadata and corresponding keyname that can be added to the PublishFormat parameter.

The same keynames are used to predefine custom payload styles in the PubTemplates.txt file.

Keyname Description
datatype Datatype of the variable. See below.
varname Name of the variable.
Present by default when using the Generic payload type.
value Value of the variable.
Present by default when using the Generic payload type.
timestamp Timestamp of the variable.
Present by default when using the Generic payload type.
quality Quality of the variable.
Present by default when using the Generic payload type.
tsType Quality of the timestamp: SV, PLC or PLC?.
unit Unit of the variable, only for register variables.
description Description of the variable.
source Source of the variable. See below.
alarmLevel Alarm level, only for alarm variables.
domain Domain of the variable.
nature Nature of the variable.
textAttr03 to textAttr16 Text attribute 03 to 16 of the variable.
binAttr Binary attribute 1 to 32, in decimal. Example: If binary attributes 1, 4 and 8 are true, then binAttr = 137.
minimumValue Minimum value of the variable, only for register variables.
maximumValue Maximum value of the variable, only for register variables.
controlMinimum Minimum control value of the variable, only for register variables.
controlMaximum Maximum control value of the variable, only for register variables.
threshold1Value Value of threshold1, only for register variables.
threshold2Value Value of threshold2, only for register variables.
threshold3Value Value of threshold3, only for register variables.
threshold4Value Value of threshold4, only for register variables.
textSize Length, only for Text variables.
