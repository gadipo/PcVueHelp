Scripting Options

Show picture

  • Disable VBA - VBA scripting is disabled and the PcVue software behaves as though it were a non-VBA version. The VBA editor is disabled and any existing VBA scripts do not run. If a mimic or symbol containing a VBA script is saved, the script is lost.
  • Variable Value returns alarm value if variable is an alarm - If enabled, the Value property of Variable object is handled in scripts as an alarm value instead of a bit value. Otherwise, the bit value is returned even if the variable is configured as alarm.
  • Script Editor - Configures the behavior of the VBA and ENA (Enable) scripting engines.
    • Show VBA Only - Only the VBA tool is shown on the toolbar.
    • Show both VBA and Enable, shortcuts to VBA - Both VBA and ENA tools are shown on the toolbar. Shortcuts (View script on the drawing element context menu and any events in the Properties List) are directed to the VBA editor.
    • Show both VBA and Enable, shortcuts to Enable - Both VBA and ENA tools are shown on the toolbar. Shortcuts (View script on the drawing element context menu and any events in the Properties List) are directed to the ENA editor.
  • Rename mimic template object IDs - See explanation below.

Using the Rename mimic template object ID's option

When selected, this option will cause the ID of any mimic template objects that are identical to the ID of mimic objects inherited from the template, to be automatically renamed. Not only is the object ID modified, but also any references to the ID in any Enable script.

This is necessary when using the Enable scripting language as each object ID must be unique. If you script objects with identical IDs, the script of the object loaded first will work but subsequent objects with the same ID that are loaded will generate an error message.

If possible, it's better to plan in advance and set-up a naming scheme that will avoid duplicate ID as the automatic renaming system will penalize the mimic loading performance.

This option must not be used in the VBA version of PcVue.

  • VBA allows duplicate object IDs.

  • VBA does not support object renaming.
