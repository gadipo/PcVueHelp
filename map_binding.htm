Binding Marker Properties to Variables

Many of the properties configured in the Map Markers Editor can be bound to variables. The property then takes its value from the variable. You can bind to a variable's real-time value or one of the extended attributes. When a property can be bound to a variable, the Variable Mode button appears adjacent to the property.

How to bind a property to a variable

  1. Click the Variable Mode button adjacent to the property. The Variable Selector button appears.
  2. Enter the name of the variable. You can either type it in directly or use the Variable Selector to select it. Show picture If a Branch has been configured the variable name is relative to that branch unless it is preceded by the direct reference character (@).

What variables and variable properties can I use?

You can bind a property to either a variable's real-time value or one of its extended attributes. To bind to an extended attribute the variable name is appended with the attribute reference. For example, UK.INFORMATION.TATT3#, references the third text attribute of the variable UK.INFORMATION.

You must respect the property type. That is, a numerical property (for example Latitude) must be bound to a register variable, and a text property (for example Mimic) to a text variable or a text attribute. The one exception to this is the color swab, which can be bound to a register variable or text variable. If you use a text variable, you can use any standard HTML color name. The standard HTML color names are widely available on the Internet.

To minimize the system resources required to display a map, markers that have been visible, but are no longer, are temporarily unbound from any variables. You can prevent this by selecting the property Keep invisible markers in memory in the Map Control's configuration dialog.
