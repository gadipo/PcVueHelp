Creating a Log Viewer and Configuring the General Features

The Log Viewer is flexible in both its appearance and operation and as a result, there are many configurable properties. The procedure for creating a Log Viewer follows.

  1. Open the mimic in which the Log Viewer will be inserted and select Edit mode.
  2. Select the Log Viewer tool from either the animation toolbar or the menu.
  3. A Log Viewer will be created and its properties box automatically opened and showing the Filters tab.

Once you have created a Log Viewer, and closed its properties box, it can be moved, sized and copied the same as any other drawing element.

The Log Viewer configuration dialog contains six tabs for the various aspects of the configuration. Show picture



  • Display - General display features. See also the topic Configuring the line format.
  • Aspect - General appearance - the same as the Aspect tab of all drawing elements.
  • Execution - Properties that select the tools available to the User at run-time. See the topic Configuring the run-time behavior.
  • Filters - Selection of the source (Log List) of the data the Log Viewer is to display. See the topic Configuring the filter.
  • Advanced - Selection of how log data is sorted according to its timestamp. Newest data at the top or bottom. See the topic Advanced behavior.
  • Menu - Configuration of the run-time context menu. See the topic Configuring the context menu.

Configuring the general display features

  • Toolbar and menu - The appearance and location of the toolbar at run-time.
    A drop-down list box is used to select the size of the icons on the toolbar. The space that the toolbar occupies depends on selected size. If None is selected the toolbar is not displayed.
    The position and configuration of the toolbar, but not the size of the icons, can be changed by the User at run-time unless you specifically disable that capability in the Execution tab.
  • Use context menu - Enable the run-time context menu.
  • Scrollbar - The availability and width of the vertical and horizontal scroll bars. The default setting of a vertical scroll bar only is used in most applications.
  • Background - The background color of the display.
  • Alternative color of text - The alternative text color for projects configured with legacy versions of PcVue that did not have the capability to select the two colors for a blinking event.
  • Without blinking - The blinking is turned off for any logs that have been configured with a blinking color (for example alarms that are not acknowledged). The text is displayed in the main color only.

The ability to define the size of the icons and the width of the scrollbars allows the application designer to adapt the viewer if touch screens are used at runtime.

You can specify more properties to control the graphical element visibility. From the Aspect tab, behind the down arrow at the bottom of the Properties dialog box:

  • Specify zoom visibility bounds so that the graphic control is only visible for predefined zoom levels.

  • Specify layers so that the graphic control is only visible if one of the selected layers is displayed.

These are even more useful when combined with user permissions and decluttering templates.
