Workspace overview

The Application Architect Workspace consists of the following major components. Show picture

  • Left pane and toolbar - Displays the available templates or parameters. Although this area has the appearance of a configuration tree, it is a toolbar. Double clicking on an object (either a template or parameter) will insert an instance of it under the object selected in primary configuration tree displayed in the right pane.
  • Right Pane - Contains three tabs - Instances, Templates and Parameters.
    • Instances - See the topic The Instances Tab.
    • Templates - See the topic The Templates Tab.
    • Parameters - See the topic The Parameters Tab.
  • Pop-up menu - A context sensitive right click pop-up menu is available from the configuration trees. The commands on the pop-up menu are repeated on the tool bar.
  • Main toolbar - The tools that are available reflect the configuration element that is selected in the configuration tree. Toolbar commands are repeated on the right click pop-up menu.
  • Menu - Non-context sensitive commands - see list below.

You can resize any of the Application Architect's panes by positioning the cursor over the border and clicking and dragging. Changes that you make to the Application Architect's look and feel are persistent - that is they are remembered even after PcVue has been shut down and restarted.

When you first open the Application Architect's workspace, all configuration trees are collapsed. All branches will have a + sign against them indicating that there are further branches - even there are none. This is because the Application Architect only loads the configuration of a branch when you attempt to expand it. If the Application Architect loaded the configuration of all branches when it was opened, it could take a considerable time and add an unacceptable delay.

Menu commands

  • File.Save (Ctrl+S) - Save the current configuration.
  • File.Exit (Alt+F4) - Close the Application Architect. Any changes that have been made are saved.
  • View.Toolbars.Parameters - Enable display of the parameters tool bar in the left pane.
  • View.Toolbars.Templates - Enable display of the templates tool bar in the left pane.
  • View.Display thumbnails - Enable display of thumbnails for graphic elements in templates and instances property grids.
  • Tasks.Check (F5) - Check the topography of the configuration.
  • Tasks.Check consistency - Check the consistency of the data model. Will detect instances for which the template is not defined, parameters used but not defined etc.
  • Tasks.Update libraries - Reload shared libraries. Run this command if you have added or removed a library and want the change to be taken into account without closing and re-opening the Application Architect.
  • Tasks.Generate (F7) - Generate the application.
  • Task.Settings - Opens the settings dialog box.

Using the libraries to store templates and parameters

Folders representing the PcVue project libraries appear in both the Templates and Parameters tabs. This allows you to choose where you want to save the configuration of any Templates or Parameters that you create. That way, you can build libraries of templates and parameters in the same way that you build libraries of symbols. Templates and parameters are both saved in the Templates folder of a library.

A reminder about libraries

  • The contents of the Local Library is only available to the current project.
  • The contents of the Common Library is available to all projects.
  • You can create custom libraries that are either only available to the project or are available to all projects.

For more information on libraries, see the book on Developing the HMI.
