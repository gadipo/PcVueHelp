Data group, Dataset, and Setting group configuration

A data group provides polled access to all data objects and data attributes of an IEC 61850 device. It is the least efficient way of collecting data from the network as each variable that you map using a data group will be serviced using a separate poll. Data groups should only be used to access data objects or data attributes that are not available from a data set or report.

A dataset provides polled access to all data objects and data attributes of an IEC 61850 device dataset. It is more efficient than a data group, as a single poll will return all data objects and data attributes that belong to the dataset.

A setting group manages configuration presets within an IEC 61850 device, allowing dynamic switching between operational settings.

Prerequisites

  • You have configured an IEC 61580 network.
  • You have configured an IEC 61580 device.

Add a data group

  1. Open the Application Explorer and select an IEC 61850 network and device.
  2. Click Add a monitoring object in the toolbar and select Add a data group from the drop-down. The IEC 61850 data group creation dialog opens. Show picture

  3. Enter a name and (Optional) a description. The name is the identity of the data group within PcVue.
  4. Enter a polling group to enable cyclic data collection at defined intervals.
  5. Tick Activate at start-up to enable automatic polling at start-up.
  6. Click OK to save the configuration.

Add a dataset

  1. Open the Application Explorer and select an IEC 61850 network and device..
  2. Click Add a monitoring object in the toolbar and select Add a dataset from the drop-down. The IEC 61850 dataset creation dialog opens. Show picture

  3. Enter a name and (Optional) a description. The name is the identity of the dataset within PcVue.
  4. Select the dataset type. Static datasets are used for consistent and unchanging data structures, whereas dynamic datasets are used for applications needing flexible configuration that can adapt to different engineering scenarios.
  5. Select the dataset name. This is the name as exposed by the physical device. Depending on the device configuration, PcVue obtains a list of the available datasets by browsing the connected physical device or by listing datasets defined in the SCL file associated to the device.
  6. (For Dynamic dataset only) Select the dynamic type. If you selected Persistent, the dataset object will be persisted at the physical device level and you are required to select a Logical Device and Logical Node, and enter a Name for the persisted dataset. These properties allow structured data grouping within the device and are essential for configurations that persist across sessions.
  7. Enter a polling group to enable cyclic data collection at defined intervals.
  8. Tick Activate at start-up to enable automatic polling at start-up.
  9. Click OK to save the configuration.

The configuration of a dynamic dataset is sent to the physical device upon connection. You need to restart the IEC 61850 device configuration element if you want such a change to be taken into account.

A non-persistent dynamic dataset is automatically deleted and unreferenced by the physical device upon disconnection.

Deleting a persistent dataset removes the associated dataset from the physical device. If an error occurs (e.g., if a report still references the dataset), a warning popup is displayed, and the dataset group will remain active until resolved. Such an error can commonly occur, in particular if a report is still referencing the dataset in the physical device.

Buffered reporting with a dynamic dataset requires the dataset to be configured for persistency at the physical device level.

Add a setting group

  1. Open the Application Explorer and select an IEC 61850 network and device.
  2. Click Add a monitoring object in the toolbar and select Add a setting group from the drop-down. The IEC 61850 setting group creation dialog opens. Show picture

  3. Enter a name and (Optional) a description. The name is the identity of the setting group within PcVue.
  4. Select a control block from the drop-down. The control block is a structure that manages and organizes the setting groups within a device. PcVue verifies the existence of the selected control block upon startup. If the control block is missing on the device, its status is set to "Invalid control block.”
  5. Enter a polling group to enable cyclic data collection at defined intervals.
  6. Tick Activate at start-up to enable automatic polling at start-up.
  7. Click OK to save the configuration.
