Configuring the Grid's General Appearance

The Grid's configuration allows selection of the size of the grid, the number of rows and columns of cells. An empty grid might look like this. Show picture

The size and general appearance is set by the properties in the Aspect tab, which is similar to those of other drawing elements used in PcVue. For details, see the Drawing book in the main help.

Configuring the cells

The cells are configured from the General tab. Show picture

You can configure the number of rows and columns, the color and font used in the cells, and the color of the grid used to separate the cells.

  • Rows/columns - The total number of rows and columns of cells respectively, including any header rows and columns. For example, if wanted a grid of 10 x 3 cells and both row and column headers the number of rows/columns would be 11 x 4. When referenced programmatically the top left hand cell, excluding any headers, has an index of 0,0. There is no coded limit for the maximum number of cells but it is recommended that the total number of cells is kept below 100,000.
  • Fixed row/col - The number of header rows and columns. When referenced programmatically, the cells of the header rows and columns are indexed negatively. A value of zero means that no row or column header is displayed.
  • Background - Sets the background color of the cells.
  • Gridline color - Sets the color of the grid displayed between the cells.
  • Text color - Sets the color of any text displayed in the grid.
  • Font - Displays a dialog from where you can set the font for all the cells including the headers.

You can only use solid, non-blinking colors in the Grid.

For information on how to configure the column widths and row heights, see the topic Using the Grid at run-time.

About the header row and/or column

The header row and column appear at the top and left of the grid respectively. The cells that make up the headers behave differently compared to the other cells.

  • You cannot change their color from the configuration dialog.
  • They always have a button style.
  • They cannot be selected and hence you cannot type text into them (although you can enter text using a script).
  • The cells behave differently at run-time.

You must have a header row and column if you want to allow the user to resize the columns and rows at run time.

Enabling the scroll bars

The scrollbars can be used to navigate through the cells at run-time if the number of cells is greater than can be physically displayed. Display of the scroll bars is controlled using the Scrollbar drop down list box and can be selected from the following.

  • None - Never display any scroll bars.
  • Horizontal - Always display a horizontal scroll bar.
  • Vertical - Always display vertical scroll bar.
  • Both - Always display both scroll bars.
  • Auto - Display scroll bars automatically whenever there are more cells than can be displayed.
