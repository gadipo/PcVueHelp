Using the Cursor Behavior Property

The Cursor Behavior property can be used with the position animations, Single Axis, 2 Axis and Free.

If the Cursor Behavior property is selected the user is able to click and drag the drawing element to set the value of the register. This is used to create what is commonly known as a slider control.

Enabling cursor behavior enables the Direct Send property and a number of other advanced properties applicable to the command. Show picture

The Direct Send property

If the Direct Send property is selected the value of the register is changed as the drawing is dragged. If the Direct Send property is not selected the value is only set when the mouse button is released.

You will notice that Direct Send is not the default behavior.

The use of the Direct Send option is highly discouraged when the animation is connected to a non-internal or non-local variable (i.e. a variable having its source in a PLC or a variable produced by another station in a networked system). As a consequence of the Direct Send option, and depending on how long the user holds the mouse button on the cursor, a very large number of write requests can be sent to the device or to the producer station. At best, it is useless, at worst the network or field device may be overwhelmed by the requests.

Advanced properties related to Cursor Behavior

Additional properties are available in the Advanced section of the property tab. They correspond to some common properties of command animations and are described in the section Overview of control zones.
