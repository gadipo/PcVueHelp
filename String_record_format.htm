String Record Format

When you translate texts to another language, the corresponding Style Index entries must be set up also. In the simplest case that consists of copying the column 2 cells to the right of an existing language.

Lines starting with the keyword MIMIC, TEMPLATE or SYMBOL contain the information about one text string in each of the languages. Each line is of the following format.

Keyword,String ID,String in language 1,String in language 2,...String in languagen

Description

Data type

Text

MIMIC, TEMPLATE or SYMBOL depending on the source of the string.

Default settings

For text elements that have no font and format specified, the style index is set to 0.

Syntax for the File ID field

The File ID field contains the name of library in which it is located and the name of the file (mimic, template or symbol) separated by an oblique (/).

Source

Manner of reference

Example
Local library

File name only.

Sym1

Common library

 Separator before the file name. /Sym3
The symbol file Sym3 in the common library.

Project library

 Separator between library name and file name. ProjLib/Sym2
The symbol file Sym2 in the project library ProjLib.
Shared library Separator before library name and between library name and file name. /MyLib/Sym4
The symbol file Sym4 in the shared library MyLib.

String record example

In this example, the file ID is MIMIC1 and the Text string ID is TEXT1.TEXT. There are strings defined for two languages.

MIMIC,"MIMIC1",TEXT1.TEXT,0,"This is text in a mimic",1,1,0,"C'est un texte dans un synoptique",1,1,0
