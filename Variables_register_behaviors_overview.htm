What is Associated Behavior?

Associated behavior adds functionality to a variable in addition to that provided by its main properties. The following associated behaviors are available.

Behavior Description Reg Text Bit Alarm
Threshold Set up to four bits when the value rises above or falls below specified values. See the topic Configuring a threshold. Yes      
Chronometer Measure the period for which a bit has been set. See the topic Configuring a chronometer. Yes      
Counter Count how many times a bit has been set. See the topic Configuring a counter. Yes      
Discrepancy Compare the status of one bit with another. See the topic Configuring discrepancy checking.     Yes  
Alarm - Ack. trace Set a trace bit when an alarm is acknowledged. See the topic Configuring an alarm acknowledge trace.       Yes
Alarm - Action Associate a program or mimic with an alarm. See the topic Configuring an associated action.       Yes
Alarm - Masking Mask an alarm with either a bit or an expression. See the topic Configuring alarm masking.       Yes
Event - Program Run a program on change of value. See the topic Configuring an event. Yes Yes Yes Yes
Event - Message Send a message on change of value. See the topic Configuring an event. Yes Yes Yes Yes
Trend Record the variable in an archive unit for future replay through a Trend Viewer. See the topic Configuring trend recording. Yes   Yes Yes
Overflow processing Set a bit when the value of a register variable, with an equipment source, exceeds its range. See the topic Configuring overflow processing. Yes      

Configuring and managing associated behaviors takes place in the Associated Behavior pane. The Associated Behaviors pane is displayed whenever the Variables Tree is selected in the configuration tree. The behavior associated with a particular variable is displayed when that variable is selected. Show picture The names of any other variables used in the Associated Behavior are also displayed. To display or edit the Associated Behavior, select its entry and then click the appropriate tool in the Associated Behavior toolbar.

Only one of each type of behavior can be added to a variable with the exception of an alarm's associated action. You can add two associated actions to an alarm. Some behaviors are exclusive - for example, a register cannot have both chronometer and counter behaviors. In addition, certain variable properties may need to be set or cleared for a behavior to be available. For example, to add discrepancy behavior to a bit its Command property must be set.

Each tool's appearance is modified by overlaying a symbol depending on what actions are available.

  • A green plus sign indicates that you can add the behavior.
  • A red cross indicates that you can delete the behavior.
  • A magnifying glass indicates that you can display the behavior's properties.
