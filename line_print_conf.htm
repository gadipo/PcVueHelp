How to configure Line printers

There are two distinct printing mechanisms within PcVue, graphical printing and text printing.

Graphical printing allows you to print the contents of a window or the entire screen on the Windows' default printer. This mechanism is used when producing screenshots:

  • From a SCADA Basic script using the instruction HARDCOPY.

  • When printing a report from the replay dialog box.

  • When printing using the File.Print menu item.

In general, graphical printing uses the default printer as set in the operating system. You cannot change print settings from within PcVue. The only exception to this is when printing a window using File menu item then Print. You can select and print to any printer configured under the operating system. You can also change the printer properties as usual with the Windows Print dialog box.

When using any of the screen printing methods, some graphic controls may not print correctly.

Text printing allows you to print information in a line-based text format on any printer configured in the operating system, either locally or over a network. This mechanism is used when printing events from a log list, from the Alarm and Log viewers and when using the SCADA Basic instruction LPRINT and requires that the printer is also configured in PcVue. PcVue supports printing on up to 12 printers from a single station. For each printer that you configure, PcVue creates a print spooler, which manages the information being sent to the printer. You can also print to a text file. Text printing is always in portrait format and is not affected by changing the printer properties in the operating system.

When a printer is used to output events on the fly, PcVue needs an exclusive access to the printer. If you print to such a printer from another application, and depending on a variety of factors such as the printer driver behavior, you may cause PcVue's print spooler to stop. Printing can then only be resumed by user intervention.

A printer can only be used to print data from one source. If your project requires the use of both printing mechanisms, you need two printers. For example, you could use a dot matrix printer connected to one of the PC's USB ports to print alarms and events and a color ink jet printer connected to another PC's port or network to make screenshots.

Changes you make to a printer's properties at the operating system level, including changing the default printer and paper defaults (size, orientation...) are permanent.

Changes you make at the operating system level affect the printing of graphics with PcVue immediately. Text printing is not affected by printer properties.

Configuring a line printer - Text printing

Configuring a line printer is a prerequisite to printing events from PcVue. Once a line printer is configured, you are able to:

  • Stream events to a printer via the built-in printer spooler.

  • Print events and alarms displayed in an alarm or log viewer to a text file.

  • Print events and alarms displayed in an alarm or log viewer to a Pdf or Xps file using Windows virtual printers.

  1. Go to Configure then Application Explorer, expand the General node and select Line printers.

  2. Click Add a line printer button in the toolbar. The Line printer creation dialog opens. Show picture

  3. Select the printer Id. This number is used as a reference when selecting the printer in PcVue and enter a description for the printer (optional).

  4. From the Print to drop-down, select whether to print to a File, to a Printer spooler, or to a Virtual printer and select the Enable box.

  5. Depending on what you selected in the Print to drop-down, the following properties apply:

    • (For Printer spooler) To select a printer connected via another host on a network, click on the ellipses button in the Printer section and select a printer from the list displaying known hosts on the network then select or enter a name for the printer in the Printer name field.

    • (For File) Select or enter a name for the file to which the output is sent. The file must already exist, PcVue does not create it.

    • (For Virtual printer) Select a printer. Next, click on the ellipsis button in the Folder field to select the folder where to save files and enter the prefix string for file names. Files are named after a predefined pattern that includes the date and time.

  6. Select the Advanced tab, depending on what you selected in the Print to drop-down, the following advanced properties apply:

    Properties Description
    Apply to all printer types
    Servers If set, it defines the list of server stations on which the printer will be active. Only used in networked applications.
    Apply to Printer spooler
    Working folder Defines the folder in which the temporary files used by the spooler are stored. By default, the TEMP folder within the project.
    Character conversion Defines the character conversion to be performed before sending to the printer.
    TrueType font option Defines how TrueType fonts must be dealt with. If the printer does not support TrueType fonts, then leave this property at its default setting.
    Spooler timeout

    Defines the period at which the spooler checks to see if it has anything to print.

    In some cases, it may be necessary to change this period to avoid overloading the printer buffer.
    Printer timeout

    Defines the maximum period during which continuous messages are sent to a printer. After this period, messages are saved until the next spooler timeout. In some cases, it may be necessary to change this period to avoid overloading the printer buffer.

    The properties Spooler timeout and Printer timeout are used together.

    For example, if the Spooler timeout was set to 3 seconds, and the Printer timeout set to 2 seconds, messages are sent to the printer for a maximum of 2 seconds in any 3.
    Retry timeout

    Defines the period at which the spooler retries printing to a failed printer (out of paper etc.). The default period is 900 seconds.
    Enable packet printing

    If set, the legacy packet printing mechanism is activated instead of the built-in printer spooler. The spooler and printer timeout periods are ignored and the messages are buffered and printed in packets.

    • Lines per packet - If set, it defines the number of messages (lines) per packet.

    • Add form feed - If set, it adds a form feed after each packet.
    Open warning boxes If set, displays of the Windows' printing warning boxes, under error conditions, allowing the user to cancel or retry printing.
    Verbose If set, activates the reporting of all printer transactions in the print spooler message area.
    Apply to Virtual printer
    Margins Defines the margins for the printed page.
    Orientation Defines the orientation of the printed page. Either portrait or landscape.
    Page header and footer

    Defines the look and feel of the header and footer of the page to be printed:

    • The font type, font size, and color.

    • The title of the page (Header 1) which could be a static text limited to 132 characters or a text variable limited to 256 characters as well as the printing timestamp format and alignment options.

    • The header 2 of the page which is an image. The image path has to be less than 260 characters. The path can be an absolute path or relative path to the project subfolder TP.

    • The footer page number.
    Page body

    Defines the look and feel of the page to be printed:

    • The font type, font size, and border color and style.

    • The line spacing.

  7. Click OK then Apply to confirm the configuration and close the dialog.

Depending on the operating system and the printer driver, a spooler may be associated with each printer at the operating system level. Whilst PcVue is able to print with the print spoolers enabled at the system level, it delays the detection of printing issues as they only notify problems once they are full. To disable the Windows' print spooler, go to the Windows printer properties and select the option Print directly to the printer.

Tasks are available from the Application Explorer.

When the node Line printers is selected, you can:

  • Access Line printers settings.

  • Add a Line printer.

  • Enable and Disable all Line printers.

When a configured Line printer is selected, you can:

  • Remove the selected Line printer.

  • Access its properties.

  • Enable and Disable it.

 PcVue print spooler - Text printing

 PcVue creates a print spooler for each printer configured. The print spooler is started with its interface window minimized. To restore the Print spooler window click on its icon in the Windows' task bar. Each print spooler is identified by displaying the associated printer number in its title bar.

The print spooler window displays events related to the spooler activity and a number of fields displaying the spooler properties. Show picture

If you close a print spooler, all printing associated with that printer stop until PcVue is restarted.

You can perform the following actions on the print spooler while printing:

  • Stop - Temporarily suspends printing. PcVue continues to send messages to the spooler files, which grows in size until the spooler Start button is clicked. The printer status system variable is set to 0.

  • Start - Resumes printing. The spooler prints the contents of the spooler files until they are empty. The printer status system variable is set to 1.

  • Cancel - Minimizes the print spooler window.

Using the Start and Stop buttons in the print spooler window does not have the same effect as using the Start and Stop mode of the PRINTER instruction in a SCADA Basic program. If you stop the printer from the spooler window, you must start it again from the spooler window. If you stop the printer from a SCADA Basic program, you must start it again from SCADA Basic.

Line printers settings - Text printing

To configure general printer settings, right-click on Line printers and select Settings, the following settings can be configured:

  • Printing period - Defines the period at which packets are sent to the spooler for all printers having Packet printing enabled. The default is 1 second.

  • Number of lines per period - Defines the number of lines that are sent at once when a packet is to be printed for all printers having Packet printing enabled. Enabling this property is highly recommended.

    For certain ink jet and laser printers, you must set the Add form feed property under Enable packet printing in the advanced properties of the line printer configuration to continue printing on the next page.

These settings determine the maximum rate at which PcVue prints to a line (text) printer if the Packet printing property is set in the printer configuration. For example, if the printing period is set to 5 seconds and the number of lines per period is set to 10, PcVue is able to print up to 10 lines once every 5 seconds. It is used to avoid congestion in the printing mechanism when attempting to print an excessive number of lines. For example on recovery from a communication failure.

It is included for compatibility only as the packet printing mechanism has been superseded by PcVue's print spooler.

Printing graphics

Using the file menu

Go to File in the main menu, you find here three buttons for printing: Print setup, Print preview, and Print. These provide the same functionality you find in most applications running under Windows (word processor, spreadsheet, etc.).

  • The Print setup button displays a standard Windows Print Setup dialog that allows you to select which printer to use and change its properties.

    Any change in printer properties are temporary and do not affect the configuration of the operating system.

  • The print preview button displays a preview of what will actually be printed.

  • The print button prints the contents of the window that currently has the focus (the one that was last clicked).

Using a SCADA Basic script

You can print the entire screen or the current mimic using a Run Program animation to call a script taking advantage of the SCADA Basic instruction WINDOW.
