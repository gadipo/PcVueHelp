SnapVue Settings View

The Settings view is used to manage the app's general settings.

Some settings can greatly affect the performance of the app and should only be changed by informed users.

General settings

Show picture

  • Time filter properties
    • Log lists - The default time period displayed by log viewers.
    • Trend - The default time period displayed by trend viewers.
  • Settings
    • Variables refresh period - The default refresh rate in seconds for real time variable values.
  • Theme
    • Theme color - The main app theme color, selected from a predefined palette. The default is red.
  • Audit
    • Diagnostic - Enable access to diagnostic to assist with troubleshooting, including the Connectivity and Environment views.
  • Version info - The app version.

Messaging settings

Show picture

  • Messaging properties
    • Messages polling interval when active - Rate in seconds at which the update requests are made to the messaging server when the app is in the foreground.
    • Message polling interval in background - Rate in seconds at which the update requests are made to the messaging server when the app is in the background.
    • Storage period for message attachments - The period for storing message attachments. When elapsed, message attachments are deleted from the device memory. Attachments can be downloaded again from the Instant Messaging Server on demand.
    • Mute messaging notifications - Mute instant message notifications.

    • Change display order - Changes the order in which messages are displayed - either the newest, or the oldest, at the top.

Geo-location settings

Show picture

  • Geo-tag remanence
    • Use remanence settings from server - Use the global remanence setting defined on the server or use local settings.
  • Bluetooth properties (Android app only)
    • Scan interval - The continuous period, in milliseconds, during which Bluetooth beacons are scanned for.
    • Sample expiration interval - The period, in milliseconds, over which distance calculations are averaged. Averaging is used as a mechanism to improve the accuracy of distance calculations.
    • Between scan period - The wait period, in milliseconds, between 2 consecutive Scan Intervals. The wait period is used to minimize the effect of Bluetooth signal echo when calculating distance.
  • Indoor positioning properties
    • Location polling period when active - Period in seconds between requests to the location web service for updates when the app is in the foreground.
    • Location polling period when in background - Period in seconds between requests to the location web service for updates when the app is in the foreground.
    • Minimum position change for tracking - The distance in cm that is considered as a significant move when refreshing the user contextual information.
    • Expiration time of tracking results - The time in seconds after which the user contextual information expires and a new refresh request is sent.

Notification settings

Show picture

  • Notifications
    • Mute alarm notifications - Mute, or not, alarm notifications. The default is Off (no muting).
    • Start the alarm notification service at boot time (Android app only) - Start the alarm notification service at device boot time. This means that, once logged in, alarm notifications will be displayed, even after a device restart, without the need to restart the app. The default is On.
    • Contextual alarming - Display only contextual alarms, that is alarms relevant to the user's current location, or display all alarms. The default is On (contextual alarms only).
  • Alarm properties
    • Change display order - Changes the order in which alarms are displayed - either the newest, or the oldest, at the top.
    • Polling interval for alarm list refresh - Period in seconds between requests for alarm list updates.
    • Ringtone (iOS only) - The ringtone associated to notifications.
