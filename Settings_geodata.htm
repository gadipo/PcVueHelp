Geographical Data Settings

The geographical data settings can be displayed from the Geographical Data entry in the Application Explorer's Main settings dialog or from the Settings task when the Geographical Data folder is selected in the Application Explorer's configuration tree. Show picture

  • Zones
    • Zone detection criteria when no geo-tag defined - This setting controls if a zone with no associated geo-tag shall be displayed or not in the SnapVue app:
      • Always display - Such a zone will always be displayed in SnapVue app as soon as you are connected.
      • Never display -Such a zone will not be displayed in SnapVue.

      A zone without a geo-tag associated makes a perfect sticky zone users can always access for actions of general interest such as displaying a production line status of energy dashboard.

  • Geo-tags
    • Default remanence - The remanence settings to be used by client app of PcVue unless overwritten by its own local settings. The remanence is the period of time, after receiving a signal from a geo-tag, that it will be considered as detected. The value set for remanence depends on the type of geo-tag. For example, Bluetooth beacons emit a signal about every second so the remanence can be low. NFC and QR codes are detected once only, so the remanence means how much time these tags should be considered part of remain in the user environment.
      • Unlimited - The remanence period is unlimited. That is once it has been scanned it is to be permanently considered part of the user environment stored in client's memory.
      • During time interval - The remanence period in hours, minutes and seconds.
