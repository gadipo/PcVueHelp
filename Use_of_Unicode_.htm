Important Note About the Use of Unicode for Character Encoding

 PcVue uses a mechanism called DBCS (double byte character sets) for encoding the characters used in its strings. The Import/Export Tool converts the DBCS characters to/from Unicode during the import/export process, unless the Create Unicode Files tick box is not ticked. If it is not ticked, single byte (ANSI) encoding is used.

Wherever possible Unicode should be used in order to support the widest range of languages (particularly those with an Asian origin).

When Unicode should be used

Unicode encoding should only be used with a third party editor that supports it.

About converting Unicode to DBCS

The precise conversion process depends on the system's native language and/or Regional Setting configuration. Some combinations of these parameters work for particular languages but not for others. Therefore, it is strongly recommended that, if after an import the character rendering in PcVue produces unexpected results, you do not use the Import/Export Tool for that particular language.
