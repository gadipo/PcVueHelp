Using the Alarm Viewer to Display Unavailable Alarms

An alarm in the unavailable state no longer responds to changes in value of its source or User actions. The unavailable state may be caused by a number of conditions and an alarm will remain in the unavailable state until the condition or conditions that caused it are reversed or removed.

In the Alarm Viewer, you can select the unavailable alarms to be displayed according to the condition that caused it. The event text that is displayed (using the #E format character) is specific to the condition, but the color is the same for all unavailable alarms of a particular level.

The following conditions can cause an alarm to be unavailable.

Condition/Event (#E)

Cause

Alarm operator masked

The User has masked the alarm using the Alarm Viewer or it has been masked using the SCADA Basic instruction ALARM.

Alarm program masked

The alarm has been masked by the SCADA Basic instruction VARIABLE.

Alarm variable masked

The mask variable, as selected in the alarm variable's configuration, is true.

Alarm inhibited

The Inhibited property has been selected in the Advanced tab of the variable's configuration or it has been inhibited by script.

Alarm not accessible

The source of the alarm variable is unavailable. For example, communication to a PLC has failed.

Alarm expression masked

The expression, as selected in the Advanced tab of the alarm variable's configuration, is true.

If you select the option to display all alarms that are unavailable, using the alarm icon on the tool bar, the event text that is normally specific to the condition is replaced with Alarm Unavailable and all unavailable alarms are displayed, irrespective of the reason.
