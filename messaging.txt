The chats list
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The chats list
 
When a mimic containing a Messaging Control is opened, by default, the chats list is displayed. The chats list displays all the chats, for which the current user has permissions, plus a tool in the top left corner for adding a new chat. Show picture
 

 
 
 
Clicking a chat displays the chat view from where messages can be read and written and, if the user has the correct permissions, the chat settings can be changed.
 
How to add a chat
 
 
Click the tool at the top left of the chats list. The Add chat dialog is displayed. Show picture

 
Enter the Chat name. The chat name is only visible in the configuration and cannot be changed once the chat has been created.
 
Enter the Chat title. The chat title appears as the identifier in the chats list and can be edited at run-time by a user having Owner privileges.
 
Select the Storage time for messages. Messages are automatically discarded after this period.
 
Select the users that will have access to the chat, and their privileges. 
 
Click the Tick button to confirm the creation of the chat. The chat view is displayed.
 
 

 
Once a chat has been created it can never be deleted.

The chat view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The chat view
 
The chat view is displayed by clicking on a chat in the chat list, or after creating a new chat. Show picture
 

 
 
 
Messages from the local user are displayed on the right, messages from other users on the left.
 
The chat settings and filter can be changed from the menu tool at the top right corner, if the chat user has the correct privileges.
 
How to send a message
 
New messages are added by typing in the text box at the bottom of the chat view and clicking the Send button. Note that the send button only appears when there is a message to send, replacing the Record button. 
 
How to change the chat settings
 
Message setting can only be changed by a user having the Owner privileges. The Settings view is displayed by clicking the menu tool and selecting Settings. Show picture
 

 
 
 
The following settings are available.
 
 
 Chat name - The name of the chat from when it was created. Read only.
 
Chat title - The title of the chat which appears in the chats list.
 
Appearance - Select the chat color. The color is allocated automatically when a chat is created and can be changed later.
 
Storage time of messages - The message storage time. Messages are deleted automatically after this period unless Unlimited is selected. Note that this setting is applied retrospectively to messages that have already been created.
 
Users - Invite and remove users, and / or change their permissions.
 
Quit the chat room - Leave the chat. A chat must have at least one owner so, if the original owner wishes to leave the chat, another user must be given owner privileges.
 
 
How to change the message filter
 
The Message Filter dialog is displayed by clicking the menu tool and selecting Filter. Show picture
 

 
 
 
 
Display last messages - Display messages sorted by the most recent first.
 
Display messages on date - Display only messages that match the time and date filter.
 
Users filter - Display only messages from the selected users.

Inserting a Messaging Control in a mimic
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Inserting a Messaging Control in a Mimic
 
You insert a messaging control in a mimic as follows.
 
 
Insert the messaging control.
Open the mimic in which the messaging control is to be inserted and select Design mode.
From the Insert menu select Messaging control. An empty rectangular drawing element appears in the mimic to represent the messaging control and its properties dialog opens. 
 
Select the Aspect tab to configure the messaging control's appearance. Show picture
Use the Border and Appearance properties to configure the style of the controls' border. These are almost identical to those used when configuring a solid drawing element such as a rectangle.
Use the Position and Size properties to fine-tune the control's position and size. 
Use the Font and colors properties to change the background color and the font used for any text displayed by the control. Note that you must first tick the corresponding boxes to enable these properties.

Messaging Control overview
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Messaging Control Overview
 
The Messaging Control provides access for a user to the Instant Messaging service. The Instant Messaging service allows a user to communicate, by sending and receiving text messages, to one or more other users on another PcVue station and / or one of the mobile applications.
 
Instant messaging uses a conversation known as a chat (or chat room). Chats are created, and owned, by a specific user. Once a chat has been created, the owner can invite other users and change user permissions. In the PcVue desktop application, chats are managed at run-time using the Messaging Control. Show picture
 

 
 
 
For more information about Instant Messaging see the book Communication.Servers.Web & Mobile back ends in the Application Explorer help.
 

 
Instant Messaging is licensed. If no license is available, or if the feature is not enabled by the license, the Instant Messaging feature runs in trial mode and stops after one hour.

Configuring the run-time behavior
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring the Messaging Control Run-time Behavior
 
The run-time behavior is configured using the Execution and Filter tabs. 
 
Selecting the run-time features
 
The features that are available, from a particular instance of the messaging control, at run-time are configured from the Execution tab. Show picture
 

 
 
 
 
Chat settings
Allow creating a new chat - Allow the user to create a new chat. The user will be the owner of the newly created chat.
Allow quitting a chat - Allow the user to leave a chat using the Settings dialog. Note that a chat must have at least one user with owner permissions. If the user is the last owner, he will be prevented from leaving.
Allow inviting / removing chat users - Allow the user to invite and remove users from chats, and to change their permissions using the Settings dialog.
Allow modifying chat properties - Allow the user to change the chat color and message storage time using the Settings dialog.
 
Navigation
Allow entering / exiting a chat - Allow the user to enter a chat, by clicking on it in the chat list. Allow a user to return from a specific chat to the chat list.
Allow filtering - Allow the user to use the chat Filter dialog.
Allow opening attachments - Allow the user to open message attachments.
 
Messages
Allow sending text messages - Allow the user to send text messages. Even if the sending of text messages is not allowed a user can still record and send voice messages and / or attach a file to a text message.
Allow record voice messages - Allow the user to record and send a voice message.
Allow attaching files - Allow the user to attach a file to a message.
 
 

 
These properties limit what users will be able to do with a particular instance of the Messaging Control in a mimic. They have precedence over user permissions on chats, but do not override them.
 
For example, if a user is part of a chat but uses a Messaging Control configured to prevent sending text messages, he will be in a view only mode, not allowed to send messages.
 
By using these properties properly, you can have several instances of the Messaging Control in different mimics, to serve different purposes:
 
 

 
In a mimic accessible in one click from the mimic header (or a function key), a Messaging Control locked on a chat dedicated to general talks among team members.
 
 

 
In mimics corresponding to process parts or sites, a Messaging Control locked on the chat corresponding to the installation.
 
 

 
In a mimic dedicated to chat, a Messaging Control allowing users to create and manage ad'hoc chats.
 
 
 
 
Configuring the opening conditions
 
The opening conditions, that is what the user will see when the mimic containing the Messaging Control is opened, are defined by the Filter tab. Show picture
 

 
 
 
 
Opening properties
Default chat - Select a specific chat that will be displayed when the Messaging Control is opened at run-time. If left blank a list of all chats, that the user has permission to access, are displayed.
Persist properties - Remember the chat that was displayed when the Messaging Control was closed and display it next time it is opened.
 
Display time range - Select a specific date and time range over which messages are filtered.

Sending and receiving other media
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Send and Receive Other Media
 
In addition to sending plain text messages, the messaging control is also able to send attachments in the form of a file, or a voice message. Note that you can send either a text message, an attachment or a voice message. You cannot send, for example, an attachment and accompany it with a text message. To do that you would have to send two separate messages.
 
Sending attachments
 
Attachment are sent using the button at the lower left corner of the chat. The button opens a browse window from where the file is selected. After it has been selected, and the browse window closed, the file is sent using the Send button. If the file is in a recognized image format, a thumbnail is displayed in the chat. Show picture
 

 
 
 
Receiving attachments
 
Received attachments are displayed in the chat in a similar manner to a text message. If the attachment is in a recognized image format, a thumbnail is displayed. Otherwise, click on it to download and open it with the Windows default application applicable to the type of file.
 
Sending and receiving voice messages
 
Voice messages are sent using the button at the lower right corner of the chat. The button opens a voice recorder from where the user records the message. After it has been recorded, and the voice recorder closed, the file is sent using the Send button. Show picture
 

 
 
 
Received voice messages are displayed in the chat in a similar manner to a text message. Clicking on a voice message will play it through the computer's default audio channel.
