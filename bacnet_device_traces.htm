BACnet Device Traces

The Traces tab contains properties that allow you to enable diagnostic trace messages associated with the BACnet device object. Enabling a trace generates a corresponding message, which is logged in the Trace Files and displayed in the Event Viewer dialog (F7) when the condition occurs.   • Trace 1: Error level 1 - Unused.
  • Trace 2: Error level 2 - Unused.
  • Trace 3: Binding - Enable traces related to device detection.
  • Trace 4: Full errors from BACnet stack - Enable traces to display more details of errors related to the communication stack.
  • Trace 5: Full details - Enable all traces related to the device, not just error messages.

  • Trace 7: Device status - Enable traces related to the state of device objects.
  • Trace 8: Add item - Enable traces related to the creation of items.
  • Trace 9: Create Notification Class - Enable traces related to the creation of Notification Class objects.
  • Trace 10: Delete step - Enable traces related to the deletion of a device.
  • Trace 11: OnWrite method - Enable traces related to the writing to a device by PcVue.
  • Trace 12: OnDataChange method - Enable traces related to change of data values.
  • Trace 13: Connection - Enable traces related to device connection status.
  • Trace 14: Reporting events - Enable traces related to event notifications transmitted by the device.
  • Trace 15: Alarm helper - Enable traces related to the helper tool.
  • Trace 16: Scan - Enable traces related to the BACnet device browsing.

  • Trace 17: Text messages by notification – Enable traces related to notification of text messages from devices.
