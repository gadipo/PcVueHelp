Overview of Context Mimics

A context mimic is created in the same way as any other mimic. It becomes a context mimic when it is opened in one of two special ways.

  • By clicking on certain animation types with the right-hand button of the mouse.
  • By using the CONTEXT instruction from a Script animation.

To differentiate it from a normal Mimic Open operation, when a context mimic is opened the drawing element supporting the originating animation is surrounded by a rectangle with a dashed border. Show picture

A context mimic has a special relationship with the variable used in the animation (or specified in the CONTEXT instruction).

  • The variable name is automatically made available in the text variable SYSTEM.CONTEXT.NAME.
  • The variable title is automatically made available in the text variable SYSTEM.CONTEXT.TITLE.
  • Substitution strings associated with the variable may be used in any script animation in the Context mimic. For example, #V will substitute the variable name.

Context mimics are created using the Mimic Editor in the same way as any other mimic. The only special requirement is that a context mimic must be configured with the Popup Behavior property enabled.

A context mimic behaves as a child of the mimic in which the originating animation resides.

  • It may only be moved within the bounds of the parent mimic.
  • The opening position is determined by the attributes of the child mimic.

A major feature of a context mimics is the way in which attributes of the context variable may be substituted using special character strings in animations. See the topic Using Substitution Strings in an Animation for more information.
