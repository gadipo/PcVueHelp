Configuring ICCP Variables

The user can create and configure ICCP variables in PcVue by adding a formatted value in the extended attributes of the variable. This value contains ICCP mapping information for that variable. It has 7 or 9 fields, depending on the type, with each field separated by a special character corresponding to the parameter BranchSeparator.

The user can configure two important parameters in the ICCP.ini file that are later used to parse the variables extended attributes:

  • BranchSeparator - The field separator in text extended attributes
  • AttributeNumber - The Id of the text extended attribute that contains the ICCP mapping

The number of characters in an extended attribute is limited to 100. To reduce the length, some of the more common fields are entered with a single or a few characters.

It is important that to use short names with care for the elements Network, LCC, Local Domain, RCC, Remote Domain and Data Set. All the name lengths come into play when using this extended attribute text.

Required parameters

In the extended attribute, the user must enter specific information that is important to define the variable mapping. All the fields must be separated by the character BranchSeparator defined in the ICCP.ini file. The specific order of these fields depends on the variables type (data or device command) and its origin (local or remote).

Field Value Description
1 ICCP

A header that must be ICCP. Mandatory for all ICCP variables.

Used to identify that this is an ICCP variable.

Case sensitive.
2 <LocalOrRemote>

A string that can be either: L or R. These letters represent Local or Remote variables.

Defines if the variable belongs to the local control center or to the remote control center.

Mandatory. Not case sensitive.
3 <DataType>

A string that defines the ICCP data type of the variable. It must be one of the 15 data types or 3 control types.

The allowed values for this field are: D, DE, DQ, DQT, DQTE, R, RE, RQ, RQT, RQTE, S, SE, SQ, SQT, SQTE, CC, DSP or RSP.

Mandatory. Not case sensitive.
4 <Network>

A string that defines the Network that the variable belongs to.

It must match a Network Name configured in the ICCP.ini file.

Mandatory. Case sensitive.
5 <LCC>

A string that defines the LCC that the variable belongs to.

It must match a LCC Name configured in the ICCP.ini file that is consistent with the Network.

Mandatory. Case sensitive.
6 <RCC>

A string that defines the RCC that the variable belongs to.

It must match a RCC Name configured in the ICCP.ini file that is consistent with the LCC and Network.

Only Remote variables require this field. Case sensitive.
7 <Domain>

A string that defines the domain name (Local or Remote) that the variable belongs to.

- If the LocalOrRemote field is L, then this field is the local domain name.

- If the LocalOrRemote field is R, then this field is the remote domain name.

It must match a domain configured in the ICCP.ini file.

- If the variable is Local. Either VCC or the Local Domain Name entry established in desired Local Domain section of the ICCP.ini file.

- If the variable is Remote. Either VCC or the Remote Domain entry established in the desired Remote Control Center section of the ICCP.ini file.

Mandatory. Case sensitive.
8 <DataSet>

A string that defines the data set that the variable belongs to.

It must match a dataset configured in the ICCP.ini file.

Only Remote data type variables require this field. Case sensitive.
9 <ICCPVarName>

A string that defines the ICCP variable name.

The name does not have to be the same as the variable name to which it is mapped. The name must be unique in the domain (Local or Remote) it is assigned to.

Case sensitive.
10 <MappingId>

A string that defines the mapping mechanism that the variable is to use. There are 3 possible options for this field: F (Full), H (High) and L (Low). This field should be F for all variable cases except for the following: State, StateExtended, StateQ, StateQTimeTag and StateQTimeTagExtended.

Only variables for monitoring require this field. This field is case sensitive.
11 <CurrentSource>

A string that defines the ICCP source of the variable.It can be: Entered, Calculated, Estimated or Telemetered.

The allowed values for this field are: EN, C, ES or T.

It specifies the nature of the value.

Only local data type variables require this field. Not case sensitive.
12 <SBOOrDE>

A string that defines the control mechanism for the variable.

The value can be either SBO (Select Before Operate) or DE (Direct Execute).

Mandatory only for device command type variables. Not case sensitive.

More information about <MappingId>

  • For ICCP States, you can select to use a bit, register or alarm variable.
  • All Discrete, Real, ControlCommand, DiscreteSetPoint and RealSetpoint variables use F as their mapping ID. ICCP State variables have 4 possible states: 0, 1, 2 and 3. Typically, 0 is intermediate, 1 is open, 2 is closed and 3 is error.
  • A register mapped to a state, functions in the following way:
    • As a register it can only have F as its mapping Id.
    • The value 0 represents a State value of 0 (00 or intermediate).
    • The value 1 represents a State value of 1 (01 or open).
    • The value 2 represents a State value of 2 (10 or closed).
    • Any other value represents a State value of 3 (11 or error).
  • A bit or alarm can function in the same way.
    • If the mapping Id is F and the bit is valid in PcVue, it can send only 2 values. This option can be inverted when used in conjunction with the InvertStateSingleBit parameter discussed previously.
    • The value of 0 represents a State value of 1 (01 or open).
    • The value of 1 represents a State value of 2 (10 or closed).
    • If the bit variable is not valid (Status NS) it represents a State value of 3 (11 or error).
  • If the mapping Id is not F, then the ICCP State must be represented with 2 bits or 2 alarms. One of the bits will represent the High part of the bitstring and the other bit will represent the Low part.
    • H and L must be used together. They must be associated to the same <ICCPVarName> but with different PcVue variable names.
    • Example: 2 PcVue bit variables with the following fields in the extended attribute:
      SVBit1 -> ICCP/MyState/H/Dom/Discrete/Telemetered

      • SVBit2 -> ICCP/MyState/L/Dom/Discrete/Telemetered
      When SVBit1 is 0 and SVBit2 is 0, they represent an ICCP state MyState of 0 (00 or intermediate)
      When SVBit1 is 0 and SVBit2 is 1, they represent an ICCP state MyState of 1 (01 or open)
      When SVBit1 is 1 and SVBit2 is 0, they represent an ICCP state MyState of 2 (10 or closed)
      When SVBit1 is 1 and SVBit2 is 1, they represent an ICCP state MyState of 3 (11 or error)

    • In this High/Low configuration of the states, the timestamp that is sent to the ICCP variable comes directly from the variable’s origin.

For local ICCP monitoring variables

The structure of the parameters should be entered in the corresponding extended attribute of the variable and contain 9 different fields as follows:

ICCP/<LocalOrRemote>/<DataType>/<Network>/<LocalControlCenter>/<Domain>/<ICCPVarName>/<MappingID>/<CurrentSource>

Example for a VCC variable:

ICCP/L/D/NET1/LCC2/VCC/SV_VCC_Disc/F/T

Example for an ICC variable:

ICCP/L/D/NET1/LCC2/LCC2_Dom1/SV_ICC_Disc/F/T

For local ICCP device command variables

The structure of the parameters should be entered in the corresponding extended attribute of the variable and must contain 8 different fields as follows:

ICCP/<LocalOrRemote>/<DataType>/<Network>/<LocalControlCenter>/<Domain>/<ICCPVarName>/<IsSBO>

Example for a VCC SBO variable:

ICCP/L/DSP/NET1/LCC2/VCC/SV_VCC_DSP_SBO/SBO

Example for an ICC DE variable:

ICCP/L/RSP/NET1/LCC2/LCC2_Dom1/SV_ICC_RSP_DE/DE

For remote ICCP monitoring variables

The structure of the parameters should be entered in the corresponding extended attribute of the variable and must contain 10 different fields as follows:

ICCP/<LocalOrRemote>/<DataType>/<Network>/<LocalControlCenter>/<RemoteControlCenter>/<Domain>/<DataSet>/<ICCPVarName>/<MappingID>

Example for a VCC variable:

ICCP/R/RQ/NET1/LCC2/LCC2_MyRCC1/VCC/LCC2_MyDataSet/SIM_VCC_RealQ/F

Example for an ICC variable:

ICCP/R/SQTE/NET1/LCC2/LCC2_MyRCC1/SIM_Domain/LCC2_MyDataSet/SIM_ICC_Reg_StateQTTE/F

For remote ICCP device command variables

The structure of the parameters should be entered in the corresponding extended attribute of the variable and should contain 9 different fields as follows.

ICCP/<LocalOrRemote>/<DataType>/<Network>/<LocalControlCenter>/<RemoteControlCenter>/<Domain>/<ICCPVarName>/<IsSBO>

Example for a VCC DE variable:

ICCP/R/DSP/NET1/LCC2/LCC2_MyRCC1/VCC/SIM_VCC_DSP_DE/DE

Example for an ICC SBO variable:

ICCP/R/CC/NET1/LCC2/LCC2_MyRCC1/SIM_Domain/SIM_ICC_CC_SBO_Reg/SBO
