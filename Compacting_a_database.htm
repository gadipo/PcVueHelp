Shrinking a Database

The Database Manager can shrink Access databases and SQL Server databases. The shrinking function seeks to reduce the size of the files that make up a database by:

  • Removing previously deleted records.
  • Reorganizing the physical structure to eliminate wasted space.

You can access the shrinking options for a database from the tree structure in the Database Manager:

  1. Click on the database's name then select the Action menu (or right-click on the required database to open its context menu).
  2. Select Shrink. The shrinking process is run immediately.
  3. When a dialog confirms that the process has finished, select Close. Show picture

For Access databases, the shrinking function requires exclusive access to database, so any other applications must first be disconnected from the database.

For SQL Server databases, applications can remain connected but exclusive access is recommended.
