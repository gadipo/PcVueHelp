Configuring the Alarm Viewer Filter

The alarm filter is used to decide which alarms will appear in the list. Show picture



  • Domain - Optional domain filter for alarms. ******** is the default and displays alarms from all Domains.
  • Nature - Optional nature filter for alarms. ******** is the default and displays alarms of all Natures.
  • SQL filter - Optional SQL-style filter for alarms. See the topic Using a SQL expression to filter variables.
  • Alarm level - Filter using the alarm level.
    • Minimum level - If selected only alarms of the lowest level are displayed. For example, if the alarm range was set from level 3 to level 10 and the lowest level alarm passing the filter criteria was level 5, only level 5 alarms will be displayed. If, later, a level 4 alarm occurred then only level 4 alarms would be displayed.
    • Maximum level - If selected only alarms of the highest level are displayed. This option is mutually exclusive to minimum level. For example, if the alarm range was set from level 3 to level 10 and the highest level alarm passing the filter criteria was level 8, only level 8 alarms will be displayed. If, later, a level 9 alarm occurred then only level 9 alarms would be displayed.
    • All -If selected all alarms between the To level and From level properties are displayed.
    • From level -Minimum alarm level to display.
    • To level -Maximum alarm level to display.
  • Alarm status -Filter using the alarm status.
    • Alarm on, not ack. -Display alarms that are on but have not yet been acknowledged by the User.
    • Alarm on, ack. - Display alarms that are on and have been acknowledged by the User.
    • Alarm off, not ack. -Display alarms that have been on and gone off before being acknowledged by the User.
    • Alarm off -Display alarms that are off.
    • Maintenance -Display only alarms that have been set to maintenance.
    • All alarms unavailable -Display alarms that are unavailable for any reason.
    • Alarm inhibited -Displays alarms that are unavailable because they have the inhibited property set in their configuration.
    • Alarm not accessible -Displays alarms that are unavailable because their value is no longer updated (NS COM).
    • Alarm user masked -Display alarms that are unavailable because they have been masked by a User (from the Alarm display)
    • Alarm program masked -Display alarms that are unavailable because they have been masked by a SCADA Basic program.
    • Alarm variable masked -Display alarms that are unavailable because have been masked by another variable (a bit or another alarm as specified in the alarm's configuration).
    • Alarm expression masked -Display alarms that are unavailable because they have been masked by an expression.

An alarm can be configured to be on when it is either in the one or zero state. See the Variables Tree book in the Application Explorer help for further information.
