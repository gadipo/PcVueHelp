IEC 60870-5 client general settings

IEC 60870-5 settings are accessed from the Application Explorer by selecting <Project>.Communication.Data acquisition.IEC 60870-5 and choosing the Settings command from the task list. Show picture



  • Invalidate variables - Defines if variable status should be set to invalid upon connection failure.
    • Never - The variables are never made invalid. This can be used when there is intermittent communication with the equipment.
    • Immediately - The variables are made invalid immediately on a communication failure. This is the default value.
    • After a delay - The variables are made invalid after the specified delay.
  • Monitoring
    • Use the local timestamp upon device reconnection - Use PcVue’s timestamp for the first value received after a device is re-connected.
    • Double-point
      • Intermediate state - Defines how Intermediate state value should be interpreted.
        • Time before invalidating bit variable - The delay of the intermediate state (0,0 or 1,1) before the object is considered to be in error.
      • Off – Defines how Off value should be interpreted.
      • On – Defines how On value should be interpreted.
      • Indeterminate – Defines how Indeterminate value should be interpreted.
        • Time before invalidating bit variable - The delay of the indeterminate state (0,0 or 1,1) before the object is considered to be in error.
      • Register/Text mapping
        • Apply the invalidating rules as for bit variables - If enabled, settings driving invalidity for bit variables are applied to register and text variables mapped on double-points.
  • Control
    • Update the variable immediately with the written value - If enabled, the variable value is updated immediately after the write rather than waiting for the device to acknowledge the command execution.
    • Single command
      • Default qualifier - The default qualifier sent with a Single command.
      • Default method - The default method, either Direct Execute or Select Before Execute, used for a Single command.
      • Time tagged - If enabled, a time-tagged command will be sent.
    • Double command, Regulating step, Bitstring, Set-point-command - See Single command.
  • Quality descriptor management - Selects which of the quality flags from the device are considered as invalid quality in PcVue. By default, only substituted (SB) is not selected.
  • Attributes and properties - Defines how relevant variable properties shall be mapped on IEC 60870-5 built-in mechanisms.
    • Deadband property
      • Control direction - Overwrite threshold parameters - If enabled, the deadband value of the variable is sent to devices to overwrite the threshold parameters.
  • Networking
    • Switchover turnaround delay - The guaranteed interval of silence during data acquisition server switchover. During this time, no request is sent to devices by the IEC 60870-5 driver. Required by some devices as part of their connection management. Default 200 ms.
