Tyco POSM Ip

This driver supports communication over Ip with the Tyco POSM XPro system.

The system is typically used to monitor bridges and tunnels using cameras, monitors, matrix displays and advanced vision systems. The driver operates in both master and slave modes.

Supported functions

In master mode, PcVue polls the system 'heartbeat' periodically (every minute is recommended). PcVue can also send a command to the POSM XPro system, to switch a camera to a particular monitor for example. In return, the XPro system sends an acknowledgment to PcVue .

In slave mode, more than one PcVue can control a POSM XPro system so it must be possible to receive unsolicited information on the status of the system. The driver supports that feature.

SCADA Basic scripting

You can control communication with this driver in SCADA Basic scripts with the CIMWAY instruction.

See the topic Message exchange mechanisms in PcVue for more information about what to optimize and how you can send commands from scripts.

The driver also supports the following specific capabilities in SCADA Basic.

Sending a control message to the driver

IntVal = CIMWAY (SENDMSG, CommObjectName, CommandString);

Return type: INTEGER.

Argument

Meaning

CommObjectName

Identifies the target object identifier:

All networks: Empty string.

A specific network: NetworkId.

A specific device: NetworkID.EquipmentId.

A specific frame: NetworkID.EquipmentId.FrameId.

Type STR.

CommandString

The command to send to the driver. Type STR.
