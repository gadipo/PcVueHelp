Advanced Properties Specific to BACnet Variables

The Advanced tab of the variable's properties dialog contains a number of properties only available when a BACnet source has been selected. Show picture

  • Monitoring type - The mechanism used to obtain a BACnet variable's value.
    • Polling - The variable value is obtained by polling the device at the rate configured in the polling interval. This is the default setting.
    • Confirmed change of value (C-COV) - The variable value is obtained using the BACnet C-COV mechanism whereby the device sends changes of value as they occur without the need for polling. The driver confirms that the change in value has been received.
    • Unconfirmed change of value (U-COV) - The variable value is obtained using the BACnet U-COV mechanism whereby the device sends changes of value as they occur without the need for polling. The driver does not confirm that the change in value has been received.
  • Write priority- The BACnet write priority, range 1 to 16, where 1 is the highest priority. The default is 16.
  • Fields - Used to specialize the mapping, including extraction of values from structured data types, specific deserialization or to specify how to map to a proprietary property. See below for more information.
  • Property - Used to define the name of the BACnet proprietary property to map to.

Using the Fields property for advanced mapping

The table below describes syntax elements that helps you configure ad'hoc BACnet value deserialization for the most advanced mapping needs.

Type Description
OctetString(x) Return the value of the byte at position x in property of type Octet_String. Can be used with text, register and bit variables. If the specified byte does not exist in the string, the variable will have the quality NS FMT.
BitString(x) Return the value of the bit at position x in a property of type Bit_String. Can be used with text, register and bit variables. If the specified bit does not exist in the string, the variable will have the quality NS FMT.
Reference(Object_Type, Property_Id)

Return the content of proprietary property in XML format that has a type already defined in the BACnet standard.

Object_type is the Id of the type of object.

Property_ID is the Id of the property within the object.

Can only be used with text variables only. The result is returned in XML format.

If the referenced type does not correspond with the format of the value, then the quality of the variable will be NS FMT.
BACnetObjectIdentifier(x)

BACnetObjectIdentifier is a specific BACnet type. Generally used for the property Object_Identifier that identifies the BACnet object instance identified by two elements Object type and instance number.

This type is defined as a data type in the BACnet specification and it is necessary to specify this type (using the position “x”) to return one of these elements. Can be used with text variables only. If the specified instance does not exist in the string, the variable will have the quality NS FMT.
BACnetObjectIdentifier Similar to BACnetObjectIdentifier(x) above except you cannot specify the instance. Can be used with text variables only.
BACnetArray(x)

Returns the value at the position “x” in a property of type BACnetArray encoded as an integer. The range for “x” is linked to the property value. If the position is defined out of the range of array, the variable will be NS FMT.

The syntax is : BACnetArray ([Position]) where Position is the position or index in the BACnet array.

IsBACnetArrayNull(x)

Returns the state of the value defined at the position “x” ( NULL (1) or not (0), encoded as an Integer.

The syntax is: IsBACnetArrayNull ([Position]) where Position is the position or index in the BACnet array. If the position is out of the range of the array, the variable quality will be NS FMT.
FormattedXML() Returns a string in XML format, including indentation. Used to deserialize structured BACnet properties such as Weekly-Schedule into a text variable. Can only be used with text variables.
XML() Similar to FormattedXML() except the deserialized string is in a raw XML format (no indentation).

Syntax example:

To use the property action-text (ID = 3 as defined in the BACnet standard ) of the object Command (ID= 7 as defined in the BACnet standard) you could use either:

Reference(command,action-text)

or

Reference(7,3)

Using the Fields property to handle priority

The table below describes syntax elements that helps you handle the priority mechanism of BACnet for commandable objects.

Type Description
SetPriority(ValueOrVariableName, Priority)

Applied to a commandable bit variable mapped on a BACnet object supporting the priority mechanism, it allows setting or resetting the value at a given priority for the object on which the bit variable is mapped.

The parameter ValueOrVariableName is either the numeric value to set or the name of a register variable that holds the numeric value to set.

The parameter Priority is the priority for writing to set or reset (1 to 16).

When the bit variable is set to 0, the value NULL will be written to the BACnet item at the Priority.

When the bit variable is set to 1, the value of the parameter ValueOrVariableName will be written to the BACnet item at the Priority.
IsPrioritySet(Priority)

Applied to a bit variable mapped on a BACnet object supporting the priority mechanism, it allows retrieving if the value at a given priority is set for the object on which the bit variable is mapped.

The parameter Priority is the priority for writing to check (1 to 16).

If the value for Priority is NULL, the bit variable is set to 0 (priority value not defined), otherwise it is set to 1 (priority value set).
ActivePriorityValue(Priority)

Applied to a bit, register or text variable mapped on a BACnet object supporting the priority mechanism, it allows retrieving the value at a given priority for the object on which the variable is mapped.

The parameter Priority is the priority for writing to check (1 to 16).

If of type register or text, the value of the mapped variable is set with the value defined for the Priority.

If of type bit, the value of the mapped variable is set to 0 if no value is defined for the Priority (NULL), and it is set to 1 if a value is defined.
ActivePriority()

Applied to a register variable mapped on a BACnet object supporting the priority mechanism, it allows retrieving the currently active priority for the object on which the variable is mapped.

The return value is a priority in the range [1..16].
IsActivePriority(Priority)

Applied to a bit variable mapped on a BACnet object supporting the priority mechanism, it allows checking if the priority passed as parameter is the highest written priority for the object on which the variable is mapped.

The parameter Priority is the priority for writing to check (1 to 16).

The value of the bit variable is set to 1 if Priority is the highest written priority, otherwise it is set to 0

If the value at Priority is NULL, the value of the bit variable is also set to 0.

Using the Fields property to map on a proprietary property

Manufacturers sometimes define a property that is not in the BACnet standard. These are called proprietary properties. There are three possibilities with proprietary properties.

  • Basic type - A simple type like boolean, unsigned integer, real, or a string of values (like byte string or bit string or character string).
  • Structured type defined in the BACnet standard. - For example, a character string array as defined for the property “action-text” of the BACnet object “command”,
  • Structured type not defined in the BACnet standard. It is currently not possible to use such proprietary properties with PcVue.

The format of a proprietary property can be defined by the Fields property to make it possible to map a variable on a proprietary property.

Since version 11.2, mapping to proprietary properties based on BACnet standard types is supported by a built-in mechanism of PcVue. Using the Fields property for this purpose is obsolete and only necessary if your project was designed with a version earlier than 11.2.

Type Description
Boolean() Return the value of a proprietary property encoded as a Boolean. It can be used with bit, text and register variables.
Double() Return the value of a proprietary property, encoded as a Double. It can be used with bit, text and register variables.
Real() Return the value of a proprietary property, encoded as a Real. It can be used with bit, text and register variables.
Signed() Return the value of a proprietary property encoded as an Signed integer. It can be used with bit, text and register variables.
Unsigned() Return the value of a proprietary property encoded as an Unsigned integer. It can be used with bit, text and register variables.
Date() Return the value of a proprietary property encoded as a Date. It can be used with text variables only.
Time() Return the value of a proprietary property encoded as a Time. It can be used with text variables only.
OctetString() Return the value of a proprietary property encoded as an Octet_String in a raw format, each byte value is separated by a comma. It can be used with text variables only.
BitString() Return the contents of a proprietary property encoded as a Bit_String, in a raw format, each bit value is separated by a comma. It can be used with text variables only.
