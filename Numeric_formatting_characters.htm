Numeric Formatting Characters

  • I - Used for input formatting only.
  • O - Used for output formatting only.
  • I/O - Used for both input and output formatting.

I/O

Character

Meaning

I/O

#

Digit placeholder. Display a significant digit or nothing. If the expression has a digit in the position where the # appears in the format string, display it; otherwise, display nothing in that position.

I/O

0

Digit placeholder. Display a significant digit or a zero. If the expression has a digit in the position where the 0 appears in the format string, display it; otherwise, display a zero in that position. If the number has fewer digits than there are zeros (on either side of the decimal) in the format expression, display leading or trailing zeros.

I/O

#e or #E

Exponential format in either lower or upper case. You can use placeholders either side to specify the format of the mantissa and exponent. Example 0.00#e00 would display 45.6789 as 4.57e+01.

I/O

#b

Binary format, bit base. You can use placeholders specify the number of bits. Example 000000#b would display 45 as 101101.

I/O

#B

Binary format, byte base. You can use placeholders to specify the minimum number of bytes. Example 0#b would display 45 as 00101101.

I/O

#bb or #BB

Binary format. The number of digits displayed depends on the number of bytes of the source variable. Example #bb would display 45 from a 2 byte unsigned integer variable as 0000000000101101

I/O

#x or #X

Hexadecimal format. You can use placeholders to specify the number of characters. Example 0000#X would display 45.6789 as 002D

I/O

#x or #XX

Hexadecimal format. The number of characters displayed depends on the number of bytes of the source variable. Example #xx would display 45 from a 4 byte unsigned integer variable as 0000002d

I/O

.

Decimal placeholder. The decimal placeholder determines how many digits are displayed to the left and right of the decimal separator.

O

%

Percentage placeholder. The expression is multiplied by 100. The percent character % is inserted in the position where it appears in the format string.

O

~

Display the absolute value (unsigned).

O

,

Thousand separator. The thousand separator separates thousands from hundreds within a number that has four or more places to the left of the decimal separator.

Important notes about numeric formatting

If a decimal input format is chosen (for example #.###), the User will not be able to enter a value in exponent format (for example 1.5e2). The maximum number of digits that can be entered depends on the number of placeholders in the format.

If an exponent input format is chosen, the user can enter the value as a normal number or as an exponent (for example 1.5e2 or 150). The maximum number of digits that can be entered depends on the number of placeholders in the format. When no decimal point placeholder is defined, a default format of 6 digits is used.

For binary, byte and hexadecimal formats, the maximum number of digits that the user can enter depends on the variable type. For example, a two byte unsigned integer would allow a maximum of four hexadecimal digits.
