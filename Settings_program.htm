Program Settings

The program settings are displayed from the Programs entry in the Application Explorer's Main settings dialog. The program settings affect the operation of the programing environments and operation within PcVue. Show picture

  • SCADA Basic
    • Errors and warnings
      • Program infinite loop threshold - The maximum period for which a SCADA Basic program can run continuously before it is stopped. Used to prevent infinite loops in SCADA Basic programs. The default value is 30 seconds.
      • Message on discarded function in a program with delay - Display an error message if a program is called again whilst still executing a delay instruction from the previous call.
      • Only log program errors once in the Event Viewer – Repetitive errors appearing in the trace window from program language are only logged once after which they do not appear again until PcVue is restarted or the trace window is cleared.
    • Compatibility
      • Use register variables as double in SCADA basic script – Treats register variables as type double in a SCADA BASIC script. Set to True by default but can be changed to False for compatibility with projects created with early versions of PcVue where a register was treated as a single.
      • Working variables scope control - See the Working Variables topic in the SCADA Basic book for full information on the behavior.
        • Strict control - Use strict control for the scope of SCADA Basic variables. This is the default setting for new projects.
        • Legacy control with warning - As below but with additional warning messages when the rules enforced by Strict Control are broken.
        • Legacy control - Use the rules for the scope of SCADA Basic variables that were used prior to version 11.2. This is the default setting after migration of existing projects.
      • 'Return' instruction scope - Control the behavior of the 'Return' instruction
        • Exit the function (Sub) - Executing the 'Return' instruction causes the function to exit even if the 'Return' is in a nested set of statements (if condition, loop, etc.)
        • Behave like 'Break' when declare in a loop (For/While) - Executing the 'Return' instruction from within a For or While loop only causes an exit from the loop (equivalent of a call to 'break'). It does not causes the function to exit.
