The Instances tab
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Instances tab
 
The Instances tab is used to display, create, edit and organize instantiations of the templates. Show picture.

 
In common with the Templates and Parameters tab, it is divided into two panes. The left pane contains the main navigation tree from where the instantiation of templates is managed. The right pane contains a properties grid for the selected instantiation, a secondary navigation tree (representing the template of the selected instantiation) and a second properties grid from where the evaluated and/or named properties of the selected configuration object can be viewed and edited.
 
The main navigation tree pop-up menu
 
The main navigation tree pop-up menu is displayed when right clicking on the folder representing the root, any template instantiation, or a topology element. The menu is context sensitive and the commands that are displayed will depend on what is selected.
 
 
Cut - Cut the current selection.
 
Copy - Copy the current selection.
 
Paste - Paste a previously cut or copied element thereby creating either a new category or template.
 
Add a topology element - Add a topology element. 
 
Add an instance - Add a new instance of a template.
 
 
For all commands that add an object, there is a corresponding remove command.
 
The secondary navigation tree pop-up menu
 
The secondary navigation tree pop-up menu is displayed when right clicking on any folder or object. The menu is context sensitive and the commands that are displayed will depend on what is selected. Its main use is to create exceptions to an instance of a template.
 
 
Unselect - Flag the current selection so that it is not taken into account when the application is generated. The configuration remains in the tree. 
 
Select - Turn off the Unselect flag (see above).
 
Add a variable - Add a variable to a template instance. 
 
Add a trend - Add trending (historical data recording) to the selected variable instance. 
 
Add an event - Add an event (which triggers a program) to the selected variable instance.
 
Add mimic - Add a mimic to a template instance. 
 
Add symbol - Add a graphical symbol to a template instance. 
 
 
For all commands that add an object, there is a corresponding remove command.

The Parameters tab
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Parameters tab
 
The Parameters tab is used to display, create, edit and organize parameters. Show picture

 
In common with the Instances and Templates tab, it is divided into two panes. The left pane contains a navigation tree from where parameters can be selected, added and organized. The right pane contains a single properties grid from where the properties of the selected parameter can be viewed and edited.
 
The navigation tree pop-up menu
 
The navigation tree pop-up menu is displayed when right clicking on any folder, category, or parameter. The menu is context sensitive and the commands that are displayed will depend on what is selected.
 
 
Cut - Cut the current selection.
 
Copy - Copy the current selection.
 
Paste - Paste a previously cut or copied element thereby creating either a new category or parameter.
 
Add a category - Add a category (sub folder) to a library folder or category. 
 
Remove a category - Delete the selected category including its contents.
 
Add a parameter - Add a new parameter to the selected folder or category.
 
Remove a parameter - Delete the selected parameter and all of its instantiations.
 
 
What is a category?
 
A category is just a convenient way of organizing parameters. Creating a category does not create a physical sub-folder within the projects library structure.

The Properties grid
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Properties grid
 
The properties of all the Application Architect's configuration objects (templates, parameters etc.) are displayed in a tabular format known as the Property grid. Properties are grouped together according to their use. Show picture

 
Main features
 
 
To expand/collapse the display of a group of properties click the +/- sign adjacent to its name.
 
To change the value of a property click in the right column. Most values can either be entered directly by typing in a new value, or selected from a list displayed by clicking the down arrow button on the right side of the field. 
 
Properties that are read only are displayed in a dimmed typeface. 
 
Properties that have been manually edited are displayed in a bold typeface.
 
 

 
 Within each category, properties are displayed in alphabetical order.

 
 
The Properties grid pop-up menu
 

 
The Properties grid pop-up menu is displayed by right clicking anywhere in the left column.
 
 
 
Description - Toggles the display of additional information about the selected property in an area below the Properties List.
 
Advanced - Toggles the Properties List advanced mode. Depending on which configuration object is selected, advanced mode may display additional properties for the object.
 
Reset - Resets the property value to its default.
 
Do not overwrite default value - If selected, the property value, if left to its default, will not be overwritten by any global parameter.
 
Define by - Allows the value of a property to be defined by either an expression or a parameter. Not available for some properties that must be defined by a value.
 
 

 
By right-clicking on a category of properties with the mouse, it is possible to reset all properties of the category at once.

The Templates tab
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Templates tab
 
The Templates tab is used to display, create, edit and organize templates. Show picture.

 
In common with the Instances and Parameters tab, it is divided into two panes. The left pane contains the main navigation tree from where templates can be selected, added and organized. The right pane contains a secondary navigation tree (representing the template selected in the left pane) and a properties grid from where the properties of the selected configuration object can be viewed and edited.
 
The main navigation tree pop-up menu
 
The main navigation tree pop-up menu is displayed when right clicking on any folder, category or object. The menu is context sensitive and the commands that are displayed will depend on what is selected.
 
 
Cut - Cut the current selection.
 
Copy - Copy the current selection.
 
Paste - Paste a previously cut or copied element thereby creating either a new category or template.
 
Add a category - Add a category (sub folder) to a library folder or category. 
 
Add a template - Add a new template to the selected folder or category.
 
Add an inherited template - Add a new inherited template to the selected template.
 
 
For all commands that add an object, there is a corresponding remove command. Removing an object will also remove all of its instantiations.
 
The secondary navigation tree pop-up menu
 
The secondary navigation tree pop-up menu is displayed when right clicking on any folder or object. The menu is context sensitive and the commands that are displayed will depend on what is selected.
 
 
Unselect - Flag the current selection so that it is not taken into account when the application is generated. The configuration remains in the tree.
 
Select - Turn off the Unselect flag (see above).
 
Copy - Copy the current selection.
 
Paste - Paste a previously cut or copied element thereby creating either a new category or template.
 
Add a variable - Add a variable to a template. 
 
Add a trend - Add trending (historical data recording) to the selected variable.
 
Add an event - Add an event (which triggers a program) to the selected variable.
 
Add alarm synthesis - Add an alarm synthesis. It allows you to create alarm counter and status variables for groups of alarms.
 
Add an embedded template - Add an embedded template to the selected template.
 
Add mimic - Add a mimic to template. 
 
Add symbol - Add a graphical symbol to the template (instantiated in the mimic).
 
 
For all commands that add an object, there is a corresponding remove command. Removing an object will also remove all of its instantiations.
 
What is a category?
 
A category is just a convenient way of organizing templates. Creating a category does not create a physical sub-folder within the projects library structure.

Workspace overview
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Workspace overview
 
The Application Architect Workspace consists of the following major components. Show picture

 
 
Left pane and toolbar - Displays the available templates or parameters. Although this area has the appearance of a configuration tree, it is a toolbar. Double clicking on an object (either a template or parameter) will insert an instance of it under the object selected in primary configuration tree displayed in the right pane.
 
Right Pane - Contains three tabs - Instances, Templates and Parameters.
 
Instances - See the topic The Instances Tab.
Templates - See the topic The Templates Tab.
Parameters - See the topic The Parameters Tab.
 
Pop-up menu - A context sensitive right click pop-up menu is available from the configuration trees. The commands on the pop-up menu are repeated on the tool bar.
 
Main toolbar - The tools that are available reflect the configuration element that is selected in the configuration tree. Toolbar commands are repeated on the right click pop-up menu.
 
Menu - Non-context sensitive commands - see list below.
 
 
You can resize any of the Application Architect's panes by positioning the cursor over the border and clicking and dragging. Changes that you make to the Application Architect's look and feel are persistent - that is they are remembered even after PcVue has been shut down and restarted.
 

 
When you first open the Application Architect's workspace, all configuration trees are collapsed. All branches will have a + sign against them indicating that there are further branches - even there are none. This is because the Application Architect only loads the configuration of a branch when you attempt to expand it. If the Application Architect loaded the configuration of all branches when it was opened, it could take a considerable time and add an unacceptable delay.
 
 
Menu commands
 
 
File.Save (Ctrl+S) - Save the current configuration.
 
File.Exit (Alt+F4) - Close the Application Architect. Any changes that have been made are saved.
 
View.Toolbars.Parameters - Enable display of the parameters tool bar in the left pane.
 
View.Toolbars.Templates - Enable display of the templates tool bar in the left pane.
 
View.Display thumbnails - Enable display of thumbnails for graphic elements in templates and instances property grids.
 
Tasks.Check (F5) - Check the topography of the configuration.
 
Tasks.Check consistency - Check the consistency of the data model. Will detect instances for which the template is not defined, parameters used but not defined etc.
 
Tasks.Update libraries - Reload shared libraries. Run this command if you have added or removed a library and want the change to be taken into account without closing and re-opening the Application Architect.
 
Tasks.Generate (F7) - Generate the application.
 
Task.Settings - Opens the settings dialog box.
 
 
Using the libraries to store templates and parameters
 
Folders representing the PcVue project libraries appear in both the Templates and Parameters tabs. This allows you to choose where you want to save the configuration of any Templates or Parameters that you create. That way, you can build libraries of templates and parameters in the same way that you build libraries of symbols. Templates and parameters are both saved in the Templates folder of a library.
 
A reminder about libraries
 
 
The contents of the Local Library is only available to the current project.
 
The contents of the Common Library is available to all projects.
 
You can create custom libraries that are either only available to the project or are available to all projects.
 
 
For more information on libraries, see the book on Developing the HMI.
