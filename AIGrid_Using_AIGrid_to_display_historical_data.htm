Using the Grid to Display Historical Data

In historical mode, the cells are populated with a list of trended variables and their historical values generated using a similar mechanism to that of the Data Export Wizard. The raw values are retrieved from the archive units and the selected variables must have Trend Behavior configured. Show picture

Main features

  • The data displayed can be selected from: Raw, sampled, statistics or aggregated statistics the same as for the Data Export Wizard.
  • In raw or sampled mode, a threshold system can be used to color the cells when the value goes outside of configurable limits.
  • A toolbar is available for the user at run-time.
  • Data can be exported.

Configuring the Grid to display historical data

To use the Grid to display historical data you must select Historical mode, in the Historical tab, and then select and Configure the data request. Show picture

  • Default header - The default header for each column of data, selected from either <none>, variable description or variable name.
  • Request data at mimic open - Request data when the mimic containing the Grid is opened. If not ticked then the user will have to make the request manually, at run-time, using the toolbar, or the initialization will be done by script.
  • Display toolbar - Display the Grid historical toolbar. The toolbar contents and position can be changed at run-time. See the topic Configuring an Alarm Viewer, Log Viewer, Trend Viewer or Grid Control toolbar.
  • Use global threshold NS colors - Use the colors configured in Preferences.Colors.Thresholds to color any cell displaying an invalid value. Show picture
  • Auto-resize columns after data refresh - Automatically re-size the grid columns to accommodate length of the strings used to display the data. If not ticked the column sizes remain as configured.
  • Descending (new data on top) - Forces a sort order to display the most recent data on top.

Configuring the date and time format used

The date and time format is taken from the project languages settings, either Configuration 1 or Configuration 2 depending on the current run-time language. Show picture
