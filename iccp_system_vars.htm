ICCP System Variables and commands

System Variables

The ICCP system variables are available to improve visibility when operating the driver. They must be created manually.

Name Type Description
System.ICCP.<NetworkName>.ON Bit

The NetworkName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

  • Value 0 indicates that the network is stopped.

  • Value 1 indicates that the network is started.
System.ICCP.<NetworkName>.<LCCname>.ON Bit

The NetworkName and LCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Value 0 indicates that the LCC is stopped.

Value 1 indicates that the LCC is started.

System.ICCP.<NetworkName>.<LCCname>.

InformationReportsSent

Register

The NetworkName and LCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the number of Reports sent by the LCC.
System.ICCP.<NetworkName>.<LCCname>.
DeviceSelectRequestsReceived 		Register

The NetworkName and LCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the number of Reports received by the LCC.

System.ICCP.<NetworkName>.<LCCname>.

DeviceOperateRequestsReceived

Register

The NetworkName and LCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the number of Operate requests received by the LCC.
System.ICCP.<NetworkName>.<LCCname>.NbClients Register

The NetworkName and LCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the number of remote clients connected to the LCC.
System.ICCP.<NetworkName>.<LCCName>.<RCCName>.ON Bit

The NetworkName, LCCname and RCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

  • Value 0 indicates that the RCC is stopped.

  • Value 1 indicates that the RCC is started.

System.ICCP.<NetworkName>.<LCCName>.<RCCName>.

STATUS

Register

The NetworkName, LCCname and RCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

  • Value 0 indicates that the RCC is connected.

  • Value 1 indicates that the RCC is disconnected.

  • Value 2 indicates that the RCC is in an error state.
System.ICCP.<NetworkName>.<LCCName>.<RCCName>.
InformationReportsReceived 		Register

The NetworkName, LCCname and RCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the number of Reports received from the RCC.

System.ICCP.<NetworkName>.<LCCName>.<RCCName>.

DeviceSelectRequestsSent

Register

The NetworkName, LCCname and RCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the number of Select requests sent to the RCC.

System.ICCP.<NetworkName>.<LCCName>.<RCCName>.

DeviceOperateRequestsSent

Register

The NetworkName, LCCname and RCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the number of Operate requests sent to the RCC.

System.ICCP.<NetworkName>.<LCCName>.<RCCName>.

ActiveAR

Text

The NetworkName, LCCname and RCCName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

Indicates the name of the AR the RCC is currently connected to. Useful when an RCC has multiple redundant AR configured.

System.ICCP.<NetworkName>.<LCCName>.<RCCName>.

<RemoteDomainName>.<DSName>.Created

Bit

The NetworkName, LCCname, RCCName, RemoteDomainName and DSName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

  • Value 0 indicates that the Dataset has not been created on the remote.

  • Value 1 indicates that the Dataset ha been created on the remote.

System.ICCP.<NetworkName>.<LCCName>.<RCCName>.

<RemoteDomainName>.<DSName>.DSTSEnabled

Bit

The NetworkName, LCCname, RCCName, RemoteDomainName and DSName must match that configured in the ICCP.ini file. The variable must not have the Command property set.

  • Value 0 indicates that DSTS linked to the Dataset has not been enabled on the remote.

  • Value 1 indicates that the DSTS linked to the Dataset has been enabled on the remote.

Commands

You can send commands to the Add-on including starting and stopping a communication object, setting the log level...
To do so, you use two Text variables which names are set with the configuration keys called VarNameCmdRequest and VarNameCmdResult.
The Request variable is used to send the command, the Result variable is used to receive the execution result. You cannot send 2 commands at the same time.

The following table describes the commands that can be sent to the ICCP driver. The text must be one string that contains the different fields separated by commas.
Example:

StopLCC,Network01.MyLCC
Command Argument 1 Argument 2 Argument 3 Argument 4 Description
StopNetwork <NetworkName> N/A N/A N/A

Stops the Network specified by <Network Name>.

Also stops all objects that are nested below (LCCs and RCCs).

Example: StopNetwork,MyICCPNetwork
StartNetwork <NetworkName> N/A N/A N/A

Starts the Network specified by <Network Name>.

Also starts all objects that are nested below (LCCs and RCCs).

Example: StartNetwork,MyICCPNetwork
StopLCC <NetworkName> <LCCName> N/A N/A

Stops the LCC specified by <Network Name> and <LCC Name>.

Also stops all objects that are nested below (RCCs).

Example: StopLCC,MyICCPNetwork,MyLCC
StartLCC <NetworkName> <LCCName> N/A N/A

Starts the LCC specified by <Network Name> and <LCC Name>.

Also starts all objects that are nested below (RCCs).

Example: StartLCC,MyICCPNetwork,MyLCC
ResetInformationReportsSent <NetworkName> <LCCName> N/A N/A

Resets the InformationReportsSent system variable for the LCC specified by <Network Name> and <LCC Name>.

Example: ResetInformationReportsSent,MyICCPNetwork,MyLCC
ResetDeviceSelectRequestsReceived <NetworkName> <LCCName> N/A N/A

Resets the DeviceSelectRequestsReceived system variable for the LCC specified by <Network Name> and <LCC Name>.

Example: ResetDeviceSelectRequestsReceived,MyICCPNetwork,MyLCC
ResetDeviceOperateRequestsReceived <NetworkName> <LCCName> N/A N/A

Resets the DeviceOperateRequestsReceivedsystem variable of the LCC specified by <Network Name> and <LCC Name>.

Example:

ResetDeviceOperateRequestsReceived.MyICCPNetwork.MyLCC
ReloadLCCSecurityConfig <NetworkName> <LCCName> N/A N/A

Reloads the security configuration for the LCC specified by <Network Name> and <LCC Name>.

Example:
ReloadLCCSecurityConfig,MyICCPNetwork,MyLCC
StopRCC <NetworkName> <LCCName> <RCCName> N/A

Stops the RCC specified by <Network Name> <LCC Name> and <RCC Name>.

Example: StopRCC,MyICCPNetwork,MyLCC,MyRCC
StartRCC <NetworkName> <LCCName> <RCCName> N/A

Starts the RCC specified by <Network Name> <LCC Name> and <RCC Name>.

Example: StartRCC,MyICCPNetwork,MyLCC,MyRCC
SwitchActiveRCC <NetworkName> <LCCName> <RCCName> <NewARName>

Sets the active RCC to the one specified by <NewAR Name>.

Command applies only to LCCs that have the calling behavior (Client or Calling Client/Server).

The <NewAR Name> must match a previously configured RemoteARX field of an RCC in the ICCP.ini configuration file.

Example: SwitchActiveRCC,MyICCPNetwork,MyLCC,MyRCC,MyBackupAR1
ResetInformationReportsReceived <NetworkName> <LCCName> <RCCName> N/A

Resets the InformationReportsReceived system variable for the RCC by <Network Name> <LCC Name> and <RCC Name>.

Example: ResetInformationReportsReceived,MyICCPNetwork,MyLCC,MyRCC
ResetDeviceSelectRequestsSent <NetworkName> <LCCName> <RCCName> N/A

Resets the DeviceSelectRequestsSent system variable for the RCC by <Network Name> <LCC Name> and <RCC Name>.

Example: ResetDeviceSelectRequestsSent,MyICCPNetwork,MyLCC,MyRCC
ResetDeviceOperateRequestsSent <NetworkName> <LCCName> <RCCName> N/A

Resets the DeviceOperateRequestsSent system variable for the RCC by <Network Name> <LCC Name> and <RCC Name>.

Example: ResetDeviceOperateRequestsSent,MyICCPNetwork,MyLCC,MyRCC
ReloadRCCSecurityConfig <NetworkName> <LCCName> <RCCName> N/A

Reloads the security configuration for the RCC by <Network Name> <LCC Name> and <RCC Name>.

Example:
ReloadRCCSecurityConfig,MyICCPNetwork,MyLCC,MyRCC
ChangeGlobalLogLevel Integer (1-4) N/A N/A N/A

Sets the Log Message Level at the Global level.

This level affects all objects below if they are using default log level.
ChangeNetworkLogLevel <NetworkName> Integer (0-4) N/A N/A

Sets the Log Message Level at the Network level. This level affects all objects below if they are using default log level.
ChangeLCCLogLevel <NetworkName> <LCCName> Integer (0-4) N/A

Sets the Log Message Level at the LCC level. This level affects all objects below if they are using default log level.
ChangeRCCLogLevel <NetworkName> <LCCName> <RCCName> Integer (0-4)

Sets the Log Message Level at the RCC level.
PrintConfigurationSummary N/A N/A N/A N/A

Generates a set of configuration summary messages in the Event Viewer. Displays the structure of created objects inside PcVue.
PrintVariableSummary N/A N/A N/A N/A

Generates a set of variable summary messages in the Event Viewer. Displays the structure of created variables inside PcVue.
PrintLogLevelSummary N/A N/A N/A N/A

Generates a set of log level summary messages in the Event Viewer. Displays the structure of the log levels inside PcVue.
