Overview of the BACnet Server Add-on

The BACnet Server Add-on turns PcVue into a BACnet Server device. The station running the Add-on is exposed on the network as a BACnet Server and selected variables from the project as BACnet objects.

The real time value of a variable is exposed as the Present Value property. Other variable properties are also exposed as BACnet properties.

The BACnet Server Add-on supports the B-ASC profile. Please refer to the PICS topic for a complete list of supported BACnet services, objects and properties.

Variables configured and exposed as BACnet objects are counted as I/O variables and must be taken into account when sizing the license.

There is no configuration dialog specific to the BACnet Server Add-on. At the time of writing, it uses a combination of dedicated text files (.dat and .xml) and regular configuration features.

The BACnet Server Add-on can be used in trial mode. Communication will stop automatically at the end of the trial period.

The BACnet Client driver and the BACnet Server Add-on must not be used on the same computer. In a way to avoid any conflict, a station running this Add-on to expose BACnet objects shall have no BACnet client configuration set.
