Overview of the BACnet Helpers

The helpers are provided to assist the application developer diagnose and troubleshoot automation objects’ behavior in a BACnet device available on the network:

  • Alarms and Event notification using Notification Class objects,
  • Trend and Event logging using Trend Log, Trend Log Multiple or Event Log objects,
  • Scheduling using Schedule and Calendar objects.

The helpers can also be used to manage PcVue configuration objects, related to these interoperability areas, at the design stage, during commissioning and even after commissioning for maintenance. The helpers provide tools to:

  • Discover existing objects in the physical BACnet device,
  • Create new objects on the fly, including the actual object creation in the end physical device,
  • Read and change any of the object's properties,
  • Retrieve data associated with those objects - notifications, trend records, logged events.

Changes that you make, including changing the value of properties, as well as adding and deleting objects using the left panel, directly affect the configuration of the physical BACnet device. Unintentional changes can have an impact on the BACnet system behavior.

Depending on the physical device's capabilities and configuration, it may be necessary to make configuration changes directly in the device itself using the software from the manufacturer. In particular, not all devices support object creation and deletion, and some properties may not be changed remotely. See the device manufacturer's documentation for more information.
