Configuring Log Viewer Advanced Behavior

The Log Viewer advanced behavior is configured using the properties found in the Advanced tab. Show picture

Configuring the time order in which events are displayed

By default, when the Log Viewer is opened, events are sorted according to their timestamp. Using the Display options in the Advanced tab you can select if the newest events are at the bottom (default) or the top.

If Enable sorting and Sort modification in the Execution tab are selected any sorting configured at run time by the User takes priority.

Using advanced color mode

If Advanced color mode is selected both the stroke and background color for each event type, as defined in Color Preferences, are used. Normally the background color is not used, the background color being that of the Log Viewer.

Configuring an automatic refresh

In versions of PcVue prior to 11, the Log Viewer refreshed only as a result of the User navigating using the scroll bar. By entering a Refresh period other than 0 the Log Viewer automatically requests the most recent records from the archive unit. In order not to overload the archive mechanism, the minimum Refresh period is 5 seconds.

If the property Suspend automatic refresh when navigating is selected then the refresh is suspended when the scroll bar is used to navigate through the display. Automatic refresh remains suspended until the Automatic refresh tool on the toolbar is clicked. As each refresh also resets the display and scroll bar to the "when first opened" position, it is highly recommended this property is always selected when using automatic refresh.

The property Display icon when automatic refresh is suspended displays an icon in the middle of the display list when refresh is suspended thereby making it more obvious to the User.

Prior to version 11 the Log Viewer could only be refreshed using SCADA Basic but the refresh could be clearly seen. The refresh using the built in facility is seamless..

Limiting the period for which historical requests are made

You can limit the period for which historical requests are made by using the Time buffer maximum length when requesting option. When this option is configured, the records retrieved are limited by their age in addition to the limitation imposed by the Display buffer property (execution tab). The limit remains active unless the Time Filter tool is used and a From time and To time are entered.

The period is configured as a number (1 to 3600) of time units (Seconds, Minutes, Hours, or Days). The default time unit of None means there is no time limit.
