Map Markers Editor

The purpose of the Map Markers Editor is to create a map marker file. You can create any number of map marker files but only one file can be used per Map Control instance. Map marker files are created in a globally recognized standard known as Global Positioning eXchange, or GPX.

The Map Markers Editor can be opened using either of the following methods.

  • From PcVue's Main Menu using the command Display.Map Markers Editor.
  • From the Map Configuration dialog of the Map Control dialog.

Map Markers Editor main features Show picture

  1. Map pane. The actual map view can be zoomed and dragged as required. You can change the area occupied by the map pane by clicking and dragging the vertical line that separates it from the properties pane. Shown on the screen shot by the dashed vertical red line.
  2. Markers toolbar.
  3. Currently selected marker. The selected markers are indicated with a translucent green highlight. The marker selected in the screen shot is a polygon.
  4. Properties pane. At the top of the properties pane is the Map Provider selector.
  5. Layers toolbar. Using this toolbar, you can add a new layer, a new service layer, and/or remove selected objects.
  6. Map objects (layers and markers) configuration tree. You can change the area occupied by the map objects tree by clicking and dragging the horizontal line that separates it from the marker properties. Shown on the screen shot by the dashed horizontal red line.
  7. Properties for the selected map marker.
  8. Save and Open command buttons.
  9. Map zoom control.

The map

  • The selected map can be panned, using click and drag, and zoomed, using the map zoom control, in a similar manner to the Map Control. Note that the map is only used as an aid to position the markers and has no link to the actual map that to be used at run time. The selected map is not saved in the map markers file and therefore the default map is always displayed when the Map Markers Editor is opened.

    • The map layers

    Map layers are a way of grouping together the map markers. The map layers themselves do not appear on map but when they are configured, they are allocated to a specific zoom range so that, at run-time, the map markers they contain appear or disappear according to the map zoom level.

    The map markers

    The map markers that you configure appear at run time in any Map Control in which a map markers file has been selected. Each map marker is attached to a map layer that controls its visibility. The following Map marker types are supported.

    • Shape - A solid shape selected from the following forms - circle, square, rounded square, triangle, inverted triangle and rhombus.
    • Text - A bilingual text string.
    • Image - An image from a library.
    • Symbol - A symbol from a library.
    • Track - A multi-segmented line. Similar to the polyline drawing element.
    • Polygon - A polygon. Similar to the polygon drawing element

    Many of the map marker's properties can be bound to the value of a variable.

    All map markers types, except symbol, have a property that allows you to select a mimic that can be directly opened by clicking on the marker in run mode (like a Link-Open animation). Symbols do not have this property as the symbol could include a Link-open animation itself.

    Supplementary information, known as User Data, can be associated with a marker. The information is displayed in a pop-up window when the marker is right clicked at run-time. User data can include fixed text, variable's real-time values, and variable's extended attributes.

    It is recommended to not exceed a maximum of 500 markers visible at the same time. Displaying more markers on a map can affect responsiveness of the user interface.

    The command buttons

  • When you open the Map Markers Editor from the menu, you must use the command buttons to open or save a specific map markers file.
