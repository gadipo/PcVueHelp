OPC settings

The OPC Settings are accessed from the Application Explorer by selecting <Project>.Communication.Data acquisition.OPC and choosing the Settings command from the task list. Show picture

  • Variable status
    • Consider bad quality as - How PcVue treats OPC items with any of the bad quality flags set.
      • Unavailable (NS) - The quality of a linked variable is set to NS.
      • Non accessible (NS COM) - The quality of a linked variable is set to NS COM.
  • Invalidate variables - Defines if variable status should be set to invalid upon connection failure.
    • Never - The variables are never made invalid. This can be used when there is intermittent communication with the server.
    • Immediately - The variables are made invalid immediately on a communication failure. This is the default value.
    • After a delay - The variables are made invalid after the specified delay.
  • Monitoring
    • Use the local timestamp upon device reconnection - Use PcVue’s timestamp for the first value received after a device is re-connected.
