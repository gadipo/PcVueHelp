How to Configure an Event to Run a Program

  1. Creating and enabling the event.
    1. Select the Events folder in the Application Explorer configuration tree and, from the task list, select Add event to trigger a program. A dialog to configure the new event will open. Show picture
    2. Enter a name for the event. You can either accept the suggested name or type in one of your own.
    3. Enter a description of the event. The description is optional.
    4. For multi-station applications, select a servers list. The event will only be activated on stations within the list. The servers list is optional.
    5. Select or enter the name of a bit to enable or disable the event. The bit must be set to 1 to enable the event. The bit is optional.
    6. Tick the Enable box. It must be ticked for the event to be enabled.
  2. Configure the trigger.
    1. Select the Trigger tab. Select or enter the name of the variable to be used for the trigger. The ellipsis button adjacent to the Variable field opens the Variable Selector dialog.
    2. Enter the trigger expression used in conjunction with the value of the variable to trigger the event. For information on trigger expressions, see the topic Event trigger Transitions.
  3. Select the program and function.
    1. Select the Program tab. Enter the name of the SCADA Basic program that the event is to run. The ellipsis button adjacent to the Program name field displays a list of available programs. The program must exist or you will not be able to create the event.
    2. Enter a variable branch for use with the program. The branch is optional.
    3. Enter the name of the function, within the program, to run. If you do not specify a function, the Main function will run.
    4. Enter a comma-separated list of arguments for the function. The arguments can be retrieved in the function using the GETARG instruction. The list of arguments is optional.
  4. Click OK to confirm the event configuration and close the dialog.

Used with Event and Cyclic actions, the SCADA Basic program language is a very powerful tool allowing many things to be done.

As with any program language, the developer needs to be aware of limitations imposed by the environment in which it runs, and of the programming techniques needed to make most efficient use of the available resources.

For more information, see the topic SCADA Basic programming guidelines in the SCADA Basic book.
