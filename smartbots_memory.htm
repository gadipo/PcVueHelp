Bot agent memory

Every bot agent has an internal memory with a set of key-value pairs. The operation SaveInMemory can be used to add or replace an item in the memory.

Almost every text property of bot conditions and operations support substitution of values stored in memory, called internal bot variables.

In order to reference such value, the following notation is used: {key}. In this case, the {key} string will be replaced with the value stored in memory before verifying a condition or executing an operation.

In the following example, State1 is associated with a set of operations as follows. Show picture

  • The first operation creates a text internal bot variable with the value of the last written text message

  • The second operation puts the Sender of the last message to the variable send

  • The last operation sends a text message to the current chat. Note that the text is defined as Hi, {send}, where the {send} will be substituted with the value of the send variable, that is the Sender of the last message. For example, Hi, Ope1 will be sent to the chat in response to the message from Ope1.
