Running a SCADA BASIC Program

Programs can do many things from providing a simple increment to a variable to changing the properties and hence behavior of some of the macro animations. With careful design and by using techniques such as passing arguments and a variable branch, generic programs may be written which may be re-used many times. The Run Program animation allows a SCADA BASIC program or function to be run from a control zone.

In the Run Program tab, you enter the name of the program to be run and an optional function name. If you do not specify a function name, the control zone will always try to start the MAIN function. Show picture

A program must be pre-loaded before it may be executed from a control zone.

Functions are not re-entrant, that is they can only be run once at any one time. In order to avoid problems with this function's execution time should be kept short and not blocking. Avoid using instructions that cause delay in execution (for example DELAY).

Using a variable branch

If you enter a variable branch in the Branch field of the Send Program tab, any variables names in the program not preceded by the @ character will be prefixed with the branch. This is especially useful when developing generic programs.

For example, if in the program you have a variable named MAX, and you run the program with the branch INLET1.FLOW the program will actually refer to the variable INLET1.FLOW.MAX.

Passing Arguments

A list of up to 10 arguments, separated by commas, may be entered in the Arguments field of the Run Program tab. At run-time, the arguments may be retrieved from within the program using the GETARG instruction. The maximum number of characters that may be entered in the Arguments field is 256.
