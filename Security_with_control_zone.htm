Using the Security Animation with a Control Zone

The Security animation is applied to a drawing element before applying a Control Zone animation to provide additional security. Show picture

Using the Security animation in conjunction with a Control Zone you can require the user to:

  • Re-enter their password before being allowed access to the control.
  • Enter an additional user name and password before being allowed access to the control.
  • Re-enter their password and enter an additional user name and password before being allowed access to the control.

You can use the Security animation in conjunction with the following animations.

  • Send - Bit, Bit with color, Double bit, Register, Text, Recipe, Timetable and Region.
  • Run - Program, Macro and Application.
  • Link - Open, Close and Note.

You cannot use the Security animation for additional security when running a VBA script.

The Security animation must be applied to the support drawing element BEFORE the control zone animation.

Creating a Security animation to force a user to re-enter their password

  1. Select the drawing element to which the animation is to be applied.
  2. From the menu, select Animate.Security.User Confirmation.
  3. In the Security tab, select OK to confirm the configuration.
  4. Apply the control zone animation as you would normally.

Creating a Security animation to force a user to enter an additional name and password

  1. Select the drawing element to which the animation is to be applied.
  2. From the menu, select Animate.Security.Double Signature.
  3. Click the Add button to display the Users & Profiles Selection dialog. Show picture.

  4. Select the authorized users and/or Profiles. To select more than one Profile and/or User hold down the Ctrl button whilst clicking the left mouse button. If you select a Profile, any User with that Profile will be authorized.
  5. Click OK to confirm the selection and close the User and Profile selection dialog.
  6. In the Security tab, select OK to confirm the configuration.
  7. Apply the control zone animation as you would normally.

Creating a Security animation to force a user to re-enter their password and enter an additional name and password

  1. Select the drawing element to which the animation is to be applied.
  2. From the menu, select Animate.Security.User Confirmation.
  3. Tick the Double Signature property.
  4. Add the authorized Users and/or Profiles as described above.
  5. In the Security tab, select OK to confirm the configuration.
  6. Apply the control zone animation as you would normally.

Other Security animation properties

  • Confirmation on each command - Ticking this property has two effects.
    • If you are using a control zone that requires confirmation, for example Send bit with label, the password confirmation dialog appears after the command confirmation dialog (otherwise it is the other way round).
    • If there are multiple control zones supported by one drawing element, the password confirmation dialog will appear for each control zone in turn (otherwise, it only appears once).
  • Double signature on each command - Ticking this property has two effects.
    • If you are using a control zone that requires confirmation, for example Send bit with label, the double signature dialog appears after the command confirmation dialog (otherwise it is the other way round).
    • If there are multiple control zones supported by one drawing element, the double signature dialog will appear for each control zone in turn (otherwise, it only appears once).
  • Message - A text string that will be displayed in the security dialog at run time.

The following properties are only available if either Confirmation On Each Command or Double Signature On Each Command is ticked.

  • Animation name - If ticked the name of the animation, for example Send Bit, will be added to the message displayed in the security dialog at run time.
  • Command comment - If ticked the text entered in the control zone comment field (also used to provide the text for the tool tip) will be added to the message displayed in the security dialog at run time.
  • Object name and value - If ticked the name of the object and the value to which it is to be forced will be added to the message displayed in the security dialog at run time. If you are forcing a variable, the name of the variable and the value to which it will be forced is displayed. For example,: 'Tank 1 Pump1 - Start command'.
