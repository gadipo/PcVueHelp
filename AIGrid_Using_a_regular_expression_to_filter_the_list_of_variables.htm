Using a Regular Expression to Filter the List of Variables

The regular expressions used in the Grid are derived from the notation used in automata theory to describe formal languages and finite state machines. Regular expressions consist of characters (operands), such as "a", "O", and ", " and meta-characters (operators), such as "+", "|", and "[".

A regular expression, which is similar to the familiar arithmetic expressions, can be either a basic expression or a complex expression that is formed by applying operators to several basic expressions. The regular expression meta-characters and their uses are as follows.

Character

Usage

\

The escape character is used in escape sequences to specify characters that would otherwise have no representation. These are similar to those used in the C language.

\b

Backspace.

\t

Horizontal tab.

\n

New line or line feed.

\f

New page or form feed.

\r

Carriage return.

\ddd

Octal value.

\c

Where c represents any character string that is taken literally.

^

The character matches the beginning of a string. For example, "^abc" matches only those strings which start with "abc".

When used as the first character of a character class it denotes a negated class (exclusion) - See example below.

$

Matches the end of a string. For example, "z$" will only match strings with "z" as the last character.

.

Period/full stop matches any single character. Be careful as ".*" will match everything.

[

Denotes the start of a character class.

]

Denotes the end of a character class.

|

The pipe character is the alternation operator, so that "a|b" will match either "a" or "b".

()

Parentheses are used to group expressions in much the same way as with arithmetic expressions.

*

The closure operator matches 0 or more instances of the specified string.

+

Indicates positive closure and matches 1 or more instances of the specified expression. For example, "+z" matches a string of 1 or more "z"s.

?

Matches 0 or 1 instances of the specified expression. For example, "9?" matches either the null string or "9".

About character classes

Character classes are shorthand for matching one of several characters. For example, [AaBb] is the same as (A|a|B|b) and matches "A", "a", "B", and "b". There are also character ranges, such as [A-Z] that matches any uppercase character.

Negated character classes specify characters that should not be matched. For example, [^A-Z] matches everything except upper case alpha characters.
