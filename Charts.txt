Configuring the axes
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring the Axes
 
The Chart animation has configurable X and Y axes. For each axis, you can configure a title, the axis and the grid lines. There are two similar corresponding tabs, X Axis and Y Axes, for configuration. Show picture

 
About the Y axes
 
The Y axis is actually two independent axes, one appearing on the left, and the other on the right. Each series that you configure has to be attached to either the left or the right axis. This allows you to have two ranges of data on the same chart, which is very useful when comparing two different types of data, for example engine speed and torque. Show picture
 
 
If the chart is configured to plot more than two series, each series still has to be attached to either the left or the right axis. The axis range must be configured to accommodate the range of values from all series attached to it. Otherwise, some of the data points will not be visible. 
 
If a Y axis range is configured so that the Minimum and Maximum are both zero, the range is calculated automatically taking into account the range of values from all series that are attached to it. For example, if an axis were attached to two series, one with a range of data points from 0 to 100 and the other from -50 to 0, the range of the axis would be -50 to 100.
 
Title
 
The text, and its font, orientation and alignment, for the axis title. The maximum length of the string is 100 characters. The text is bilingual so a different string can be entered for Language Configuration 1 and Language Configuration 2 if the Bilingual option is selected in the Project Language settings.
 
If changing the font be sure to check that the selected font is available of the target platform.
 
Axis
 
The range (minimum, maximum), color and thickness of the axis line. If the Maximum and Minimum properties are left at their default of zero, the range auto scales according to the range of plotted values. Not the maximum and minimum of the variable configuration as in a Trend.
 
Grid
 
The interval, color, thickness and style of the lines forming the grid. The ticks and corresponding values are always displayed, but the lines are only displayed if Display Lines is selected. It is not possible to configure the font in which the grid tick values are displayed, however the size of the font is configured automatically depending on how many digits have to be displayed and the space that is available.
 
The Offset allows the lines making up the grid to be offset from their theoretical position. This is useful to adjust the position of the lines in the case that the range and interval of the two Y axes would result in the lines being drawn in the same position. Chart Overview
 
The Chart (XY Plot) is a control that plots the historical values of one to eight variables (Y-Axis) against one other (X-Axis). Show picture
 

 

 
 

 
 
When the mimic containing the Chart is opened, the Chart makes a historical request for the values of the variables, over a pre-configured period, and plots the variables representing the Y-axis against the single variable representing the X-axis. 
 
Taking the above screen shot as an example, the X-axis represents the engine speed and the Y-axis the torque and power.
 
 
When the value for engine speed was 1000 RPM, the corresponding torque was 180 Nm and the power 20 kW.
 
When the value for engine speed was 2000 RPM, the corresponding torque was 225 Nm and the power 50 kW. 
 
Etc.
 
 
The plot of each Y-Axis variable against the X-Axis variable is known as a data series.
 

 
The Chart is not automatically refreshed. It is populated with data points only when the containing mimic opens, after making a historical request, or by program.
 
 
Data series
 
The plot of the Y-axis variables against the X-Axis variable. Each point of a plot is represented by an X and Y value (co-ordinates). The collection of co-ordinates for one Y-axis variable is known as a series. Each series can be drawn in one of several ways - points, lines, bars, area etc.
 
Y-Axes
 
The left and right Y_Axes are individually configured so they can each represent a different series with its own color, name and range. The range can be configured to that it is auto scaling according to the values of the series, or have a fixed range. 
 
X-Axis
 
 The range can be configured to that it is auto scaling according to the values of the X-Axis variable, or have a fixed range. Toolbar
 
Allows the User to zoom in, refresh the chart, make a new historical request etc. The tools that appear on the toolbar are configurable. The same tools are also available from a right click pop-up menu. 
 
Legend
 
Displays the name of each variable (series) and the color in which it is represented. 
 
Tooltip
 
Displays the value and timestamp of the data point over which the pointer is hovering.
 
Scripting
 
The Chart Control can be manipulated programmatically by both SCADA Basic and VBA. 
 
A specific instruction, CHART, is provided in SCADA Basic. See the help for SCADA Basic for information on its use.
 
Other features
 
 
Highly configurable look and feel.
 
A Chart animation can be embedded in any mimic
 
Unlimited number 
 of Chart animations in each project.
 
Historic data can be retrieved from either proprietary or HDS archive units.
 
Up to 8 series each representing a register or a bit.
 
A further 2 reference series can be generated programmatically.
 
Ranges (Y-Axis) independently 
 configurable for each series.
 
The historic request from which the series are generated is configurable 
 from 1 second to 3600 days.
 
Toolbar to change 
 run-time configuration.
 
Zoom on both X and 
 Y axes.
 
Access from both SCADA Basic and VBA for custom features.
 
 

 
The main use of the Chart is to plot the historical value of one variable against another. However, it is also possible to populate the chart series programmatically.

Making a historical request

About the sampling and how it works
 
When plotting more than one series it is mandatory to have the same number of data points for each series, plotted at identical time intervals. However, by default PcVue's historical system uses exception recording which records data when it changes and not at regular intervals. Therefore, the Chart animation employs a sampling mechanism that takes PcVue's raw data and generates data points at a known interval. This has the further advantage that you know exactly how many data points you will have. The operation of the sampling mechanism can be explained by a simple example.

Configuring the sampling
 
The main sampling properties are the Display period and the Sampling rate.
 
 
Display period - The period of time for which historical data is requested by the chart. If the Start date is not configured then the request is made using the actual time the mimic is opened, when the refresh tool is used, or when a refresh is requested programmatically. Range 1 second to 3600 days.
 
Sampling - The sampling period. Configurable from 10 milliseconds to 3600 days.
 
 

 
The maximum number of points for one series is 25,000. You cannot configure values for the Display period and the Sampling rate that would cause this to be exceeded.
 
 
Using the Start date property you can force the start time of the Chart to a particular day and time relative to the current day. 
 
 
Every day - The current day. For example, if the current date and time was Tuesday 5th February 2013 09:15 the start would be 05/02/2013 (plus the configured hours and minutes).
 
Yesterday - Yesterday. For example, if the current date and time was Tuesday 5th February 2013 09:15 the start would be 04/02/2013 (plus the configured hours and minutes).
 
Monday to Sunday - The first previous day corresponding to that selected. For example, if the current date and time was Tuesday 5th February 2013 09:15, and Thursday was selected, the start would be 31/01/2013 (plus the configured hours and minutes).
 
First day of the month - The first day of the current month. For example, if the current date and time was Tuesday 5th February 2013 09:15 the start would be 01/02/2013 (plus the configured hours and minutes).

Configuring a Pie Chart's Display
 
This topic is for the features configured by the pie chart's Chart tab that configures the look and feel of the dynamic part of the chart. Show picture

 
Configuring the chart type
 
The chart can be selected as either pie/doughnut (radial segments) or pyramid (horizontal segments) type. You can also choose either a two dimensional (2D) or three dimensional (3D) appearance.
 
Configuring the label used for each segment
 
In addition to the font and position, you can also choose, using the Label format property, the information that each label contains.
 
 
Label and value - Display the segment label, and value as a number.
 
Label and value percent - Display the segment label, and value as a percentage of the total.

Angle - The angle at which the first configured segment is displayed. With the default of zero the first segment starts at ninety degrees to the right. Angles other than zero cause clockwise rotation.
 
 
Properties specific to the pyramid type
 
 
Value type - The way in which the value for each segment is represented. Linear means the value is represented as the segment's height. Surface means that the value is represented as the segment's area.
 
Label inside alignment and Label outside alignment - The position of the label.
 
Gap - The gap between each segment.
 
Base - The segment colors can be shaded to give the pyramid base the appearance of square or circle form.
 
Angle (3D) - The segment colors can be shaded so the pyramid appears to be viewed from a horizontal angle.

Configuring a pie chart's colors
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring a Pie Chart's Colors
 
This topic is for the pie chart's Colors tab that configures the colors used for the chart segments. Show picture

 
The colors used for the chart segments can be configured in one of three ways using the Palette drop down list box.
 
 
Custom palette - The segment colors can be selected from the color palette. The colors are used in the order in which they appear in the palette (left to right, top to bottom). Any colors that are deselected (unticked) are ignored and unused.
 
Global palette - The segment colors are defined according to the global color preferences: Configuration.Preferences.Colors.Chart control. The colors are used in the order in which they appear in the palette (left to right, top to bottom). Any colors that are deselected (unticked) are ignored and unused.
 
Pre-defined palettes - The segment colors are defined by one of the pre-defined colors palettes, Berry, Bright, Pastel etc.
 
 
In addition, you can enter a transparency figure, zero being solid colors, and 100 completely transparent colors.

Configuring a pie chart's contents
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring a Pie Chart's Contents
 
This topic is for the features configured by the pie chart's Content tab that configures the chart segments. Show picture

 
Each segment is configured by entering a Label, Variable name or Value, and Legend. 
 
 
Each label appears with its corresponding segment, overlaid or adjacent to it depending on the configuration in the Chart tab.
 
The legends are grouped together at the position configured in the Chart tab.
 
The values determine the size of each segment. Fixed values and variable values can be mixed in the same chart.
 
The maximum number of segments is 32.
 
 
Using fixed values
 
To populate a segment with a fixed value, the Label, Value and Legend are entered directly in the corresponding fields. 
 
Using variable values
 
To populate a segment using a variable value, the variable name is selected or entered in the Variables / Values field. The Label and Legend can be directly entered or provided by the variable itself. Segments can be updated either when the variable value changes or using a timer. Only register variables can be used.
 
 
Default legend - Provides the Legend text for segments where none is configured. Can be selected from Variable name, Variable description or None.
 
Default text - Provides the Label text for segments where none is configured. Can be selected from Variable name, Variable description or None.
 
Refresh mode - The refresh mechanism for all variable values used in the Pie chart: 
Variable change - Each time the value of a variable changes.
Timer - Periodically according to the value of Refresh timer.
 
Display unavailable variable warning - If ticked, the variable name along with its status is displayed on the chart if it is invalid. Otherwise invalid variable are ignored and not taken into account in the chart.
 
 
Populating the content from a file
 
The content can be configured from an external text file. The file and hence the content is loaded when the containing mimic is opened. You can also manually (in design mode) force the contents to be saved in, or loaded from, a file.
 
 
You can use the same file for any number of instances. This ensures consistency and is particularly useful when the content has many segments.
 
You can manage the file and hence the contents by using a third party application (text editor, database export...).
 
 
The file name is entered in the File name field. The file name can be entered in a number of ways.
 
 
You can type it directly into the field.
 
You can use the file select tool to open a standard Windows file selection dialog.
 
You can use the variable selection tool to select a text variable. The file name is supplied by the value of the text variable.
 
 
You can enter a relative or absolute. If you enter the file name only, PcVue assumes that the file is located in the project's TP folder.
 
You can cause the contents to be loaded automatically from a file at run-time. For that, the Source must be set to File.
 

 
If you choose to automatically load the content from a file, and then manually edit the content using the table in the Content tab, you must manually save the changes using the Save in file button. 
 
 
To load the contents of a Pie chart from a file manually (in design mode), use the Load from file button. A file must be specified before this button is enabled.

Configuring a pie chart's general features
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring a Pie Chart's General Features
 
This topic is for the features configured by the Pie Chart's Display tab that affect its general appearance. The Display tab contains three sub tabs General, Title and Legend. Show picture

 
General
 
 Configuration of the background color, plot area color and selection of optional background images. 
 
The images, if used, are selected from the libraries. Different images can be used for the background and plot areas.
 
Title
 
The text, its position and its style, for the pie chart's title. The text is bilingual so a different string can be entered for Language Configuration 1 and Language Configuration 2 if the Bilingual option is selected in the Project Language settings. If changing the font be sure to check that the selected font is available on the target platform.
 

 
The Style property is different to the other viewers and can be selected from Default, Embed, Emboss, Frame and Shadow. Default should be used for a graphic rendering similar to that of the other controls.
 
 
Legend
 
The legend displays a description and the corresponding color for each segment.
 
Color swatches are used to select the background and text colors for the legend. The position can be select from Top, Bottom, Left, Right or None (no legend).
 
If a Legend is not entered in the Contents tab then the Title is used.

Pie chart overview
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Pie Chart Overview
 
The Pie chart displays one or more values as pie, doughnut or pyramid style chart. The values can be entered as part of the configuration, loaded from a file or set by scripting. Show picture
 

 

 
 

 
 
Scripting
 
The Pie Chart Control can be manipulated programmatically by both SCADA Basic and VBA.
