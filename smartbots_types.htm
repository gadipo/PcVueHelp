Smart Bots types

The smart bots property Bot Type defines how the bot is handled on start-up and during development.

Smart bots

Bots of type Smart Bot are bots designed to interact in a chat room. These bots are loaded from the “<projectFolder>/MsgData/Bots” folder at the Instant Messaging Server start-up. Bots of this type must be added to the chat room in order to be started.

System bots

System Bots are designed to work in the background without any user interaction. When deploying a System Bot, the Instant Messaging Server automatically creates a chat room for it. The following values are used for the chat properties.

  • Id - The bot’s Id.

  • Name - The bot’s name.

  • Description - The bot’s description.

  • Color - A random color from the list of available colors.

  • Storage time - One day.

  • Associated Zone - None.

The deployed bot is automatically invited to this chat room as the owner and therefore gets started.

Assistant Bots

Assistant Bots are designed to work one to one with a user. The configuration specifies which users, profiles and profile roles it will serve. Usually these bots are used to provide a specific HMI to mobile users. An assistant bot may be associated with the following sets of users:

  • A set of users.

  • A set of user profiles.

  • A set of profile roles.

Multiple users / profiles / profile roles can be defined by separating them with a comma or semicolon. Multiple users / profiles / profile roles can also be defined using wild-card notation with * corresponding to any amount of any characters and ? corresponding to any single character.

When deploying an assistant bot, the Instant Messaging Server automatically creates a chat room for every associated user. If the profile is specified, the chat room is created for each user with the given profile.

The following values are used for the chat properties.

  • Id - The bot’s Id.

  • Name - The bot’s name.

  • Description - The bot’s description.

  • Color - A random color from the list of available colors.

  • Storage time - One day.

  • Associated Zone - None.

The deployed bot is automatically invited to these chat rooms as the owner and therefore gets started.

Template Bots

Template bots are custom parametrized operations for use by other bots. A template bot is not visible in the chat interface and therefore cannot be added to a chat room. A template bot has a set of argument / parameter definitions. When a bot calls a template, the specified argument values are written to internal bot variables and thus can be used in the template bot. Show picture

The following table enumerates the possible parameter types.

Parameter type Expected value Type Remarks
String Text String Arbitrary string, can be used as a parameter for most operations.
Number Number Double  
Datetime Date and Time DateTime  
Database name Internal variable name (Key) containing the Database context String To be used in local database related operations.
Report name Internal variable name (Key) containing the Report context String To be used in reporting related operations.
Function Expression Expression  
File path Path to the file or folder String  
Variable name  PcVuevariable name String  
Branch Branch name String  
Result Internal variable name (Key) String This type is used to define output results of the template. When the template process is completed, the resulting value is written to the internal variable specified in parameter value.

All parameters support the use of internal bot variables. The following properties must be specified for each parameter.

  • Key – The internal variable name that will contain the input parameter value.

  • Description – The label of the input parameter field.

  • Parameter type – The type of the input parameter.

  • Default value – The input parameter value that will be used by default. That is, if the Input field remains empty.

  • Required – A flag indicating if the parameter value is mandatory.

When deploying a template bot, the Instant Messaging Server saves it in the <projectFolder>/MsgData/Bots folder. The deployed templates appear in the Smart Bots Manager UI along with other operations. Show picture

When adding a template to a bot, the user sets properties according to the template input parameter definitions.
