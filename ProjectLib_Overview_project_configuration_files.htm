Project configuration files

Much of the configuration information entered by the designer of the PcVue application using the configuration tools is saved in either comma-separated ASCII files or Xml structured files stored in the project’s C directory.

The table below contains the main list of files.

Files

Use

Action.dat

Configuration of actions associated to alarms.
AlarmConf.dat Settings for the alarm management.

Comm.dat

Configuration of the data acquisition networks, nodes and frames for the Equipment drivers.

Cron.dat

Configuration of Cron timetables and scheduled actions.

Cyclic.dat

Configuration of cyclic actions.
DataConnectionsConf.xml Configuration of the Sql connections and pre-defined Sql requests.
DataExportsConf.xml Configuration of the data exports.

Db.dat

Configuration of ODBC databases.

DrawDefault.dat

Default properties for drawing elements (drawing property manager).
DrawModels.dat Configuration of models for drawing elements (drawing property manager).
Enumala.dat Configuration of alarm enumerations.

Event.dat

Configuration of event-driven actions.

Exprm.dat

Configuration of expression templates.

Exprv.dat

Configuration of expressions on variables.
FileReferences.dat Configuration of file references.

Formula.dat

Configuration of formulas.
GeoDataConf.xml Configuration of geographical data (zones, actions and geo-tags).

GPConf.dat

Configuration of filters for alarm and log viewers for the Web & Mobile apps.
HDS.ini Settings for the Historical Data Server.

HDSConf.dat

Configuration for the Historical Data Server.

HDSTrend.dat

Configuration of the trends for the Historical Data Server.
HisConf.dat Settings for the archiving features.

Histo.dat

Configuration of the archiving.
HMIVariables.dat Configuration of HMI variables.
I104Server.dat Configuration of the mapping of variables to the points exposed via the IEC 60870-5-104 outstation interface.
Icons.dat Configuration of the palette of icons.
IMConfig.xml Configuration of the Instant Messaging feature.

ImportReferences.dat

Information about files imported by Smart generators (for the purpose of future synchronizations).

Key.dat

Configuration of function keys (built-in keyboard shortcuts).
LanConf.dat Settings for the client/server networking.

LPrinter.dat

Configuration of line printers.
MailConfig.xml Configuration of the e-mail and SMS features.

MRUFiles.dat

List of the most recently used mimics.
OcxStore.dat List of favorite ActiveX controls.

Options.dat

HMI settings.

Palcol.dat

Configuration of the color palette.

Param.dat

General settings.

ParamWS.dat

Mimic workspace settings.

Popu.dat

Configuration of the population feature.
Prefcol.dat Configuration of the color preferences.

Recipe.dat

Configuration of the list of recipes. The recipes are saved separately in the R folder.
Scheduler.xml Configuration for the HDS task scheduler for maintenance plans.

Script.dat

Configuration of the list of custom modules for the Cypress Enable scripting engine.

ScriptVar.dat Configuration of the list of variables referenced for scripting with the Cypress Enable scripting engine.
SqlVariables.xml Configuration of the mapping of variables onto Sql requests.

Station.dat

Configuration of the client/server networking.
SvConf.dat General settings.
TemplatesConfig.xml Settings for the Application Architect.

TSPrefs.dat

Settings for the task scheduler (maintenance plans).
UIConf.dat General settings.

UIVConf.dat

General settings.

User.dat

Settings and configuration of the user directory.

VarConf.dat

General settings related to variable management.

Varexp.dat

Main configuration file. It includes the configuration of variables, data acquisition drivers, and a variety of other configuration elements.

VarTreat.dat

Configuration of alarm synthesis and other alarm processing behaviors.
VisKbdOptions.xml Settings for the software keypads.

The table below contains the main list of folders.

Folders

Use

MIBs MIB files specific to the project.
TEMP Temporary files.
TMP Temporary files.

While it can be handy to modify configuration files directly, in particular to copy and paste elements from one project to another one, any mistake in doing so can lead to incorrect or undefined behavior of the product.
