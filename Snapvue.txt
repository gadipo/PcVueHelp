Chat view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
The Chat View
 
The Chat view is displayed by tapping on a chat in the Chat List, or after creating a new chat. Show picture
 

 
 
 
Messages from the local user are displayed on the right, messages from other users on the left.
 
The chat settings and filter can be changed from the menu tool at the top right corner if the chat user has the correct privileges.
 
How to send a message
 
New messages are added by typing in the text box at the bottom of the chat view and clicking the Send button. Note that the send button only appears when there is a message to send, replacing the Record button. 
 
How to change the chat settings
 
Changing the chat setting requires Owner privileges. The Settings view is displayed by tapping the Settings tool and contains two tabs, Settings and User. Show picture
 

 
 
 
The following settings are available.
 
 
 In the Settings tab
Chat title - The title of the chat, which appears in the chats list.
Appearance - Select the chat color. A color is allocated automatically when a chat is created but may be changed later.
Storage time of messages - The message storage time. Messages are deleted automatically after this period, unless Unlimited is selected. Note that this setting is applied retrospectively to messages that have already been sent.
 
In the Users tab
Users - Invite and remove users, and / or change their permissions.
Quit the chat room - Leave the chat. A chat must have at least one owner, so if the original owner wishes to leave the chat, another user must be given owner privileges.
 
 
Tap the Done button once the changes are ready.
 
How to change the message filter
 
The Message Filter dialog is displayed by tapping the Filter tool and contains two tabs Date Time Range and Users Filter. Show picture
 

 
 
 
 
Date Time Range - Display only messages that were sent between the start and end dates.
 
Users filter - Display only messages from the selected users.

Chats list view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Chats List View
 

 
The availability of Instant Messaging is subject to license options.
 
 
The Chats List view provides access for a user to the Instant Messaging service. The Instant Messaging service allows a user to communicate, by text message, with one or more other users of the PcVue desktop application or SnapVue.
 
Instant messaging uses a conversation known as a chat. Chats are created, and owned, by a specific user. Once a chat has been created, the owner can invite other users and change user permissions. Chats are managed on SnapVue from the Chats view. Show picture

 
Tapping a chat displays the Chat view from where messages can be read and written and, if the user has the correct permissions, the chat settings can be changed.
 
How to add chat
 
 
Tap the large '+' sign at the top of the Chats view. The New Chat view is displayed. Show picture

 
Enter the Chat name. The chat name is only visible in the configuration and cannot be changed once the chat has been created.
 
Enter the chat Title. It appears as the identifier in the Chats List and can be edited at run-time by a user with Owner privileges.
 
Select the Storage time for messages. Messages are automatically discarded after this period.
 
Select the users that will have access to the chat, and their privileges. 
 
Click the Done button to confirm creation of the chat or Cancel. The Chats view is displayed.
 
 

 
Once a chat has been created, it is never deleted.

Home view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Home View
 
The Home view is where the SnapVue user will spend the majority of their time. It has a navigation bar at the top and one or more tiles representing the available actions. The tiles that are displayed depend on the context, the zone in which the user is detected and his/her profile. This is configured on the PcVueWeb & Mobile back end server. Tapping a tile executes the action associated with it. Some tiles display information, for example a Trend view, others perform actions such as sending a command on a variable or an alarm. Show picture

 

 
If no zone is detected, the tile area will be empty and a message confirming that it is not an error is displayed.
 
 
Navigation bar
 
The navigation bar contains the following tools.
 
 
 
 
 
 

 
 
 
 Zone selection
 If the user has been detected as being in more than one zone at the same time, the left and right arrows can be used to move the tile display between them. Tapping the zone name displays the list of accessible zones, and among others, allow access to the actions of the parent zone.
 
 

 
 
 
 QR code scanner
 Opens the device's QR Code scanner to scan a QR code used as a geo-tag.
 
 

 
 
 
 Diagnostics view
 Displays the Diagnostics view. Only available if the Diagnostic mode is enabled in the app Settings.
 
 
 
 
Tiles in general
 
The action type, position and appearance of the tiles are configured in the PcVue project.
 
 
Action types
Send command
Variable details
List of variables
Trends
List of alarms
List of logged events
Symbol
Mimic
Execute Program
Open Url
 
Position - Tiles are sorted on the screen according to a priority that is part of their configuration in PcVue.
 
Appearance - The size, color and visual representation of each tile is configured in the PcVue project. The screen-shot at the top of this topic contains tiles with default settings. 
 
 
The views displayed by some of the actions are similar to those in TouchVue except that, for most of the action types, the information that is displayed depend on the context. 
 
Send command
 
Forces a variable to a value, a command or setpoint. The value is pre-configured in PcVue and cannot be entered directly via SnapVue.
 
 
A bit variable can be forced to 0, to 1 or toggled.
 
A register variable can be forced to a fixed value, incremented or decremented.
 
A text variable can be forced to a specific string.
 
 
Variable details
 
Display a view of detailed attributes of a variable. See the topic Variable details view. 
 
List of variables
 
Display a list of the variables corresponding to the detected location. See the topic Watch list view.
 
Trends
 
Display a trend view of up to 16 variables. See the topic Trend view. 
 
List of alarms
 
Display a list of alarms corresponding to the detected location. See the topic Alarm list view.
 
List of logged events
 
Display a list of events from a selected Log List. See the topic Logged events list view. This action is not context sensitive.
 
Symbol
 
Display a pre-selected graphic symbol from the PcVue symbol library. 
 
Mimic
 
Display a pre-selected mimic from the PcVue mimic library.
 
Mimic and symbol rendering in mobile apps is similar to that of WebVue except for the following restrictions.
 
 
Mimics are displayed full size and cannot be scaled to fit the device screen. However, you can scroll a mimic using touch and drag.
 
The Link-open animations are not executed, other than to open a single pop-up mimic.
 
The Link-close animations are not available to the user.
 
 

 
For detailed information on the available mimic functionality, see the book Configuring a project for WebVue.
 
 
Execute Program
 
Execute a function from a SCADA Basic script on the PcVue Web& Mobile back end server.
 
Open Url
 
Open a specific Url. This action is not context sensitive.

Sending and receiving other media
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Sending and Receiving Other Media
 
In addition to sending plain text messages, you can also send attachments in the form of a file, or a voice message. Note that you can send either a text message, an attachment or a voice message. You cannot send, for example, an attachment and accompany it with a text message. To do that you would have to send two separate messages.
 
Sending attachments
 
Attachment are sent using the button at the lower left corner of the chat view. The button opens a browse view from where the attachment is selected. After it has been selected, and the browse window closed, the file is sent using the Send button. Show picture
 

 
 
 
Received attachments are displayed in the chats view in a similar manner to a text message. If the attachment it is a recognized image format, tapping it will display it, otherwise it will be downloaded.
 
Sending and receiving voice messages
 
Voice messages are sent using the button at the lower right corner of the chat view. The button opens a voice recorder widget from where the user records the message. After it has been recorded, and the voice recorder widget closed, the file is sent using the Send button. Show picture
 

 
 
 
Received voice messages are displayed in the chat view in a similar manner to a text message. Tapping a voice message will play it through the device's audio channel.

Snapvue overview
 

 
 
 
 
 
 
 
 
 
 
 
 
 
SnapVue Overview
 
SnapVue is a mobile application that displays contextual information on a mobile device such as a smart phone or tablet. By contextual we mean that the information depends on the location of the device, known as the zone, and the user's profile. The location of a device is detected using geo-tag technologies including Bluetooth beacons, QR codes and NFC tags.
 
The information is supplied by the PcVue Web & Mobile back end server and primarily comprises of the real-time and historical value of process variables. Show picture

 
See Web and Mobile Client Applications Overview for an overview of Web & Mobile client applications, including their common requirements and deployment steps.
 
The configuration steps specific for SnapVue access to PcVue are as follows:
 
 
Configure the Mobile Server settings in the Application Explorer. It includes the variable and alarm properties that should be displayed in the SnapVue app detailed views, and the default filter for alarm notification. See the book Mobile server.
 
Configure one or more SnapVue users. See the topics Properties that affect access to Web Apps and Web Services in the Application Explorer book and Configuring mobile applications users.
 
Configure the geographical data in the Application Explorer. The geographical data comprises of zones, the geo-tags corresponding to those zones and the actions available from each zone. See the book Geographical Data.
 
Have users install the SnapVue mobile application on their mobile device.
 
 
Summary of the main SnapVue views
 
The views are all accessed from a drawer menu opened by a swipe from the left border of the screen, or from the menu tool found at the top left of most views. Show picture

 
 
Home - The primary view comprising of one or more context sensitive tiles, one for each action available for the detected location. See the Home view topic.
 
Chats - Access to the optional Instant Messaging system.
 
Settings - SnapVue user configurable settings. See the Settings view topic.
 
Accounts - Configure user accounts and display their status. See the Accounts view topic.
 
 
SnapVue makes use of the Notification Area to notify the user of the status of selected alarms and current location detection. 
 
Licensing SnapVue
 
The components necessary to run SnapVue are included 
 on the PcVue distribution media or the Windows distribution media. 
 
The maximum number of simultaneous SnapVue sessions is license-protected.
 
Terminology
 
For readers not familiar with mobile platforms terminology here are some of the terms in common use:
 
 
View - The term used for a graphic display page occupying the entire screen.
 
Notification area - A small area, not used by any app's views, reserved for notifications requiring the user's attention. Normally at the top of a smart phone or the bottom of a tablet.
 
Gestures - The finger movements to interact with a touch screen:
Touch or Tap - Touch the screen Press and lift immediately.
Long touch or Long tap - Touch the screen, wait, lift.
Swipe - Touch the screen, move, lift.
Drag - Long touch, move, lift.
Double touch or Double tap - Two touches in quick succession.
Pinch open - 2-finger touch, move outwards, lift.
Pinch close - 2-finger touch, move inwards, lift.
 
Drawer menu - A menu generally opened by a left swipe. Known as a drawer because of the way in which it opens.

Settings view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
SnapVue Settings View
 
The Settings view is used to manage the app's general settings.
 

 
Some settings can greatly affect the performance of the app and should only be changed by informed users.
 
 
General settings
 
 Show picture

 
 
Time filter properties
Log lists - The default time period displayed by log viewers.
Trend - The default time period displayed by trend viewers.
 
Settings
Variables refresh period - The default refresh rate in seconds for real time variable values.
 
Theme
Theme color - The main app theme color, selected from a predefined palette. The default is red.
 
Audit
Diagnostic - Enable access to diagnostic to assist with troubleshooting, including the Connectivity and Environment views.
 
Version info - The app version.
 
 
Messaging settings
 
 Show picture

 
 
Messaging properties
Messages polling interval when active - Rate in seconds at which the update requests are made to the messaging server when the app is in the foreground.
Message polling interval in background - Rate in seconds at which the update requests are made to the messaging server when the app is in the background.
Storage period for message attachments - The period for storing message attachments. When elapsed, message attachments are deleted from the device memory. Attachments can be downloaded again from the Instant Messaging Server on demand.
Mute messaging notifications - Mute instant message notifications.

Change display order - Changes the order in which messages are displayed - either the newest, or the oldest, at the top.
 
 
Geo-location settings
 
 Show picture

 
 
Geo-tag remanence
Use remanence settings from server - Use the global remanence setting defined on the server or use local settings. 
 
Bluetooth properties (Android app only)
Scan interval - The continuous period, in milliseconds, during which Bluetooth beacons are scanned for.
Sample expiration interval - The period, in milliseconds, over which distance calculations are averaged. Averaging is used as a mechanism to improve the accuracy of distance calculations.
Between scan period - The wait period, in milliseconds, between 2 consecutive Scan Intervals. The wait period is used to minimize the effect of Bluetooth signal echo when calculating distance. 
 
Indoor positioning properties
Location polling period when active - Period in seconds between requests to the location web service for updates when the app is in the foreground.
Location polling period when in background - Period in seconds between requests to the location web service for updates when the app is in the foreground.
Minimum position change for tracking - The distance in cm that is considered as a significant move when refreshing the user contextual information.
Expiration time of tracking results - The time in seconds after which the user contextual information expires and a new refresh request is sent.
 
 
Notification settings
 
 Show picture

 
 
Notifications
Mute alarm notifications - Mute, or not, alarm notifications. The default is Off (no muting).
Start the alarm notification service at boot time (Android app only) - Start the alarm notification service at device boot time. This means that, once logged in, alarm notifications will be displayed, even after a device restart, without the need to restart the app. The default is On.
Contextual alarming - Display only contextual alarms, that is alarms relevant to the user's current location, or display all alarms. The default is On (contextual alarms only).
 
Alarm properties
Change display order - Changes the order in which alarms are displayed - either the newest, or the oldest, at the top.
Polling interval for alarm list refresh - Period in seconds between requests for alarm list updates.
Ringtone (iOS only) - The ringtone associated to notifications.

Starting SnapVue
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Starting SnapVue
 
The SnapVue app is started by tapping the app icon. Show picture

 
If it is the first time ever that the app has been started on a device, you must configure an account. After the splash screen has been displayed for a few seconds, the Accounts view is displayed from where the account is be configured. 
 
If an account has already been configured, but you have not previously logged in or the login has expired, you must log in. After the splash screen has been displayed for a few seconds the Accounts view is displayed from which the account to log in with can be selected.
 
If you have already logged in recently, and have not explicitly logged out, the Home view is displayed. Login are remembered even if the app has been stopped and the device turned off.
 

 
Starting the SnapVue app also starts the notification service and the geolocation detection service that run in the background.
