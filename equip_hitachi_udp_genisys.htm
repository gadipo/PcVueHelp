Hitachi Rail STS UDP Genisys MicroLok

This driver supports communication over UDP with VHLC and IXS devices, specifically designed to read Indication Bits and write Control Bits.

At startup or when the device reconnects, the driver initiates a Recall (0xFD) and automatically switches to Poll (0xFB) upon receiving a valid response. A VHLC or IXS system consists of two redundant devices: Normal and Standby.

The driver fully supports redundancy by sending requests to both addresses and includes an Indication Bit to show which device is currently returning the Indication Bits.

Standby Ip address

Ip address of a standby device used to provide redundancy, enabling the system to switch if the primary device becomes unavailable.

The ip4sameclassstandby parameter must be manually set to 0 in the COMM.DAT file for the ipxstandby fields to be applied. For example:

10#33#86#22#50003#122#1 indicates that both devices listen on port 50003, where:

The normal device TCP address is 10.33.86.22.

The standby device TCP address is 10.33.86.122.

If all values are set to 0, the default values are used.
