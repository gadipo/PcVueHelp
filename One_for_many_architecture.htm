One for Many Architecture

One for Many architecture uses a single server station to provide redundant operation for two or more associations. For example using three stations, Station1, Station2 and Station3.

  • Station3 is included in an association with Station1 to provide redundant operation for Station1.
  • Station3 is ALSO included in an association with Station2 to provide redundant operation for Station2.

This architecture is a way to minimize the number of stations to deploy and maintain, while ensuring a certain level of redundancy.

Instead of the entire variables tree in Station3 being active (producing the value of variables) or passive (consuming the value of variables) some variables can be passive and others can be active depending on which association produces them.

For an example of this type of architecture and more information, see the topic 1 for many redundant servers and 2 clients in the Architecture Examples book.
