Types of Archive Unit

Several different types of archive unit may be configured. The type of archive units that you use is dependent on how the recorded data is to be retrieved.

Database

General-purpose archive unit.

  • Main advantage - Records data in an industry standard database.
  • Main disadvantages - Can use a large storage space. Access speed can be compromised with larger databases files.

Data source

Trend data from variables added to the Trends folder and log data from the Log Lists.

Data retrieval

Historic Trend Viewer.
Log Viewer.
Programmatically using SCADA Basic.

Recording system

Industry standard database. Maintenance mechanism for controlling file size and/or age of data.

Location

In the Databases folder in the root of the installation folder.

Record format

Sql relational database tables. Format of tables partially configurable.
Redundancy and consolidation Supported.

Proprietary

General-purpose archive unit.

  • Main advantage - Highly optimized data recording for maximum performance.
  • Main disadvantage - Proprietary format so access by 3rd party products limited.

Data source

Trend data from variables added to the Trends folder, Log data from the Log Lists and report data.

Data retrieval

Historic Trend Viewer.
Log Viewer.
Mimics configured as reports.
Programmatically using SCADA Basic.

Recording system

File based. Each archive unit is composed of a number of files of pre-defined size. The oldest file is discarded when the unit becomes full and a new file is created. Files may be automatically copied to another location before they are discarded.

Location

Normally within the TH folder of the project.

Record format

Proprietary. Binary or ASCII data is recorded in blocks. The record format is partly configurable.
Redundancy and consolidation Not supported.

(Printers)

A pre-configured virtual archive unit that allows log and report data to be sent directly to a printer without being recorded in an archive unit.

Free

Special purpose archive unit only used if access to data is required in a simple ASCII format by 3rd party product.

Data source

Trend data from variables added to the Trends folder and Log data from the Log Lists.

Data retrieval

3rd party products, word processor, spreadsheet, database etc. - depending on the chosen recording format.

Recording system

File based. Sequential system with new uniquely named file created each time the current file becomes full. No file management - files must be moved or deleted by some means external to PcVue.

Location

Anywhere within the Windows' file system.

Record Format

ASCII. Each data point is recorded on a single line. The record format is fully configurable.
Redundancy and consolidation Not supported.

ODBC

Special-purpose archive unit only used if access to data is required in database format by a 3rd party product. Superseded in version 8.10 onwards by the database type archive unit.

Data Source

Trend data from variables added to the Trends folder, Log data from the Log Lists and report data.

Data Retrieval

Through the HMI using a historic Trend Viewer.
Through the HMI using a Log Viewer.
Through the HMI using mimics configured as reports.
Any program with connectivity to the selected database.

Recording System

ODBC database.

Automatic reconnection to the ODBC data source is not supported. If the connection is lost, PcVue must be restarted.

Location

Anywhere within the Windows' file system.

Recording Format

ODBC database in fixed format tables.
Redundancy and consolidation Not supported.

VCR

Special-purpose archive unit only used for projects configured for the VCR facility.

Data Source

The values of register, bit, alarm and text variables.

Data Retrieval

Through the HMI using a specially configured PcVue station. The data is replayed in pseudo real time controlled using a VCR style (Video Cassette Recorder) dialog.

Recording System

File based. New file created periodically each time a snapshot of the variable values is taken.

Location

Normally within the TH folder of the project.

Recording Format

Proprietary.
Redundancy and consolidation Not supported.

If you need to archive data timestamped at the device level, it is recommended to use Database archiving (HDS). In general, the native recording system (proprietary archive unit) does not provide a reliable data sequence.
