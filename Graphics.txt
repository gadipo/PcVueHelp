Graphic Explorer context menus
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Context Menus in the Graphic Explorer 
 
The Graphic Explorer provides two context menus for access 
 to tools associated with a graphic object:
 
 
With the mimic open 
 in Design mode, click on a graphic object's name in the Graphic Explorer's 
 objects list. 
 
Right-click on the 
 graphic object's name to open a context menu. Show picture

 
 
The Edit context menu
 
The Edit context menu is available only for objects that 
 are located in the mimic. It is not available for an object located inside 
 a group or symbol.
 
It is the same context menu as is displayed by right-clicking 
 on a graphic object in a mimic.
 
Animations context menu
 
The Animations context menu is available for objects in 
 a group or symbol. It can be used to add an animation to an object inside 
 a group or symbol without having to un-group the group or symbol.
 
It is the same context menu as is displayed from Animate 
 in the main menu or in the context menu displayed by right-clicking on 
 the object and selecting Animate.

The top three rows of buttons in the Graphic Explorer 
 turn filter commands on or off. 
 
When a filter is turned on it allows all objects of a 
 particular type to be displayed in the Graphic Explorer. When a filter 
 is turned off, it prevents all objects of a particular type from being 
 displayed in the Graphic Explorer. 
 
 
When a filter is 
 turned on, its button has a gray background 
 
When a filter is 
 turned off, its button has a white background 
 
 
When all of the filters are turned off, only the mimics 
 are listed in the Graphics Explorer.

Overview of the Graphic Explorer
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Overview of the Graphic Explorer
 
The Graphic Explorer provides you with a hierarchical 
 view of the elements forming a window:
 
 
In the main menu, 
 select Display.Graphic Explorer to open the Graphic Explorer. Show picture

 
 
The Graphic Explorer shows:
 
 
A list of all windows 
 that are currently open.
 
For each window, 
 a list of drawing elements and the co-ordinates of their anchor points.
 
For each drawing 
 element, any animation that has been applied.
 
For each animation, 
 the name of any variables and branches used.
 
 
You can display and change the properties for any drawing 
 element:
 
 
Double-click on the 
 property's control in the Graphic Explorer. Show picture

 
 
The tools at the top of the Graphic Explorer enable you 
 to: 
 
 
Filter the drawing 
 elements to be displayed. The polyline and B�zier curve objects can be 
 filtered independently of the polygon and B�zier shape objects respectively.
 
Use the Graphic object 
 ordering command.
 
Use the Graphic object 
 Edit context menu.
 
Use the Graphic object 
 Animations context menu.
 
 
VBA tools
 
There are VBA tools associated with the Graphic Explorer:
 
 
VBA Control objects 
 marker command.
 
VBA Events filtering 
 commands.
 
Flag for symbol declared as VBA controls.
 
Flag for elements with VBA events.

VBA Tools for the Graphic Explorer
 
The Graphic Explorer provides tools to:
 
 
Mark each object 
 configured as a VBA Control. 
 
Filter the list of 
 objects according to whether there is an associated event.
 
Directly access an 
 object's VBA event script.
 
 
VBA control marker command
 
You can mark the entries for objects configured as VBA 
 controls:
 
 
In the Graphic Explorer 
 click on the button, Highlight VBA Control Objects. Show picture

 
 
When the VBA Control Marker is turned on, each object 
 listed in the Graphic Explorer and configured as a VBA Control is displayed 
 with the informational text (position, identity etc.) in BLUE.
 
VBA events filtering
 
You can filter the contents of the Graphic Explorer's 
 objects list according to VBA event scripting.
 
There are two commands for command objects related
 to VBA events: Show picture

Complex filtering
 
The VBA event filtering commands work in the same way 
 as the animation filtering commands: 
 
 
Any filtering by 
 object type (i.e. Rectangle, Ellipse, Group etc.) takes place first. 
 
The sub-filters for 
 Events are applied to its result.
 
The filtering process 
 applies a logical 'AND' between the object type filters and the animation 
 and VBA event filters (e.g. display objects [Rectangle AND scripted with 
 VBA Event]). 
 
The VBA event and 
 animation filtering commands are combined by using a logical OR (e.g. 
 display objects [Rectangle AND [without animation OR with scripted with 
 VBA event]]).
 
An object VBA event 
 is regarded as scripted as soon as the object event procedure is declared 
 in the VBA scripting editor (with or without code).
 
 
Displaying Event details
 
When displayed, each event is represented by a sub-item 
 that is placed directly below the VBA Control object or VBA Project to 
 which it belongs. The sub-items display an Event icon with the event name 
 next to it.
 
There are two kinds of event:
 
 
Associated with a 
 VBA project, i.e. for a Mimic or Symbol (if defined as a VBA Project).
 
Associated with a 
 VBA Control object.
 
 
The Graphic Explorer distinguishes the two kinds of events

Graphic Objects ordering command
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Graphic Objects Ordering Command
 
The objects in a mimic have a particular order. At first 
 that is the order in which they were added to the mimic. When you use 
 the ordering tools, they modify that initial order. 
 
You can reorganize and maintain the list of objects in 
 the Graphic Explorer to the order in which they appear in the mimic. To 
 apply the ordering command:
 
 
Click on the button 
 Display according to Mimic Objects ordering. Show picture

 
 
When ordering is turned off
 
When the Ordering command is turned off, the Graphic 
 Explorer works as follows.
 
 
It is not notified 
 of any change of the order of objects in the mimic.
 
When you create a 
 group or symbol, the new graphic object (group or symbol) is always added 
 (if the corresponding filtering command is turned on) at the end of the 
 Graphic Explorer's objects list. 
 
When you un-group 
 a group or symbol, the drawing elements that it contains are always added 
 (if the corresponding filtering command is turned on) at the end of the 
 Graphic Explorer's objects list.
 
When you turn on 
 a filtering command, the objects that match the filter are always displayed 
 at the end of the Graphic Explorer objects list.
 
 
When ordering is turned on
 
When the ordering command is turned on:
 
 
The Graphic Explorer 
 reorganizes its objects list according to the object ordering in each 
 mimic. 
 
This order is automatically 
 updated when the order of objects in the mimic changes or when a filtering 
 command is turned on.
