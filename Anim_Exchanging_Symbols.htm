Exchanging Symbols

Simple changes in process status may be indicated by change of color. For example, a pump may be displayed in green when it is running, red when it is stopped and blue when faulty. In some cases however, changing color is not adequate.

Consider the case where a float switch is to be displayed. The requirements are that it should appear horizontal when its corresponding bit in the variables tree is true, and at an angle of 45° when the bit is false. This can easily be achieved by creating two symbols, each representing one of the conditions, and exchanging them depending on the state of the bit variable. In practice, exchanging symbols can create many visual effects.

It is also possible to use the Visibility animations.

Anchor points

When exchanging symbols the support drawing element is always another symbol, normally one of those used in the animation. The location of all symbols used in the animation is determined by aligning their anchor points with that of the support symbol.

Using symbols with a Branch

If the symbols being displayed contain animations which use relative variable referencing, then a branch must be supplied. The Branch may be entered directly, by using the Branch selector or by using substitution strings. For further information, see the topic Overview of Animation. For further information on symbols see the topic Overview of Symbols.

Specifying a symbol for invalid Status

On all the symbol animations, you have the option to select a special symbol that will be displayed when any of the variables used becomes invalid. A variable may become invalid for a variety of reasons, the most common one being the loss of communication to external equipment such as a PLC or RTU.

It is highly recommended that you make use of this option.

If you do not use this option, when a variable becomes invalid the animation will display the symbol representing the last valid value of the variable. The user will then be unaware that they may be viewing out of date information.
