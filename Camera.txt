3D camera First Person view
 

 
 
 
 
 
 
 
 
 
 
 
3D Camera First Person View
 
In First Person View mode, the camera simulates a person by allowing movement to anywhere in the scene. The following keyboard
 and mouse actions are available.
 
 
 
 

 
 
Mouse/keyboard action 
 
 
 
Result 
 
 
 

 
 
Hold down the mouse's right-hand button and move the mouse sideways
 
 
 
Changes the camera azimuth as if the person is turning to the right or left.
 
 
 
 

 
 
Hold down right-hand mouse button and move the mouse forward or backward
 
 
 
Changes the camera pitch as if the person is facing down or up respectively.
 
 
 
 

 
 
Left or Right Arrow key 
 
 
 
Changes the camera position as if the person is moving sideways to the left or to the right. 
 
 
 
 

 
 
Up or Down Arrow key 
 
 
 
Changes the camera position as if the person is moving straight ahead or backward. 
 
 
 
 

 
 
Page Up or Page Down key 
 
 
 
Changes the camera position as if the person is moving along the 3D world Y axis.
 
 
 
 

 
In First Person View mode, the camera position is always
 the same as the target position so the distance is zero.

Locking a parameter
 
When an Anchor is ticked, it indicates
 that the associated parameter (Eye, Target or Distance) is locked. There
 is always one (and only one) parameter locked at a time. 
 
When
 a parameter is locked, its values cannot be modified by the HMI when
 it re-calculates the parameters (following the modification of one parameter
 by the user).
 
 
For example, if the Target parameter
 is locked and you modify the Distance, the HMI re-calculates the Eye parameter.

The main purpose of the Orbit View mode is to make a
 window camera orbit around a target. To do so you hold down the right-hand mouse button and move the mouse.
 
 
 
 

 
 
Mouse movement
 
 
 
Result
 
 
 

 
 
Sideways
 
 
 
The camera orbits around the target
 Y-axis while always keeping the same target. This changes the camera position
 and azimuth.
 
 
 
 

 
 
Forward
 and backward
 
 
 
 
The camera orbits around the target
 X-axis while always keeping the same target. This changes the camera position
 and pitch.
 
 
 
 
 
Other mouse or keyboard actions can be used to move the
 camera as follows. 
 
 
 
 

 
 
Mouse/keyboard action
 
 
 
 
Result
 
 
 

 
 
Hold down the mouse's
 central button then move the mouse sideways or forward and backward.
 
 
 
Moves the camera so as to reduce or
 increase the distance between the camera and the target. The camera moves
 toward or away from the target as you move the mouse to the left (or away)
 or to the right (or toward you) respectively.
 
 
 
 

 
 
Roll the mouse's wheel
 
 
 
As above. 
 
 
 

 
 
Use the Left or Right Arrow key
 
 
 
Moves the camera horizontally to left
 or right respectively. This changes the camera target.
 
 
 
 

 
 
Use the Up or Down Arrow key
 
 
 
Moves the camera forward or backward
 respectively. This changes the camera target. 
 
 
 

 
 
Use the Page Up or Page Down key 
 
 
 
Moves the camera along the 3D world Y-axis. This changes the camera target.

Overview of the 3D Camera tools
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Overview of the 3D Camera Tools
 
The content of a 3D mimic is displayed in one or more
 3D windows. A 3D window is actually displayed in 2D on the surface of
 a display screen. It shows in real time what a camera is viewing in the
 3D world. That depends on the window's camera position, target position
 (the point the camera is aimed at), distance (between camera and target) and
 mode. Each window has its own camera.
 

 
You can insert and configure camera objects in 3D windows so as to pre-define custom points of view. 
 
 
Camera modes are used to define what one sees and how
 one interacts with the camera of a 3D window. To set a 3D window camera
 to a particular camera mode, the HMI provides these features.
 
 
3D Mode toolbar -
 Tools to select the camera view.
 
Camera model - A
 3D model that provides pre-configured parameters including the camera
 mode, position, target and distance. 
 
 
Camera views available from the 3D mode toolbar
 
There are eight basic camera modes (as presented in the toolbar, from left to right).

There are three further camera tools to control the camera view.
 
 
 
 
 

 Icon
 Meaning
 
 

 
 
 
 Camera View: displays a selected camera 'view'.

Camera Reset: resets the window camera's current settings to the default settings. 
The command is available from both the 3D Mode toolbar and the window's contextual pop-up menu at run time.
 
 
 
 
 

 
The top, bottom, front, rear, left and right view camera
 modes are called orthographic modes since what the viewer sees is an orthographic
 projection of the 3D scene (i.e. at right angles to two axes).
 
 
See the Glossary topic for definitions of 3D terms.
 
The Camera View command
 
This is only active when a window is in Design mode. 
 
 
Select a camera model.
 
Run the command.

 
 
The HMI applies the camera model settings to the window camera that is active in the 3D window, which then displays the view of the 3D world at which the window camera is currently aimed.

 
The Camera Menu command
 
This displays a pop-up menu that lists the camera models configured in the current active window.
The content of the list, for the same window, may be different depending on the window mode (Design or Runtime) as follows.
 
 
 
 

 
 
Mode 
 
 
 
Content
 
 
 
 

 
 
Design
 
 
 
All of the camera models are listed.
 
 
 

 
 
Runtime
 
 
 
Only the camera models configured to appear in the menu at run time are listed.
 
 
 
 
The menu is available from both the 3D Mode toolbar and the runtime contextual pop-up menu of a window. Show toolbar list Show context list
 

 
 
 

 
 
 
In them, the camera model references are automatically numbered and identified using the camera model labels (see the topic 3D Camera Model Settings).

 

 
Whilst it is possible to insert more than 64 cameras in a mimic, the menu can only reference up to 64 camera models.

CAMERADISPLAY example
 

 
 
 
 
 
 
 
 
 
 
 
 
 
 
CAMERADISPLAY Example
 

 
 

 

 

 Applies to
 
 
This program illustrates the use of the CAMERADISPLAY instruction.
 
Example prerequisites
 
 

 
A mimic containing an ONVIF control and a List-box control must be open. Show picture
 
 

 
One text variable called @SB_EXAMPLES.PresetName is required.
 
 
 

 
 
 

 

 
 
 

 
'-------- Program wide variables
 
 
 

 
DIM sWindow AS STR, sBranch AS STR, sIdentity AS STR;
 
 
 

 
DIM sAIListBox AS STR;
 
 
 

 
&#160;
 
 
 

 
'-------- Must be ran first to initialize program wide variables&#160;&#160;
 
 
 

 
SUB Main ()
 
 
 

 
&#160;&#160;sWindow = "Mimic2"; 'Name of the mimic containing the ONVIF control
 
 
 

 
&#160;&#160;sBranch = "";
 
 
 

 
&#160;&#160;sIdentity = "SecDevice1"; 'Identity of the ONVIF camera
 
 
 

 
&#160;&#160;sAIListBox = "AIListBox1"; 'Identity of the list box used to display the camera presets
 
 
 

 
&#160;&#160;Exec_GetDeviceInfo();
 
 
 

 
END SUB&#160;&#160;
 
 
 

 
&#160;
 
 
 

 
'-------- Get device info and display in results window&#160;&#160;
 
 
 

 
SUB Exec_GetDeviceInfo ()&#160;
 
 
 

 
&#160;&#160;DIM sDevInfoName AS STR, sResult AS STR;
 
 
 

 
&#160;&#160;sDevInfoName = "Model";
 
 
 

 
&#160;&#160;sResult = CAMERADISPLAY("GETDEVICEINFO", sWindow, sBranch, sIdentity, sDevInfoName);
 
 
 

 
&#160;&#160;PRINT("Device model ", sResult);
 
 
 

 
&#160;&#160;sDevInfoName = "Firmware";
 
 
 

 
&#160;&#160;sResult = CAMERADISPLAY("GETDEVICEINFO", sWindow, sBranch, sIdentity, sDevInfoName);
 
 
 

 
&#160;&#160;PRINT("Device firmware ", sResult);
 
 
 

 
&#160;&#160;sDevInfoName = "Hardware";
 
 
 

 
&#160;&#160;CAMERADISPLAY("GETDEVICEINFO", sWindow, sBranch, sIdentity, sDevInfoName);
 
 
 

 
&#160;&#160;PRINT("Device hardware version ", sResult);
 
 
 

 
&#160;&#160;sDevInfoName = "Serial";
 
 
 

 
&#160;&#160;CAMERADISPLAY("GETDEVICEINFO", sWindow, sBranch, sIdentity, sDevInfoName);
 
 
 

 
&#160;&#160;PRINT("Device serial number ", sResult);
 
 
 

 
END SUB
 
 
 

 
&#160;
 
 
 

 
'-------- Get camera presets and display in a list box
 
 
 

 
SUB Exec_GetPresets()
 
 
 

 
&#160;&#160;DIM iPresetCount AS INTEGER, iIndex AS INTEGER;
 
 
 

 
&#160;&#160;DIM sText AS STR, sUserData AS STR;
 
 
 

 
&#160;&#160;LISTBOX("CLEAR", sWindow, sBranch, sAIListBox);
 
 
 

 
&#160;&#160;iPresetCount = TOI(CAMERADISPLAY("GETPRESETCOUNT", sWindow, sBranch, sIdentity));
 
 
 

 
&#160;&#160;For (iIndex=0; iIndex < iPresetCount; iIndex++)
 
 
 

 
&#160;&#160;&#160;&#160;sText = CAMERADISPLAY("GETPRESETNAME", sWindow, sBranch, sIdentity, iIndex);
 
 
 

 
&#160;&#160;&#160;&#160;sUserData = sText;
 
 
 

 
&#160;&#160;&#160;&#160;LISTBOX("INSERT", sWindow, sBranch, sAIListBox, sText, sUserData, iIndex);
 
 
 

 
&#160;&#160;Next
 
 
 

 
END SUB
 
 
 

 
&#160;
 
 
 

 
'-------- Go to a preset. Preset name comes from the list box populated by Exec_GetPresets
 
 
 

 
SUB Exec_GoToPreset()
 
 
 

 
&#160;&#160;DIM sPresetName AS STR;
 
 
 

 
&#160;&#160;DIM iIndex AS INTEGER;
 
 
 

 
&#160;&#160;iIndex = TOI(LISTBOX("GETSELECTEDINDEX", sWindow, sBranch, sAIListBox));
 
 
 

 
&#160;&#160;sPresetName = LISTBOX("GETUSERDATA", sWindow, sBranch, sAIListBox, iIndex);
 
 
 

 
&#160;&#160;CAMERADISPLAY("GOTOPRESET", sWindow, sBranch, sIdentity, sPresetName);
 
 
 

 
END SUB
 
 
 

 
&#160;
 
 
 

 
'-------- Go to home
 
 
 

 
SUB Exec_GoToHome()
 
 
 

 
&#160;&#160;CAMERADISPLAY("GOTOHOME", sWindow, sBranch, sIdentity);
 
 
 

 
END SUB
