How to add a data export to a template
 
The data export element exports PcVue historical data. This topic explains how to add a data export to a template using the Application Architect.
 

 
Refer to the sub-book Data analysis for more information.
 
 
Adding a data export 
 
 
In the right pane of the Application Architect, select the Templates tab.
 
Using the templates configuration tree select the template to which the data export will be added.
 
Using the secondary navigation tree select the Configuration elements folder or the element to which the data export is to be added.
 

 
Select Add a data export. A new data export with a default name is created. Show picture
 

 
 
 

 
You can edit the name of the created configuration element by directly double clicking it in the Properties pane.
 
 
 

 
Configure the data export's properties using the properties grid. Right-click the grid and select Advanced to display the advanced properties.
 

 
The Naming Rule property is useful to ensure the uniqueness of the names of the generated elements, by default, the name includes branch segments from all levels of the instantiation tree. The available property values are:
 
 

 
Full branch - Generates the full name including all branch segments. This is the default.
 
 

 
Short branch - Generates the name using the primary instance. That is the branch segment of the instance containing the element that is generated.
 
 

 
Leaf only - Generates the name using only the name of the element as defined in the template.
 
 
 
Alternatively, you can enter an expression in the naming rule that is used to generate the name.
 
If you use a naming rule other than Full Branch, then you must ensure the uniqueness of the generated name.
 
 
 

 
After you are done, click File menu item then Save.
 
 
 
You can add a log page, a trend page or a file element to a data export.
 
Adding a log page or a trend page to a data export
 
 

 
A log page generates data from a log list.
 
 

 
A trend page generates data from one or more trended variables.
 
 
 
 
In the right pane of the Application Architect, select the Templates tab.
 
Using the templates configuration tree select the template to which the page will be added.
 
Using the secondary navigation tree select the Configuration elements folder or the element to which the page is to be added.
 

 
Select Add a log page or Add a trend page. A new log or trend page with a default name is created. Show picture
 

 
 
 

 
You can edit the name of the created configuration element by directly double clicking it in the Properties pane.
 
 
 

 
Configure the log or trend page properties using the properties grid. Right-click the grid and select Advanced to display the advanced properties.
 

 
The Naming Rule property is useful to ensure the uniqueness of the names of the generated elements, by default, the name includes branch segments from all levels of the instantiation tree. The available property values are:
 
 

 
Full branch - Generates the full name including all branch segments. This is the default.
 
 

 
Short branch - Generates the name using the primary instance. That is the branch segment of the instance containing the element that is generated.
 
 

 
Leaf only - Generates the name using only the name of the element as defined in the template.
 
 
 
Alternatively, you can enter an expression in the naming rule that is used to generate the name.
 
If you use a naming rule other than Full Branch, then you must ensure the uniqueness of the generated name.
 
 
 

 
After you are done, click File menu item then Save.

How to add a mimic to a template
 

 
 
 
 
 
 
 
 
 
 
 
 
 
How to add a Mimic to a Template
 
When you add a mimic to a template, one mimic is generated for each instantiation of the template. The mimics are created using default settings unless you select a mimic template. 
 
 
In the right-hand pane of the Application Architect, select the Templates tab.
 
Using the templates configuration tree select the template to which the mimic will be added.
 
Using the secondary navigation tree select the Graphic Objects folder.
 
Select Add mimic. A new mimic with a default name is created.
 
Edit the name to one appropriate to your application. You can only edit the name directly by clicking on it in the configuration tree. Show picture

 
Configure the mimic properties.
 
 
Mimics generated by the Application Architect use the ASCII file format. If they are manually edited, they must be saved in the same format or there will be errors the next time the Application Architect is run.
 
Mimic properties
 
To configure a mimic's properties select the mimic in the secondary navigation tree and its properties appear in the properties grid. Show picture

 
 
Description - A description of the mimic. Only used within the Application Architect and has no effect on the application itself. Optional
 
Location - The library in which the mimic is to be saved selected from Local (default), Shared Libraries or Local Libraries. If Shared Libraries or Local Libraries is selected an additional property, Library, is displayed from where you can select the library name.
 
Mimic title - The title that is to appear in the mimic title bar. Optional
 
File extension - An optional file extension to be used when creating the mimic file from the mimic name.
 
Tags to include - See the topic Controlling Instantiation Using Tagging.
 
Mimic template - The name of a mimic template which will be attached to all instantiations of the mimic and hence define its look and feel. Optional.
 
Use context branch - If set to Yes the context branch will be set to the branch of the template that contains the mimic and the local branch will be set to the property of the symbol. No specific concatenation is done. Show picture If set to No generates the symbol without a context branch and the local branch is set to that configured in the symbol. Show picture 

 

 
 
How the mimic name is calculated
 
So as to ensure the mimic name uniqueness, mimic name is prefixed with the context branch with any branch separator replaced by an underscore.

How to add a symbol to a template
 

 
 
 
 
 
 
 
 
 
 
 
 
 
How to add a Symbol to a Template
 
When you add a symbol to a template, the symbol must already exist in PcVue libraries. The variables used in the symbol should match those of the template. Each instantiation of a template containing a symbol will generate an instance of the symbol in one or more mimics generated from an instance higher up or equal in the instances tree. As you may not want a symbol to appear in all mimics, a simple filter system can be used to control symbol instantiation. See the topics Controlling Instantiation Using Tagging and Controlling symbol instantiation using manual deselection.
 
 
In the right pane of the Application Architect, select the Templates tab.
 
Using the templates configuration tree select the template to which the mimic will be added.
 
Using the secondary navigation tree select the Graphic Objects folder.
 
Select Add symbol. A new symbol with a default name is created.
 
Edit the name to one appropriate to your application. You can only edit the name directly by clicking on it in the configuration tree. Note that this name is only used internally by the Application Architect. Show picture

 
Configure the symbol properties.
 
 
Configuring a symbol's properties
 
To configure a symbol's properties select the symbol in the secondary navigation tree and its properties appear in the properties grid. Show picture

 
 
Description - A description of the symbol. Only used within the Application Architect and has no effect on the application itself. Optional
 

 
Local branch - Local branch for the symbol. Used in conjunction with the mimic property Use context branch to set the Local and Context branches in the symbol instantiation. See explanation in the topic How to add a Mimic to a Template.
 
 
Tag - See the topic Controlling Instantiation Using Tagging.
 
Layers - The drawing layers in which the symbol will be visible. By default, it will be visible in all layers.
 
Symbol - The name of a symbol from PcVue libraries. The ellipsis button adjacent to the field displays the library browser from where the symbol can be selected.
 
Visibility - The zoom range in which the symbol will be visible. By default, it will be visible for all zoom ranges.
 
Position X and Position Y - The symbols location within the mimic. If specified with overwrite the position defined in the symbol.
 
 
Substitutions
 
The Substitutions dialog (opened from the ellipsis button adjacent to the Substitutions property) allows you to substitute the variables defined in the symbol with others - typically those that will be generated by the template. Show picture

 
If the symbol is modified and any of the substituted variables are removed, these will appear in a separate list - Nonexistent variables - in the Substitutions dialog.

Adding Topology Elements
 
A topology element can be considered similar to the instance of a template that has no configuration elements. Its purpose is to add structure to the Instances tree and, as a result, a branch to the variables tree.

Once you have added a topology element you can add instances and create topology elements sub-ordinate to it in the same way that you would for an instance of a template.
 

 
The Branch property creates an explicit branch in the variables tree. If a Description is entered, it is used as the branch description.
 
 

 
The name of the topology element is for reference only and does not affect the structure of the application.
