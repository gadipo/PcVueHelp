Linking a Variable to an IEC 60870-5 Object

Before you can link a variable to an IEC 60870-5 object, the IEC 60870-5 network, one or more devices as well as sectors must have been added to the IEC 60870-5 folder in Data acquisition. For further information, see the help book on Data Acquisition.

You can also link a variable to an IEC 60870-5 object using the IEC 60870-5 Mapping dialog. See the topic Mapping IEC 60870-5 information objects to variables in the book on Data Acquisition for information.

How to link a variable to IEC 60870-5 object

  1. Display the variable's Source tab.
    1. In the Application Explorer expand the variables tree and select the variable.
    2. Select the Properties task. The Variable Properties dialog opens.
    3. Select the Source tab.
  2. Select the Device, Sector and Object type.
    1. Using the Source list box select an IEC 60870-5 network. The network is referenced using the Alias that was entered when it was configured in PcVue. Once you have selected an IEC 60870-5 network as the source the IEC 60870-5 tab appears in the dialog. Show picture
    2. Select the Device and Sector using the drop down list boxes. The list boxes are populated according to the IEC 60870-5 configuration.
    3. Select the Object type. See below for a list of object types and corresponding variable types in PcVue. If you are mapping a bit variable to a Bitstream object, you must also select the Bit rank.
  3. Enter the Information object address. If the variable is read and write (has the control property set), you must also enter a Write information object address. IEC 60870-5 objects use a separate address for read and write. For information on how the address is formatted, see the note on Using a structured address format below.
  4. Select OK to confirm the configuration and close the dialog.

Using the double-point data type

In the equipment, the double point data type has two bits. It is typically used for equipment that has two discreet positions but may not move from one to the other instantly - for example a large circuit breaker. In PcVue, a double-point data type can be mapped to either a single bit or register. The values produced are as follows.

Equipment  PcVue 
B2 B1 Bit Register
0 0 Invalid 0
0 1 0 1
1 0 1 2
1 1 Invalid 3

Using a structured address format

When using a structured address format the three address fields each represent one byte. One, two or three fields are available depending on if you are using one, two or three byte addressing. To convert a decimal address into the three byte oriented fields required for PcVue configuration, use the following method.

  1. Convert the decimal address into a hexadecimal number. Pad out the number with zero's until the number has six digits. For example 210010 > 00083416.
  2. Split the hexadecimal number into three sets of two digits. Each two digits represents one byte. For example 00083416 > 0016 0816 3416.
  3. Convert each of the three two digit numbers back to decimal. For example 0016 0816 3416 > 010 810 5210. These numbers are entered in the three address fields.

The maximum value in each field is 255.

Using an unstructured address format

If using unstructured addressing only one address field is available. The address is entered as it is. The maximum value for the address is 16777215.

Advanced properties

The Advanced tab of the variable's properties dialog contains a number of properties only available when an IEC 60870-5 source has been selected. The exact list of properties depends on the type of variable, the type of ASDU it is mapped to and the fact that it is controllable or not.

  • Monitoring
    • Time tagging - The method used to generate the timestamp for the variable.
      • Default - Use the default method as configured in the IEC 60870-5 settings.
      • Time tagged - Use the time tag from the device.
      • Not time tagged - Do not use the time tag from the device and use PcVue's timestamp instead.
  • Control
    • Control parameters - The control model to be used.
      • Default - Use the default control model as configured in the IEC 60870-5 settings for the corresponding ASDU.
      • Direct execute (DE) - Use the Direct execute control model.
      • Select before operate (SBO) - Use the Select before operate control model.
    • Qualifier (QU or QL) - The control qualifier to be used, or Default to apply the qualifier as defined in the IEC60870-5 settings for the corresponding ASDU.
    • Time tagging - Defines if commands for the variable have to be time-tagged or not.
      • Default - Use the default method as configured in the IEC 60870-5 settings for the corresponding ASDU.
      • Time tagged - If enabled, commands for the variable will be time-tagged.
      • Not time tagged - If enabled, commands for the variable will not be time-tagged.
  • Format
    • Unsigned - If enabled, the value received from the device is interpreted as unsigned.
    • Signed - If enabled, the value received from the device is interpreted as signed. Default.
