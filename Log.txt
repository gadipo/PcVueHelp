Adding a Log page
 

 
 
 
 
 
 
 
 
 
 
 
 
 
How to Add a Log Page
 
Each Log Page can include data from one Log List. A single Log Page can generate two outputs.
 
 
Data output - Basic table of the logged data as extracted from a Log List in an archive unit.
 
Statistics output - Statistical information about the extracted log data. 
 
 
 
Open the Application Explorer and expand the configuration tree to select a previously configured Data Export.
 
From the task list, select Add a log page. The Log Page Creation dialog opens. Show picture

 
Configure the main properties.
Enter the Name for the log page. The name is the unique identifier of the log page within PcVue and can also be used in the expression used to generate the output name.
Enter the optional Description.
Select the output to be created by ticking the Export data and Statistics output properties as appropriate.
Enter the Output name. This property is used to generate the sheet name when using an Excel output or the file name when using a Csv output. It can either be an expression or plain text. See the topic Using Expressions in the Folder, Workbook, Template or Sheet Names. Note that the name for the statistics output is configured in the Statistics tab.
Click Apply to create the log page.
 
Select the Period tab to configure the default period. The properties in the Period tab are only used if you generate the export using SCADA Basic and do not specify the period in the EXPORT instruction. See the topic Understanding the time period used by Data Export for information about how the period properties are used.
 
Select the Columns tab to configure the columns that will appear in the output. See the topic Configuring the log page columns.
 
If you have chosen to generate the statistics output, select the Statistic tab to configure what will be generated. See the topic Configuring the Log Statistics Output.
 
Click the OK button to close the dialog. The final configuration step is to add a Log List to the log page. See the topic How to Add a Log List filter to a Log Page.

Configuring the log output columns
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring the Log Output Columns
 
The configuration of the log output columns is similar to that of the Log Viewer line format. Show picture

 
To edit an existing column, or add a new one, you type in the Header name and then use the ellipsis button to open the Format Editor dialog and select the format characters that define the contents of the column. Show picture

 

 
Although you can choose any combination of format characters, remember that Data Export can only extract the same data (or a subset of it) as was configured in the selected Log List for archiving. For example if you select #@A3 (extended attribute 3) and this was not configured to be archived in the Log List, then that column will be empty in the corresponding page output.

Log files overview
 

 
 
 
 
 
 
 
 
 
 
 
Log files overview
 

 &#160;PcVue generates a number of log files during operation containing messages from the various managers and processes. The messages include general information, debug, warnings and errors. The log files are found in Log Files sub folder of the PcVue main folder Bin and can be broken down into the following categories.
 
 
Trace.Log files - All events produced by the PcVue main program as seen in the Event Viewer window. See the topic Trace log files for more information.
 
MacroTrace.Log file - A record of the PcVue major events only. See the topic Other log files.
 
HDS.Log file - Events produced by the Historical Data Server. See the topic Other log files.
 
OPCSrv.log file - Events produced by the PcVue main program dedicated to OPC server activities. See the topic Other log files.
 
Sv32.exceptions.log file - A file generated in the unlikely event of a fatal error of the PcVue main program. See the topic Other log files.
 
 

 &#160;PcVue also generates various audit files corresponding to the information in the Audit diagnostics part of the Application Explorer. These are also found in the Log Files folder. See the book on Audit & diagnostics in the Application Explorer help.
 

 
Use of traces can affect PcVue performances. If you enable any of the optional traces, they should be disabled once the issue has been diagnosed.

Configuring the Log Statistics Output
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring the Log Statistics Output
 
The Log Statistics output generates statistical information about the log data. Using the Statistics tab, you can select the number of occurrences and/or the duration for each of the states of a bit or alarm. Show picture

 
Using the sorting order
 
Using the Sorting order list box you can choose one of the statistical data as the sort criteria for the rows of data. Data is sorted in descending order. If you choose None then data is not sorted and appears in the order in which the variables first appear in the historic data file within the configured time period.
 
Time counter format
 
The time counters format allows selection of the units in which the durations are displayed.
 

 
Be careful to use a realistic time unit if you use Rounded to the requested unit or you may get zero in all the duration counters. Example period of 10 minutes with a requested unit of hours will produce zero.
 
 
Understanding how the duration is calculated
 
When Data Export calculates the duration for the first point of the requested period it is not possible to know the previous state of the variable. Therefore, the following principles are applied.
 
 
If the first point is a change to 0 it is assumed that the previous value was 1.
 
If the first point is a change to 1 it is assumed that the previous value was 0.
 
If the first point is a change to NS it is not possible to accurately calculate the duration of each state which will result in an empty cell in Excel.
 
 
Furthermore, for the calculations to be made properly, all transitions (0, 1 and invalid) must be selected in the original Log List to make sure they are recorded in the archives.
