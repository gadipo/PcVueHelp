Using the Log Viewer's Scroll Bar

The scroll bar is used to select which data in the display buffer is displayed on the list. The behavior of the scrollbar is different depending on whether you have chosen the default display mode of Display new events at the bottom of the list or the alternative Display new events at the top of the list.

The following abbreviations have been used to make the descriptions easier to read.

  • Buffer Size = the number of lines specified by the Display Buffer property of the Log Viewer's configuration.
  • Display Size = the number of lines visible on the Log Viewer, determined by its size and display font.

If the Display Size is greater than or equal to the Buffer Size then the , and buttons are not active.

Scroll bar behavior when using Display new events at the bottom of the list

Tool

Action

Scroll to the oldest data in the archive unit. The view in the Log Viewer is scrolled so that the oldest data appears on the first line of the display.

Retrieve the next older Buffer Size lines of data from the archive unit and store them in the buffer. The view in the Log Viewer is scrolled so that the oldest data in the buffer appears on the first line of the display.

Scroll the display back in time by one line.

Scroll the lines from the buffer that are visible in the display by clicking and dragging with the mouse.

Scroll the display forward in time by one line.

Retrieve the next newer Buffer Size lines of data from the archive unit. The view in the Log Viewer is scrolled so that the newest data in the buffer appears on the last line of the display.

Scroll to the newest data in the archive unit. The view in the Log Viewer is scrolled so that the newest data in the buffer appears on the last line of the display.

Display or hide the Log Viewer filter dialog.
