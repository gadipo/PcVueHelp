How the Import/Export Tool Works

The Import/Export Tool can be used to export strings from, or import strings to, an individual mimic, mimic template, symbol or entire library.

  • Local library - all mimics, mimic templates and symbols of the local project.
  • Common Library - all mimics, mimic templates and symbols of the common library.
  • Shared libraries - all mimics, mimic templates and symbols of the named shared library.

For more information on libraries, see the book called The Project Libraries in the main Help.

The exported strings are stored in one or more text files in CSV (comma separated value) format from where they can be manipulated by a third party package such as a word processor or spreadsheet. One file can include strings from any number of sources (mimics, mimic templates and symbols). For details, see the topic on Configuring the Import/Export Tool and the book on Import/Export File Formats.

How the Import/Export Tool interacts with PcVue's language capability

Normally during the exporting process, strings for both base and alternative languages are exported. The resulting file will contain records for strings in both languages even if one of them is not used and the strings are null.

When importing you select the two languages to import, one for the base and one for the alternative language, from the selection of languages that the file contains. For example, a file may contain strings for English, German, French and Spanish. You could select French for the base language strings and German for the alternative language string.

Importing strings from different languages does not affect PcVue's language configuration.

For example, if PcVue was installed as French/English and you imported strings for French/German, the software configuration and appearance will still be as French/English.
