Configuration of the PcVue Variables for the KNX Data Acquisition Driver Add-on

To define a variable as having a KNX source you have to enter its KNX address in the extended attribute, as specified in the KNXInterfaces.ini file. It takes the form of a string with the following syntax.

KNX#InterfaceName#GroupAddress#Format#Access

  • KNX - The key that indicates it is a variable with a KNX source.
  • InterfaceName - The name of the KNX/IP interface from which the KNX point is accessible as defined in the KNXInterfaces.ini file, Name parameter.
  • GroupAddress - The KNX group address. All three possible addressing types available within KNX are supported.
    • 3 level addressing format: x/y/z
    • 2 level addressing format: x/y
    • Free addressing format: x
  • Format - The KNX DPT format of the point. See table below.
  • Access - The access method. Specifies if the variable need to be read at start up, or cyclically, and / or if the variable can be written. In all cases, the variable is also refreshed when it changes.
    • W - The variable can be written/controlled. If used this must be the first character in the string.
    • R - The variable is initialized at start-up.
    • Rx - The variable is read cyclically every x seconds.

    See examples below.

Format table

U8 Yes To enable scaling, the DPT range defined in the KNX document must be entered in the min & max of the variable definition.
For example:
DPT_Scaling - variable min = 0, variable max = 100

Access method examples

#WR - The variable can be written from PcVue and is read once at start-up. The variable is also refreshed on change.

#R60 - The variable cannot be written but is read every minute. The variable is also refreshed on change.

# - The variable is only refreshed on change.

#R60W - Invalid syntax. The W must be placed first. The correct syntax would be #WR60.

The syntax KNX#InterfaceName#Disable can be used to map a bit variable for controlling the communication on the specified interface:

  • Bit variable at 0 - Communication on the interface is enabled

  • Bit variable at 1 - Communication on the interface is disabled
