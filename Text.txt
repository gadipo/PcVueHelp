Overview of the Text animations
 

 
 
 
 
 
 
 
 
 
 
 
Overview of the Text Animations
 
The text animations provide real-time information to 
 the user in the form of a character string. All the text animations use 
 a text drawing element as the support. The text drawing element provides 
 the font, color and the style (inverse, button etc.) properties. The following 
 text animations are provided.
 
 
Label animations 
 - displays text from variable's configuration (associated label 
 or description) or context sensitive information such as the branch of 
 a symbol.
 
Text animations 
 - Displays the real time value of a text variable.
 
Register animations 
 - Displays the real time value of a register variable
 
Pre-selected text - Displays 
 text that is configured in the animation itself.
 
 
What is the difference between the Label and Pre-selected Text 
 animations?
 
The label animations display texts that are configured 
 elsewhere - not in the animation itself. If the source of the texts is 
 edited (for example an associated label), the texts that the animation 
 displays change without having to re-configure the animation.
 
The pre-selected text animations display texts that are configured 
 as part of the animation. The only way to change the texts that are displayed 
 is to re-configure the animation.

Changing the appearance of text
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Changing the Appearance of Text
 
Changing the font
 
The font in which the text is displayed may be changed 
 by clicking on the font tool in the Text Properties dialog. You can display 
 text in any font installed on the host PC.
 

 
Be careful when selecting a font that it will be available 
 on the PC on which the application will finally run.
 
 
As long as you do not have the property Do Not Auto Size 
 selected the support rectangle will automatically adjust to suit the size 
 of the text.
 
The color of the text may be selected from any of those 
 in the palette.
 
Changing the appearance of the support rectangle
 
The support rectangle is the rectangular area surrounding 
 a text string. Normally this is transparent but it can be colored and 
 given one of a number of different styles. The style is selected using 
 the appearance properties in the Aspect tab of the text Properties dialog. 
 Show picture
 

 

 
 
 
 
The following options are available.
 
 
 
 

 
 

 
 
 
 
 
Regular. The default appearance for text. The background 
 fill can be transparent or colored.
 
 
 

 
 

 
 
 
 
 
Button. The text takes the appearance of a button. The 
 fill color is fixed. The border size can be changed using the Thickness 
 property.
 
 
 

 
 

 
 
 
 
 
Colored button. The text takes the appearance of a colored 
 button. You can change the fill color and the border size.
 
 
 

 
 

 
 
 
 
 
Relief. The text takes a sunken appearance. You can change 
 the fill color and the border size and color.
 
 
 

 
 

 
 
 
 
 
Inverse Relief. The text takes a raised appearance. You 
 can change the fill color and the border size and color.
 
 
 

 
 

 
 
 
 
 
Shadow. The text support rectangle is displayed with a 
 colored shadow. You can change the fill color and the shadow width and 
 color.

Using the Do Not Auto Size property
 

 
 
 
 
 
 
 
 
 
 
 
Using the Do Not Auto Size Property
 
Selecting the property Do Not Auto Size has the 
 following effects.
 
 
If you change the 
 size of the font the support rectangle does not change (and vice versa)
 
The alignment of 
 the text can be selected from center (default) left or right.
 
A margin, in pixels, 
 can be applied to the side to which the text is aligned.
 
The Multiline property can be 
 selected. Text that is too long to be displayed on one line will wrap 
 onto further lines. The text is automatically broken at the most convenient 
 space.

Overview of inserting text
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Overview of Inserting Text
 
Text is created in much the same way as any other drawing 
 element except that, as part of the process, the text string to be displayed 
 must be typed in. When text is inserted, its dimensions are supplied by 
 a supporting rectangle. This rectangle may or may not be visible depending 
 on the appearance chosen for the text.
 
 
Select the Text command 
 from the Draw toolbar.
 
Position the cursor 
 on the screen where the text is to start.
 
Type in the text 
 to display.
 
Press the Enter key 
 on the keyboard to confirm the text.
 
 
The text will initially appear with a default font and 
 the colors currently selected in the Palette. The text may be edited and 
 other properties changed from the text Drawing Properties dialog. Show picture

Overview of 3D Text
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Overview of 3D Text
 
3D mimics can display text in three dimensions. You create, edit and animate such text objects in a similar way to 2D texts but with added features similar to those of other 3D drawing objects. Show picture
 

 

 
 
 
 
You have to choose appropriate properties for a 3D text object to ensure that it is visible in all required conditions and from all applicable points of view. This Help book includes some basic examples and guidance for doing that.

Re-sizing text
 

 
 
 
 
 
 
 
 
 
 
 
Resizing Text by Dragging
 
Resizing text is another way of changing the font size. 
 If the Do Not Auto Scale property is not selected, the point size of the 
 font is changed automatically to fit the support rectangle.
 
 
Select the text.
 
Click and drag one 
 of the corner handles to its new position. The width and height of the 
 support rectangle and the point size of the font will change simultaneously.

Text string IDs
 

 
 
 
 
 
 
 
 
 
 
 
Text String IDs
 
Each text string ID uniquely identifies a text string
 within a file. The text strings that the Import/Export Tool produces come
 from three sources.
 
 
Mimic properties
 - for example the mimic title.
 
Drawing element properties
 - for example the text displayed by a text drawing element.
 
Animation properties
 - for example the format string of a label animation.
 
 
The Translation Import/Export Tool uses the same identifier
 that is used by the scripting language to reference the drawing elements.
 When a drawing element is included in a group, the identifier is composed
 of the group and element identifiers separated by a dot, for example GROUP2.TEXT1.
