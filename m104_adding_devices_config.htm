Adding Devices to an IEC 60870-5 Network

An IEC 60870-5 device object in PcVue configuration represents, and provides access to, an IEC 101 or an IEC 104 device. In practice, a device may be a data concentrator collecting together several discrete sectors corresponding to underlying devices, or it may have sectors for data it handles directly. You can choose from either an IEC 101 or an IEC 104 compatible device depending on your network architecture.

In PcVue, an IEC 60870-5 device is identified by the Name that you give it during the configuration.

Adding a Device

  1. Select an IEC 60870-5 network in the Application Explorer. From the task pane, select Add an IEC 101 device or Add an IEC 104 device to open the Device dialog. The general configuration is the same for either. Show picture


  2. In the Name field, enter a name for the device.
  3. Enter the optional descriptive text in the Description field. The description is only used in the Application Explorer.
  4. In the IP Address field, enter the device's unique IP address.
  5. Click on OK to confirm the configuration and close the dialog.

The Advanced tab

The Advanced tab contains several device properties pre-set to recommended defaults and in most cases will not need to be changed. A different set of properties is available for the IEC 101 and IEC 104 devices. Please refer to the documentation provided by the device manufacturer for properties that must match the device configuration. Show pictures





The following properties are available for IEC 104 devices:

  • Connection parameters
    • TCP/IP parameters
      • Remote port - The TCP port used to communicate with the device. Default 2404.
    • Originator address - The address to be used by the driver to identify the PcVue IEC 104 client. Default 0.
    • Link layer properties
      • T0 - Timeout of connection establishment. Default 30 sec.
      • T1 - Timeout of send or test APDUs. Default 15 sec.
      • T2 - Timeout for acknowledges in case of no data messages. T2 < T1. Default 10 sec.
      • T3 - Timeout for sending test frames in case of a long idle state. Default 20 sec.
    • APDU sequence control parameters - Maximum number of outstanding I format APDUs
      • k - Maximum difference between the receive and send sequence numbers. Default 12.
      • w - Latest acknowledge after receiving w I format APDUs. w should not exceed 2/3 of k. Default 8.
    • Application layer properties
      • Response timeout - Timeout for Application layer response messages. Default 10 sec.

The following properties are available for IEC 101 devices:

  • Connection parameters
    • Connection type
      • TCP/IP - If enabled, communication over TCP will be used with the device.
        • Remote port - The TCP port used to communicate with the device. Default 2404.
      • UDP/IP - If enabled, communication over UDP will be used with the device.
        • Listening port - The UDP port used to listen to incoming messages. Default 2405.
          The listening port must be unique for all IEC 101 devices over UDP produced by the same server list.
        • Remote port - The UDP port used to send messages to the device. Default 2404.
    • Link layer properties
      • Link address - The Link layer address of the device. Default 0.
      • Link address size - The size of the link address. Either 1 or 2 bytes. Default 2.
      • Link mode - Unbalanced. Fixed value.
      • T0 - Timeout of connection establishment. Default 30 sec.
    • Session layer
      • Common address of ASDU size - Size of the common address of ASDU. Either 1 or 2 3 bytes. Default 2.
      • Cause of transmission (COT) size - Size of the cause of transmission. Either 1 or 2 bytes. Default 2.
      • Information object address (IOA) size - Size of the Information object address. Either 1, 2 or 3 bytes. Default 3.
    • Application layer properties
      • Response timeout - Timeout for Application layer response messages.
  • Classes polling
    • Class 1 polling delay - Polling interval for class 1 data. Default 0 means a new poll request is sent as soon as the previous one is completed.
    • Class 2 polling delay - Polling interval for class 2 data. Default 500 ms.
    • Class 1 polling delay when an application layer response is pending - Polling interval for class 1 data when a request is pending. Default 0 means a new poll request is sent as soon as the previous one is completed.
    • Class 2 polling delay when an application layer response is pending - Polling interval for class 2 data when a request is pending. Default 0 means a new poll request is sent as soon as the previous one is completed.
  • Field devices redundancy
    • Link status watchdog for standby devices - The interval at which the link status is checked for standby devices. Default 1 sec.

The Traces tab

The Traces tab is used to enable debugging traces in the event of any problems with the device. Only use this if advised to do so by your support agency.
