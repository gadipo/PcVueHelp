The SNMP Mapping Dialog

The SNMP Mapping dialog is opened from the Application Explorer by using the Mapping task whenever an SNMP Network or Device is selected. It contains four display panes plus a status line. Show picture



[A] Device Tree

See the topic Understanding the SNMP device tree.

[B] Variable mapping

The Variable Mapping pane displays the OID's belonging to the selected branch of the Device Tree. Right clicking in the pane displays a context menu with the following commands.

  • Variable selector - Opens the Variable Selector from where you can map an existing variable to the selected OID.
  • Create variable - Creates a new variable and maps it to the selected OID. See the topic Mapping an OID, Trap or PING.
  • Unmap item - Removes the variable mapping from the select OID.
  • View variable - Open the properties dialog for the variable mapped to the selected OID.
  • Select columns - Select the columns displayed in the Variable Mapping pane from the following.
    • Name - The symbolic name of the OID (or numeric identifier if the symbolic name is not available). This column is mandatory.
    • Mapped variable - The name of the variable that is mapped to the OID.
    • Native type - The SNMP native data type.
    • Access - Access type.
      • Read only - for read-only OIDs.
      • Read-write - for read-write OIDs.
      • Accessible-for-notify - For OIDs that cannot be read but can be sent in a trap
    • Full numeric OID.
    • Snapshot value - The value of the OID when it came on-line.

[C] Enumerated variable mapping

List of the mapped variables for the selected enumerated OID. Empty if the selected OID is not enumerated. See the topic Mapping an enumerated OID.

[D] OID information

Detailed OID information from the MIB file including the description, range, access and data type.

[E] Status bar

Displays the branch that is currently selected in the Device Tree and allows you to change the browsing method from Offline to Online (and vice versa).
