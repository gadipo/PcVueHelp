DNP3 Inputs and Outputs, Object Groups and Variations

The following table lists the DNP3 input and output types that are supported by PcVue. An input or output can correspond to a physical input or output or it can be calculated by the device. In addition to the inputs and outputs, there two other data types known as Internal Indications and Device Attributes that provide information about the device itself.

The Object Groups define how the inputs and outputs are read and written. The actual messaging in DNP3 is quite complex but a highly simplified explanation of the groups is that, for each input and output type there are two groups, one that is used for polled messages and the other for event (exception driven) messages. For binary and analog outputs there is an additional group used for the write process.

Type Group Description Variables tree
Binary input 1 A two state value such as a simple on off switch. Bit
Binary input (event) 2
Binary output 10 A two state output such as the open and close control for a circuit breaker. Bit
Binary output (event) 11
Binary output (CROB) 12 Control Relay Output Block. The mechanism used to write a binary output.
Running counters 20 A value that counts the number of occurrences of an event. For example the number of times a circuit breaker has tripped. Register
Running counters (event) 22
Frozen counters 21 A value that counts the number of occurrences of an event at a particular point in time. Register
Frozen counters (event) 23
Analog input 30 A single analog input such as a phase voltage. Register
Analog input (event) 32
Analog output 40 A single analog output such as a set-point. Register
Analog output (event) 42
Analog output (AOB) 41 Analog Output Block. The mechanism used to write an analog output.
Internal indications 80 A two state value that indicate the operational status of the device. Bit
Device attributes 0 String values representing static properties, and analog values representing configuration attributes, of the device. For example the manufacturer's name or the quantity of analog inputs. Register or text as appropriate

The double bit binary input is not currently supported.

Variations

Variations are a specific range of formats in which the data can be presented in each Group. For example, analog inputs can be a 32 bit integer with flag, a 16 bit integer with no flags, etc. For a full list of the supported variations, see the topic DNP3 implementation table.

For more information on the operation of the Variable Selector, see the book The Variable Selector.
