Application Architect Settings
 
The Application Architect's settings can be displayed either from the project settings in the Application Explorer or from Tasks.Settings in the Application Architect menu. Show picture

 
 
Show generation option box - Show the Options dialog when generating the configuration elements.
 
Consistency checking
 Check consistency at startup - If enabled, the consistency check is performed during the startup of PcVue. While it comes with a performance penalty and is not recommended in production, it can be helpful on an engineering station.
Treat warning as error - If enabled, the occurrence of a warning has the same effect as the occurrence of an error. Saving is blocked and other actions may be interrupted.
Allow blank branch - Allows an empty (blank) branch at instantiation.*
Allow duplicate branch - Allows the use of duplicate branches within the same tree.*
Allow recursive property - Allows one property to be used in the expression of another parameter. See below. Must also be enabled if a parameter or expression is used in the Description property of any of the elements.
 
Type of synchronization
 Full synchronization - Uses full synchronization when generating the application. See the topic on Generation for details.
Fast synchronization - Uses fast synchronization when generating the application. See the topic on Generation for details.
 
Time optimization - Options to improve the responsiveness of the Application Architect's HMI.
Skip reference calculations in the Application Architect's HMI - This setting affects the templates tab. Selecting it means that the calculation of properties requiring access to an Excel function (by parameter, expression, combination of two etc.) are not made. The properties are displayed as empty.
Skip reference calculations for generation - This setting affects the generation of properties requiring access to an Excel function. The properties are not generated in the output from the Application Architect. The affected properties do not exist - they are not just empty. This is to ensure the correct synch when de-selecting the option.**
Expand template composition one level only - This setting affects the templates tab. When displaying the composition of a template, if this setting is selected, the tree view is only expand to one level. So, for example, you will see variables but not the trends associated with the variables. This is used to improve the display of the Application Architect's HMI when the template contains many elements for which and the user does not need to see the entire expanded tree.
Do not display detail of included templates for instances - This setting affects the instances tab. If not selected when you click on a template instance, the inputs, input parameters and expressions appear in the lower part of the screen for both the selected instance and any child objects particularly those from included templates. If set, inputs, inputs parameters and expressions for the included templates are not displayed. This is used to improve the display of the Application Architect's HMI, particularly for templates including other templates that have many parameters to evaluate.
 
 

 
* If you enable these settings it may result in the generation of duplicate variables.
 
 

 
** If you select, and then at a later time, de-select the Skip reference calculations for generation and then do a synchronization, it can be equivalent to a full synchronization if there are many properties affected. 
 
 
Allow recursive property
 
The Allow recursive property setting should be used with caution as it can cause an infinite loop during generation. This is best explained by two examples.
 
Example 1 - The following can only be evaluated if Allow recursive property were enabled.
 
Parameter01: "=3 + 4"
 
Parameter02: "=Params["Parameter01"] + 7"
 
Example 2 - The following would cause an infinite loop if Allow recursive property is enabled.
 
Parameter01: "= Params ["Parameter02"] + 4 "
 
Parameter02: "= Params ["Parameter01"] + 7 "

Audit & diagnostics settings
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Audit & Diagnostics Settings
 
Audit file configuration
 
The number of audit files and the frequency at which they are created can be configured using the following properties. Show picture

 
 
Rotating files
Number of rotating audit files - 
The maximum number of audit files that can exist at any one time in the Log Files folder. When the limit is reached, each time a new audit file is created the oldest file is discarded. The default value is 1000, minimum value = 100 and maximum value = 9999.
Write period for the Audit.Coumtersxxxx.txt files - The period at which each new general audit file is created. The default value is 30 minutes, minimum value 1 minute, maximum value 24 hours.
Audit Specific Counters period - The period at which each new memory specific audit file is created. The default value is 10 minutes, minimum value 1 minute, maximum value 24 hours.
 
Trace files
Max size for the Trace.Log file - The maximum size of the Trace.Log file in megabytes. When the file reaches this size, it is backed up and a new file is created.
Max size for the MacroTrace.log file - The maximum size of the MacroTrace.Log file in megabytes. When the file reaches this size, it is backed up and a new file is created.
 
Dump files

The maximum number of dump files that are persisted at any one time in the Log Files folder (in addition to the last one). When the limit is reached, each time a new dump file is created, the oldest file is discarded. The default value is 2 for large dumps (max 9) and 9 for small dumps (max 20). Because dump files can be large, it is not recommended to use a large value, in particular for large dump files. 
 
 
Enabling the audit system variables
 
If you want to use any of the audit system variables in the application, they must be enabled using the System Variables tab of the Audit Settings dialog. For each variable, you can use the default name or choose your own. The Create all system variables tick box can be used to create all the variables, but before you do this you should consider the impact on system performance. There are over 1000 variables. Color preferences
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Color Preferences
 
The Color Preferences dialog is opened from the menu using the command Configure.Preferences.Colors. Show picture

 
The Color Preferences dialog contains five tabs that are used to select the colors for certain objects and features in PcVue's HMI. Some of the color preferences are used as is - that is the color defined in the preferences is the color that is used. Others are used as a default that can optionally be changed in each instance of the object. Changing a color in the Color Preferences dialog automatically changes the colors in the HMI's objects where it is used as is.
 
Each color preference defines a pair of colors, one for the background and the other for the text (or border depending on the use). Non-indexed or indexed colors can be used.
 
Alarm on text tab
 
Used to color alarms in the alarm viewer and log viewer, and the text drawing element when animated using color-alarm on text.
 
 
In an alarm viewer or log viewer, the background color is only used if the Advanced color mode property is selected. The text color is always used to color the text of alarms. The colors are used as is.
 
In a text drawing element, both the background and text colors are used. The colors are used as a default.
 
 
Alarm tab
 
Used as the default colors whenever a shape is animated using a color-alarm animation. For a solid shape, rectangle, polygon, b�zier shape ..., both background and border colors are used. For a line, polyline or b�zier curve only the border color is used.
 
Bit tab
 
Used to color bit transitions in the log viewer, and a shape drawing element when using a color-bit animation.
 
 
In a log viewer, the text color is always used to color the text of bit transitions. The background color is unused. The colors are used as is.
 
For a solid shape, rectangle, polygon, b�zier shape ..., both background and text colors are used. For a line, polyline or b�zier curve only the border color is used. The colors are used as a default.
 
 
Action tab
 
Used to color user actions in the log viewer. The colors are used as is. Only the text colors are used.
 
Thresholds tab
 
Used to color the cells of the grid, when used in historical mode, when the register values exceed their thresholds. The colors are used as a default.
 
Navigator
 
Used to color the borders of the navigator, run-time navigator and mimic navigator.
 
Trends
 
Used to color the traces in a trend viewer. The colors are used as a default.
 
Chart Control
 
Used to provide the default palette colors for the Pie Chart.

Interoperability issues
Ack time sync request from clients - If enabled, the IEC 104 outstation will acknowledge positively the reception of clock synchronization requests.
The clock synch should be based on a system-wide master clock. Because setting the Microsoft Windows system clock requires privileges higher than those usually given to the account used to run PcVue, the IEC 104 outstation never sets the system clock upon reception of such a clock synch command.
Timestamp
UTC - If enabled, all timestamps send by the selected sector are expressed in UTC.
Local - If enabled, all timestamps send by the selected sector are expressed in the Local time zone.
 
 

 
While the IEC 104 standard assumes the use of timestamps expressed in the local time zone, many devices supports the use of either local or UTC timestamps. The Timestamp interop setting allows you to set how the IEC104outstation should interpret timestamps. This setting must be carefully set in accordance with the behavior of the installed devices.
 
In case of a discrepancy on this setting, and in addition to monitored data being wrongly timestamped, any received time-tagged command will be ignored if the timestamp is in the past compared to the current time. How a Quality Sequence Works
 
A previously configured sequence is assigned to the animation 
 in question. For example, the sequence Standard is assigned to a Color 
 On Bit animation. At Run time, when the quality of any variable in the 
 animation changes each row in the sequence is checked in turn. The quality 
 value is ANDed with the Mask and the result checked against the Value. 
 
 
If there is a match the properties defined in the condition 
 are substituted for those of the animation, otherwise the next row is 
 checked and so on. If there are no matches then the properties configured 
 in the animation are used.
 
The properties that are used from the sequence will depend on the animation type.

Overview of OPC Quality
 
An OPC server provides a minimum of three properties 
 for each of its variables, known as Value, Quality and Timestamp.
 
 
The Value is the 
 real-time value of the variable, for example the level of a tank.
 
The Timestamp is 
 the time at which the value last changed.
 
The Quality provides 
 information about the reliability and status of the value.
 
 
The Quality is provided as a 16-bit word. The higher 
 8 bits are vendor specific and the lower 8 bits always follow a standard 
 format.
 
The extent to which the Quality property is supported 
 depends on the server you are using. For further information on the server 
 you are using see the vendor documentation.

Using the Rename mimic template object ID's option
 
When selected, this option will cause the ID of any mimic template objects that are identical to the ID of mimic objects inherited from the template, to be automatically renamed. Not only is the object ID modified, 
but also any references to the ID in any Enable script.
 
This is necessary when using the Enable scripting language as each object ID must be unique. If you script objects with identical IDs, the script of the object loaded first will work but subsequent objects with the same ID that are loaded will generate an error message.
 
If possible, it's better to plan in advance and set-up a naming scheme that 
will avoid duplicate ID as the automatic renaming system will penalize 
the mimic loading performance.
 

 
This option must not be used in the VBA version of PcVue.
 
 

 
VBA allows duplicate object IDs.
 
 

 
VBA does not support object renaming.
