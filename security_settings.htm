Security settings

Security settings control various access permissions and system behaviors, ensuring secure operation and data integrity. These settings allows configuration of elements such as expiration warnings, security alteration, and server interfaces activation. Proper configuration is essential for enabling features like alarm and historical data access while maintaining system security.

The Security settings can be displayed from the Security entry in the Application Explorer's Main settings dialog or from the Settings task when the Security folder is selected in the Application Explorer's configuration tree.

Prerequisites

  • You are logged in with a user having the Security administration permission enabled. See the topic Profile properties that affect operating system and development access for more information.

Security settings reference

The following settings can be configured for the project:

  • Settings:
    • Warning delay before expiration (in days) - Defines the number of days before a security-related expiration triggers a warning. This helps administrators take preventive actions before access restrictions, certificate validity or credential expiration occurs.
  • Security alteration:
    • OPC DA servers:
      • Allow security altering configuration options - Grants permission to modify security-related configuration elements for OPC DA servers. Enabling this option allows changes to OPC DA servers access control policies, which may impact the system exposure and communication with external OPC clients.
    • Networking:
      • Allow security altering configuration options - Grants permission to modify security-related configuration elements for stations in a networking environment. This setting is only available in the project's settings.
    • COM servers:
      • Allow alarm server - Activates the Alarm COM server, allowing external applications to access alarm data from PcVue. This is essential for integrating alarm management with third-party systems such as FrontVue.
      • Allow historical server - Activates the Historical COM server, allowing external applications to retrieve stored historical data from PcVue. This is essential for integrating historical management (logs and trends) with third-party systems such as FrontVue.
    • DDE clients and servers:
      • Allow feature - Activates the DDE client and server features. This option requires a restart of PcVue to take effect after being enabled.

Activating the COM servers is essential if you are using FrontVue as runtime front-end on top of PcVue.
