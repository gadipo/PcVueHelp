Data group, Dataset, and Setting group configuration
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Data group, Dataset, and Setting group configuration
 
A data group provides polled access to all data objects and data attributes of an IEC 61850 device. It is the least efficient way of collecting data from the network as each variable that you map using a data group will be serviced using a separate poll. Data groups should only be used to access data objects or data attributes that are not available from a data set or report.
 
A dataset provides polled access to all data objects and data attributes of an IEC 61850 device dataset. It is more efficient than a data group, as a single poll will return all data objects and data attributes that belong to the dataset.
 
A setting group manages configuration presets within an IEC 61850 device, allowing dynamic switching between operational settings.
 
Prerequisites
 
 
You have configured an IEC 61580 network.
 
You have configured an IEC 61580 device.
 
 
Add a data group
 
 
Open the Application Explorer and select an IEC 61850 network and device.
 
Click Add a monitoring object in the toolbar and select Add a data group from the drop-down. The IEC 61850 data group creation dialog opens. Show picture
 

 
 
 
Enter a name and (Optional) a description. The name is the identity of the data group within PcVue.
 
Enter a polling group to enable cyclic data collection at defined intervals.
 
Tick Activate at start-up to enable automatic polling at start-up.
 
Click OK to save the configuration.
 
 
Add a dataset
 
 
Open the Application Explorer and select an IEC 61850 network and device..
 
Click Add a monitoring object in the toolbar and select Add a dataset from the drop-down. The IEC 61850 dataset creation dialog opens. Show picture
 

 
 
 
Enter a name and (Optional) a description. The name is the identity of the dataset within PcVue.
 
Select the dataset type. Static datasets are used for consistent and unchanging data structures, whereas dynamic datasets are used for applications needing flexible configuration that can adapt to different engineering scenarios.
 
Select the dataset name. This is the name as exposed by the physical device. Depending on the device configuration, PcVue obtains a list of the available datasets by browsing the connected physical device or by listing datasets defined in the SCL file associated to the device.
 
(For Dynamic dataset only) Select the dynamic type. If you selected Persistent, the dataset object will be persisted at the physical device level and you are required to select a Logical Device and Logical Node, and enter a Name for the persisted dataset. These properties allow structured data grouping within the device and are essential for configurations that persist across sessions.
 
Enter a polling group to enable cyclic data collection at defined intervals.
 
Tick Activate at start-up to enable automatic polling at start-up.
 
Click OK to save the configuration.
 
 

 
The configuration of a dynamic dataset is sent to the physical device upon connection. You need to restart the IEC 61850 device configuration element if you want such a change to be taken into account.
 
A non-persistent dynamic dataset is automatically deleted and unreferenced by the physical device upon disconnection.
 
Deleting a persistent dataset removes the associated dataset from the physical device. If an error occurs (e.g., if a report still references the dataset), a warning popup is displayed, and the dataset group will remain active until resolved. Such an error can commonly occur, in particular if a report is still referencing the dataset in the physical device.
 
 

 
Buffered reporting with a dynamic dataset requires the dataset to be configured for persistency at the physical device level.
 
 
Add a setting group
 
 
Open the Application Explorer and select an IEC 61850 network and device.
 
Click Add a monitoring object in the toolbar and select Add a setting group from the drop-down. The IEC 61850 setting group creation dialog opens. Show picture
 

 
 
 
Enter a name and (Optional) a description. The name is the identity of the setting group within PcVue.
 
Select a control block from the drop-down. The control block is a structure that manages and organizes the setting groups within a device. PcVue verifies the existence of the selected control block upon startup. If the control block is missing on the device, its status is set to "Invalid control block.�
 
Enter a polling group to enable cyclic data collection at defined intervals.
 
Tick Activate at start-up to enable automatic polling at start-up.
 
Click OK to save the configuration.

A data report provides access to all data objects and attributes of the linked dataset. It is the most efficient method to access data as data changes are sent automatically by the device each time the specified trigger conditions are met. No polling by PcVue is required.

Depending on the device configuration, PcVue obtains a list of the available data reports by browsing the connected physical device or by listing report control blocks defined in the SCL file associated to the device. See below for more information on how to configure the Original report name in redundant data acquisition architectures.
 
Tick Activate at start-up if the data group is to start automatically at start-up.

The Advanced tab
 
See the topic IEC 61850 Data Report - Advanced tab. Note that the properties in the advanced tab are important as they determine how the Report Control Block is handled.
 
Configuration for data acquisition redundancy
 
As specified by the IEC 61850 standard, handling reporting requires exclusive access to a Report Control Block. If 2 clients access the same RCB, correct reporting cannot be guaranteed and data may be lost.
 
To cope with this constraint, when 2 or more redundant stations are in charge of data acquisition for a given Data Report, one need to define one RCB (original report name) per redundant data acquisition server. In practice, you have to define one redundant RCB per station in the association producing the IEC 61850 network:
 
 
Set the property Original report name to (Network-dependent configuration) on the Data report item.
 
Select the data report in the data report list (top panel of the Application Explorer), go to the behavior panel (bottom panel) and click the Add button.
 
Select the first data acquisition in the Server station list, and select the Original report name this station will be in charge of.
 
Repeat steps 2 and 3 for each data acquisition server in the list, each time selecting another report name.
 
 

 
We recommend avoiding non-persistent datasets in a redundant context because they would be deleted and recreated with each disconnection or station switch. Persistent datasets or redundant reports, however, maintain existing redundancy behavior.
 
 
Data acquisition redundancy for reporting requires each IEC 61850 device to expose as many Report Control Block as there are redundant data acquisition servers in the association producing the network.
 
Refer to the device manufacturer documentation for more information about support and configuration of report control blocks, support for multiple client connections and report control block reservation.
 

 
It is strongly recommended that such 'redundant' report control blocks be attached to the same dataset, which PcVue is able to enforce if the data report is configured to send the dataset configuration at start-up (see IEC 61850 Data Report - Advanced tab).

Data Sets, Data Reports, Data Groups and Setting groups
 

 
 
 
 
 
 
 
 
 
 
 
Data Sets, Data Reports, Data Groups and Setting groups
 
&#160;PcVue can collect data from an IEC 61850 device by three mechanisms.
 
 
Using a Data Set. A Data Set is created as part of the physical device configuration (using the IED configuration tool) and is normally a group of data objects. You can configure PcVue so that it polls a Data Set at regular intervals. A single poll will return all attributes of the Data Set making it a very efficient mechanism. PcVue supports static and dynamic datasets.
 
Using a Data Report. A Data Report or Report Control Block is created as part of the physical device configuration (using the IED configuration tool) and is a set of conditions applied to data belonging to a Data Set. When the conditions are met, the data is sent as an unsolicited report to PcVue. There are two forms of Data Report.
Buffered. If PcVue is unavailable, Buffered Data Reports are stored locally in the device and then sent once PcVue becomes available.
Un-buffered. Un-buffered reports are not stored locally in the device if PcVue is unavailable and any data sent will be lost.
 
Using a Data Group. A Data Group is configured in PcVue and defines only a poll rate. The IEC 61850 objects and attributes, mapped to variables allocated to the group, are polled at that rate. As each attribute is polled individually this is a very inefficient method and should only be used if the data cannot be obtained by one of the other mechanisms.
 
Using a Setting group. A Setting group is configured in PcVue and defines only a poll rate. The IEC 61850 attributes of a Setting Group Control Block can then be mapped to variables allocated to the group, and polled at that rate. It allows monitoring and controlling settings groups activation.

IEC 61850 communication can be entirely event driven. That is PcVue does not poll a device, it just waits for the device to send it any changes. This is the most efficient way of communicating. However if communication to the device fails PcVue may be unaware.

Mapping IEC 61850 data objects and data attributes
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Mapping IEC 61850 Data Objects and Data Attributes
 
The procedures required to map data objects or data attributes are very similar. Any differences are noted in the text.
 
The Mapping dialog can be used online and offline:
 
 
To use it online, PcVue must be connected to the actual network, physical devices must be operational, and the discovery services must be enabled.
 
To use it offline, SCL files must have been associated to the network and used as part of the PcVue configuration.
 
 
You switch from one mode to the other by clicking the Online/Offline button available in the status bar.
 
Behavior and requirements when mapping data objects and data attributes
 
When mapping a variable to a data object, the value, timestamp and quality are all supplied by the object. Therefore, the data object must support these attributes. PcVue uses the object's Common Data Class (CDC) to identify which of its attributes to use. The supported CDC's are defined in the profile that was selected when the Physical Device was configured. See the topics Adding a Physical Device to an IEC 61850 network and IEC 61850 device profile catalog for further information. You can only map data objects that match a CDC PcVue supports.
 
When mapping a variable to a data attribute, only the value is supplied by the attribute. The timestamp and quality are provided by PcVue. All data attributes can be mapped.

IEC 61850 Network - SCL Files
 
An SCL file is an Xml file whose content is based on the Substation Configuration Language format as defined in the IEC 61850-6 standard.
 
SCL files are a basic brick of IEC 61850 system engineering and come in a variety of forms: Scd, Cid, Ssd, Sed, Iid etc.
Although all may contain information about IED's that PcVue is able to extract, each file type has a particular purpose and the use of Scd files is recommended.
 
In PcVue, you can use SCL files to define a variety of configuration elements:
 
 

 Data acquisition: Devices, dataset groups, data report groups and data groups, 
 

 Variable mapping: On Data Objects and Data Attributes. 
 
 
SCL files can be added to an IEC 61850 network configuration in PcVue using the SCL Files tab of the IEC 61850 Network Properties dialog. 
 Show picture

 
How to add an SCL file
 
 
Open the IEC 61850 Network properties dialog and select the SCL Files tab.
 
 Click the Add tool. A file browser dialog opens. By default, the contents of the project TP folder is displayed but you can navigate anywhere visible to the host PC. 
 
Select the SCL file to be added and click Open. The IEC 61850 SCL file creation dialog opens. 
 Show picture

 
The reference Name under which the file will be added to the network configuration defaults to that of the SCL file but you can edit it to choose one of your own. 
 
Click Apply to read the SCL file and extract the information pertinent to PcVue. This can take a few seconds if the file is large. The information that is extracted is displayed in the Content tab. Show picture

 
Click OK to complete the process of adding the SCL file to the network.
 
Repeat steps 2 to 6 to reference additional SCL files (if using Cid files for example).
 
 

 
 When associating SCL files with the IEC 61850 network for offline configuration, it is strongly recommended to store the SCL files in the TP folder of the project and to add them to the list of files managed as part of Central Project Management (if in use).

Overview of IEC 61850
 

 
 
 
 
 
 
 
 
 
 
 
Overview of IEC 61850
 
IEC 61850 is a communication standard developed for the power substation market. A derivative of it is used in wind farm generation where it is known by the standard IEC 61400-25. The full IEC 61850 standard, comprising of some 20 documents, is available from the IEC website (chargeable).
 
IEC 61850 defines a standardized method for describing power system devices that enables all IED's (Intelligent Electronic Devices - the microprocessor-based controllers of power system
equipment) to present data using identical structures directly related to their function. The abstract data model defined in IEC 61850 can be mapped to a number of protocols. Current mappings in the standard include MMS (Manufacturing Message Specification), GOOSE and SMV. For Supervisory purpose, MMS over TCP/IP is used and it is PcVue's interface to this that is described in this help book.
 
IEC 61850 modeling
 
The IEC 61850 data model begins with a physical device that connects to the network. The physical device is defined by its network address. Within each physical device may be one or more logical devices. The IEC 61850 logical device model allows a single physical device to behave as a proxy or gateway for multiple devices thereby behaving as a data concentrator.
 
Each logical device contains one or more logical nodes. A logical node is a named grouping of data and services that is logically related to some power system function. The following table represents the XCBR class.

Each logical node has an instance Id as a suffix to the logical node name. For example, a device has two measurement inputs to measure two 3-phase feeders. The standard name of a logical node for measurements in 3-phase is MMXU and so the two logical nodes would be MMXU1 and MMXU2. Each logical node may also have an optional application specific prefix to provide further identification of its purpose. A list of compatible logical nodes can be found in the document IEC61850 part 7-4.
 
Each logical node contains one or more data elements. Each data element has a unique name determined by the standard and related to its purpose. For example, a circuit breaker is modeled as an XCBR node. It contains a variety of data including Loc (local or remote mode), OpCnt (operations count), Pos (position), BlkOpn (block breaker open command), BlkCls (block breaker close command) and CBOpCap (operating capability).
 
Each data element conforms to the specification of a Common Data Class (CDC). The CDC determines the type and structure of the data. For example, there are CDCs for status information, measured information, controllable status information etc. Each CDC has a defined name and set of attributes, also each with a defined name, type and purpose. Each attribute also belongs to a set of functional constraints (FC) that groups the attributes into categories. For example, ST for status attributes. In the following table, representing the SPS class, the status attributes consist of a status value (stVal), a quality flag (q) and a timestamp (t).
