SNMP general settings

The SNMP General Settings are accessed from the Application Explorer by selecting Project.Communication.Data acquisition.SNMP and choosing the Settings command from the task list. Show picture



  • Communication
    • Maximum number of concurrent requests - The maximum permitted number of pending requests, across all the configured networks, at any given time. Default 150, maximum 500.
  • Data
    • Convert timetick values to milliseconds - The SNMP timetick unit is 100 milliseconds. Ticking this box converts any timetick values to milliseconds, which is PcVue's standard time unit. The default is ticked.
  • Configuration
    • Use numerical OID in variable configuration - If ticked use the numerical OID format in the OID property of the Variable Configuration dialog (source tab) and in the trace. If un-ticked the symbolic OID used. Changes to this property are only taken into account after network or equipment start. The default value is ticked.
    • MIB browser sorting
      • Using the symbolic OID - If selected, the MIB browser displays data sorting according to their symbolic OIDs.
      • Using the numerical OID - If selected, the MIB browser displays data sorting according to their numerical OIDs.
    • Maximum number of port displayed - The number of link status displayed in the mapping.

SNMP Traces tab

The SNMP Traces tab contains properties to enable additional error messages that can be useful when diagnosing errors. The errors appear in the Event Viewer (F7). You should only select properties in this tab if advised to do so by your support authority.
