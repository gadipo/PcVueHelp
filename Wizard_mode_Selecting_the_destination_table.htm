Selecting the Destination Table

The next step is to complete the Import Destination Configuration dialog. It enables the import process to connect to the data server that contains the table into which the data is to be imported. Show picture.

  1. In the Server Instance drop-down box, select the data server that contains the table.
  2. In the Authentication section, select the mode of authentication for access to the server as either Windows Authentication or SQL Server Authentication. The latter requires a User name and Password.
  3. Select Encrypt connection if you want to ensure confidentiality and integrity
  4. Click on the Test Connection button. A dialog confirms whether the connection is made. That must occur to enable the next step.
  5. In the boxes of the Destination section, type in the Database Name and the Table Name for the destination table.
  6. The option Purge Table Before Import enables you to delete any previous contents of contents of the destination database before the import takes place.
  7. Click on the Import button to proceed with the import.

To take advantage of an encrypted communication channel, SQL Server must be setup to work with TLS and digital certificates.
For more information, refer to https://learn.microsoft.com/sql/database-engine/configure-windows/certificate-overview

Make sure you select the correct type of table when importing records. The wizard does not differentiate between them at this stage and you will get an error if you try to import log records into a trend table (or vice-versa)

If you tick the Purge Table Before Import option ALL DATA in the selected table will be PERMANENTLY DELETED before the import.
