Using Mimics with a Branch

When a mimic is opened or created, it may be given a variable branch dynamically at opening time. Subsequently any reference to variables is relative to that branch. This technique can be used to reduce the number of mimics required and the subsequent development time.

An example of using a mimic with a variable Branch

Consider an application where there are four identical gas turbines to be supervised. Each turbine has more than 100 variables associated with it. In PcVue, each variable name starts with the turbine number.

@turbine1.speed, @turbine1.pressure etc.
@turbine2.speed, @turbine2.pressure etc.
@turbine3.speed, @turbine3.pressure etc.
@turbine4.speed, @turbine4.pressure etc.

A mimic named Turbine is created initially with the Branch turbine1. Subsequently all variable references are relative to the branch turbine1, so speed actually refers to @turbine1.speed. A further mimic named Menu is created with four buttons each one with an animation to open the Turbine mimic but using four different Branches each corresponding to one of the turbines (turbine1, turbine2, turbine3 and turbine4).

At runtime, the Turbine mimic may be opened four times, each time displaying a different group of variables.

With this example, instead of creating four mimics it was only necessary to create one. This has the dual advantage of reducing development time and making maintenance much easier.
