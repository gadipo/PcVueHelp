How to import a configuration subset

The Import feature of the Application Architect makes it possible to import a subset of the configuration elements you have modeled and exported. It facilitates the reuse of application functions in other projects, reinventing therefore, the way projects are configured. This feature also enables the creation of large projects out of smaller ones, improving thus the production and generation of projects.

We recommend you make a backup of you configuration before carrying out the import process. Having a backup at hand helps manage any error that might occur.
The backup can be done using the Save button on the import task bar.

  1. From the PcVue menu, go to Configure.Application Architect and select the Import tab.

  2. Click on the button representing the type of element you want to import: Instances, templates or parameters. The Reference data pane on the left displays the list of elements of the selected type in the current configuration of the project. Show picture

  3. In the right pane, select the file containing the configuration to import - A file created using the Export feature. The content of the file will appear in the Data to import pane. If needed, you can use the Reload file button to update the content of the pane.

    An empty list of elements to import after you have selected and loaded a file to import is the sign that the selection of element type in the left pane does not match the type of elements you are trying to import.

    For example, if you have selected Instances in the left pane and the file you are trying to import only contains templates, then the list of elements to import remains empty because there is no instance to import in the selected file.

  4. (Optional) Right click in the Data to import pane to check the Advanced mode to see the properties modified and their values for configuration elements contained in templates. This mode is not available for parameters.

  5. (Optional) Check Merge with existing to combine the imported elements with the ones in your current configuration. Without merging, the elements you import replace the current configuration.

    For every element you import, if the element already exists in your current configuration, its properties will be overwritten by those defined for the element you import.

  6. Drag and drop the elements from the Data to import pane to the Reference Data pane.

  7. Click Check consistency to check if there is a configuration conflict.

    Inconsistent configurations are due to various factors. Refer to Troubleshooting configuration import for more information.

  8. Click the Save drop-down list and select a save mode. Standard location saves the configuration file in its default location (same as the File.Save menu item) and Export location saves it to the folder \TEMPLATES\EXPORT of the project. Show picture

If necessary, you can restore a previous configurations using the Restore drop-down: Restore from Standard location or from Export location.

Manually deleting a configuration element leads to cascading deletions due to dependencies between elements. These dependencies will not be recreated automatically by re-importing the deleted element.
