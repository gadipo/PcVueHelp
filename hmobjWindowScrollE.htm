WindowScroll Event  

 Example Applies to 

Syntax

object. WindowScroll(ByVal ScrollDirection As fvScrollDirection, ByVal ScrollRequest As fvScrollRequest,

ByVal PreviousPosition As Long, ByVal NewPosition As Long, ByVal Window As Integer)

Part

Description

ScrollDirection

Indicates for which scroll bar control the event has been fired:

  • fvScrollHorizontal: Horizontal scroll bar control
  • fvScrollVertical: Vertical scroll bar control.

ScrollRequest

Indicates the scroll action the User has requested:

  • fvScrollLine: Line scrolling.
  • fvScrollPage: Page scrolling.
  • fvScrollWheel: Mouse wheel scrolling.
  • fvScrollTrackStart: The User selected the scroll box, ready to move it. In that case the PreviousPosition and NewPosition values are both equal to the current scroll bar position.
  • fvScrollTrack: The User is moving the scroll box.
  • fvScrollTrackStop: The User released the scroll box. In that case the PreviousPosition and NewPosition values are both equal to the current scroll bar position.

PreviousPosition

Provides the previous scroll bar position, before the User scroll request has been executed. It is set with the scroll bar current position when the ScrollRequest parameter equals fvScrollTrackStart or fvScrollTrackStop.

NewPosition

Provides the new (current) scroll bar position, after the User scroll request has been executed.

Window

Provides the index of the window (in the Mimic Windows Collection) for which the event has been fired.

Each time a User scrolls the contents of a Window, the WindowScroll event is fired in both VB/VBA. That event is associated with the Mimic object to which the Window object belongs.

Note The WindowScroll event is not fired when you use the Zoom tools. (Even though the contents may be scrolled, there is no scroll request.) Use the WindowZoom event to manage the scroll bar controls while zooming.
