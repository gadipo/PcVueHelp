Map Control displays interactive geographic maps, with an overlay of animated symbols representing real world objects, in mimics. This type of system is often referred to as a Geographical Information System (or GIS). Show picture

The geographic maps are provided by third party services such as OpenStreetMap and are generated using a set of graphic images known as tiles. There are normally several sets of tiles, each covering a range of zoom levels. The higher the zoom level the more detailed are the tiles and the more tiles you need for a particular view. The tiles can either be downloaded on demand from the Internet, or they can be loaded from local storage.

The map is overlaid with configurable symbols known as markers. Markers are attached to layers and each layer is assigned to a range of zoom levels. The layers, and hence the markers, are hidden and revealed according to the map zoom level.

The Map Control layers and zoom are unrelated to a mimic's layers and zoom.

The Main Map Components

The Map facility has three main components.

  • Map Control - The special animation that enables maps can be embedded in any mimic. Maps are rendered using the World Geodetic System (WGS 84) standard. Using the Map Control the developer can configure the following features.
    • The space that the map occupies within the mimic.
    • The provider of the map tiles and if they are to be downloaded from the Internet or come from a local cache.
    • The initial position and zoom level.
    • The bounds between which the map panned and zoomed.
    • The name of the file that provides the map markers overlay.

    In addition, the Map Control supports a programmatic interface permitting manipulation by both SCADA Basic and VBA.

  • Map Markers Editor - A tool to configure the map marker overlays. Each overlay contains one or more layers to which markers are attached. The visibility of each layer (and hence the map markers) is controlled by selecting a zoom minimum and maximum. The map marker overlays are saved in a modified version of a format known as GPX.
    • Map markers - A map marker is a symbolic representation of a real world object.
      • Each map marker is attached to a layer in a map markers overlay.
      • Appearance can be selected from a shape, text, track (polyline), polygon, or a symbol.
      • Several properties of a marker can be bound to the value of a variable.
      • Can be configured so that, at run time, a left click opens a mimic.
      • Can be associated with the value of one or more other variables that are then displayed in a pop-up at run time (right click).
  • Map positioning animation - A position type animation that allows the user to position and zoom the map using a pre-configured (by the developer) set of co-ordinates.

In addition, to these three main components, a standalone tool is provided to download map tiles from the Internet and save them in a map cache for use offline (with no Internet access).

Map Storage

By default, local maps, map marker files and the map cache are saved in the GIS folder located in PcVue's root. Map files can be very large (many terabytes) and by locating them here, the same maps can be accessed by several projects minimizing storage requirements.

Scripting

The Map Control can be manipulated programmatically by both SCADA Basic and VBA.

A specific instruction, MAPDISPLAY, is provided in SCADA Basic. See the help for SCADA Basic for information on its use.
