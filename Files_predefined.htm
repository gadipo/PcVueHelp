File and File Item Properties Specific to the Predefined Format

Predefined format

The following properties are for File and File Item objects when using the predefined format. This format is to support the PcVue form controls: combo box, list box, tree view.

The tree-view is the most complex because it can manage the tree structure with hierarchy and behaves in a similar manner to the XML format.

XML is a hierarchical format but it should not be confused with the hierarchy of the AA. While in the AA there is a direct hierarchical link between parent and child (for example Building and Floor), this does not imply a direct hierarchical link between an XML element positioned on Building and an XML element positioned on Floor.

File:

There are no file properties specific to the predefined format.

File item:

Property Description Default value or example

UserData

 The value of the item's UserData field. Example: =Me.Name

Text lang 1

 The item's Text field in language 1. Example: =Me.DescriptionLang0

Text lang 2

 The item's Text field in language 2. Example: =Me.DescriptionLang1

Default selection (0 or 1)

 Set to 1 if this item must be selected by default when opening the control. Default: 0
Collapsed icon type The type of icon to display when the node is collapsed.  
Expanded icon type The type of icon to display when the node is expanded.  

Child of

 The name of the parent element. If empty, it will be the last hierarchically. See note

Animations and the tree view control:

Only the following animations are currently supported by the file item when used with a tree view.

  • Link open
  • Send program
  • Send bit
  • Send register
  • Send text
  • Send macro
  • Recipe

The animation is created as child of a file item object. If multiple animations are added to the same object, they are processed in alphabetical order. Any animations added manually using the PcVue mimic editor will be lost at the next generation.

Notes on using the ChildOf property

The ChildOf property makes it possible to have grouping that does not depend on the topological hierarchy. If using this property, it is strongly advised that all applicable elements use it. See File Example 5 - Use of the ChildOf property for examples specific to the use of the ChildOf property.

Predefined advanced format

The following properties are for File and File Item objects when using the advanced predefined format. This format is to support the Pie/Doughnut/Pyramid chart control.

File:

There are no file properties specific to the advanced predefined format.

File item:

Property Description Default value or example

Text lang 1

 The item's Text field in language 1. Example: =Me.DescriptionLang0

Text lang 2

 The item's Text field in language 2. Example: =Me.DescriptionLang1
Legend lang 1 The item's Legend field in language 1.  
Legend lang2 The item's Legend field in language 2.  
Value type The item's value type.  
Value If By value, the item's value. If By variable, the variable name.
