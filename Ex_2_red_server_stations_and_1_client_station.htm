Architecture Examples - 2 Redundant Server Stations and 1 Client Station

Project requirements

  • Station1 communicates with equipment and produces the value of variables.
  • Station2 communicates with equipment and produces the value of the same variables as Station1.
  • Station1 and Station2 are to behave as redundant servers.
  • Station3 consumes the value of variables from Association1 and displays mimics.

The project configuration is identical on all stations.

License requirements

  • Station1 requires a server license.
  • Station2 requires a server license.
  • Station3 requires a client license.

Network configuration

  • The default network configuration is used with all stations configured as both client and server and with all links between all stations made.
  • A single association, called Association11, which contains Station1 and Station2 is configured.
  • Two station lists are required.
    LASSOC11, a server list containing the name Association11.
    LCLIENT3, a client list containing the name Station3.

Variables tree configuration

  • Variables with values produced by Station1 and Station2 are allocated to server list LASSOC11 and client list LCLIENT3.
