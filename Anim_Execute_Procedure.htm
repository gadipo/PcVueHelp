Using the Execute Procedure Animation

The Execute Procedure animation executes a VBA procedure (Sub or Function) defined in a mimic's project module or in the Main project module. Show picture

VBA data types

A procedure parameter can be defined with one of the following VBA data types:

  • Byte.
  • Boolean.
  • Integer.
  • Long.
  • Single.
  • Double.
  • Date.
  • String.

The parameter value can be initialized with a text (which may represent a numeric value) or be linked to a variable value.

Object types

A procedure parameter can be defined with one of the following object types:

  • Mimic.
  • Window.
  • Graphic.

Whatever the object type, the parameter value cannot be modified, and it is automatically initialized with an instance of the object. The object instance used depends on the context in which the animation exists:

Undefined data type

VBA permits a procedure parameter to be defined without a data type. In that case, the parameter value can be initialized with a text or linked to a variable value. However, when the animation is executed and the procedure called, the parameter value is automatically converted into a value with a VBA data type:

    • Text - converted to a String value.

    • Link to a variable value - converted according to the variable type:

Register variable: a Double value.

Text variable: a String value.

Bit variable: a Boolean value.

No array of any kind is permitted.

Parameter value configuration

A parameter's value can be edited by selecting the parameter in the list. If permitted (according to the parameter type), an edit control for typing the parameter value (or linking it to a variable value) is displayed with a button to open the Variable Selector (browser):

For instance, you could cause the parameter value to be initialized:

  • By typing in a text value of 150.
  • Or by linking it to the value of a variable e.g. Cluster1:@Branch1.M1.

Whether the source of a parameter value is a text value or variable value, if the conversion of that value into the parameter type fails, the animation execution is stopped and the procedure not called.

For instance, an attempt to convert a double value of 123456.52 into an integer value will fail since that value is outside of the integer range.

Whatever the quality of a variable linked to a parameter value may be, the animation is executed and the procedure called.

Return Value type and variable

If a procedure returns a value, that value can be used (optionally) to write a variable.

The Type field displays the value type returned by the procedure. The Variable edit control is used to link a variable value to the returned value:

The only permitted value types are VBA data types :

  • Byte.
  • Boolean.
  • Integer.
  • Long.
  • Single.
  • Double.
  • Date.
  • String.

Using an unauthorized return value type does not prevent execution of the animation. In addition, whatever the quality of the variable may be, the animation is executed and the procedure is called.

When you have configured the animation:

  • Click on OK to apply the settings and close the Text Properties dialog.
