Using Time and Date Formatting When Not Using GMT0

The host operating system's time zone is taken into account when using date and time formatting. The effect of this can be seen in the following examples.

If the host operating system is set to a time zone other than GMT0 the time zone offset (in hours) is added to the formatted value. This is best explained by an example.

  • A register formatted as dd/mm/yyy hh:nn:ss@s with an input value of 1483228800 will display:
    • In a GMT0 time zone - 01/01/2017 00:00:00.
    • In a GMT+1 time zone - 01/01/2017 01:00:00
  • A register formatted as hh:nn:ss@s with an input value of 3600 will display:
    • In a GMT0 time zone - 01:00:00.
    • In a GMT+1 time zone - 02:00:00.

This has an unexpected effect if you want to enter a time, including hours, on a non-GMT0 system. You will find that you are unable to enter a time less than the time zone offset. For example, for GMT+1 you cannot enter a time less than 01:00:00. The solution is to configure the hours as a duration rather than a time. That is, enclose the hours formatting character in square brackets. For example, with a received value of seconds the required format is: [hh]:nn:ss@s-[hh]:nn:ss.
