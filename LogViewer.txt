TOPIC: LogViewer_Advanced.htm
========================================
Configuring advanced behavior
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring Log Viewer Advanced Behavior 
 
The Log Viewer advanced behavior is configured using the properties found in the Advanced tab. Show picture

 
Configuring the time order in which events are displayed
 
By default, when the Log Viewer is opened, events are sorted according to their timestamp. Using the Display options in the Advanced tab you can select if the newest events are at the bottom (default) or the top.
 

 
If Enable sorting and Sort modification in the Execution tab are selected any sorting configured at run time by the User takes priority.
 
 
Using advanced color mode
 
If Advanced color mode is selected both the stroke and background color for each event type, as defined in Color Preferences, are used. Normally the background color is not used, the background color being that of the Log Viewer.
 
Configuring an automatic refresh
 
In versions of PcVue prior to 11, the Log Viewer refreshed only as a result of the User navigating using the scroll bar. By entering a Refresh period other than 0 the Log Viewer automatically requests the most recent records from the archive unit. In order not to overload the archive mechanism, the minimum Refresh period is 5 seconds.
 
If the property Suspend automatic refresh when navigating is selected then the refresh is suspended when the scroll bar is used to navigate through the display. Automatic refresh remains suspended until the Automatic refresh tool on the toolbar is clicked. As each refresh also resets the display and scroll bar to the "when first opened" position, it is highly recommended this property is always selected when using automatic refresh.
 
The property Display icon when automatic refresh is suspended displays an icon in the middle of the display list when refresh is suspended thereby making it more obvious to the User.
 

 
Prior to version 11 the Log Viewer could only be refreshed using SCADA Basic but the refresh could be clearly seen. The refresh using the built in facility is seamless..
 
 
Limiting the period for which historical requests are made
 
You can limit the period for which historical requests are made by using the Time buffer maximum length when requesting option. When this option is configured, the records retrieved are limited by their age in addition to the limitation imposed by the Display buffer property (execution tab). The limit remains active unless the Time Filter tool is used and a From time and To time are entered.
 
The period is configured as a number (1 to 3600) of time units (Seconds, Minutes, Hours, or Days). The default time unit of None means there is no time limit.

Using the context menu
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Using the Log Viewer Context Menu
 
The Log Viewer context menu is available if the property Use context menu has been enabled in the Log Viewer's configuration. It is opened by right clicking anywhere in the display area. The available tools depend on the configuration of the log viewer (Menu tab) and if one or more log events are selected. Commands available in the context menu correspond to those available in the toolbar, and have the same behavior.
 
The following screen shot shows all available tools. Show picture
 

 
 
 
The tools are divided in 3 sections:
 
 
Filtering - The 1st section comprises buttons to change the log display filter for alarm transitions. Any change in the filter applies to the log viewer as a whole.
 
Filtering - The 2nd section comprises button to change the log display filter for user actions and bit variable transitions: Commands, Acknowledgment, masking, log to 0, log to 1...
 
Commands - The 3rd section comprises general command buttons: To change the time filter, print selected log events, change the blinking mode and activate/de-activate the auto-refresh.
 
 
Interaction with other Log Viewer actions
 
There is one other form of action from the Log Viewer triggered by clicking with the mouse. How this behaves will depend on how Mouse inputs have been configured in the HMI General Options. 
 
 
Context mimic - If the context mimic action is configured for a right mouse click, and the context menu is enabled, on a right mouse click the name of the context mimic appears in the context menu above the other tools.
 
 

 
The context mimic and the context menu are two entirely separate features other than the interaction mentioned noted above.

Creating a Log Viewer
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Creating a Log Viewer and Configuring the General Features
 
The Log Viewer is flexible in both its appearance 
 and operation and as a result, there are many configurable 
 properties. The procedure for creating a Log Viewer follows.
 
 
Open the mimic in 
 which the Log Viewer will be inserted and select Edit mode.
 
Select the Log Viewer 
 tool from either the animation toolbar or the menu.
 
A Log Viewer will 
 be created and its properties box automatically opened and showing the Filters tab.
 
 
Once you have created a Log Viewer, 
 and closed its properties box, it can be moved, sized and copied the same 
 as any other drawing element.
 
The Log Viewer configuration dialog contains six tabs for the various aspects of the configuration. Show picture
 

 

 
 

 
 
 
Display - General display features. See also the topic Configuring the line format.
 
Aspect - General appearance - the same as the Aspect tab of all drawing elements.
 
Execution - Properties that select the tools available to the User at run-time. See the topic Configuring the run-time behavior.
 
Filters - Selection of the source (Log List) of the data the Log Viewer is to display. See the topic Configuring the filter.
 
Advanced - Selection of how log data is sorted according to its timestamp. Newest data at the top or bottom. See the topic Advanced behavior.
 
Menu - Configuration of the run-time context menu. See the topic Configuring the context menu.
 
 
Configuring the general display features
 
 
Toolbar and menu - The appearance and location of the toolbar at run-time.
A drop-down list box is used to select the size of the icons on the toolbar. The space that the toolbar occupies depends on selected size. If None is selected the toolbar is not displayed.
The position and configuration of the toolbar, but not the size of the icons, can be changed by the User at run-time unless you specifically disable that capability in the Execution tab.
 
Use context menu - Enable the run-time context menu.
 
Scrollbar - The availability and width of the vertical and horizontal scroll bars. The default setting of a vertical scroll bar only is used in most applications.
 
Background - The background color of the display.
 
Alternative color of text - The alternative text color for projects configured with legacy versions of PcVue that did not have the capability to select the two colors for a blinking event.
 
Without blinking - The blinking is turned off for any logs that have been configured with a blinking color (for example alarms that are not acknowledged). The text is displayed in the main color only.
 
 

 
The ability to define the size of the icons and the width of the scrollbars allows the application designer to adapt the viewer if touch screens are used at runtime.
 
 

 
You can specify more properties to control the graphical element visibility. From the Aspect tab, behind the down arrow at the bottom of the Properties dialog box:
 
 

 
Specify zoom visibility bounds so that the graphic control is only visible for predefined zoom levels.
 
 

 
Specify layers so that the graphic control is only visible if one of the selected layers is displayed.
 
 
 
These are even more useful when combined with user permissions and decluttering templates.

Log Viewer overview
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Log Viewer Overview
 
The Log Viewer is a control that displays 
 the data that has been archived through a single Log List. You can display 
 all of the data from the Log List, or a sub-set of it. A Log Viewer can only display 
 data from one Log List at a time. 
 Any number of Log Viewers can be embedded in a mimic, the only limitation 
 being the amount of physical space and memory available.
 
By selecting the type of data recorded 
 by a Log List, you can easily create historic alarm lists, process event lists and operator action lists. For more information, on how the Log Lists 
 are configured see the book about Archives. Show picture
 

 

 
 

 
 
Summary of Features
 
 
Log Viewers can 
 be embedded in any mimic.
 
Unlimited number 
 of Log Viewers in each project.
 
The size of Log Viewer 
 is configurable from 1 line to the maximum allowed by the screen resolution.
 
Log events can be 
 selected from: Alarm changes of state, Bit changes of state and Operator 
 actions.
 
Data can be filtered 
 by Domain and Nature.
 
Optional scroll bar.
 
Optional Toolbar and context menu to filter 
 which log events are displayed.
 
The format of the log lines 
 in the display list is 100% configurable.
 
Can be configured so that the newest logs appear at the bottom, or at the top, of the list.
 
Can be scripted by 
 SCADA Basic for more advanced features.
 
Can have a context mimic. For details, see the topic Opening a Context Mimic Using a Mouse Button.

Overview of the Log Viewer at Run-time
 
A Log Viewer displays data that has been recorded through 
 a particular Log List. The Log List is attached to the Log Viewer 
 when the Log Viewer is configured. The types of data that can be displayed 
 are as follows :
 
 
Alarm changes of 
 state.
 
 
On and not acknowledged.
 
On and acknowledged.
 
Off and not acknowledged.
 
Off.
 
Unavailable.
 
 
Bit changes of state.
 
 
Change to zero.
 
Change to one.
 
Change to invalid.
 
 
Operator actions.
 
 
Commands.
 
 
 
Logon and Logoff.
 
 
 
Alarm acknowledgement.
 
 
 
Alarm masking.
 
 
 
Program start.
 
 
 
By using different Log Lists, Log Viewers can be configured 
 to create historic alarms lists, process event logs and operator action 
 logs. The Domain and Nature options may be used to configure different 
 Log Viewers for different process areas.
 
When a Log Viewer is opened, log data is extracted from 
 the archive files through the Log List and saved temporarily in the 
 display buffer. Data is extracted for the current time backwards until 
 the display buffer is full or there is no more data. Data is sorted for display according to the sort column that was chosen for sorting, by default the date and time.
 
Selecting Logs
 
Individual logs are selected by clicking on the line 
 in the display list. A selected log is indicated 
 by displaying it in reverse video. Selected logs can be printed using the Print tool on the toolbar or context menu, and copied to the Windows clipboard using the Ctrl+C keyboard shortcut.

Size of Columns - If selected, the User can change the width of columns at run-time by clicking and dragging with on the header.
 
Toolbar Position - If selected the User can change the toolbar position at run-time.
 
Toolbar Modification - If selected, the User can change the toolbar configuration at run-time.

Sort modification - If selected the User can change the sort column and direction.

If you want to change the column width or modify the toolbar, but do not want the user to do so, enable the properties and make the changes at run-time and then disable the properties again. The changes to the display will be saved.
 
 
Changing the way that the list is sorted
 
To permanently sort the Log Viewer by a particular column and direction:
 
 
In design mode, tick both the Enable Sorting and Sort modification properties.
 
Switch to run mode and click the header of the column to be used for sorting. Clicking more than once will toggle between ascending and descending.
 
Switch back to design mode and un-tick Sort modification.
 
Save the changes.
 
 
To allow the User to change the column and direction for the sort:
 
 
In design mode, tick both the Enable Sorting and Sort modification properties.
 
Save the changes.

Sorting the contents of the Log Viewer by column
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Sorting the Contents of the Log Viewer by Column
 
The Log Viewer contents can be sorted order according to the contents of any column. By default, it is sorted according to the date and time with either the most recent logs at the top, or the bottom, of the list. If configured to permit it, the sort direction and/or the column on which the sort is made can be changed at run-time by clicking on the column header. 
 
The sorting is intelligent and depends on the contents of the sorted column.
 
 
A column containing numeric data only will cause a numeric sort. For example, using ascending sorting, 2 would be above 3. Show picture
 
A column containing alphanumeric data will cause a character sort. For example, using ascending sorting, 20 RED CARS, would be above 3 GREEN CARS. The first character 2, comes before 3 in the ASCII character set. Show picture
 
A column containing timestamp (time and/or date) information will cause a time/date sort. For example, using an ascending sort 12:08:37 is above 12:08:43. Show picture
 
 

 

 
 
Ascending sort on a numeric column
 
 

 

 
 
Ascending sort on an alphanumeric column
 
 

 

 
 
Ascending sort of a time/date column
 
 

 
By ascending, we mean values get greater from the top to the bottom of the list.
 
 
Understanding the effect of a historical request on the sorting
 
When a Log Viewer makes a historical request, it asks for records according to their timestamp. For example, when the container mimic is opened it will request Buffer Size older records back from the current time. Only when the historic data arrives at the Log Viewer is it then sorted, for the display, according to the sort criteria that is active. It is important to understand the effect of this.
 
For example,: The User has sorted the display according to the name of the variable and, because there are a lot of logs for a particular variable in the current display buffer, only logs for that one variable are visible. The User then makes a historical request. The new records that arrive at the Log Viewer do not include any for the variable that was previously visible in the display. The records are still sorted to the same criteria, but there will be none visible for the variable that previously occupied the display.
 
To summarize - A historical request is always made according to timestamp NOT according to the sort criteria.
 
 

 
<Buffer Size> is the number of lines specified by the Display Buffer property of the 
 Log Viewer's configuration. Not the number of visible lines.
 
 
Understanding the effect of using field equipment that supports timestamping
 
Both Proprietary and ODBC archive units record two timestamps for each record.
 
 
The time of PcVue when the record was actually made.
 
The time from the field equipment that is the source of the event - for example a PLC.
 
 
Most traditional types of field devices, (PLC, I/O etc.), do not support timestamping. In this case, PcVue's time is used for both timestamps. However, for equipment that does support timestamping it means that the two timestamps in each record could be different. It is important to understand the effect this has on the Log Viewer.
 
When the Log Viewer makes a historical request, the timestamp in the request is matched with that of PcVue in each record. However, when the historic data arrives at the Log Viewer, it is the Equipment timestamp that is used to sort the data and also used in the display. The effect of this is that you could see data in the Log Viewer with a timestamp that is apparently outside of that requested.
 
In the following screen shot, the Historical Request tool of the Log Viewer is used to make a request for records between 17:00:00 and 17:05:00. However, Log Viewer has actually displayed some records that have, apparently, an older timestamp than that. Show picture
