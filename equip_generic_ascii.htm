Generic Ascii

This driver supports sending ASCII requests and receiving ASCII responses over serial communication.

It has only one type of frame, designed to map Text variables.

It can send ASCII requests in two ways:

  • Cyclic reading - The value of the mapped variable is the ASCII string received on the serial port.

  • One-time writing - The value set for the mapped text variable is sent on the serial port.

Frame addressing

Depending on the equipment type you have selected when configuring the device, you have access to the following type of frames.

The Starting address and Quantity properties define the memory address range for each frame, how it is read from and written to, and how it is interpreted in PcVue.

Data type Access Max quantity Starting address Description
Byte
Message

Read/Write

 2048

0..65535

A text variable of length 2048 mapped on address 0 will contain the full reply payload.

If only a part of the reply payload is of interest, you can map a text variable of the desired length starting at the offset corresponding to the first interesting character.

A frame for this driver has the following specific advanced properties:

Property Description
Characteristics
Read request payload The payload sent cyclically by the driver in read requests.
Append CR Used to add a Carriage Return character at the end of a write request.
Append LF Used to add a Line Feed character at the end of a write request.

Driver status

The driver status provides driver-specific information to supplement the frame general status. For more information on status in general, refer to the topic General communication status.

Driver status Description
01 Timeout reading or writing
02 On a write request with Write reply different from write request, the reply to the write request is not echoed
03 The message in the read request (as configured on the frame) exceeds 255 characters
