The WebVue Workspace

Toolbar

Click and drag to move, or double click to dock, the toolbar.

Reload WebVue. Can be used to restart WebVue after a disconnection or to return to the first mimic with a single click.

Show the settings view. Note that the settings are saved in the browser itself and are therefore common to different tabs, and instances, of the same browser.

  • Do not fit rendering to the web browser viewport - Disable the automatic scaling of mimics that may be imposed by default by the Web back end.
  • Automatically hide the toolbar when docked - The system menu is hidden if it is docked at the top. Move the cursor to the top to show the menu again.

  • Keep WebGL buffer for printing - For Firefox users only. Enable this setting if you face issues when printing a Trend viewer with Firefox (black area instead of curves). Because this setting can have a significant impact on performance, it is recommended to activate it only if necessary.

  • UI scaling - Selection of the scaling factor to apply:

    • Auto - Apply the UI scaling settings of the operating system (at least on Microsoft Windows and Apple macOS).

    • 100% - Override the system UI scaling settings to favor a rendering at the scale mimics were designed on the PcVue desktop application. On 2k or 4k screens, mimics may become too small.

    • 125% - 175% - Similar behavior as 100%, mimics are just rendered a bit...larger.
Show the window list. Clicking on a window name will move it to the foreground or restore it if it has been minimized.
Show the WebVue event viewer.

Opens the SimuLock view.

The SimuLock view shows the list of variables used as part of animations in the current mimic.
It provides variable filtering and allows the user to control variable simulation:

  • Set and Reset variable value simulation

  • Set simulation value

  • Set simulation expiration time

The exact behavior of the SimuLock view depends on variable simulation properties. Refer to the topic Properties common to all variable types for more information.

By making it easy to set the value of any variable, the SimuLock view can be useful for mimic testing.
Full screen mode. The same as if using the F11 keystroke.
Logout.

Opening WebVue in additional browser tabs, or another instance of the same browser

A successful login to WebVue starts a WebVue session. In general, further copies of WebVue using the same Url in additional tabs in the same browser instance, or in further instances of the same browser on the same computer, use the same session. One WebVue session uses one WebVue license.

  • You do not need to login when starting WebVue in each additional tab.
  • Logging out of WebVue, on any tab or browser instance on the same computer, logs you out of the WebVue session and hence logs you out of WebVue in all tabs or browser instances.

This behavior depends on the way that a web browser manages session cookies and may be different in different web browsers or in future versions of a web browser.

Display language and Project language

The Display language is taken from the web browser language. If not suitable, users can always change the order of priority at the web browser level to ensure they are displayed the WebVue generic text in the language they prefer. This affects the following elements:

  • Login page, Change password page
  • Messages related to connection status and failure
  • The workspace toolbar
  • Setting dialog etc.

The Project language is taken from the Web back end and user profile configuration and cannot be switched from the WebVue client.
