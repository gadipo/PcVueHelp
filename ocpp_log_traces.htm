OCPP Add-on traces

Event logging

Information regarding the OCPP Add-on is logged in the traces and displayed in the Event Viewer with lines starting with '[OCPP]'.

Diagnostic traces

Additional traces can be activated using the SCADA Basic instruction TRACE as follows.

TRACE(Mode, ManagerId, HexFlag);

  • Mode - Turn the additional trace on or off
    • 1 - On
    • 0 - Off
  • ManagerID - The identifier of the OCPP Add-on task. Set to 11 if OCPP is the only Add-on in use.
  • HexFlag - Hexadecimal combination flag. The level of traces to activate/deactivate as per the table below.
Type Bit mask
Raw message exchange with charge points 0x0001
Raw message exchange with CSMS (proxy) 0x0002
Incoming requests from charge points 0x0400
Outgoing requests to charge points 0x0800
Incoming responses from charge points 0x1000
Outgoing responses to charge points 0x2000
Incoming requests from CSMS (proxy) 0x0800
Outgoing responses to CSMS (proxy) 0x10000
Incoming responses from CSMS (proxy) 0x20000

Example of use of SCADA Basic TRACE instruction

TRACE(1,11,”001”); ‘Turn on additional trace for raw message exchange with charge points

TRACE(1,11,”400”); ‘Turn on additional trace for incoming requests from charge points

TRACE(1,11,”1000”); ‘Turn on additional trace for incoming responses from charge points

 

TRACE(0,11,”001”); ‘Turn off additional trace for raw message exchange with charge points

TRACE(0,11,”400”); ‘Turn off additional trace for incoming requests from charge points
