3D Transformations

Geometrical transformation is expressed in terms of 3 kinds of motion - Translation, Rotation and Scaling.

Translation

A translation is a transformation that changes the position of an object without changing its orientation or size. It is described by a 3D vector (a direction in space), that can be measured along the 3 axes: X, Y and Z. Show picture

Rotation

Rotation is a transformation that changes an objects orientation. Rotation can be described by a number of methods, two of the most common of which are supported by the HMI.

  • Euler XYZ angles - This method is the easiest to understand but has major limitations and is not contiguous. It is described by three angle one for each of the X, Y and Z axis. The axis are sometimes referred to as yaw, pitch and roll. Show picture

  • Rotational vectors - This is the most universal and best representation, although not the easiest to understand. The axis is a 3D vector and the angle is given about that axis. Rotational vectors are described using X, Y and Z values (for the vector) and a single angle.

Scaling

Scaling is a transformation that resizes (scales) the object. The HMI uses 3 coefficients (or ratios) to describe the scaling along the local X, Y and Z axes of the objects. Local scaling is used to avoid any skewing. Show picture

How transformation is specified

Transformation may be specified in two ways.

  • World transformation - The change in an objects origin is with respect to the world space origin that is common to all objects in a scene. In the case of the HMI, the world space is an entire 3D mimic.
  • Local transformation - The change in an objects origin is with respect to the parent objects origin.

The following diagram illustrates the relationship between the world coordinate system and an object's local coordinate system. Show picture

In a parent-child hierarchy, the transformation of the parent is always applied to the objects it contains. For a top-level object (no parent), its world transformation is identical to its local transformation. When editing a child object, you will see its local transformation information (relative to its parent).

In terms of the HMI, a 3D model is normally a top-level object and hence uses world transformation. The one exception is when a model has been incorporated in a symbol in which case the symbol is the top level object and the parent of the 3D model. A frame within a model is a child of either the model or another frame and hence uses local transformation.
