Configuring a message template
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring a Message Template
 
A message template allows you to predefine some or all 
 of the fields that are used when configuring an event to send a message. When configuring 
 the event you select the message template and it automatically 
 completes the fields with the information you have defined. 
 
Field options
 
Each field in a message template has the following options selected by a button marked with a forward arrow adjacent to the field.
 
 
Not defined by the 
 template - The template does not provide information for that field. This is the default setting.
 
Imposed by the template 
 - The text that you enter will be used in any message where the template 
 is used and cannot be changed there.
 
Suggested by the template - The text that you enter will be used in any message where the template is used but can be edited there.
 
 
How to create a message template
 
 
Select the Message templates folder in the Application Explorer configuration tree and, from the task list, select Add a message template. A dialog to configure the message template will open. Show picture
 

 

 
 

 
 
Enter a name for the template. You can either accept the suggested name or type 
 one of your own.
 
Enter a description of the template. The description appears in the contents pane of the Application Explorer and can be useful to identify the template later. The description is optional.
 
If the message is to be sent by e-mail, use the drop-down field E-mail profile to select a profile.
 
If the message is to be sent by SMS, use the drop-down field Telephony profile name for SMS to select a profile.
 
Complete the remaining fields in the Message, E-mail and SMS tabs as required. For information on the fields, see the following topics.
Message tab - How to configure an event to send a message and Using substitution strings with messages
E-mail tab - Configuring e-mail options.
SMS tab - Configuring SMS options.

 
Select Imposed by the template or Suggested by the 
 template for each completed field as appropriate.
 
Click on OK to confirm the settings and close the dialog.
 
 

 
Leave blank any fields that are to be determined solely via the message itself.

Configuring an e-mail profile
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring an E-mail Profile
 
An e-mail profile contains information about the account to be used by PcVue when sending an e-mail. The account 
 supplies information such as the SMTP server name and is similar to the account configuration for e-mail software such as Microsoft Outlook Express. In this version, there is one account per profile. 
 

 
Provision has been made for future versions to have multiple accounts selected in a priority order.
 
 
How to configure an e-mail profile
 
 
Creating the profile.
Select the E-mail profiles folder in the Application Explorer configuration tree and, from the task list, select Add a new e-mail profile. A dialog to configure the new e-mail profile will open. Show picture

Enter a name for 
 the profile. You can either accept the suggested name or type 
 in one of your own.
Enter a description of the profile. The description appears in the contents pane of the Application Explorer and can be useful to identify the profile later. The description is optional.
Click the OK 
 button to confirm the creation of the profile. The E-mail profile dialog 
 closes and new dialog opens from where the e-mail account is configured.
 
Configuring the account - E-mail Account dialog, General tab. Show picture 

Enter a name for 
 the account. You can either accept the suggested name or type 
 in one of your own.
Enter a description of the account. The description appears in the contents pane of the Application Explorer and can be useful to identify the account later. The description is optional.
Enter a valid 
 address in the E-mail 
 address field. This will appear in the From 
 field of the recipient's e-mail client and is mandatory.
Enter a display name. If configured, this will be displayed in 
 the From 
 field of the recipient's message client instead of the source e-mail address. The Display name is optional.
Enter a valid message 
 address in the Reply-to 
 address field. This will 
 be used by the recipient's message client if the recipient chooses to reply 
 to the message. The Reply-to address is optional.

 
Configuring the outgoing server - E-mail Account dialog, Outgoing Server tab. Show picture 

Enter the name of 
 your outgoing (SMTP) mail server in the Server 
 name field.
Enter the outgoing 
 mail server port number in the Port 
 number field. The default is 25.
Select the delivery method, either From IIS directory if using a local IIS server, or Network if using an external SMTP server (default).
If your server requires 
 a secure connection, select the secure transport it supports, either Explicit TLS or Implicit TLS. If it does not require a secure connection, select Never.
If your server requires 
 authentication tick the Authentication 
 box and enter a user name and password. 
 
Click OK 
 to confirm the configuration and close the dialog.
 
 

 
The Priority property in the General tab is not used in this version.
 
 

 
Explicit TLS usually requires the use of port 587 and Implicit TLS port 465. Please check with your service provider to select the appropriate secure connection type and associated port.

Configuring an SMS profile
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Configuring an SMS Profile
 
An SMS profile contains information about the account to be used by PcVue when sending an SMS. The account 
 supplies information about the telephony modem to be used for sending short messages. In this version, there is one account per profile. 
 

 
Provision has been made for future versions to have multiple accounts selected in a priority order.
 
 
How to configure an SMS profile
 
 
Creating the profile.
Select the SMS profiles folder in the Application Explorer configuration tree and, from the task list, select Add a new SMS profile. A dialog to configure the new SMS profile will open. Show picture

Enter a name for 
 the profile. You can either accept the suggested name or type 
 in one of your own.
Enter a description of the profile. The description appears in the contents pane of the Application Explorer and can be useful to identify the profile later. The description is optional.
Click the OK 
 button to confirm the creation of the profile. The SMS profile dialog 
 closes and new dialog opens from where the SMS account is configured.
 
Configuring the account - SMS Account dialog, General tab. Show picture 

Enter a name for 
 the account. You can either accept the suggested name or type 
 in one of your own.
Enter a description of the account. The description appears in the contents pane of the Application Explorer and can be useful to identify the account later. The description is optional.

 
Configuring the modem connection - SMS Account dialog, Advanced tab. Show picture 

Configure the modem's connection to the PC. The defaults are: Port name Empty, Baud rate 115,200 bps, Parity None, Data bits 8, Stop bits One, Flow control Wavecom: Fastrack Supreme 20, Timeout 10,000 seconds.
Enter the personal identification number (PIN) for the subscriber identity module (SIM) card to give PcVue access to a mobile telephony network. Use the default of 0000 if no PIN is required. 
 
Click OK 
 to confirm the configuration and close the dialog.
 
 
Checking an SMS account
 
You can check the operation of an SMS account and its corresponding modem by selecting the account in the Application Explorer's configuration tree and then clicking the Check SMS account task. A dialog opens from where you can enter a recipient's telephone number, subject and message. When the Check button is sent, PcVue will attempt to send the message and the return status is displayed in the dialog. Show picture

 

 
This checks message sending, not the actual transmission or delivery. For the latter you will need to check the SMS-enabled device to which the message was addressed.

Message Profile and Templates Overview
 

 
 
 
 
 
 
 
 
 
 
 
Message Profile and Templates Overview
 
Message profiles and templates are used when sending messages using an event, SCADA Basic code or VBA code. 
 
 
A profile provides information about the user accounts and the servers to be used. 
 
A template pre-configures some of the properties and contents of a message.
