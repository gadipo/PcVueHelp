IEC 61850 Data Report - Advanced Tab

The Data Report advanced properties are found in the Advanced tab of the Data Report properties dialog. The majority of the properties are directly related to the operation of the Data Report and should only be changed if you fully understand their use and the consequences of changing them. The default values will be adequate for most applications. Show picture

  • Report properties
    • Buffer time (BufTm)
      • Send buffer time to the device - Tick to enable writing the buffer time parameter to the physical device.
      • Buffer time (ms) - Specifies the time interval for the buffering of changes at the physical device level, for inclusion into a single report. See the document IEC61850-7-2 for more information.
    • DataSet (DatSet)

      • Send dataset configuration to the device - Tick to enable writing the dataset parameter to the device. It must be enable if using a dynamic dataset.

      • Original - Tick to select the datasets associated with the Report Control Block at the physical device level.

      • Dynamic - Tick to select the dataset groups linked to a dynamic dataset.

    • General Interrogation (GI)

      • Send GI after enable report - Tick to enable sending a GI after the enable report command.

    • Integrity period (IntgPd)
      • Send integrity period to device - Tick to enable writing the integrity period to the physical device.
      • Integrity period - An interval at which a report will be sent independent of the trigger conditions. Used to maintain a keep-alive with PcVue even if no data is to be reported (the trigger conditions are not met). If set to 0 then it is disabled.
    • Optional fields (OptFlds) - The default settings should not be changed unless advised to do so by the technical support team. Changing them can affect the identification of data and may lead to incorrect variable updates if the configuration of the physical device does not match that of PcVue (including referenced SCL files not matching physical device configuration).
      • Send optional fields to device - Tick to enable writing the optional fields to the device.
      • Optional fields
        • sequence-number - If ticked, the property SqNum is included in the report.
        • report-timestamp - Not used.
        • reason-for inclusion - If ticked, the property ReasonCode is included in the report.
        • data-set-name - If ticked, the property Data-set-name is included in the report.
        • data-reference - If ticked, the property DatSet is included in the report.
        • conf-revision - If ticked, the property ConfRev is included in the report
    • Report identifier (RptID)
      • Send report identifier to the device - Tick to enable writing the buffer time parameters to the physical device. The attribute RptID is the client-specified report identifier of the report that has caused the generation of the report.
    • Reserve URCB (Resv)
      • Use the reservation function - If enabled, PcVue takes an exclusive access to the Report Control Block. This property only applies to un-buffered RCB. For buffered RCB, reservation is used if the device supports it.
    • Trigger options (TrgOps) - The conditions under which the report is sent to PcVue. The default setting will be adequate for most applications.
      • Send trigger options to the device - Tick to enable writing the trigger options parameters to the physical device.
      • Trigger options
        • data-change - Generate a report when a data value changes.
        • quality-change - Generate a report when a data quality changes.
        • data-update - Generate a report on a data update (timestamp change).
        • integrity-period - Generate the report periodically using the Integrity period.
        • general-interrogation - Generate the report whenever PcVue sends a GI (General Interrogation) request. Used by PcVue when starting the network or device to get the initial value of all variables.
  • Miscellaneous
    • Reconnection polling period - The client response when a URCB/BRCB is already reserved. Default 5 sec. Upon connection, the URCB initialization phase includes an attempt to write the RptEna attribute to False and then to True. If the URCB is already reserved, the attempt fails, and the Client tries to set the RptEna attribute to False on a timer, by default every 5 seconds (configurable). Upon the first error during this phase, variables are set invalid (NSCOM) immediately.
    • General interrogation polling period - The period at which the general interrogation request is made. Default 0 sec (no GI except for initialization).
