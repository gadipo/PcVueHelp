Alarm notifications
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Alarm Notifications
 
The Alarm Notifications mechanism runs as a background service and, once the TouchVue app has been started, it runs even if the TouchVue app is closed. The service polls the server for alarms at the rate configured in the Settings view. Notified alarms appear in the notification area of the mobile device. Show picture

 
Tapping a notification opens the TouchVue app and displays the Inbox view. 
 

 
Not all versions of Android support TouchVue notifications. For information on which platforms are supported by TouchVue, see the topics Operating System and PC Requirements or the readme file.

Browsing view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Browsing View
 
The Browsing view allows you to navigate the PcVue project variables starting at the root. Show pictures
 

 &#160;&#160;
 
Screen features
 
 
Displays one branch at a time from the variables tree starting at the root and ending with the variables for the chosen branch. 
 
Only variables that match the user's Browse Level (or levels) are displayed. Branches that do not contain any matching variables are not displayed.
 
Branches and variables are designated by their Description. The Name is displayed if the Description is not present in the PcVue project configuration . The use of the Description property for branches and variables is normally preferential, as it does not have the constraints that affect a variable's Name.
 
Branches and variables (register, text, bit and alarm) are clearly differentiated with different icons.
 
Tapping a branch will navigate to the next level of that branch which may contain more branches, variables or a mixture of both.
 
Tapping a variable's name will open the Variable Details view for that variable. Which details are displayed depends on the configuration of the Mobile Server in PcVue.
 
Tapping a variable's icon will select the variable. The menu actions can then be used on that variable.
 
Tapping the trend icon, adjacent to any register variable, adds it to the Trend view, and the icon color changes to red.
 
Variables that have been added to the Watch List are indicated with a star adjacent to their icon.
 
For branches and / or variables that have an associated mimic or symbol, an additional view, Visual, is available. See the section below.
 
 
Menu actions
 
Menu actions are applied to all selected variables.
 

 - Watch selected variables.
 

 - Unwatch selected variables.
 

 - Select or un-select all variables.
 
Visual tab
 
If the configuration is based on templates with the Application Architect, and there is a mimic and / or symbol associated with the instance that generated the branch or variable, then you can display the mimic or symbol by tapping the Visual tab. Show pictures

Inbox view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Inbox View
 
The Inbox view displays a list of the most recent 50 alarm notifications. The notifications are stored locally on the device and are displayed even if the device is not connected to the Web Server. Show picture

 
Alarms can have either a read or unread status. Unread alarms are displayed in a bold font and a count of the unread alarms is displayed at the top of the view.
 

 
Newly received notifications whilst the Inbox view is opened are not added immediately. However, the Inbox can be refreshed using a down swipe.
 
 
How to mark an alarm as read
 
An alarm can be marked as read in two ways:
 
 
By displaying the Details view, for an alarm, by tapping the alarm name.
 
By selecting one or more alarms, by tapping the icon adjacent to the alarm name, and using the Mark As Read command on the toolbar at the top of the view.

TouchVue overview
 

 
 
 
 
 
 
 
 
 
 
 
 
 
TouchVue Overview
 
TouchVue is a mobile application that displays information about PcVue variables, and alarm notifications, on a mobile device such as a smart phone or tablet. The information is supplied by the PcVue Web & mobile back end server. The variables that are eligible for display are primarily selected by matching their Browse Level to the Browse Level configured in the profile of the TouchVue user. Show picture

 
See Web and Mobile Client Applications Overview for an overview of Web & Mobile client applications, including their common requirements and common deployment steps.
 
The configuration steps specific for TouchVue access to PcVue are as follows:
 
 
Make sure you have properly configured the Mobile server settings available in the Application Explorer. It includes the variable and alarm properties that should be displayed in the TouchVue App detailed views, and the default filter for alarm notification,
 
Make sure the variable Browse level matches your need, so that the proper variables and alarms are accessible to the intended users via TouchVue,
 
Have users install the TouchVue mobile application on their mobile device.
 
 
This book includes detailed information about how to design your project so that users can successfully take advantage of the TouchVue mobile application:
 
 
For information about developing a project for TouchVue, see the book Mobile server.
 
For information about TouchVue user configuration, see the topic Properties that affect access to Web Apps and Web Services in the Application Explorer book and Configuring mobile applications users.
 
For information about using TouchVue, see the book Starting TouchVue.
 
 
Summary of the main TouchVue views
 
The views are all accessed from the drawer menu opened by a swipe from the left border of the screen, or from the menu tool found at the top left of most views. 
 
 
Inbox - List of recent alarm notifications from all connected accounts. See the Inbox view topic.
 
Watch list - List of watched variables from all connected accounts. See the Watch list view topic.
 
For each configured account
Browsing - The root of PcVue variables, from where you can navigate to the views that display the variables lists and individual variable details for the account. See the Browsing view topic.
Watch list - A list of the watched variables for the account. See the Watch list view topic.
Alarmlist - A list of current alarms for the account. See the Alarm list view topic.
 
Logs - Selection of a log list and the display of the events it has recorded. See the Logs view topic.
 
Settings - TouchVue user configurable settings. See the Settings view topic.
 
Accounts - Configure user accounts and display their status. See the Accounts view topic.
 
 
TouchVue makes use of the Notification Area to notify the user of the status of selected alarms. 
 
Licensing TouchVue
 
The components necessary to run TouchVue are included 
 on the PcVue distribution media or the Windows distribution media. 
 
The maximum number of simultaneous TouchVue sessions is license-protected.
 
Terminology
 
For readers not familiar with mobile platforms terminology here are some of the terms in common use:
 
 
View - The term used for a graphic display page occupying the entire screen.
 
Notification area - A small area, not used by any app's views, reserved for notifications requiring the user's attention. Normally at the top of a smart phone or the bottom of a tablet.
 
Gestures - The finger movements to interact with a touch screen:
Touch or Tap - Touch the screen Press and lift immediately.
Long touch or Long tap - Touch the screen, wait, lift.
Swipe - Touch the screen, move, lift.
Drag - Long touch, move, lift.
Double touch or Double tap - Two touches in quick succession.
Pinch open - 2-finger touch, move outwards, lift.
Pinch close - 2-finger touch, move inwards, lift.
 
Drawer menu - A menu generally opened by a left swipe. Known as a drawer because of the way in which it opens.

Settings view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
TouchVue Settings View
 
The Settings view is used to manage the app's general settings. The settings are divided into two categories, General Show picture and Alarm notifications Show picture. The settings are persistent.
 
General settings
 
 
Time filter properties
Log lists - The default time period for the List of logged events view.
Trends - The default time period for the Trend view.
 
Settings
Variables refresh interval - The period, in seconds, at which variable values are polled when the Watchlist view is open.
 
Theme
Theme color - The main app theme color, selected from a predefined palette. The default is red.
 
Audit
Diagnostic - Enable access to diagnostic to assist with troubleshooting, for example network name on the connection view.
 
Version info - The version of the installed TouchVue.
 
 
Alarm notification settings
 
 
Notifications

Mute alarm notifications - Mute, or not, alarm notifications. The default is Off (no muting).
Start the alarm notification service at boot time (Android app only) - Start the alarm notification service at device boot time. This means that, once logged in, alarm notifications will be displayed, even after a device restart, without the need to restart the app. The default is On.
 
Alarm properties

Change display order - Changes the order in which alarms are displayed - either the newest, or the oldest, at the top.
Polling interval for alarm list refresh - Period in seconds between requests for alarm list updates.
Ringtone (iOS only) - The ringtone associated to notifications.

Starting TouchVue
 

 
 
 
 
 
 
 
 
 
 
 
 
 
Starting TouchVue
 
TouchVue is started by tapping the app icon. Show picture

 
If it is the first time ever that the app has been started on a device, you must configure an account. After the splash screen has been displayed for a few seconds the Accounts view is displayed from where an account can be configured. 
 
If an account has already been configured, but you have not previously logged in or the login has expired, you must log in. After the splash screen has been displayed for a few seconds the Accounts view is displayed from which the account to log in with can be selected.
 
If you have already logged in recently, and have not explicitly logged out, the Inbox view is displayed. Login are remembered even if the app has been stopped and the device turned off.
 

 
Starting the TouchVue app also starts the alarm notification service, which runs in the background.

Dashboard view
 

 
 
 
 
 
 
 
 
 
 
 
 
 
TouchVue dashboard view
 
The Dashboard view displays a summary of the variables from a given account. It is opened by tapping the account name in the Menu Drawer. Show picture

 
Screen features
 
The screen is divided into three areas.
 
 
Trend. A simplified version of the Trend View. Tapping the Trend takes you to the Trend View.
 
Alarms. A summary of the most recent alarms. Tapping a single alarm takes you to the Variable details view for that alarm. Tapping Go To Alarms takes you to the Alarms list view.
 
Lower menu bar. Tapping the icons takes you to the Browsing, Watch List or Logged events views.
 
 
Menu actions
 
None
