Configuring the Content of a Text-box

The initial content and behavior of a Text-box is defined in the Content tab. Show picture

Properties and commands

  • Initialization
    • Text - The text that is displayed in the text-box when the container mimic is opened at run-time. It can be hard-coded at design time or be the value of a text variable.
    • Lines - The line configuration, either Multiline or Single Line. Multiline allows text to be displayed on one or more lines with the Enter keystroke starting a new line. Single Line forces the text to be displayed on one line only. The Return keystroke has no effect.
    • Case - Force all text to be lower case, upper case or allow both upper and lower case (default)
  • Properties
    • Scrollbars - Select which scroll bars are displayed at run-time. Scroll bars are not displayed in Single Line mode.
    • Text alignment - Select the text alignment from Left (default) Right or Center.
    • Text max length - The maximum number of characters that can be entered. The default of 0 means no limit.
    • Read only - Contents is read only. The User is unable to change the text at run-time.
    • Enable shortcuts - Enable the use of keyboard shortcuts such as Ctrl+c, Ctrl+v etc.
    • Password style (single line only). Actual characters are hidden. Enable this property if the text box is used as a password entry field.
    • Use soft keypad - If enable, the soft keypad will be displayed at run-time for user inputs.
