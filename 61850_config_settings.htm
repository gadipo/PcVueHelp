IEC 61850 settings

IEC 61850 settings are accessed from the Application Explorer by selecting <Project>.Communication.Data acquisition.IEC 61850 and choosing the Settings command from the task list. Show picture

  • Invalidate variables – Defines if variable status should be set to invalid upon connection failure.
    • Never - The variables are never made invalid. This can be used when there is intermittent communication with the server.
    • Immediately - The variables are made invalid immediately on a communication failure. This is the default value.
    • After a delay - The variables are made invalid after the specified delay.
  • Invalidate variables (while producer switchover) – Defines if variable status should be set to invalid upon producer switchover.
  • Monitoring
    • Double point
      • Off – Defines how Off value should be interpreted by PcVue.
      • On – Defines how On value should be interpreted by PcVue.
  • MMS layer configuration
    • Maximum size of an assembled MMS message
      • Max PDU size – Defines the maximum size of PDUs. Depending on the devices you are communicating with, this setting may need to be changed.
  • Data
    • Use timestamp quality – If not enabled, the quality of timestamp associated to data will be ignored.
    • Default originator category for select/operate of controls – Defines the value used for the OrCat parameter when sending Controls – Default to station-control.
    • Default originator identifier for select/operate of controls – Defines the value used for the OrIdent parameter when sending Controls (max 32 characters). Default to an empty string.

      • The originator identifier and category can be handled with the extended attributes of variables mapped onto controllable IEC 61850 objects. Refer to the Linking a variable to an IEC 61850 data object or data attribute topic for more information.

    • Default Test for select/operate of controls – If ticked, all controls sent will be by default flagged as ‘Test’.
    • Default Check bitstring for select/operate of controls
      • Synchrocheck – If ticked, all controls sent will be by default flagged for synchro-check.
      • Interlock-check – If ticked, all controls sent will be by default flagged for interlock-check.
  • Interoperability issues
    • Enable discovery services. If disabled, the IEC 61850 driver will not attempt to check if the configuration of the physical devices fit the PcVue configuration. This setting only applies to devices configured to use SCL-based offline configuration, it is ignored for the others.

Disabling the use of discovery services affects elements such as browsing and checking the existence of Logical devices, Logical nodes, Data objects, Data attributes, Datasets, Dataset members...
If disabled, it is the responsibility of the application designer to make sure that the configuration is consistent system-wide to ensure proper operations of the IEC 61850 driver and data mapping.

In particular, when SCL files are associated with the IEC 61850 network for offline configuration, it is strongly recommended to store the SCL files in the TP folder of the project and to add them to the list of files managed as part of Central Project Management (if in use). See the topic IEC 61850 Network - SCL Files for more information.

If in addition to disabling the use of discovery services, you wish to fully disable the use of IEC 61850 services that lead to changes of the configuration of physical devices, you must make sure that none of the following Data report advanced properties are activated:

  • Send buffer time to the device,

  • Send dataset configuration to the device,

  • Send integrity period to the device,

  • Send optional fields to the device,

  • Send report identifier to the device,

  • Send trigger option to the device.
