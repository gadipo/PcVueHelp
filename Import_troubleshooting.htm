Troubleshooting configuration import

The table below details the possible configuration inconsistency issues that might occur when importing a subset of modeled configuration into PcVue.

When a configuration inconsistency occurs with severe warnings, the save button is deactivated and the dialog box listing the inconsistencies is displayed. This is not true for warnings due to minor inconsistencies. You can configure the save button to be deactivated even with less severe warnings from the settings.

Problem description

 Possible causes Possible solutions Expected behaviors

Missing parameters on a template (or inherited template) or

Missing parameters as "Named parameters" on a template (or inherited template)

 Templates imported before dependencies. Make sure you import the necessary parameters.

Configuration becomes consistent and saving is possible.

The parameter values, if they existed, will also be restored without re-importing the template or instance.
Missing parameters on an instance (or in an element in an instance) Instances imported before dependencies.

Make sure you import the necessary parameters.

Re-import the template if the parameter (with the exception of global and unitary) is not found during an import. This could be because it has been taken out of the template used to create the instance. Does not block saving.

Configuration becomes consistent and saving is possible.
Missing included template in a template Templates imported before dependencies. Make sure you import the necessary templates.

Configuration becomes consistent and saving is possible.

The included template is automatically updated as well as the concerned instances.
Missing instance's configuration element in the original template.

The configuration element exists in an imported instance, but not in the corresponding template.

If the template is the source of the problem, the solution is to re-import the template with the configuration element in question as well as the instance.

If the instance is the source of the problem, the solution is to re-import the instance that is consistent with the template.

Configuration becomes consistent and saving is possible.

Missing instance's parameter in the original template.

The template structure is no longer supported, or the imported template is corrupted.

If the structure is no longer supported, the solution is to remove the parameter from the instance. Otherwise, re-importing the template should solve the problem.

 Configuration becomes consistent and if the parameter is still supported, references will automatically be restored between the instance parameter and that of the template without reloading the instance.

Other reasons for an inconsistent configuration are:

  • Importing a configuration into a library that is not its original library.

  • Using files from previous, more permissive PcVue versions.

Depending on the issue, restoring a previously saved configuration or continuing to import more element may help troubleshooting.
