SNMP Network Advanced Properties

The SNMP Network advanced properties are accessed from the SNMP Network configuration dialog, Advanced tab. The majority of properties in this tab are defaults that are propagated to sub-ordinate devices.Show picture.



  • Trap/inform
    • Listening port - The port number on which devices must send any traps. Default 162.
  • Device default properties - The properties to be used for each sub-ordinate device unless configured otherwise in the device configuration Advanced tab.
    • SNMP port - The port used by the device to communicate using SNMP.
    • Number of retries - The number of retries after a timeout. Default 2.
    • Number of OID's per request - The maximum number of OIDs that the device supports in one SNMP request. Default value is 25.
    • Timeout (ms) - Device timeout in milliseconds. Default 1000.
    • Host name resolution mechanism
      • Resolution period (ms) - When devices are addressed based on their host name, resolved names may be reset at runtime due to external actions. Setting a resolution period forces host name resolution to be executed periodically. Default to 1 hour.
      • Enable dynamic resolution upon reconnection - In addition, enabling this setting forces a device's host name to be resolved upon reconnection to avoid failure if the resolved name was reset during the disconnection.
    • SNMP V1 and V2/2c
      • Community - The read community string. If the manager does not have the same string as the device, any request times out.
    • SNMP V3
      • User name - The V3 User Name string. The default is null.
      • Authentication properties
        • Type - The Type when using AuthNoPriv. Either MD5 or SHA. The default is MD5.
        • Key - The password string when using AuthNoPriv. Default is empty.
      • Privacy properties
        • Type - The Type when using AuthPriv. Either AES or DES. The default is DES.
        • Key - The password string when using AuthPriv. Default is empty.
      • Interoperability issues
        • Reset persistent Engine Time upon reconnection - Because many SNMP Agent devices do not persist the Engine Boots counter correctly, the SNMP v3 hand check may fail upon reconnection. Enabling this setting allows for working around the issue by resetting the Engine Time upon reconnection.
  • Networking
    • Servers - The name of the Servers List when configuring a multi-station application. The network will only be activated on those stations named in that list.
