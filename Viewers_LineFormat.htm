Configuring the Line Format for an Alarm or Log Viewer

The active part of an Alarm or Log Viewer is the display list that contains one or more lines each displaying one item of logged data.

The number of lines that are available depends on the size of the viewer and the font chosen. The font used is selected using the Line font button. The color in which the text of each line is displayed is configured from the Preferences menu.

The display list is arranged in up to eight columns. The information displayed in each column is configured from the Display tab using special format characters. You can also enter a caption (title) for each column that will appear in the list header. Any columns that do not have format characters, and do not have columns with format characters to the right of them, are not displayed. Show picture

Any characters not preceded by # are interpreted literally. For example, the format string Pri #A - #E would produce something like this in the alarm list:

Pri 8 - Alarm off - not ack

Configuring the column widths

The width of each column is configured at run-time by clicking and dragging the column boundaries in the header. After you have changed the column widths, the mimic containing the viewer must be saved or the changes will be lost when the mimic is closed. To change the column width the property Size of Columns, in the Execution tab, must be ticked. To prevent the User from changing the column widths later, Size of Columns must be un-ticked when the Viewer configuration is finished.
