Configuring User Data

User Data is supplementary information that can be associated with a marker. The information is displayed in a pop-up window when the marker is right clicked at run-time. User data can include fixed text, variable's real-time values, and variable's extended attributes. Show picture

How to configure user data

  1. In the Map Markers Editor, select the marker to which the User Data is to be added.
  2. Click the Ellipsis button adjacent to the User Data field. The User Data Management dialog opens. Show picture
  3. Select the + tool to add a new item.
    1. Enter the Label. This appears in the left hand column of the run-time window.
    2. Enter the name of a variable or variable's extended attribute in the Variable field, or enter a fixed string in Value. See the topic Binding marker properties to variables for information about how to reference a variable's extended attributes.
    3. Repeat a and b until the user data is complete.
  4. Click OK to confirm the configuration and close the dialog.

The Name field must be unique although it does not appear in the User Data window at run-time. It is only used when managing user data programmatically.

If a label is not defined, the user data will not appear in the pop-up window.

If the Variable and Value are both configured, when the marker is displayed in run mode, the variable is forced to the value even if the variable is not configured with the Command property set.

About the Label field

The Label field is a multi-lingual string. Clicking the Ellipsis button opens the Configuring Multilingual Strings dialog where you can enter strings for the various languages. The Label field supports the standard substitution strings. For example:

  • #V - Substitutes the variable name
  • #T - Substitutes the variable's Description property.
