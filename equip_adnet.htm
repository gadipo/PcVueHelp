AD-Net

This driver supports serial communication with Advanced devices, including the Mx4000 and Mx5000 series panels.

Supported functions

Functions Description
Sending messages
Request information Send commands

  • Zone Text

  • Device Status

  • Node Status

  • Analog Value

  • Reset / Mute / Silence / Resound

  • External Fire, Alarm, Acknowledgment

  • Disable / Enable Zone

  • Disable / Enable Device

  • Disable / Enable Group

  • Activate / Deactivate Output
Receiving unsolicited messages

  • Text Mode

  • Device Status

  • Node Status

  • Network Configuration Change

  • Zone Text

  • Analog Value

  • Output Activated / Deactivated by BMS
Sending acknowledgment when a message is received

 Broadcast request on all devices on the network

You can configure a Broadcast device to avoid doing individual cyclic requests for each device configured and thus minimize traffic on the network traffic. It allows using a broadcast mechanism that enables devices to send unsolicited status update.

Make sure that the broadcast device is the last one configured.

Driver status

The driver status provides driver-specific information to supplement the frame general status. For more information on status in general, refer to the topic General communication status.

Status Meaning Description
0x41 Format error Received a command that does not match the format
0x42 Command error Unsupported command code
0x60 Parameters error Specified parameters does not exist or can’t be used
0x61 Data error Incorrect contact, data area, data number, size, range or format specification
0x62 Registration over error Number of registrations exceeded or operated in an unregistered state
0x63 PC mode error A command that cannot be processed was executed while in RUN or PROG mode
0x71 Exclusion control error A command that cannot be processed simultaneously with the command being processed has been executed
0x80 CRC Code error The warranty data (CRC code) is wrong
0x91 No additional slave unit error A slot number in the range where the expansion slave unit is not mounted was specified

The driver can also return error codes if the configuration is incorrect:

0x8002 Command not supported

0x8005 Incorrect equipment address
0x8006 Error on SendRcv
0x8007 Incorrect packet number

SCADA Basic scripting

You can control communication with this driver in SCADA Basic scripts with the CIMWAY instruction.

See the topic Message exchange mechanisms in PcVue for more information about what to optimize and how you can send commands from scripts.
