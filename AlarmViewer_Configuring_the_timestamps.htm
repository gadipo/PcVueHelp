Displaying the Alarm On, Ack and Off Timestamps on the Same Line

From version 8.20 onwards, it is possible to display the timestamp at which an alarm came on, was acknowledged and went off, all on the same alarm line. Show picture



This is achieved using the following format strings in the line format.

Description Format (Range) On Ack. Off
Year 2 digits #YAlaOn #YAlaAck #YAlaOff
Year 4 digits #yAlaOn #yAlaAck #yAlaOff
Month 2 digits (1 - 12) #MAlaOn #MAlaAck #MAlaOff
Day 2 digits (1 - 31) #DAlaOn #DAlaAck #DAlaOff
Hour 2 digits (0 - 23) #hAlaOn #hAlaAck #hAlaOff
Minutes 2 digits (0 - 59) #mAlaOn #mAlaAck #mAlaOff
Seconds 2 digits (0 - 59) #sAlaOn #sAlaAck #sAlaOff
Milliseconds 3 digits (0 - 999) #lAlaOn #lAlaAck #lAlaOff

When creating an Alarm Viewer with the three timestamps on the same line it should be configured so that it can operate in List mode only.

By default, timestamp of future transitions are represented with a question mark. This character can be replaced by another one by modifying the following HMI option: Replacement symbol if alarm timestamp does not exist.
