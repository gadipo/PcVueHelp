Configuring a Scheduled Action

A new action is configured by selecting the Create command button in the Scheduler Management dialog box. If an existing action is selected in the action list, the dialog box that is displayed will contain the configuration of the selected action enabling a new action to be created with similar characteristics. An action may set a Bit, run a SCADA BASIC function or send a Recipe.

Actions may be enabled and disabled by a database Bit. The action is enabled when the Bit is set to 1 and disabled when 0.

Setting an action's agenda

An action's agenda determines when and with what frequency it will be initiated.

  • Frequency - The frequency at which the action is triggered
    • Timetable - According to a user defined time table.
    • Once - Once only at the specified date and time.
    • Monthly - Once a month on the specified day and time.
    • Weekly - Once a week on the specified day and time.
    • Daily - Every day at the specified time.
    • Hourly - Every hour at the specified minute.
  • Time - The time in hours and minutes at which the event is to be triggered.
  • Day - The day on which the event is to be triggered (weekly events only).
  • Date - The date at which the event is to be triggered.
  • Month - The month in which the event is to be triggered (once only events).
  • Year - The year in which the event is to be triggered (once only events).

For each time table action a Time Table must be created. This is covered in the section on Configuring a Time Table Action.

The behavior of an action for Bits and Recipes is different when using a Time Table. See the topic Configuring Time Table Actions for more information.

Using a scheduled action to set a bit

Show picture

  • Name - The name of the bit to set.
  • Inverse - Set the bit to the opposite of its current state. There is no way to know the final state of the bit, only that it will change.
  • To 0 - Set the bit to 0.
  • To 1 - Set the bit to 1.
  • Pulse - Set the bit to the chosen state for a pre-defined period (in seconds).

Using a scheduled action to send a recipe

Show picture

  • Num start - The number of the recipe to be sent at the start of a period.
  • Title start - The title of the recipe to be sent at the start of a period.
  • Num end - The number of the recipe to be sent at the end of a period (optional).
  • Title end - The title of the recipe to be sent at the end of a period (optional).

Either a recipe number or recipe title may be entered - not both.

Using a scheduled action to run a function

Show picture

  • Program - The name of the SCADA BASIC program that contains the function to execute.
  • Branch - The database branch to be used by the function. Optional.
  • Function - The function to be executed. If not specified then the Main function will be executed.
  • Arguments - A list of arguments that may be retrieved within the function using the GETARG instruction. A comma is used as a separator between arguments.
