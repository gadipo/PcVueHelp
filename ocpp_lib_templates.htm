How to use the OCPP library

Charge point variables are the carriers of information exchanged between charge points on the network and PcVue. A library of predefined templates is available to ease the mapping of OCPP charging stations data and management messages.

The predefined library for OCPP includes:

  • Images

  • SCADA Basic scripts

  • Animated symbols

  • Templates and Parameters

  • Mimics

  • An archive unit

Prerequisites:

  • You have installed and activated the OCCP driver, and performed Url namespace reservation as detailed in the topic How to install and configure the OCPP driver.

  • You have the OCPP shared library installed (folder \LIB\SH_EV_Charging).

  • You know how to use the Application Architect to instantiate existing templates.

Instantiating OCPP templates

  1. Click Configure in the main menu of PcVue then Application Architect.

  2. Click the plus button next to OCPP in the Templates tab. The categories OCPP 1.6 and 2.0 appear, containing templates for all OCPP variables. Show picture

  3. Add an instance of the ChargePointProject template and give it a name. This instance provides:

    • Variables global to your charge point central monitoring system,

    • The essential keys for the General and OcppBinding sections of the OcppConfiguration.ini file,

    • The mimics global to your CSMS,

  4. Set the value of input parameters for the essential keys in the .ini file, at least the Http endpoint parameter.

  5. Add an instance of the ChargePoint template as a child of the ChargePointProject instance and give it a name that matches the Id of the actual charge point deployed on the field (for example CPxxxx). This instance provides:

    • Variables to monitor the charge point,

    • Inherited templates to support the related of OCPP messages: Status, authorization, handling of reservation, availability, charging profile, firmware update, etc.

    Make sure that the name of the ChargePoint instance you add matches the name (or id) set in the corresponding charge point device. Refer to the charge point vendor documentation for more information.

  6. Set the value of input parameters as required by your set-up.

  7. For each connector on the charge point you have instantiated, add an instance of the Connector template as a child of the ChargePoint instance (index starts at 1). This instance provides variables and inherited templates to monitor the connector and support the related OCPP messages: Availability, electrical meter values, status, transaction, etc.

  8. Set the value of input parameters as required by your set-up. At least the ConnectorId according to the charge point vendor documentation.
    The phase number is also important if the charge point has more than one phase (possible values are: L1, L2, L3, L1L2, L2L3, L3L1, L1N, L2N, L3N, N).

  9. Repeat steps 7 and 8 for each connector available on the charge point.

  10. Repeat steps 5 to 9 for each charge point.

  11. Save the configuration by clicking File.Save in upper menu then Tasks.Generate to generate the configuration.

  12. Click Configure.Application Explorer in upper menu of PcVue then open the charge point project from Variables tree. All charge point messages and variables generated are displayed. Pre-animated mimics are also generated. Show picture

    13. An alarm is set by the OCPP driver and is activated when the charge point status is Faulted and deactivated when it is any other value. Alarm and bit variable values are defined in relation to the values set in the associated labels of the Status variable.
    The OCPP driver also writes label string representations in extended attributes of alarm and bit variables such as ChargePointErrorCode written into extended attribute 3.

    We recommend you include additional details about charge point errors in other extended attributes.
