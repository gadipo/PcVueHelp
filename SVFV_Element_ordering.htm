Drawing Element Ordering

What is drawing element ordering?

Drawing element ordering is the order in which the HMI renders drawing elements on the screen. The order determines which element appears in the front when two or more elements are overlapped. The initial drawing element order is decided by the order in which the developer draws them.

In the following picture, the drawing elements were drawn in the order indicated by their number. When the drawing elements are moved so that they overlap, the element that was drawn last appears in the front. Show picture

Four tools are available on the Arrange toolbar for changing the ordering of drawing elements.

Bring to Front

Send to Back

Bring Forward

Send Backward

Front and back ordering

  • The Bring to front tool moves one or more selected drawing elements to the front. Show picture

  • The Send to back tool moves one or more selected drawing elements to the back.

If more than one drawing element is selected their relative ordering is retained.

Forward ordering

The behavior of the Bring forward tool depends on how the drawing elements are selected.

  • If there is only one drawing element selected it is moved in front the drawing element immediately in front of it.
  • If two or more drawing elements are selected, all selected elements are moved in front of the reference element. The element with solid handles, usually the one selected last. The relative order of the moved drawing elements is retained. Show picture

Backward ordering

The behavior of the Send backward tool depends on how the drawing elements are selected.

  • If there is only one drawing element selected it is moved behind the drawing element immediately behind it.
  • If two or more drawing elements are selected, all selected elements are moved behind the reference element. The element with solid handles, usually the one selected last. The relative order of the moved drawing elements is retained. Show picture

Not all objects that can be inserted in a mimic support ordering. In particular, ActiveX controls do not usually support it.
