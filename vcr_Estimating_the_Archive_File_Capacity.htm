Estimating the Archive File Capacity

Estimating the archive file capacity can be difficult as you must know:

  • The format in which you are recording the timestamps
  • The length of your variable names
  • The length of any other attributes you are recording.

You also need to know the rate at which you expect events to occur.

It is important to at least estimate the worst case, else you may find that the archive unit does not store information for a sufficient number of days.

You should provision the correct amount of storage space on the station that is the archive unit server, and also on all replay stations (clients of the archive unit) because archived data files are too be synchronized on such computers.

Example

The following example estimates the amount of space required to record 10 events per second for 7 days using the default date format and no additional attributes.

Field

Characters

Timestamp1

16

Timestamp2

16

VarName

20

Flag

2

Value

3

Attributes (none)

0

Total

57

Including the end of line character, each event will require about 60 characters. Each character requires 1 byte. The total storage required therefore is:

  • 10 (events) x 60 (seconds) x 60 (minutes) x 24 (hours) x 7 (days) x 60 (bytes) = 360MB

Assuming that you create one file an hour, this will produce:

  • 168 files (24x7) of approximately 2.2MB each (10x60x60x60).
