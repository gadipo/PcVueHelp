Creating an Alarm Viewer

The Alarm Viewer is extremely flexible in its appearance and operation, so there are many configurable properties.

The procedure for creating an Alarm Viewer follows. Show picture

  1. Open the window in which the Alarm Viewer will be inserted and select Design mode.
  2. Select the Alarm Viewer tool from either the animation toolbar or the menu.
  3. An Alarm Viewer will be created and its properties dialog automatically opened.

Once you have created an Alarm Viewer, and closed its properties box, it can be moved, sized and copied the same as any other drawing element.

The Alarm Viewer configuration dialog contains five tabs for the various aspects of the configuration.



  • Display - General display features. See also the topic Configuring the line format.
  • Aspect - General appearance - the same as the Aspect tab of all drawing elements.
  • Execution - Properties that select the tools available to the User at run-time. See the topic Configuring the run-time behavior.
  • Filters - Selection of the alarm transitions to display. See the topic Configuring the filter.
  • Menu - Configuration of the run-time context menu. See the topic Configuring the context menu.

Configuring the general display features

  • Toolbar and menu - The appearance and location of the toolbar at run-time.
    A drop-down list box is used to select the size of the icons on the toolbar. The space that the toolbar occupies depends on selected size. If None is selected the toolbar is not displayed.
    The position and configuration of the toolbar, but not the size of the icons, can be changed by the User at run-time unless you specifically disable that capability in the Execution tab.
  • Use context menu - Enable the run-time context menu.
  • Scrollbars - The availability and width of the vertical and horizontal scroll bars. The default setting of a vertical scroll bar only is used in most applications.
  • Background - The background color of the display.
  • Alternative color of text - The alternative text color for projects configured with legacy versions of PcVue that did not have the capability to select the two colors for blinking alarms.

The ability to define the size of the icons and the width of the scrollbar allows the application designer to adapt the viewer if touch screens are used at runtime.

You can specify more properties to control the graphical element visibility. From the Aspect tab, behind the down arrow at the bottom of the Properties dialog box:

  • Specify zoom visibility bounds so that the graphic control is only visible for predefined zoom levels.

  • Specify layers so that the graphic control is only visible if one of the selected layers is displayed.

These are even more useful when combined with user permissions and decluttering templates.
