Configuring the Grid's Run-time Behavior

This topic explains how to configure options that effect the user's ability to interact with a Grid at run-time. The properties are all found in the General and Execution tabs. Show picture

Not all behaviors are available in all modes. For example, Editable and Drag & Drop have no useful function when a Grid is used in variable tracking or historical modes.

Cell selection

One or more cells may be selected at run-time by pointing and clicking. Cells that are selected change color so that the background is blue and the any text is white. These colors are not configurable. The following properties effect cell selection.

  • Selectable - Enables the selection of cells by pointing and clicking.
  • Frame selection - Display a black frame around each selected cell.
  • Track selection - Causes the headers to indicate the cell that is currently selected by adopting a relief appearance.
  • Multiselection - Allow more than one cell to be selected at once. For information on how to select multiple cells, see the topic Using the Grid at run-time.

Text entry

A user can enter text into a Grid's cell by double clicking it. The following properties effect the ability to enter text.

  • Editable - Allow text to be entered into cells by the user.
  • Enable drag and drop - Allow the contents of one cell to be dragged and dropped to another.

Text entered at run-time is not persisted when the mimic containing the grid is closed. If necessary, it can be saved by script.

Other run-time behaviors

  • Sort on header click - Enables sorting of the contents of the entire grid at run-time, using to the content of a single column.
  • Autosize - The width and height of each cell is automatically adjusted to fit the available space. All rows will have the same height and all columns will have the same width. The effect of any previous resizing is lost.

Other run-time behaviors in the Execution Tab

  • Size of Columns - If selected, the User can change the width of columns and height of rows at run-time by clicking and dragging with the mouse on the headers.
  • *Toolbar position – Allows the User to change the location of the toolbar.
  • *Toolbar modification – Allows the User to select the tools that appear on the toolbar.

* Normally these options would be disabled when the project has been completed to prevent the end-user from changing the toolbar.
