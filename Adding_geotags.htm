How to a Geo-tag to a Template

This topic explains how to add a geo-tag to a template using the Application Architect.

How to add a geo-tag to a template

  1. Navigate to a template's Configuration elements folder.
    1. In the right pane of the Application Architect, select the Templates tab.
    2. Using the templates configuration tree select the template to which the geo-tag will be added.
    3. Using the secondary navigation tree select the Configuration elements folder.
  2. Add and configure the geo-tag.
    1. Select Add a geographical data > Geo-tag. A new geo-tag with a default name is created.
    2. Edit the name to one appropriate to your application. You can only edit the name directly by clicking on it in the configuration tree.
    3. Configure the geo-tag's properties. Show picture

Which Physical ID is associated to a geo-tag depends on how they will be initialized as part of the installation process on site. For this reason, the Physical ID property is likely to be defined by an expression taking advantage of a look-up in an Excel file where the list of deployed geo-tags will be maintained.
