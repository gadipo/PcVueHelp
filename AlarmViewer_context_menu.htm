Using the Alarm Viewer Context Menu

The Alarm Viewer context menu is available if the property Use context menu has been enabled in the Alarm Viewer's configuration. It is opened by right clicking anywhere in the alarm line display area.
The available tools depend on the configuration of the alarm viewer (Menu tab), and if one or more alarms are selected. Commands available in the context menu correspond to those available in the toolbar, and have the same behavior.

The tools are divided in 3 sections:

  • Filtering - The 1st section comprises buttons to change the alarm display filter. Any change in the filter applies to the alarm viewer as a whole.
  • User actions - The 2nd section comprises button to control the alarm status: Acknowledge, mask... Unless otherwise stated, the action applies to all selected alarms.
  • Commands - The 3rd section comprises general command buttons: To change the mode or print alarms.

Interaction with other Alarm Viewer actions

There are two other forms of actions from the Alarm Viewer triggered by clicking with the mouse. How these behave will depend on how Mouse inputs have been configured in the HMI General Options.

  • Associated actions - If associated actions are configured for a right mouse click, and the context menu is enabled, on a right mouse click they appear in the context menu above the other tools.
  • Context mimic - If the context mimic action is configured for a right mouse click, and the context menu is enabled, on a right mouse click the name of the context mimic appears in the context menu above the other tools.

The context mimic and the context menu are two entirely separate features other than the interaction mentioned above.
