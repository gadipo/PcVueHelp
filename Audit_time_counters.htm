Time Counters

Each time counter contains the immediate value plus the highest five values and their timestamp's. Show picture

To reduce the amount of information displayed in the Audit dialog, flow counters for which all values are zero when the dialog is opened are hidden. If, subsequently, they become non-zero you must use the Refresh button to reveal them.

Processing time - By manager

A series of time counters to measure the time taken for certain tasks performed by all managers. These time counters are given with a breakdown by manager:

  • Duration in the phase Start Project - By manager: The duration required in each of PcVue managers to start the project. This includes the load phase and start phase.
  • Duration for processing time of the watchdog message (similar to ping in TCP/IP) - By manager: Watchdog messages are the internal keep-alive mechanism, allowing managers to know if other managers are responsive.
  • Duration for processing time of messages - By manager.

Duration for processing time on specific treatment in Variable Manager

A series of time counters to measure the time taken for certain tasks performed by the variable manager (Var). This includes:

  • Stop project
  • Delete project
  • Loading file Varexp.dat
  • Saving file Varexp.dat
  • Architect - load templates
  • Architect - load parameters
  • Architect - load instances
  • Architect - generation
  • Loading treatment (VarTreat.dat)
  • Starting treatment

Duration for processing time on specific treatment in User Interface

A series of time counters to measure the time taken for certain tasks performed by the User Interface manager (UI). This includes:

  • Load variables: The time taken to load the variable definitions from Variable manager into memory.
  • Variable subscriptions: The time taken to process subscription requests for variables to the Variable manager and their acknowledgments.
  • Load expression on variable: The time taken to load expression on variable items.
  • Load expression used to mask alarm: The time taken to load expression used to mask alarms.
  • Load events: The time taken to load event items.
  • UI timer: The total time spent in the timer function of the User Interface manager.
  • SCADA Basic timer: The time taken in the timer function of the User Interface manager to process SCADA Basic functions.
  • SCADA Basic function: Time taken to process SCADA Basic functions.
  • SCADA Basic Expression Recalculation: Time taken to process expression recalculation when requested via a SCADA Basic script.

Duration for processing time opening and closing Web sessions and client contexts

A series of time counters to measure the time taken for certain tasks performed when opening and closing Web sessions and client contexts. This includes:

  • Session context creation/deletion
  • Client context creation/deletion
  • Session-related variables creation/deletion in Var manager
  • Loading/unloading of alarm synthesis
  • Starting/stopping alarm synthesis
  • Events creation/deletion
  • Expressions creation/deletion
  • Cyclic functions creation/deletion

Duration with flow regulation active - By source

A series of time counters to measure the time spent with flow regulation active. These time counters are given with a breakdown by source:

  • All sources
  • External source
  • Equipment source
  • DDE source
  • OPC source
  • LonWorks source
  • BACnet source
  • IEC 60870-5 source
  • IEC 61850 source
  • DNP3 source
  • SNMP source
  • OPC UA source
  • Sql variables source

Other auditable processing time counters

Series of time counters are available to measure the time taken by some modules for some of their specific tasks. This includes:

Tasks specific to the CimWay component:

  • CimWay Start: The time taken to load the driver definitions from the COMM.DAT file into memory and starting all communication objects.
  • CimWay Stop: The time taken to stop all communication objects and unload the driver definitions from memory.

Tasks specific to the LNS component:

  • LNS Global Start
  • LNS Lca Start
  • LNS Network Start
  • LNS Node Start
  • LNS Variables Start
  • LNS Monitoring Start

Tasks specific to the Alarm manager:

  • Alarm counters initialization

Tasks specific to the Startup manager:

  • Timer function
  • Audit object counters update
  • Audit flow counters update
  • Audit system variables update

Other miscellaneous tasks:

  • Proprietary log record: The time taken processing recording a log in a proprietary archive unit.
  • Save Audit.Counters.xxxx file: The time taken saving the audit counters to file.
  • Save Audit.SpecificCounters.xxxx file: The time taken saving the audit specific counters to file.
  • Beep: The time spent when the SCADA Basic function Beep is used to sound the PC's speaker.
  • Online print for log: The time taken to process the online print for a log.
  • Graphic print: The time taken to process a graphic print.
  • SvBatch select: The time taken to process a select operation in a batch process.
  • SvBatch insert: The time taken to process an insert operation in a batch process.
  • DDE - Transaction: The time taken to process a DDE transaction.
  • SvBatch update: The time taken to process an update operation in a batch process.
  • Global key read: The time taken to read the protection key.
